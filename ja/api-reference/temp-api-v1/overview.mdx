Temp API は、Raydium メイン API にまだ正式な場所が定まっていないエンドポイントを一時的に収容する領域です。このサービスは、迅速なプロトタイピング・内部ツール・開発中の機能を目的として提供されています。

## ホスト

| 環境 | ホスト |
|------|--------|
| メインネット | `temp-api-v1.raydium.io` |
| デブネット | `temp-api-v1-devnet.raydium.io` |

## 安定性に関する注意

**このサービスには API の安定性保証がありません。** エンドポイントは予告なく変更・非推奨化・別サービスへの移行・削除が行われる場合があります。このAPIを使用するのは以下の目的に限定してください。

- 内部ツールおよびスクリプト
- 開発中の機能
- 障害が発生しても許容できる非クリティカルなパス
- テストおよびプロトタイピング

本番 dApp やクリティカルな統合にこの API を依存させないでください。

## 概要

このサービスが現在公開しているエンドポイントは以下の通りです。

- **`/cp-creator-fee`** — ウォレットが CPMM プール全体で蓄積したクリエイターフィーを取得します。
- **`/limit-order/order/list`** — インデクサーキャッシュから提供される、ウォレットの現在のCLMM指値注文（オープン・部分約定済み）の一覧。
- **`/limit-order/history/order/list-by-user`** — ウォレットの過去（クローズ済み）のCLMM指値注文の一覧。`poolId` / `mint1` / `mint2` / `hideCancel` フィルターをオプションで指定でき、`nextPageId` によるカーソルページネーションに対応。
- **`/limit-order/history/event/list-by-pda`** — 1つまたは複数の指値注文PDASのイベントログ（`open` / `increase` / `decrease` / `settle` / `close`）を取得します。`nextPageId` によるカーソルページネーションに対応。

## 共通パターン

### ウォレットのクリエイターフィーを確認する

```bash
curl -s "https://temp-api-v1.raydium.io/cp-creator-fee?wallet=<your-wallet-pubkey>"
```

ウォレットがクリエイターであるプールの一覧と、Token A・Token B における未請求のクリエイターフィーの累計額を返します。

### ウォレットの現在の CLMM 指値注文一覧を取得する

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/order/list?wallet=<your-wallet-pubkey>"
```

未約定および部分約定の注文を両方返します。各行はデコードされた `LimitOrderState`（PDA・プール・オーナー・ティック・サイド・FIFO コホートフェーズ・合計/約定済み数量）と、`SettleLimitOrder` を呼び出すために必要なプールコンテキスト（事前計算された `pendingSettle` 額を含む）で構成されます。

アクティブ注文エンドポイントはページネーションなしで、ウォレットの現在キャッシュされた全注文を1回のレスポンスで返します。

### ウォレットのクローズ済み指値注文一覧を取得する

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/order/list-by-user?wallet=<your-wallet-pubkey>&size=50"
```

過去の注文の1ページ分を返します。各注文には `fillStatus`（`NONE` / `PARTIAL` / `FULL`）と、注文がアクティブセットから外れた時点でのインプット/アウトプット数量が含まれます。次のページを取得するには、`&nextPageId=<前のレスポンスの nextPageId>` を追加してください。

### 特定の注文のイベントタイムラインを取得する

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/event/list-by-pda?pda=<limit-order-pda>"
```

`pda` には単一のキー、またはカンマ区切りのリストを指定できます。各行はオンチェーンの1つの変更操作（`open` / `increase` / `decrease` / `settle` / `close`）を表し、txid・スロット・ブロックタイム・オーナーまたはキーパーによる署名フラグ（`autoRunner`）・ヒューマンユニットでのインプット/アウトプットのデルタが含まれます。

### 結果が空の場合

ウォレットのレコードがキャッシュにない場合、アクティブ注文エンドポイントは以下を返します。

```json
{
  "id": "...",
  "success": true,
  "data": { "rows": [] }
}
```

履歴エンドポイントも同様に `{ rows: [], nextPageId?: ... }` の形式で空の `rows` 配列を返します。レガシーの `/cp-creator-fee` エンドポイントは後方互換性のため、`"data": []` を返します。
