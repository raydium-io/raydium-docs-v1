## Burn & Earn とは

Burn & Earn は、プール作成者（または任意の LP）がデポジットをオンチェーンで**永続的**にすることができる Raydium の機能です。一方でトレーディング手数料を収集する権利を保持します。ポジションは償還不可能になりますが、新たに鋳造された**Raydium Fee Key** NFT がロックされたポジションから `collectFees` を行う継続的な権利を表します。

一般的なユースケース：

- **ローンチ信頼性。** プロジェクトはデプロイヤーの LP をロックすることで、ラグプルを行わないことを約束します。ホルダーはリクイディティが永続的であることをオンチェーンで確認できます。
- **手数料生成型コレクティブル。** 利回りを生成する譲渡可能な NFT。NFT を売却する際に手数料ストリームが譲渡されます。
- **トレジャリーグレードの LP。** プロトコルがブランド / ディスカバリーのためにリクイディティを永遠に保持したいが、手数料側で収益化したい場合。

## サポートされるプールタイプ

| プログラム | Burn & Earn サポート |
|---|---|
| CPMM | はい |
| CLMM | **フルレンジのみ**（推奨） |
| AMM v4 (Hybrid) | サポートされていません |

CLMM については、フル価格範囲にまたがるポジションのみをロックすべきです。ロック後は再バランスできません。価格が選択した範囲外に移動すると、そのポジションで永続的に手数料を稼ぐことができなくなります。タイトレンジの Burn & Earn ポジションはほぼ常に間違いです。

## プログラムレベルでのロックの仕組み

1. 元の LP トークン（CPMM）またはポジション NFT（CLMM）が、Burn & Earn プログラムによって所有されるプログラム所有 PDA に転送されます。
2. **Fee Key NFT** が呼び出し元にミントされます。そのメタデータはロックされたポジションの PDA にポイントバックします。
3. Burn & Earn は `collectFees`（またはプールタイプごとの同等の呼び出し）を公開します。これは以下を行います：
   - 署名者が指定されたロックされたポジションに対する Fee Key NFT を保有していることを検証します。
   - CPI を通じて基盤となるプールプログラムの手数料請求命令を呼び出します。
   - 収集された手数料を Fee Key 保有者の ATA にルーティングします。
4. 対応する `withdraw` は存在しません。基盤となる LP / ポジションがエスクローを離れることはありません。

Fee Key は標準的な SPL NFT です。Solana NFT マーケットプレイスで売却したり、ウォレット間で移動したり、NFT を受け入れるレンディングプロトコルで担保として使用したり、別のコントラクトにラップしたりできます。`collectFees` 呼び出しの時点でこれを保有している者が手数料を受け取ります。

## UI ウォークスルー

1. 通常どおり CPMM プール、またはフルレンジポジション付き CLMM プールを作成します。
2. **Liquidity → Create → Burn & Earn** に進みます。
3. ロックするポジションを選択します。
4. ポジションの詳細（NFT アドレス、サイズ、プール）を確認し、UI が表示する確認文を入力します。この操作は取り消せないため、意図的に摩擦が大きくなっています。
5. **Confirm, lock the liquidity permanently** をクリックします。
6. Fee Key NFT があなたのウォレットに届きます。ポートフォリオページのロックされたポジションに表示されます。

手数料の請求は、任意のポジションから請求する場合と同じ方法で行います。ポートフォリオページを通じて、または `collectFees` を通じて。

## API サーフェス

プール情報エンドポイントは、Burn & Earn を通じてロックされたリクイディティのシェアを示す `burnPercent` フィールドを返します：

```http
GET https://api-v3.raydium.io/pools/info/list
  ?poolType=standard
  &poolSortField=default
  &sortType=desc
  &pageSize=1
  &page=1
```

```json
{
  "data": [
    {
      "id": "...",
      "burnPercent": 99.17,
      ...
    }
  ]
}
```

`burnPercent` は信頼シグナリングにとって便利なサーフェスです。LP が 99% 以上 burn されているトークンは、LP が 0% burn されているトークンと比べて、ラグプル可能性が劇的に低いです。

## 落とし穴

- **取り消せません。** LP 側は決して取得されません。二度と触らないことに同意している金額であることを確認してください。
- **CLMM タイトレンジはトラップです。** フルレンジ CLMM ポジションのみをロックしてください。ただし、範囲制限ロックポジションが価格帯外で手数料を稼ぐのを停止することを本当に理解している場合は除きます。
- **Fee Key NFT を失うことは手数料を永遠に失うことです。** 登記簿のように扱ってください。誤ってこれをバーンしたり、敵対的なコントラクトにステーキングしたりしないでください。
- **Transfer fee 機能付き Token-2022 ミント**：Burn & Earn はこれらをサポートしていますが、転送手数料のヘアカットは手数料請求のたびに適用されます。予算に組み込んでください。
- **部分的なロックはありません。** 50% ロック、50% 移動可能にしたい場合は、ポジションをまず 2 つのポジションに分割し、その後 1 つだけロックしてください。

## LaunchLab は内部で Burn & Earn を使用します

[LaunchLab](/ja/products/launchlab) から卒業したトークンは、マイグレーション後の LP に Burn & Earn を使用します。Raydium 独自の LaunchLab のデフォルト分割：**LP トークンの 90% が burn**（Burn & Earn にロック）され、**10% がクリエイターに Fee Key NFT として送付されます**。LaunchLab 上に構築している他のプラットフォームは、異なる burn / クリエイター / プラットフォーム分割を設定できます。[`products/launchlab/platforms`](/ja/products/launchlab/platforms) および [`products/launchlab/creator-fees`](/ja/products/launchlab/creator-fees) を参照してください。
