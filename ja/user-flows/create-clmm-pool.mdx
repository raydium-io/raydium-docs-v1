  CLMM プールには 2 つの永続的な選択が必要です：**手数料ティア**と**初期価格**です。また、プールに流動性を供給するために初期ポジションを開く必要があります。署名する前に十分注意してください。

## 必要なもの

- Solana ウォレット（以下を含む）：
  - ペアリングしたい 2 つのトークンミント
  - プールおよびポジション作成のレント、優先度手数料で約 0.3 SOL
- 手数料ティアの決定
- 明確な初期市場価格と最初のレンジ

CLMM は SPL Token および Token-2022 ミントの両方をサポートしています。

## 手数料ティア

| 手数料ティア | 最適な用途 |
|----------|----------|
| 0.01% | ステーブルコイン・ステーブルコインペア |
| 0.05% | 相関ペア |
| 0.25% | ほとんどのボラティルな主要ペア |
| 1% | ロングテール、エキゾチックペア |

より低い手数料ティアは、より緊密な価格帯とより高いボリュームが必要です。より高い手数料ティアはより多くのボラティリティに対応できますが、ボリュームを減らす可能性があります。

## UI ウォークスルー

[raydium.io/clmm/create-pool](https://raydium.io/clmm/create-pool) で：

1. **トークンペアを選択します。** 両方のミント アドレスを貼り付けます。
2. **手数料ティアを選択します。** UI の推奨事項を出発点として使用します。
3. **初期価格を設定します。** 既存の市場価格を使用します。そのペアのプールが既に存在する場合は、それと比較します。
4. **最初のレンジを選択します。** 初期ポジションの下限価格と上限価格を入力します。
5. **預金額を入力します。** UI はレンジから必要なトークン比率を計算します。
6. **確認して署名します。** ペア、手数料ティア、初期価格、レンジ、預金額、およびレントを確認します。

## 初期価格を正しく設定する

同じペアの他の CLMM または CPMM プールが存在する場合は、その現在の価格を出発点として使用します。Raydium プールが存在しない場合は、信頼できる市場ソースを使用します。

初期価格を間違えて設定した場合、最初の取引によってプールが裁定され、安く評価されたサイドが流出する可能性があります。簡単なリセット ボタンはありません。通常は流動性を引き出してプールを再作成する必要があります。

## 最初のポジション：レンジ選択

- **緊密なレンジ：** 相関ペアに最適ですが、アクティブな管理が必要です。
- **中程度のレンジ：** SOL/USDC などのボラティルな主要ペアで一般的です。
- **幅広いレンジ：** 資本効率は低いですが、放置しやすいです。
- **フルレンジ：** CPMM のように動作しますが、集中型流動性の多くの利点を失います。

現在の価格がレンジ外にある場合、初期ポジションはレンジ外となり、価格がレンジ内に移動するまで手数料を獲得しません。

## 作成後チェックリスト

1. **プールページが読み込みます。** プール アドレス、ペア、および手数料ティアを確認します。
2. **ポジションが表示されます。** Raydium ポートフォリオでポジション NFT を確認します。
3. **レンジが正しいです。** 下限価格と上限価格を確認します。
4. **流動性が 0 ではありません。** プールは選択したレンジのアクティブな流動性を表示する必要があります。
5. **小さなスワップをテストします。** プールを公開で共有する前に、ルートが機能することを確認します。

## トラブルシューティング

### 作成コストが予想より高い

幅広いレンジと小さな手数料ティアには、より多くのティック アレイ レントが必要な場合があります。開始レンジを狭めるか、再試行する前により多くの SOL を追加します。

### トークン順序が間違っている

UI はトークン順序を正規化するため、表示されるトークン A/トークン B は入力した順序と異なる場合があります。署名する前に価格見積もりを注意深く確認します。

### 手数料ティアが間違っている

プール作成後は手数料ティアを変更できません。間違っている場合は、流動性を閉じて、正しいティアで新しいプールを作成します。

### ゼロ流動性プール

ポジションなしでプール状態を作成すると、スワップにはプールが使用できません。プールを共有する前に初期ポジションを開きます。

### レンジ外の初期ポジション

最初のレンジに初期価格が含まれていない場合、ポジションは手数料を獲得しません。現在の価格の周りにポジションを再度開きます。
