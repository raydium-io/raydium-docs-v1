## 移行する理由

| 項目 | AMM v4 | CPMM |
|-----------|--------|------|
| フィーの配分先 | プロトコル + LP の分割 | LP がデフォルトのスワップフィーシェアを受け取る |
| OpenBook への依存 | あり | なし |
| 作成コスト | 高い | 低い |
| Token-2022 対応 | いいえ | はい |
| 新規リスティングのルーティング先 | 稀 | デフォルト |

ペアの取引量が多い場合、LP フィーの上昇により移行が正当化される可能性があります。旧プールが十分な流動性を保持していて、プロジェクトが特定の移行先 CPMM プールを推奨していない場合、移行はコーディネートコストに見合わないかもしれません。

## 移行前のチェックリスト

プール運営者とプロジェクト財務部向け：

- [ ] LP に移行のタイムラインを告知する。
- [ ] 可能な限り最低 48 時間の余裕を設ける。
- [ ] 移行先の CPMM プールとフィーティアを確認する。
- [ ] 移行後の告知（新しいプール ID と LP ミント）を準備する。
- [ ] 新しいファームが必要かどうかを決定する。

個別の LP 向け：

- [ ] プロジェクトが推奨する移行に従っていることを確認する。
- [ ] 複数のトランザクションに必要な SOL を保有する。
- [ ] AMM v4 LP の残高と期待される基礎資産額を記録する。
- [ ] AMM v4 LP トークンがファームにステークされていないか確認する。

## UI での移行：個別の LP

移行は 2 つのアクションです：AMM v4 から流動性を削除してから、CPMM に流動性を追加します。

### ステップ 1：AMM v4 から引き出す

1. AMM v4 プールまたはポートフォリオのポジションを開く。
2. LP トークンがファームにステークされている場合は、まずアンステークする。
3. **流動性を削除** をクリック。
4. 引き出す割合を選択。
5. 受け取るトークン額を確認。
6. 確認して承認。

### ステップ 2：CPMM に預ける

1. 移行先の CPMM プールを開く。
2. **流動性を追加** をクリック。
3. 一方のトークン額を入力。
4. ペアになるトークン額と余剰残高を確認。
5. 確認して承認。

引き出しと預け入れの間に、プール価格が変動する可能性があります。2 つのプールの比率が異なる場合、いくつかのトークンがウォレットにダストとして残ることがあります。

## UI での移行：プロジェクト財務部

プロジェクト財務部の場合、プロセスは同じですが、調整が必要です：

1. **事前に移行を告知** する。
2. **CPMM プールをシードする**（ユーザーに移行を依頼する前）。
3. **財務部の LP を最初に移行** してトランザクションリンクを公開。
4. **新しいファームを作成** する（インセンティブが CPMM LP ミントで続くべき場合）。
5. **個別 LP の移行を監視** する（数日～数週間）。
6. **旧プールを廃止する**（ユーザーが十分な時間をかけて移動した後のみ）。

## 検証

### 引き出し後

- AMM v4 LP トークン残高が減少するか、ゼロになるはず。
- 2 つの基礎資産が ウォレットに表示されるはず。
- トランザクションは Solana エクスプローラーで **Success** と表示されるはず。

### 預け入れ後

- CPMM LP トークン残高がウォレットに表示されるはず。
- Raydium のポートフォリオに新しいポジションが表示されるはず。
- CPMM プールページに自分の流動性シェアが表示されるはず。

### 移行の証明

透明性のため、両方のトランザクションリンクを公開します：

- AMM v4 からの引き出し。
- CPMM への預け入れ。

## ロールバックの検討

CPMM から AMM v4 へのロールバックは、CPMM から引き出して AMM v4 に預け入れることを意味します。特定の製品上の理由がない限り、通常は実行する価値がありません。AMM v4 プール作成はより高額で、OpenBook マーケットのセットアップが必要です。

## よくある質問

### 自分の AMM v4 LP はファームにステークされています。移行できますか？

はい。最初にアンステークし、AMM v4 から LP を引き出し、CPMM に預け入れ、新しいファームが存在する場合は新しい CPMM LP トークンをステークします。

### CPMM プールがまだ存在しません

ほとんどの LP はプロジェクトが移行先プールを作成して推奨するまで待つべきです。自分で作成するということは、初期価格と流動性リスクを引き受けることを意味します。

### 大きなポジションを持っています。スリッページについてはどうですか？

ポジションがプール TVL に対して大きい場合、引き出しと預け入れを小分けにしてください。各クォートを確認し、受け取る額が期待より大きく悪い場合は署名を避けてください。

### 移行を忘れた LP トークンはどうなりますか？

旧プールに留まり、AMM v4 のルールに従って引き続き報酬を得ます。移行を強制するものはありません。

## トラブルシューティング

### 移行先の LP トークンアカウントが新規です

最初の CPMM 預け入れは、新しい LP トークンアカウントの作成が必要な場合があります。レントとフィーのための追加の SOL をウォレットに保有してください。

### 2 番目のトランザクションが期限切れになります

引き出しは成功したが預け入れが期限切れになった場合、トークンはウォレットに残ります。CPMM プールを再度開き、高い優先度フィー設定で再試行してください。

### 間違った CPMM プール

同じペアに対して複数の CPMM プールが存在することができます。プロジェクトが推奨するプールか、意図したフィーティアと流動性を持つプールを使用してください。

### 税務上の影響

一部の法域では、LP 引き出しが課税対象となる場合があります。大きなポジションを移行する前に、現地の規則を確認してください。
