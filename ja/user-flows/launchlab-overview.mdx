LaunchLab は、クリエイターがボンディングカーブを通じてトークンを発行し、初期リクイディティをブートストラップできます。買い手はグラデュエーション前にカーブに対して取引します。ローンチがグラデュエーション閾値に達すると、リクイディティは Raydium AMM プールに移行し、取引は通常の Raydium ルーティングで続行されます。

## LaunchLab の利用者

| ユーザー | 一般的なタスク |
|---|---|
| トレーダー | 初期段階のトークンを発見し、ボンディングカーブで売買し、グラデュエーション進捗を追跡します。 |
| クリエイター | コードなしでトークンをローンチし、カーブパラメータを設定し、オプションでクリエイターフィーを獲得します。 |
| プラットフォーム | 独自のフィー設定、ブランディング、ローンチ発見機能を備えたブランド化されたローンチ環境を構築します。 |

## ローンチの仕組み

1. クリエイターがトークンメタデータ、供給量、カーブ設定、グラデュエーション目標、およびオプションのフィー設定を設定します。
2. トークンはボンディングカーブで開きます。買い手と売り手はそのカーブに直接取引します。
3. 買いと売りが発生するにつれてカーブ価格が変わります。
4. カーブがグラデュエーション目標に達すると、リクイディティは Raydium プールに移行します。
5. グラデュエーション後、ユーザーは Raydium Swap とこれを通じてルーティングするアグリゲーターを通じてトークンを取引します。

## 作成モード

| モード | こんなときに使用 | 主要設定 |
|---|---|---|
| JustSendit | デフォルト設定で最速のノーコードローンチが必要な場合。 | 名前、ティッカー、画像、オプションのソーシャルリンク、オプションの初回購入。 |
| LaunchLab | 供給量、グラデュエーション目標、カーブアロケーション、ベスティング、またはクリエイターフィーをより細かく制御する必要がある場合。 | 供給量、レイズ目標、カーブアロケーション、ベスティング、マイグレーション後フィー配分。 |

JustSendit はデフォルトのグラデュエーション目標を使用します。LaunchLab モードではクリエイターが UI およびプログラム制限に従ってグラデュエーション目標を設定できます。

## グラデュエーションとリクイディティ

グラデュエーションはボンディングカーブを閉じ、資産を Raydium プールに移行させます。マイグレーション後のプールタイプと LP 処理は、プラットフォーム設定とクリエイターフィー設定に依存します。

- マイグレーション後クリエイターフィーが有効な場合、ローンチは CPMM に移行し、Fee Key NFT はクリエイターのフィー権を表すことができます。
- プラットフォームがマイグレーション後クリエイターフィーなしのレガシーパスを使用する場合、ローンチは AMM v4 に移行する可能性があります。
- プラットフォーム設定により、マイグレーション後の LP トークンがどのようにバーン、ロック、または割り当てられるかを変更できます。

## セーフティの基本

- ティッカーやロゴだけでなく、トークンミントを検証してください。
- 取引前にグラデュエーション目標とカーブ進捗を確認してください。
- ローンチまたは購入前にクリエイターとプラットフォームのフィー設定を確認してください。
- Fee Key NFT をフィー権資産として扱ってください。転送またはバーンされた場合、請求権は移動するか失われる可能性があります。
- ウォレットを接続またはトランザクションに署名する前に、[信頼とセーフティ](/ja/getting-started/trust-and-safety)を参照してください。
