## 必要な準備

- Solana ウォレット：
  - 初期流動性をシードするのに十分な各ミント。
  - プール作成レント、トークンアカウント作成、優先度フィー用に約 0.2 SOL。
- 2 つのトークンミントアドレス。
- フィー階層。デフォルトの 0.25% 階層はほとんどのボラティル性ペアに適しています。

## UI ウォークスルー

[raydium.io/liquidity/create-pool](https://raydium.io/liquidity/create-pool) で：

1. **プールタイプを選択します。** **Standard AMM (CPMM)** を選択してください。CLMM を使用するのはレンジ管理を理解している場合のみです。
2. **両方のトークンを入力します。** ミントアドレスを貼り付けるか、シンボルで検索してください。Token-2022 ミントの場合は、警告を確認してください。
3. **フィー階層を設定します。** 一般的なオプションは 0.01%、0.25%、1% です。より高いフィーは、よりボラティルまたはロングテール ペアに適しています。
4. **初期価格と預金を設定します。** 片方を入力し、初期価格を入力してください。UI が反対側を計算します。
5. **確認します。** ミントアドレス、フィー階層、初期価格、LP トークン、プール シェア、および一回限りのレント を確認してください。
6. **確認して署名します。** ウォレットがプール作成トランザクションに署名します。
7. **インデックス作成を待ちます。** プールはチェーン上ですぐにライブになりますが、Raydium UI が検索に表示されるまでに数分かかる場合があります。

## 初期価格の選択

プールの最初の価格は、初期預金比率から生じます。10 SOL と 1000 USDC を預金すると、初期 SOL 価格は 100 USDC になります。

市場に近い価格を選択してください。価格が高すぎたり低すぎたりすると、アービトレージ トレーダーはプールを再バランスし、初期在庫を通じてその差分を支払うことになります。

## 作成後のチェックリスト

1. **プールページを開きます。** ペア、フィー階層、プールアドレスを確認してください。
2. **LP トークンを確認します。** ウォレットまたは Raydium ポートフォリオに新しい LP トークンが表示されるはずです。
3. **小さなテスト スワップを実施します。** 少量を使用して、プールが期待どおりにルーティングされることを確認してください。
4. **発見を待ちます。** プールが検索に直ちに表示されない場合は、数分後に更新して、ミントまたはプール ID で検索してください。

## Token-2022 の考慮事項

ミントが転送フィー付き Token-2022 の場合、実効ユーザーコストはプール フィーとミントの転送フィーの合計です。UI は署名する前に Token-2022 警告を表示します。

ミントが転送フックを持つ場合、スワップはそのフック プログラムに依存します。フック プログラムが変更または転送をブロックした場合、プールの使用が難しくなる可能性があります。未知の転送フック ミントをより高いリスク として扱ってください。

転送不可のミントは CPMM プールで使用できません。

## トラブルシューティング

### 初期価格が間違っている

署名する前に間違いに気付いた場合は、戻って預金比率を修正してください。既にプールを作成している場合は、通常、修正された価格で流動性を追加し、アービトレージで落ち着くのを待つか、新しいプールを作成する必要があります。

### 優先度フィーが低すぎる

プール作成は混雑中に期限切れになることがあります。ウォレットまたは Raydium UI で優先度フィー設定を高くして再試行してください。

### 初期流動性が小さすぎる

非常に小さいプールは悪い価格影響を生じ、ユーザー体験が悪くなります。プールを公開で共有する前に、より多くの流動性を追加してください。

### 重複するプール

複数の CPMM プールが異なるフィー階層で同じトークン ペアに対して存在できます。作成前に、Raydium で両方のトークン ミントで検索し、プロジェクトが既にプールをサポートしているかどうかを確認してください。

### 権限付きミント

フリーズ権を持つミントはトークンアカウントをフリーズできます。未知のミントの場合は、フリーズ権とプロジェクト コントロールを検証した後、意味のある流動性を提供してください。
