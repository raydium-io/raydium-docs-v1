## オーダータイプ

Raydium Perpsは以下のオーダータイプをサポートしています：

| オーダータイプ | 動作 | 使用時 |
|---|---|---|
| **マーケット** | オーダーブックの最良気配値に対して即座に売買を実行します。ブックがフルサイズを埋められない場合、未約定分はキャンセルされます。 | 即座の執行を望み、スリッページを受け入れられる場合。 |
| **リミット** | リミット価格またはそれ以上で売買します。市場がその価格に到達した場合のみオーダーが約定します。 | 価格管理を望み、約定を待つことができる場合。 |
| **ストップマーケット** | トリガー価格に到達するまで非活性状態を維持し、その後マーケットオーダーになります。 | 執行トリガーを望み、トリガー後の出入りを優先する場合。 |
| **ストップリミット** | トリガー価格に到達するまで非活性状態を維持し、その後リミットオーダーになります。 | トリガーは望むがリミット価格も適用したい場合。 |
| **スケール** | 1つの大きなオーダーを価格幅全体に複数の小さなリミットオーダーに分割します。 | 段階的にエントリーまたはエグジットしたい場合。 |

## オーダーフラグ

オーダーフラグはリミット形式のオーダーの動作を変更します。

| フラグ | 動作 |
|---|---|
| **IOC** | Immediate-or-cancel。リミット価格またはそれ以上で可能な限り即座に約定し、未約定分をキャンセルします。 |
| **FOK** | Fill-or-kill。リミット価格またはそれ以上でオーダー全体を即座に約定させるか、部分約定なしでキャンセルします。 |
| **ポストオンリー** | メイカーとして流動性を提供します。即座にマッチする場合はキャンセルします。 |
| **リデュースオンリー** | オーダーによってポジションが増加することを防ぎます。その市場の既存ポジションを削減または決済することのみが可能です。 |

## オーダー送信前の確認

オーダーを送信する前に、以下を確認してください：

- 市場と方向が正しいこと。
- サイズとレバレッジが意図と一致していること。
- ストップトリガー価格とリミット価格が逆転していないこと。
- ポジションを決済または削減するつもりの場合はリデュースオンリーが有効になっていること。
- メイカー動作を望む場合はポストオンリーが有効になっていること。
