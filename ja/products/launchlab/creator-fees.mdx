## マイグレーション前：直接的なフィー蓄積

トークンがボンディングカーブ上にある間、すべての買いと売りはクリエイターフィー部分を支払い、トークンごとのヴォーライトに蓄積されます。**クリエイター**（トークンを立ち上げたウォレット）はいつでもヴォーライトから請求できます。

特性：

- **クォートトークン**（通常は SOL）で表示されます。
- **プラットフォーム**によって設定されます（プラットフォーム設定がクリエイターフィーのオン・オフ、および基本ポイントを制御します）。
- LaunchLab UI のポートフォリオページからいつでも請求可能。プログラム的には LaunchLab の `claim_creator_fees` 命令経由（IDL の正確な名前を参照）。
- 卒業時まで継続。卒業時には、残っているマイグレーション前フィーは請求可能ですが、新しいマイグレーション前フィーは蓄積されません。

## マイグレーション後：Burn & Earn を通じたフィーキー NFT

トークンのボンディングカーブがターゲットに達し、リクイディティが CPMM（または AMM v4）プールにマイグレーションされると、LP トークンがミントされ、**プラットフォーム設定に従って分割されます**。分割は常に 100% に合計され、常に以下の組み合わせを含みます：

- **バーン済み** — [Burn & Earn](/ja/user-flows/burn-and-earn) に転送され、永久にロックされます。この部分のフィーキーは、プラットフォーム制御またはバーン指定アカウントで保持されます。
- **クリエイター** — トークンクリエイターのウォレットに直接ミントされたフィーキー NFT。このフィーキーは、クリエイターの LP トークンシェアから LP トレーディングフィーを請求する権利を表します。
- **プラットフォーム** — オプション。プラットフォームトレジャリーに流れるシェア（別のフィーキー NFT として）。

### Raydium 独自の LaunchLab デフォルト

| シェア | raydium.io/launchlab のデフォルト |
|---|---|
| バーン済み（Burn & Earn、永久ロック） | 90% |
| クリエイター（フィーキー NFT） | 10% |
| プラットフォーム | 0%（Raydium のプラットフォームはプラットフォーム LP シェアを取得しません） |

これらのデフォルトでは、**クリエイターのみがそのシェアからトレーディングフィーを請求します**。他の 90% はロックされ、そのフィーは Burn & Earn に流れ（個人のポケットではなく、信頼シグナリングメカニズムに戻ります）。

### 他のプラットフォーム

LaunchLab の上に構築するサードパーティプラットフォームは、独自の分割を設定できます。マイグレーション後 LP フィーを通じて資金を調達したいプラットフォームは、たとえば 70 / 10 / 20（バーン・クリエイター・プラットフォーム）を設定できます。このドライブについては[`products/launchlab/platforms`](/ja/products/launchlab/platforms)を参照してください。

分割に関わらず、**メカニズムは同じです**：バーンされていないシェアはそれぞれフィーキー NFT として表現され、取引、転送、または請求に対して使用できます。

### マイグレーション後のクリエイターフィーが無効化された場合

プラットフォームが `creator_fee_enabled = false` を設定している場合、卒業したトークンは Burn & Earn を通じません。代わりに、レガシー LaunchLab マイグレーションパスを使用して **AMM v4** プールにマイグレーションされます。そのモードではクリエイターはフィーキー NFT を受け取らず、プログラムレベルのマイグレーション後クリエイターフィーはありません。このモードの一部のクリエイターは、オフチェーン CLI ツールを使用して `creatorFeeOn` 経由で設定されたオプションのマイグレーション後フィーを請求しています。[`products/launchlab/instructions`](/ja/products/launchlab/instructions)でオンチェーンサーフェスを参照してください。

## 0.1% トークンシェア紹介報酬

LP フィー共有とは別に、LaunchLab クリエイター（および実際には任意のユーザー）は、ウォレットを紹介者として持つ**トークンページリンク**を共有できます。共有されたリンク経由でそのトークン（または同じセッション内の他の LaunchLab トークン）を誰かが取引する場合：

- **すべてのトレードの 0.1%** が紹介者のウォレットにエアドロップされます。
- SOL またはプールのクォートトークンで支払われます。
- すべてのトレードで自動的に支払われます。請求ステップはありません。

これは[`user-flows/referrals-and-blinks`](/ja/user-flows/referrals-and-blinks)でカバーされている 1% スワップ紹介報酬とは異なる製品です。1% 紹介報酬は Raydium のスポットスワップ UI に適用され、0.1% トークンシェア紹介報酬は LaunchLab トークンページに限定して適用されます。

## フィーの請求

### クリエイターヴォーライト（マイグレーション前）

```ts
import { Raydium, TxVersion } from "@raydium-io/raydium-sdk-v2";
const raydium = await Raydium.load({ owner, connection });
const { execute } = await raydium.launchlab.claimCreatorFees({
  mint,           // launched token mint
  txVersion: TxVersion.V0,
});
await execute({ sendAndConfirm: true });
```

### フィーキー NFT（マイグレーション後）

ウォレットでフィーキーを保有することで十分です。ポートフォリオページから請求するか、関連するプールプログラムのフィー請求命令経由でプログラム的に請求してください。オンチェーンサーフェスについては[`products/cpmm/instructions`](/ja/products/cpmm/instructions)または AMM v4 相当を参照してください。

## インテグレーター用チェックリスト

- **両方のフィーストリームを表示してください。** 「マイグレーション前ヴォーライト」/「マイグレーション後フィーキー」のどちらか一方だけを表示するクリエイターダッシュボードは、他の存在を知らないユーザーのお金を逃します。
- **フィーキー所有権転送を尊重してください。** クリエイターがフィーキー NFT を売却または転送した場合、前の所有者は将来のフィー請求にアクセスできなくなります。UI は毎回元のクリエイターアドレスをキャッシュするのではなく、現在の NFT ホルダーを読む必要があります。
- **プラットフォーム設定ドリフト。** Raydium のデフォルトは変更される可能性があります。Raydium 独自のプラットフォーム以外の場合、常にプラットフォーム PDA をライブで読んで、その特定のローンチの実際のバーン・クリエイター・プラットフォーム分割を確認してください。
- **Token-2022 コラテラルと手数料トークン。** クリエイターフィーは転送手数料を持つ Token-2022 トークンで表示できます。表示される「請求可能なフィー」の数字に転送手数料ヘアカットを計上してください。
