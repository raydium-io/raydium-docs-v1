LaunchLab トークンが CPMM に移行されると、移行されたプールは LP トークンをミントします。プラットフォーム設定は、これらの LP トークンに対して何が起こるかを制御します。

## LP 分割

`migrateCpLockNftScale` には 3 つのフィールドがあります。値の合計は `1_000_000` である必要があり、これは 100% を表します。

| フィールド | 意味 |
|---|---|
| `platformScale` | LP トークンは Burn & Earn を通じてロックされます。プラットフォームが Fee Key NFT を受け取ります。 |
| `creatorScale` | LP トークンは Burn & Earn を通じてロックされます。トークンクリエイターが Fee Key NFT を受け取ります。 |
| `burnScale` | LP トークンは永続的にバーンされます。この分配に対して Fee Key NFT は作成されません。 |

これはプラットフォームレベルの設定です。同じプラットフォームを通じて起動されたトークンは、プラットフォームが設定を更新するまで同じ LP 分配を継承します。

## 例

```ts
// 40% プラットフォーム、50% クリエイター、10% バーン。
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% クリエイター Fee Key。
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% バーン。
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## CPMM フィーとの関係

`migrateCpLockNftScale` は、**誰が LP フィー権を請求できるかを制御**します。スワップ手数料レートは設定しません。移行されたプールのスワップ手数料動作は、`cpConfigId` で選択された CPMM `AmmConfig` から来ています。

例えば、移行された CPMM プールが 0.25% のトレードフィー設定を使用し、`creatorScale` が 500,000 の場合、クリエイターの Fee Key はロックされた LP フィー権の 50% を表します。`platformScale` が 400,000 の場合、プラットフォームの Fee Key は 40% を表します。

## 分割の更新

プラットフォーム設定の更新はエポックによってレート制限されています。LP 分配を重要なプラットフォームポリシーとして扱い、気軽な UI 設定ではありません。

更新する前に：

- 3 つのスケール値の合計が `1_000_000` であることを確認してください。
- プラットフォームが、将来の移行後に Fee Key NFT を受け取る人を理解していることを確認してください。
- 新しい起動がこの設定を使用する前に、クリエイターに変更を文書化してください。
