デフォルトでは、LaunchLab から移行した CPMM プールのクリエイターフィーは、元のトークンクリエイターに帰属します。プラットフォームは `platformCpCreator` を設定することで、将来の移行に対してこの動作をオーバーライドできます。

## クリエイターフィーをリダイレクトする理由

プラットフォームがクリエイターフィーの再ルーティングを使用するのは、以下が必要な場合です：

- クリエイター、コミュニティ、プラットフォーム収益間のカスタム配分ロジック
- オンチェーン再配分用の PDA で制御されたフィー・コレクター
- プラットフォームの下で移行したすべてのプール間での統一された会計処理

<Warning>
  クリエイターフィーをリダイレクトすると、誰がそれを回収できるかが変わります。この設定がアクティブになった後に移行するプール場合、トークンクリエイターはプラットフォームがフィーを彼らに再配分しない限り、直接回収にアクセスできなくなります。
</Warning>

## 仕組み

1. プラットフォーム管理者が `updatePlatformConfig` を呼び出します。
2. `updateInfo.type` は `updatePlatformCpCreator` に設定されます。
3. 値は、将来移行されるプールの CPMM プールクリエイターになるウォレットまたは PDA です。
4. そのプラットフォームの下での将来の LaunchLab 移行は、CPMM クリエイターフィーを設定されたアドレスに割り当てます。
5. プラットフォームはクリエイターフィーを収集し、独自のルールに従って再配分します。

```ts
await raydium.launchpad.updatePlatformConfig({
  programId,
  platformAdmin: owner,
  updateInfo: {
    type: "updatePlatformCpCreator",
    value: new PublicKey("<PLATFORM_WALLET_OR_PDA>"),
  },
  txVersion: TxVersion.V0,
});
```

## 注意点

- **プラットフォームごとのオール・オア・ナッシング。** この設定は、そのプラットフォームのすべての将来の移行に適用されます。
- **先行き指向のみ。** すでに移行したプールは、既存のプールクリエイターを保持します。
- **エポック制限の更新。** プラットフォーム設定の更新は Solana エポックによってレート制限されます。
- **クリエイターフィーが存在する必要があります。** 選択された CPMM `AmmConfig` に `creator_fee_rate = 0` がある場合、収集する CPMM クリエイターフィーはありません。
- **WSOL の取り扱いが重要。** 収集したフィーが WSOL で、ネイティブ SOL が必要な場合は、収集後に WSOL トークンアカウントをクローズします。

## 再配分チェックリスト

元のクリエイターに再配分する場合：

1. CPMM プールミントに関連付けられた LaunchLab プールアカウントから各元のクリエイターを解決します。
2. プラットフォームウォレットまたは PDA の保留中のクリエイターフィーを収集します。
3. 各クリエイターの合意された配分を計算します。
4. SPL トークンまたは SOL をクリエイターに転送し、必要に応じて ATA を作成します。
5. クリエイターが配分を検証できるよう、十分な会計詳細を公開します。
