## 概要

**Raydium パーペチュアル**はガスレス中央集約型オーダーブック（CLOB）パーペチュアルプロトコルです。ポジションは Solana 上の SPL ウォレットから資金提供されますが、マッチングエンジンとオーダーブックは[Orderly Network](https://orderly.network/)により運営され、複数チェーンのベニュー（Solana、Ethereum、Base、Arbitrum、BSC）全体のリクイディティをプールします。

主な特性：

- **ガスレストレード。** オーダー作成、変更、キャンセルは Solana 上のオンチェーントランザクションを必要としません。署名はローカルで無料です。デポジットと出金のみがチェーンにタッチします。
- **最大 100 倍レバレッジ**（マーケットによる）。
- **マルチチェーンコラテラル** — USDC・SOL・USDT・ETH・BNB・WBTC・YUSD・USD1 をそれぞれのネイティブチェーンからデポジット可能。すべての損益は USDC で決済されます。
- **クロスマージンのみ。** アカウント内のすべてのコラテラルは、すべてのオープンポジション全体で共有されます。
- **片方向モード。** 同じマーケットで同時にロングとショートを保有することはできません。反対側のオーダーは既存ポジションを減らすか反転させます。
- **自己管理型。** 資金はあなたのウォレットのパープスアカウント内に留まります。Raydium 管理型カストディアンウォレットはありません。
- **CEX グレードのオーダータイプ。** マーケット、リミット、ストップ（マーケット・リミット）、スケール、および IOC・FOK・Post-Only・Reduce-Only フラグ。

## 独立したプロダクトである理由

Raydium AMM（スポット）と Raydium パープス（パーペチュアル先物）はブランドと UI のエントリーポイントを共有していますが、プロトコルレベルではほぼ何も共有していません：

| | Raydium スポット AMM | Raydium パープス |
|---|---|---|
| ベニュー | Solana 上のオンチェーンプール | Orderly により運営されるオフチェーン CLOB |
| マッチングエンジン | AMM カーブ（CPMM・CLMM・AMM v4） | 中央集約型オーダーブック |
| 決済 | スワップ時のオンチェーン | オフチェーン。デポジット・出金はオンチェーン |
| 価格発見 | AMM カーブ + アービトラージ | オーダーブック + 外部価格 |
| トークンユニバース | すべての SPL・Token-2022 ミント | リストされているパープマーケットのみ（BTC、ETH、SOL、その他主要資産） |
| [`reference/program-addresses`](/ja/reference/program-addresses) のプログラム ID | あり | なし — Orderly のコントラクトを参照 |
| Raydium SDK v2 カバレッジ | 完全 | カバーなし |
| Raydium バグバウンティ | あり | Orderly を通じてルーティング |

このドキュメントサイトの残りはほぼ完全にスポット AMM と Raydium org の GitHub のプログラムに焦点を当てています。パープスはプロダクト表面レベル — オーダータイプ、フィー、コラテラル、制限 — でドキュメント化されているため、読者はそれが何であるかいつ使用するかを理解できますが、詳細なプロトコルリファレンスは Orderly 側にあります。

Raydium パープスは、米国、アフガニスタン、ベラルーシ、中央アフリカ共和国、DRC、DPRK（北朝鮮）、ウクライナのクリミア・ドネツク・ルハンスク地域、キューバ、イラン、リビア、ソマリア、スーダン、南スーダン、シリア、イエメン、ジンバブエ、またはその他の Raydium プロトコルが禁止されている地域の居住者には**利用できません**。

## インテグレーションのメモ

- スポット Raydium SDK（`@raydium-io/raydium-sdk-v2`）はパープスをカバー**しません**。プログラム的なアクセスには[Orderly の SDK](https://orderly.network/docs/build-on-omnichain/evm-api/introduction)を使用してください（TypeScript、REST、WebSocket）。
- 「スワップをパープスポジションを通じてルーティングする」というプリミティブはありません。スポットとパープスの両方のエクスポージャーが必要なクライアントは、2 つのトランザクションを独立して構築します。
- Orderly のリスクパラメータ（レバレッジキャップ、清算しきい値、コラテラルリミット）は時間とともに変わります。ユーザーにリスクを見積もる前に、常に Orderly の API からライブパラメータを取得してください。
