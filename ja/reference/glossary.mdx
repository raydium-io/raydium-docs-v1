## A

**AcceleRaytor** — Raydium の 2021～2023 年のオリジナル トークンロンチパッド。LaunchLab に置き換わるとともに廃止されました。[`introduction/history-and-milestones`](/ja/introduction/history-and-milestones) を参照してください。

**Account** — Solana 上のデータ行。公開鍵で識別され、プログラムが所有し、データバイトとランポートを保有します。[`solana-fundamentals/account-model`](/ja/solana-fundamentals/account-model) を参照してください。

**Address Lookup Table (ALT)** — アカウントを 1 バイトのインデックスで参照できるようにする公開アカウント。バイト数を節約できます。Raydium SDK は複数ホップのルートに ALT を使用しています。[`solana-fundamentals/transactions-and-fees`](/ja/solana-fundamentals/transactions-and-fees) を参照してください。

**AmmConfig** — CPMM/CLMM の構成アカウント。フィー・ティアと手数料配分を定義します。複数の構成が同時に存在でき、プール作成時に 1 つを参照します。[`ray/protocol-fees`](/ja/ray/protocol-fees) を参照してください。

**AMM v4** — Raydium のオリジナル定積 AMM（2021 年）。当初は OpenBook ミラーリング設計でしたが、OpenBook 連携は無効化され、現在は純粋な定積 AMM として動作します。完全に機能していますが、新しいプールの推奨デフォルトではなくなりました。[`products/amm-v4/overview`](/ja/products/amm-v4/overview) を参照してください。

**Anchor** — Solana プログラムの Rust フレームワーク。アカウント検証、エラー処理を行い、IDL を生成します。CPMM、CLMM、Farm v6、LaunchLab で使用されています。[`solana-fundamentals/programs-and-anchor`](/ja/solana-fundamentals/programs-and-anchor) を参照してください。

**Anchor discriminator** — Anchor 管理アカウントと命令に付与される 8 バイトの接頭辞。アカウント名または命令名から派生します。アカウントの種類を識別します。[`solana-fundamentals/account-model`](/ja/solana-fundamentals/account-model) を参照してください。

**Arbitrage** — 異なるヴェニュー（Raydium ↔ Orca ↔ CEX）間の価格差を利用して、価格を揃える行為。AMM 価格発見の基本です。

**Associated Token Account (ATA)** — （ユーザー、ミント）ペアのための PDA 派生トークンアカウント。慣例的な方法です。[`solana-fundamentals/spl-token-and-token-2022`](/ja/solana-fundamentals/spl-token-and-token-2022) を参照してください。

## B

**Bonding curve** — トークンの価格設定関数。供給量の関数として価格を決定します。LaunchLab のグラデュエーション前に使用されます。[`products/launchlab/bonding-curve`](/ja/products/launchlab/bonding-curve) を参照してください。

**BPF Loader (Upgradable)** — Solana ローダー プログラム。プログラムをデプロイ、アップグレードします。[`solana-fundamentals/programs-and-anchor`](/ja/solana-fundamentals/programs-and-anchor) を参照してください。

**Bump** — PDA をオフカーブで派生させるための末尾に追加される単一バイト。正規 bump は、オフカーブ アドレスを生成する最大値です。[`solana-fundamentals/pdas-and-cpis`](/ja/solana-fundamentals/pdas-and-cpis) を参照してください。

## C

**Canonical bump** — 255 からデクリメントしてみつけた bump。Raydium のプログラムは常に正規 bump を使用します。[`solana-fundamentals/pdas-and-cpis`](/ja/solana-fundamentals/pdas-and-cpis) を参照してください。

**CLMM** — Concentrated Liquidity Market Maker。Raydium の 2022 年の Uniswap V3 スタイルのプロダクト。[`products/clmm/overview`](/ja/products/clmm/overview) を参照してください。

**Compute Budget** — トランザクションあたりのコンピュート ユニット制限。デフォルト 200k、最大 1.4M。`ComputeBudget::SetComputeUnitLimit` で設定します。[`solana-fundamentals/transactions-and-fees`](/ja/solana-fundamentals/transactions-and-fees) を参照してください。

**Compute Unit (CU)** — Solana の計算コストの単位。優先手数料は CU あたりに請求されます。[`integration-guides/priority-fee-tuning`](/ja/integration-guides/priority-fee-tuning) を参照してください。

**Constant product** — AMM 不変式 `x × y = k`。[`algorithms/constant-product`](/ja/algorithms/constant-product) を参照してください。

**CPI (Cross-Program Invocation)** — 単一トランザクション内で 1 つの Solana プログラムが別のプログラムを呼び出すこと。[`solana-fundamentals/pdas-and-cpis`](/ja/solana-fundamentals/pdas-and-cpis) を参照してください。

**CPMM** — Raydium の最新の定積 AMM（2024 年）。新しいプール向けに AMM v4 に置き換わります。[`products/cpmm/overview`](/ja/products/cpmm/overview) を参照してください。

**Creator fee (LaunchLab)** — ボンディング カーブ フェーズ中にスワップ手数料のオプション パーセンテージがトークン作成者にルーティングされます。[`products/launchlab/overview`](/ja/products/launchlab/overview) を参照してください。

## D

**DLMM** — Dynamic Liquidity Market Maker。Meteora の CLMM バリアント。Raydium ではありません。

**Discriminator** — 「Anchor discriminator」を参照してください。

## E

**Emissions (Farm)** — ファーム報酬の配布レート。トークン／秒で測定されます。[`products/farm-staking/overview`](/ja/products/farm-staking/overview) を参照してください。

**Escrow** — 条件が満たされるまで資金を保持する PDA 保有のトークンアカウント。一般的な統合パターンです。[`integration-guides/cpi-integration`](/ja/integration-guides/cpi-integration) を参照してください。

**Executable account** — `executable = true` を持つ Solana アカウント。プログラム バイトコードを保有します。[`solana-fundamentals/account-model`](/ja/solana-fundamentals/account-model) を参照してください。

## F

**Farm v3 / v5 / v6** — Raydium の報酬ファーミング プログラムの連続世代。v6 が最新版。v3/v5 はレガシーです。[`products/farm-staking/overview`](/ja/products/farm-staking/overview) を参照してください。

**Fee growth (CLMM)** — グローバル単位流動性あたりの手数料カウンタ。`fee_growth_global_X` と `fee_growth_global_Y` で追跡されます。[`products/clmm/math`](/ja/products/clmm/math) を参照してください。

**Fee tier** — プールが提供する特定のスワップ手数料の割合。例：CLMM 0.05% ティア。[`reference/fee-comparison`](/ja/reference/fee-comparison) を参照してください。

**Finalization** — Solana の最強のコミットメント レベル。トランザクション着陸の約 13 秒後。[`solana-fundamentals/transactions-and-fees`](/ja/solana-fundamentals/transactions-and-fees) を参照してください。

**Freeze authority** — 特定のミントに対するトークンアカウントをロック（凍結）する権限。アクティブな凍結権限を持つトークンに LP するユーザーのリスク。

## G

**Graduation (LaunchLab)** — ボンディング カーブ ローンチが閾値に到達し、自動的に CPMM プールにマイグレーションするイベント。[`products/launchlab/overview`](/ja/products/launchlab/overview) を参照してください。

## H

**Hybrid AMM** — 外部オーダーブックと統合し、より良い価格を提供する方を通じてトレードをルーティングする AMM。AMM v4 は元々ハイブリッド（OpenBook に対して）でしたが、その オーダーブック側は無効化されているため、AMM v4 は純粋な AMM として動作します。CPMM はハイブリッドではありません。

## I

**IDL** — Interface Description Language。Anchor のプログラムの命令、アカウント、イベント、エラーの JSON 記述。[`sdk-api/anchor-idl`](/ja/sdk-api/anchor-idl) を参照してください。

**Impermanent loss (IL)** — LP のトークン価値と基礎トークンの保有比較における差。価格の乖離が原因です。AMM の基本です。[`algorithms/constant-product`](/ja/algorithms/constant-product) を参照してください。

**Invoke / invoke_signed** — CPI 用の Solana システムコール。`invoke_signed` は PDA 認可の CPI に使用されます。[`solana-fundamentals/pdas-and-cpis`](/ja/solana-fundamentals/pdas-and-cpis) を参照してください。

## J

**Jito** — Solana ブロック エンジン。MEV 保護のためのバンドル トランザクションをサポートします。[`integration-guides/routing-and-mev`](/ja/integration-guides/routing-and-mev) を参照してください。

**Jupiter** — Solana の主流 DEX アグリゲーター。ボリュームの約 60% を Raydium 経由でルーティングします。[`introduction/ecosystem-position`](/ja/introduction/ecosystem-position) を参照してください。

## K

**Kudelski** — Raydium プログラムを監査した監査会社。[`security/audits`](/ja/security/audits) を参照してください。

## L

**Lamport** — 最小の SOL 単位。1 SOL = 1,000,000,000 lamports。

**LaunchLab** — Raydium のトークン ロンチ プラットフォーム（2024 年）。ボンディング カーブ ローンチが CPMM にグラデュエーション。[`products/launchlab/overview`](/ja/products/launchlab/overview) を参照してください。

**Liquidity L (CLMM)** — ポジションのプール価格設定への貢献。Uniswap V3 の数学で定義される単位です。[`algorithms/clmm-math`](/ja/algorithms/clmm-math) を参照してください。

**LP (Liquidity Provider)** — プールにトークンを預ける LP トークン（または CLMM の NFT）と、将来の手数料への請求権を交換するユーザー。

**LP token** — CPMM または AMM v4 プールの LP シェアを表す SPL トークン。CLMM は代わりに NFT を使用します。

## M

**MadShield** — Raydium プログラムを監査した監査会社。[`security/audits`](/ja/security/audits) を参照してください。

**Master-chef pattern** — SushiSwap の MasterChef コントラクトに基づく報酬追跡設計。`reward_per_share` アキュムレーター スタイル。Farm v6 で使用されます。

**Memecoin** — コミュニティ／投機駆動トークンの非公式な用語。多くは LaunchLab 経由でロンチされます。

**MEV (Maximal Extractable Value)** — トランザクション順序変更で抽出可能な価値（フロントランニング、サンドイッチ、バックランニング）。[`integration-guides/routing-and-mev`](/ja/integration-guides/routing-and-mev) を参照してください。

**Mint** — トークンを定義する Token Mint アカウント。「ミント」の動詞（新規供給の発行）と混同しないでください。

**Multisig** — 複数署名アカウント。M 署名者中 N 人が必要です。Raydium は権限に Squads マルチシグを使用します。[`security/admin-and-multisig`](/ja/security/admin-and-multisig) を参照してください。

## N

**Non-transferable (Token-2022)** — トークン転送を防ぐ拡張機能。Raydium プール作成でブロックされます。

**NFT** — Non-Fungible Token。CLMM ポジションは NFT。LaunchLab の成果物の一部も同様です。

## O

**Observation / ObservationState** — CLMM のオンチェーン価格履歴アカウント。時間加重の累積価格を保存。TWAP オラクルを有効にします。[`products/clmm/accounts`](/ja/products/clmm/accounts) を参照してください。

**OpenBook** — 旧称 Serum。Solana の中央集約型オーダーブック。AMM v4 は元々 OpenBook と統合していますが、その統合は無効化されています。CPMM と CLMM は統合していません。[`products/amm-v4/overview`](/ja/products/amm-v4/overview) を参照してください。

**Oracle** — 外部価格フィード。Pyth、Jupiter、および（内部使用）CLMM の ObservationState TWAP。[`security/oracle-and-token-risks`](/ja/security/oracle-and-token-risks) を参照してください。

**Orca** — ライバル Solana DEX。Whirlpools は Orca の CLMM プロダクト。

## P

**PDA (Program-Derived Address)** — プログラム＋シードから決定論的に派生したアドレス。秘密鍵がありません。所有プログラムが `invoke_signed` で署名します。[`solana-fundamentals/pdas-and-cpis`](/ja/solana-fundamentals/pdas-and-cpis) を参照してください。

**Permanent delegate (Token-2022)** — 指定ウォレットに無条件の転送／バーン権限を付与する拡張機能。Raydium プール作成でブロックされます。

**Phoenix** — ネイティブ Solana オーダーブック DEX。ピア向け。直接的な競合ではありません。

**PnL (AMM v4)** — 歴史的には：OpenBook フィルから得た損益。AMM v4 プールに蓄積され、`MonitorStep` クランク経由で決済されました。OpenBook 統合が無効化されたため、新しいオーダーブック PnL は蓄積されません。`state_data.total_pnl_*` カウンタは歴史値のみを保持します。`need_take_pnl_*` を使用する 0.03% プロトコル手数料パスは影響を受けず、まだアクティブです。

**Pool authority** — プール資産移動に署名する PDA。プログラムごとに 1 つ（プール ごとではありません）。[`solana-fundamentals/pdas-and-cpis`](/ja/solana-fundamentals/pdas-and-cpis) を参照してください。

**Pool state** — プール準備金、手数料、設定を保有するオンチェーン アカウント。プロダクトごとの中央データ構造。

**Priority fee** — CU あたりの手数料。バリデーターに支払い、ブロック内のトランザクション優先順位を上げます。[`integration-guides/priority-fee-tuning`](/ja/integration-guides/priority-fee-tuning) を参照してください。

**Program account** — プログラムのメタデータ／バイトコードを保有するアカウント。[`solana-fundamentals/programs-and-anchor`](/ja/solana-fundamentals/programs-and-anchor) を参照してください。

**Program Derived Address** — PDA を参照。

**Protocol fee** — スワップ手数料の一部。LP ではなく RAY バイバック または財務にルーティングされます。[`ray/protocol-fees`](/ja/ray/protocol-fees) を参照してください。

**Pyth** — Solana オラクル ネットワーク。フロントエンドの USD 表示に使用。Raydium プール数学では使用されません。

## Q

**Q64.64 (sqrt_price_x64)** — 固定小数点表現：64 ビット整数 + 64 ビット小数部。CLMM がルート価格に浮動小数点ドリフトを回避するために使用します。[`algorithms/clmm-math`](/ja/algorithms/clmm-math) を参照してください。

## R

**\$RAY** — Raydium のネイティブ SPL トークン。[`ray/index`](/ja/ray) を参照してください。

**Realized fee (CLMM)** — 最後の `collectFee` 以降にポジションに蓄積された手数料。`collectFee` で請求されます。[`products/clmm/math`](/ja/products/clmm/math) を参照してください。

**Remaining accounts** — 命令のアカウント リストの末尾に追加される追加アカウント。位置で解釈されます。可変長アカウント リスト（ティック配列、転送フック エクストラ）に使用されます。[`solana-fundamentals/pdas-and-cpis`](/ja/solana-fundamentals/pdas-and-cpis) を参照してください。

**Rent** — Solana はストレージ レント を請求しますが、すべてのモダン アカウントはレント エクセンプション（レント永遠前払いに十分なランポート保有）である必要があります。[`solana-fundamentals/account-model`](/ja/solana-fundamentals/account-model) を参照してください。

## S

**Sandwich attack** — MEV 戦術：ユーザーのスワップをフロントラン、その後バックランで価格インパクトを抽出。[`integration-guides/routing-and-mev`](/ja/integration-guides/routing-and-mev) を参照してください。

**Serum** — OpenBook の前身。現在アクティブではありません。「OpenBook」を参照してください。

**Slippage** — 提示と実際の出力の差。価格インパクト＋価格変動から。スワップの `min_out` パラメータで制御します。

**SOL** — Solana のネイティブ トークン。

**SPL Token** — レガシー Solana トークン プログラム。アドレス `Tokenkeg...`。[`solana-fundamentals/spl-token-and-token-2022`](/ja/solana-fundamentals/spl-token-and-token-2022) を参照してください。

**sqrt_price_x64** — Q64.64 を参照。

**Squads** — Raydium が運用およびアップグレード権限に使用するマルチシグ プラットフォーム。[`security/admin-and-multisig`](/ja/security/admin-and-multisig) を参照してください。

**Swap / SwapV2** — Raydium 命令バリアント。`SwapV2` は Token-2022 をサポート。レガシー `Swap` はサポートしません。[`solana-fundamentals/spl-token-and-token-2022`](/ja/solana-fundamentals/spl-token-and-token-2022) を参照してください。

## T

**Tick (CLMM)** — 離散価格レベル。ポジションはティック範囲に及びます。[`algorithms/clmm-math`](/ja/algorithms/clmm-math) を参照してください。

**Tick array** — 60 連続ティックの状態を保有する PDA。ティック格子の分割ストレージを有効にします。[`products/clmm/ticks-and-positions`](/ja/products/clmm/ticks-and-positions) を参照してください。

**Tick spacing** — プール内の使用可能ティック間の整数ギャップ。フィー ティアに結び付きます（0.01% → 1、0.25% → 60 など）。[`algorithms/clmm-math`](/ja/algorithms/clmm-math) を参照してください。

**Timelock** — 機密アクション承認から実行までの遅延。Raydium アップグレードは 24 時間のタイムロックを備えています。[`security/admin-and-multisig`](/ja/security/admin-and-multisig) を参照してください。

**Token-2022** — 拡張機能を備えた新しいトークン プログラム。アドレス `TokenzQ...`。[`solana-fundamentals/spl-token-and-token-2022`](/ja/solana-fundamentals/spl-token-and-token-2022) を参照してください。

**Transfer fee (Token-2022)** — すべてのトークン転送に手数料を請求する拡張機能。Raydium CPMM/CLMM で `SwapV2` 経由で考慮されます。

**Transfer hook (Token-2022)** — 転送のたびにプログラムを呼び出す拡張機能。Raydium でサポートされていますが注意事項があります。

**TVL (Total Value Locked)** — プール／ファーム残高の USD 合計。Raydium の TVL（2026 年 4 月時点）：約 \$1.8B。

**TWAP (Time-Weighted Average Price)** — 時間で加重された平均価格。CLMM ObservationState から派生します。[`products/clmm/accounts`](/ja/products/clmm/accounts) を参照してください。

## U

**UserLedger (Farm v6)** — ファーム内のユーザーのステーク残高と報酬負債を格納する PDA。

## V

**Vault** — プール プログラムの PDA 権限が所有するトークンアカウント。プール準備金を保有します。各プールは 2 つの資金保管庫（片側各 1 つ）を持ちます。

**Versioned transaction (v0)** — ALT をサポートする現在の Solana トランザクション形式。レガシー トランザクションはバージョン 0xff です。[`solana-fundamentals/transactions-and-fees`](/ja/solana-fundamentals/transactions-and-fees) を参照してください。

## W

**Whirlpools** — Orca の CLMM プロダクト。Raydium CLMM に類似した設計。異なるコードベース。
