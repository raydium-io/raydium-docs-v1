## Raydium 対応ウォレットに必要な機能

ウォレットが以下の機能をすべて備えていれば、Raydium 対応ウォレットです：

- Solana キーペアをローカルに保管し、**決して**秘密鍵をデバイス外に送信しない。
- **バージョン管理トランザクション（V0）に署名できる。** Raydium スワップはアドレスルックアップテーブルを使用しており、V0 対応が必須です。レガシートランザクションにしか署名できないウォレットは、Raydium スワップを完了できません。
- **Wallet Adapter 標準経由で dApps に接続できる。** これが raydium.io の「ウォレットを接続」ボタンを動作させています。
- **署名前に明確なトランザクションプレビューを表示できる**（呼び出されるプログラムとトークンの移動を含む）。
- **関連トークンアカウント**（新しいミントを初めて受け取る際に ATA を作成）に対応している。

これらはすべてのモダン Solana ウォレットが満たすべき基本要件です。これらを満たさないウォレットで Raydium トランザクションに署名することは危険です。

## 基本要件を超えた重要な機能

ウォレットが基本要件を満たしたら、違いはあなたのワークフローとリスク許容度によって決まります。以下の機能のうち、**あなたにとって**重要なものを決めて、それに対応したウォレットを選びます：

| 機能 | 重要になる場面 |
|---|---|
| **ハードウェアウォレット ペアリング** | かなりの金額を保有している場合。ソフトウェアウォレットはトランザクションビルダーになり、ハードウェアデバイスが鍵を保持して署名します。ほとんどの信頼できる Solana ウォレットは少なくとも Ledger とペアリングでき、中には Keystone や SafePal などのデバイスとペアリングするものもあります。 |
| **モバイル dApp ブラウザ** | 主にスマートフォンから Raydium を使用する予定がある場合。ウォレットの内蔵ブラウザが raydium.io を読み込み、モバイルでトランザクションに署名します。 |
| **複数アカウント管理** | ホット / コールド / 実験的ウォレット全体で資金を分離し、拡張機能を切り替えることなく素早く切り替えたい場合。 |
| **Token-2022 拡張機能への対応** | Token-2022 ミント（例：転送手数料トークン）を使用するプールと相互作用する予定がある場合。ウォレットがサイニング UI でこれらの拡張機能を表示すれば、署名前に予期しない手数料を発見できます。 |
| **優先手数料の表示** | 混雑時にトレードを行い、承認前にウォレットが設定したコンピュート予算価格を確認したい場合。 |
| **Solana Actions / Blinks 対応** | Raydium がソーシャルメディアで公開しているワンクリックスワップリンクを使用したい場合。[`user-flows/referrals-and-blinks`](/ja/user-flows/referrals-and-blinks) を参照してください。 |
| **オープンソースコードベース** | ウォレットクライアントを監査したい（またはコミュニティが監査したことに依存したい）場合。 |
| **マルチチェーン対応** | Solana 以外のチェーンに資産を保有しており、すべてに1つのウォレットを使用することを希望する場合。マルチチェーン対応が実際に使用するチェーンで成熟していることを確認してください。対応状況はチェーンごとに異なります。 |

## 人気の Solana ウォレット

以下は、アクティブな Solana サポートを備えた有名なウォレットプロジェクトです。アルファベット順に列挙されており、**掲載は情報提供のためのものであり、推奨ではありません**。リストは網羅的ではありません。インストール前に公式 URL を確認してください。フィッシングクローンは定期的に本物のドメイン名に対してスポンサード広告を購入しています。

### ソフトウェアウォレット

| ウォレット | 動作環境 | 公式サイト |
|---|---|---|
| Phantom | ブラウザ拡張機能、iOS、Android | [phantom.app](https://phantom.app) |
| Solflare | ブラウザ拡張機能、iOS、Android、Web | [solflare.com](https://solflare.com) |
| OKX Wallet | ブラウザ拡張機能、iOS、Android | [okx.com/wallet](https://www.okx.com/wallet) |
| Backpack | ブラウザ拡張機能、iOS、Android、デスクトップ | [backpack.app](https://backpack.app) |

この4つはすべてノンカストディアルで、Wallet Adapter に対応し、V0 トランザクションに署名でき、少なくとも1つのメインストリームハードウェアデバイスとペアリングできます。それ以降は、違いはワークフロー固有のものです。前のセクションの基準を使用して、どれがあなたに適しているかを決めてください。

### ハードウェアウォレット

ハードウェアウォレットは、上記のソフトウェアウォレットのいずれかとペアリングします（ソフトウェア側はトランザクションビルダーになり、ハードウェアデバイスが鍵を保持して署名します）。

| デバイス | タイプ | 公式サイト |
|---|---|---|
| Ledger | USB / Bluetooth | [ledger.com](https://www.ledger.com) |
| Keystone | エアギャップ（QR コード署名） | [keyst.one](https://keyst.one) |

各ソフトウェアウォレットは、それがサポートするハードウェアデバイスを文書化しています。購入前に確認してください。

## ホット / コールドウォレットパターン

選択したウォレットに関係なく機能する一般的なプラクティス：

- **ホットウォレット**には、失われる可能性があることを受け入れる小額を保有します。日常的なスワップと試行用です。
- **コールドウォレット**（ハードウェアデバイスとペアリング）には、失われると困る金額を保有します。ホットウォレットで実際に必要な場合のみ資金を移動します。

Raydium は各ウォレットを完全に独立したものとして扱います：ポジション、LP トークン、ファーム報酬、履歴がそれぞれ独立しています。ウォレットの切り替えは1クリックです。

## ウォレットオプションを見つける場所

Raydium UI の「ウォレットを接続」モーダルには、Wallet Adapter 経由で Raydium に統合したすべてのウォレットが表示されます。それを参照して、現在サポートされているものを確認してください。Solana の公式ウォレットディレクトリ（[`solana.com/ecosystem/explore?categories=wallet`](https://solana.com/ecosystem/explore?categories=wallet)）は、もう1つの中立的な出発点です。

ウォレットを評価するときは、以下のチェックリストを確認してください：

- [ ] メンテナンス中：最近のリリース、アクティブな問題トラッカー、セキュリティ開示チャネル。
- [ ] レビュー済み：信頼できる企業による公開監査、またはアクティブなコミュニティのあるオープンソースコード。
- [ ] V0 トランザクション対応が確認済み（実際の資金を移動させる前に devnet でテストしてください）。
- [ ] ハードウェアデバイスの使用を予定している場合：ウォレットがサポートするデバイスと、ペアリングの方法が文書化されている。
- [ ] スポンサード広告や転送リンクではなく、プロジェクトの検証済みドメイン経由で公式ダウンロードを見つけた。

## ウォレットセキュリティの基本

選択したウォレットに関係なく：

- **シードフレーズを紙に書き留めます。** その紙を物理的に異なる2つの場所に保管します。**決して**シードフレーズをクリーンインストール時のウォレット復旧を除いてどこにも入力しないでください。
- **ウォレットのロック解除パスワードにはパスワードマネージャーを使用し**、そのパスワードをシードフレーズとは完全に分離して保管します。
- **暗号資産アクティビティ用に専用ブラウザプロファイルを使用し**、インストールする拡張機能を最小限にします。フィッシング拡張機能は最大の流出ベクトルです。
- **すべてのウォレット接続プロンプトを疑ってかかってください。** 署名しようとしていることを読みます。トランザクションが特定の金額ではなくトークン全体を転送する場合は、拒否します。
- **`raydium.io` をブックマークします。** 検索エンジンの広告からではなく、常にブックマークから移動します。

より詳しいセキュリティガイダンスは[`getting-started/trust-and-safety`](/ja/getting-started/trust-and-safety)を参照してください。
