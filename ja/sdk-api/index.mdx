## このセクションの対象者

オフチェーン（TS/JS、Python、バックエンド）から Raydium と統合する開発者、または別の Solana プログラム（Rust CPI）から統合する開発者向けです。また、AI エージェントおよびコーディングツール統合者向けです。詳しくは [AI 統合](/ja/sdk-api/ai-integration) をご覧ください。

<Info>
  **何か実行してみたいですか？** スワップ、CPMM プール のデプロイ、CLMM プール のデプロイの 3 つのコピー＆ペーストスクリプトは [クイックスタート](/ja/quick-start) にあります。5 分で実際のトランザクションを実行できます。その後、こちらで完全なリファレンスをご覧ください。
</Info>

## 統合パスを選択する

5 つのパスから選択できます。どのパスが最適かは、Raydium を呼び出すものと必要なコントロール量によって異なります。

| パス | 最適な用途 | レイテンシー | カスタムロジック | 言語 |
|---|---|---|---|---|
| [Trade API](/ja/sdk-api/trade-api) | バックエンド、ボット、Blink—サーバーで構築されたトランザクションが必要な場所 | 最低 | 制限あり（プリセットルート） | 任意 |
| [TypeScript SDK](/ja/sdk-api/typescript-sdk) | フルビルダーアクセスが必要な Node/TS アプリとフロントエンド | 中程度 | フル | TS / JS |
| [Python 統合](/ja/sdk-api/python-integration) | Python バックエンドと ML パイプライン | 中程度 | フル（手動命令構築） | Python |
| [Rust CPI](/ja/sdk-api/rust-cpi) | Raydium をオンチェーンから呼び出すプログラムの構成 | N/A（オンチェーン） | フル + コンポーザブル | Rust |
| [Anchor IDL](/ja/sdk-api/anchor-idl) | ファーストパーティサポートのない言語（Go、Kotlin、Swift など）向けのコード生成 | 可変 | フル | Anchor コード生成ツール付きの任意の言語 |

判断がつかない場合：バックエンドとボット → **Trade API**、フロントエンドとフルコントロールアプリ → **TypeScript SDK**、オンチェーン構成 → **Rust CPI**。

## バージョン管理

このセクション内のすべてのページのすべてのコードサンプルの上部には、次のようなバナーがあります：

```
SDK:       @raydium-io/raydium-sdk-v2 vX.Y.Z
Program:   see /reference/program-addresses
Cluster:   mainnet-beta
Verified:  YYYY-MM-DD
```

コードサンプルを変更する場合は、バナーを更新してください。
