## フロー概要

```text
スワップ手数料
  -> LP シェア
  -> RAY バイバック シェア
  -> トレジャリー シェア（該当する場合）
```

CLMMおよびCPMMプールの場合、取引手数料の分割は以下の通りです：

```text
84% を LP に
12% を RAY バイバック に
4% をトレジャリーに
```

Standard AMM v4プールの場合、取引手数料の分割は以下の通りです：

```text
88% を LP に
12% を RAY バイバック に
```

## バイバックシェアの意味

バイバック率は**取引手数料**に適用され、取引全体の金額ではありません。

例：プールが`0.25%`のスワップ手数料を請求します。バイバックシェアはその手数料の`12%`です。

```text
取引額：           1,000 USDC
スワップ手数料：   1,000 * 0.25% = 2.50 USDC
RAY バイバック：   2.50 * 12% = 0.30 USDC
```

この例では、実質的なバイバック額は取引額の`0.03%`です。

## 回収アドレス

プロトコル側の手数料は、変換またはルーティングされる前に、プログラム固有のアドレスで回収されます。

| ソース | 回収アドレス |
|---|---|
| CLMM | `projjosVCPQH49d5em7VYS7fJZzaqKixqKtus7yk416` |
| CPMM | `ProCXqRcXJjoUd1RNoo28bSizAA6EEqt9wURZYPDc5u` |
| Standard AMM v4 | `PNLCQcVCD26aC7ZWgRyr5ptfaR7bBrWdTFgRWwu2tvF` |

回収アドレスは多くの異なるトークンを保有できます。各プールは取引するトークンで手数料を支払うため、回収アドレスはルーティング前にUSDC、wSOL、RAYなどのSPLトークンを保有する可能性があります。

## バイバックがオンチェーンである理由

Raydiumのバイバックフローは公開アドレスとトランザクション履歴から監査可能です。

- **残高が可視化される：** 回収アドレスと保有アドレスは誰でも検査できます。
- **転送は追跡可能：** 着信手数料転送と発信ルーティングトランザクションは公開されています。
- **プライベート会計は不要：** チェーンがフローの台帳です。

## バイバックフローを検証する方法

1. Solscan または Solana Explorer で Raydium スワップトランザクションを開きます。
2. プールプログラムと手数料転送を特定します。
3. プロトコル側の手数料の動きを関連する回収アドレスと照合します。
4. 回収アドレスからの発信トランザクションを検査します。
5. `DdHDoz...VEZaz` での RAY 蓄積を確認します。

完全なアドレスマップについては、[`ray/treasury`](/ja/ray/treasury)を参照してください。
