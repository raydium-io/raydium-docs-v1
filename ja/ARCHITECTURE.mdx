このページは、Raydium ドキュメンテーション・セットがいかに組織されているかの**信頼できる情報源**です。任意のチャプターの詳細なコンテンツを執筆する前に、このドキュメントを参照してください。これはオーディエンス・トラック、チャプター・バウンダリー、保つべき一貫性のある相互参照、ディスク上のディレクトリ・レイアウトを定義しています。

## デザイン原則

1. **1 つのトピックは 1 つの家。** 1 つの概念は正確に 1 つのチャプターで定義され、他からリンクされます。フィーの場合、製品ごとに定義されますが（各製品は独自のフィー・モデルを持つ）、比較フィー・テーブルは `reference/fee-comparison` にあります。
2. **段階的な深さ。** 各製品チャプターは 1 段落の平易英語の概要で始まり、その後、アカウント → 数学 → インストラクション → フィー → コードに進みます。読者はどの深さでも止めることができます。
3. **実行可能なコードはいたるところに。** すべてのインストラクション・レベルのセクションは、最小限のコピー・ペーストできる TS・Rust・Python の例で終わります。長いデモは各製品の `code-demos.mdx` の下にあるため、ナレーティブはコードに埋もれません。
4. **バージョン固定。** コードを表示するページでは、SDK バージョン、プログラム ID、Solana クラスターが名前を記します。古い例は DeFi ドキュメンテーションの最大の失敗モードです。
5. **「どのように機能するか」を「どのように使うか」から分離。** アルゴリズムは独自のチャプターにあります。ユーザーフロー自体のチャプターにあります。製品チャプターは両方の方法にリンクします。
6. **セキュリティは最上級のチャプター、脚注ではない。** 監査人は製品チャプターで検索する必要がありません。
7. **オーディエンス・セグメント化されたユーザーフロー。** エンドユーザー向けウォークスルーはリーダー・ロール（トレーダー、流動性プロバイダー、トークン・クリエイター）でグループ化されます。プログラムやフィーチャーではなく。

## プロダクト・テンプレート

プロダクト・ラインを比較しやすくするため、すべてのプロダクト・フォルダーは同じ形式に従います。オンチェーン・アカウントを公開しないプログラム（例：Orderly Network 上のホワイトラベル・バックエンドであるパーペチュアル）は同等のサーフェス固有のファイルを代替します。

| ファイル | 目的 | 必須 |
|------|---------|--------------|
| `index.mdx` | チャプター・ランディング：1 段落の紹介、使用時期、サブページへのリンク、キー・プログラム ID。 | すべて |
| `overview.mdx` | 概念モデル — このプロダクトが何をする、なぜするか、図付き。 | オンチェーン・プロダクト |
| `accounts.mdx` | すべてのオンチェーン・アカウント：PDA シード、フィールド・レイアウト、不変条件、IDL リファレンス。 | オンチェーン・プロダクト |
| `math.mdx` | 不変条件、価格設定式、数値例。`algorithms/` への相互リンク。 | オンチェーン・プロダクト |
| `instructions.mdx` | 完全なインストラクション・リファレンス：引数、アカウント・リスト、事前・事後条件、エラー・コード。 | オンチェーン・プロダクト |
| `fees.mdx` | フィー・ティア、フィー・スプリット（LP・プロトコル・ファンド・クリエイター）。CLMM の場合、報酬エミッション（CLMM の報酬システムがポジションごとのフィー・グロース・アカウンティングを再利用するため）。 | すべて |
| `code-demos.mdx` | プロダクトが公開するすべてのフロー用のコピー・ペーストできる TypeScript・Rust 例。 | すべて |

標準テンプレートで十分でない箇所のプロダクト拡張：

- **CLMM** は `ticks-and-positions.mdx` を追加します。ティック数学が `math.mdx` に埋め込むには大きすぎるため。
- **LaunchLab** は `bonding-curve.mdx`、`creator-fees.mdx`、`global-config.mdx`、`platform-config.mdx`、`platforms.mdx`、`vesting.mdx`、および `tips-and-gotchas/` サブフォルダーを追加します。
- **パーペチュアル** は `trading-basics.mdx`、`order-types.mdx`、`collateral.mdx`、`fees.mdx`、`code-demos.mdx` を使用します — `accounts.mdx`・`math.mdx`・`instructions.mdx` はなし（ベニューがオフチェーン（Orderly Network）で Solana 側ラッパーがあるため）。

## クロス・チャプター規約

- **プログラム ID とクラスター** は常に `reference/program-addresses.mdx` で定義されます。他のページは値をハードコードするのではなくリンクします。
- **エラー・コード** は常に `reference/error-codes.mdx` で定義されます。インストラクション・ページは特定のアンカーにリンクします。
- **数学** は `algorithms/` にあります。プロダクト `math.mdx` ページはプロダクト固有のインスタンシエーションを示し、リンク・バック。
- **フィー経済** は 2 つの家に分割されます。意図により：プロダクト固有の仕組みは `products/<x>/fees.mdx`、プロトコル・レベニュー・RAY 買い戻しフロー は `ray/`。クロス・プロダクト比較テーブルは `reference/fee-comparison`。
- **RAY トークノミクス** は `ray/` にあります。RAY を言及する他のページ（ステーキング・フロー、フィー・スプリット、ガバナンス）はリンクします。
- **コード・バージョニング。** すべてのコード・ブロックにはバージョン・バナー：SDK バージョン、プログラム ID、Solana クラスター、最後の検証日。
- **ダイアグラム。** アーキテクチャ・ダイアグラムは `/images/architecture/` にチェックイン済みの SVG です。ソース（Mermaid または draw.io）は SVG の隣にコミットされます。
- **ユーザーフローはオーディエンス・タグ付けされ、フィーチャー・タグ付けではない。** ウォークスルーは「トレーダー向け・流動性プロバイダー向け・トークン・クリエイター向け」の下に属します — タッチするプログラムの下ではなく。

## マルチランゲージ

デフォルト・ロケール（英語）はリポジトリ・ルートにパス・プリフィックスなしで存在します。各非英語ロケールは自身のトップレベル・フォルダー（`zh/`、`zh-Hant/`、`ja/`、`ko/`、`ru/`、`es/`、`de/`、`fr/`）の下にあり、英語ツリーをミラーリングします。

- **コンテンツ（MDX）：** Mintlify の自動翻訳により同期保持。新規・変更英語ページは自動的に翻訳され、各ロケール・フォルダーにプッシュされます。
- **ナビゲーション（`docs.json`）：** 自動翻訳**されません**。各言語は `navigation.languages` の下に独自の `tabs` ブロックを持ち、グループ・ラベル、タブ名、ページ・リストは英語ナビが構造的に変更されるたびに手で更新される必要があります。
- **タブ・グループ・ラベル** はロケール自身の言語です。**ページ・パス** はロケール・コードでプリフィックス付けられます（例：`zh/products/cpmm/overview`）。
- **レガシー URL 処理。** `/en/<path>` は `/<path>` にリダイレクトされます（`docs.json` `redirects` で設定）。ページが移動またはリネームされるとき、自動翻訳機に頼るのではなくリダイレクト・エントリーを追加してください。

## 明確に範囲外

以下は**この**ドキュメンテーション・セットでカバーされておらず、ユーザーが社内で要求する場合は断るべきです：

- 取引戦略・市場アドバイス。
- トークン価格予測。
- コードを読むのに必要な以上の詳細トークノミクス・モデリング。
- シークレットまたは管理手順を含む内部ランブック — 非公開リポジトリに存在します。

## ステータス

これはアーキテクチャの v0.2 です。v0.1 との比較：

- 古い `operations/` チャプター（DAO、財務、フィー・パラメータ、緊急手順）は `ray/` に置き換えられ、RAY トークノミクスとプロトコル・レベニューの周囲にスコープされ、**セキュリティ・オペレーション** タブに移動されました。
- 3 つの新しいプロダクト・ライン — `perps/`、`stable/`、`routing/` — は `products/` の下で最上級エントリー。
- `getting-started/`、`quick-start/`、`resources/`、`api-reference/` は現在最上級チャプター。
- `user-flows/` はナビゲーション内でオーディエンス・ロール（トレーダー・流動性プロバイダー・トークン・クリエイター）でリエンジニアリングされました。フィーチャーではなく。
- LaunchLab は専用の **ローンチ・ツール** タブと運用上のエッジケース向け `tips-and-gotchas/` サブフォルダーを取得しました。
- マルチランゲージ・サポートは正式化されました：英語はリポジトリ・ルートのデフォルト、8 つのロケール・ミラーはコンテンツの Mintlify 自動翻訳およびナビゲーションは手で保持。

リーフ・ページ・オーサリングはチャプターごとに続行されます。スケーフォルディングとして存在するスタブはその前マテリアルで `status: draft` ノートでマークされます。
