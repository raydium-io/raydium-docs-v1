이 페이지는 참고 표입니다. "풀 X에서 스왑하는 데 드는 비용이 얼마인가?" 또는 "CLMM 풀을 만드는 데 SOL이 얼마나 필요한가?"라는 질문에 답하려면 각 상품 장을 자세히 읽을 필요 없이 이 표를 활용하세요. 프로토콜 수준의 수수료 분배에 대해서는 [`ray/protocol-fees`](/ko/ray/protocol-fees)를 참조합니다.

## 상품별 스왑 수수료

수수료 값은 입력 금액의 백분율이며, 풀 계산 전에 적용됩니다.

| 상품 | 사용 가능한 등급 | 기본 / 가장 많이 사용 | 분배 |
|---------|-----------------|---------------------|-------|
| **AMM v4** | 0.25% (유일) | 0.25% | 88% LP, 12% 프로토콜 (25 중 22/3) |
| **CPMM** | 0.01%, 0.25%, 1% | 0.25% | 100% LP (프로토콜 지분은 AmmConfig별로 조정 가능, 현재 0%) |
| **CLMM** | 0.01%, 0.05%, 0.25%, 1% | 0.25% | 88% LP, 12% 프로토콜 |
| **LaunchLab (사전 졸업)** | 1% (기본값) | 1% | 50% 풀 시드, 50% LaunchLab 재무부 |
| **LaunchLab (사후 졸업 CPMM)** | 0.25% | 0.25% | CPMM 분배 상속 |

### 수수료 분모 규약

- **AMM v4**: 수수료는 x/10,000으로 인코딩됩니다. "25"는 0.25%를 의미합니다.
- **CPMM/CLMM**: 수수료는 x/1,000,000으로 인코딩됩니다. "2500"은 0.25%를 의미합니다.

온체인 데이터를 직접 읽을 때는 풀 프로그램에 맞는 분모를 사용하세요.

## 프로토콜 수수료 분배

각 수수료의 "프로토콜" 지분 내에서 추가 분배가 적용됩니다:

| 상품 | 스왑 수수료 중 프로토콜 지분 | 프로토콜 내 분배 |
|---------|----------------------------|------------------------|
| AMM v4 | 12% (3/25) | 100% 재무부 |
| CPMM | AmmConfig별로 조정 가능 (현재 0%) | 해당 없음 |
| CLMM | 스왑 수수료의 12% | 100% 재무부 |
| LaunchLab | 졸업 전 스왑 수수료의 50% | 100% LaunchLab 재무부 |

**LP 지분**은 LP 토큰 보유자에게 자동으로 흐릅니다:

- AMM v4 / CPMM: 풀 리저브에 누적되고 출금 시 비례적으로 분배됩니다.
- CLMM: `fee_growth_global_X/Y`에 누적되고 `collectFee`를 통해 청구 가능합니다.

## 생성 비용

풀을 배포하기 위해 SOL로 지불하는 일회성 비용입니다.

| 상품 | 렌트 비용 | 생성 수수료 | 일반적인 총액 |
|---------|-----------|--------------|---------------|
| AMM v4 | ~0.15 SOL (여러 계정) | 0.2 SOL | ~0.35 SOL |
| CPMM | ~0.04 SOL (풀 + 금고 + 관찰) | 0.15 SOL | ~0.19 SOL |
| CLMM | ~0.075 SOL (풀 + 관찰 + 초기 틱 배열) | 0 SOL | ~0.075 SOL |
| Farm v6 | ~0.02 SOL (팜 상태 + 보상 금고) | 0.1 SOL (스팸 방지) | ~0.12 SOL |
| LaunchLab 런칭 | ~0.015 SOL (본딩 곡선 계정) | 0.1 SOL (LaunchLab 수수료) | ~0.115 SOL |

모든 렌트는 회수 가능합니다 — 풀/포지션/팜을 닫으면 람포트가 반환됩니다.

## 졸업 / 마이그레이션 비용

| 이벤트 | 비용 | 지불자 |
|-------|------|----------|
| LaunchLab → CPMM 졸업 | ~0.04 SOL (새 CPMM 풀 생성) | 본딩 곡선 리저브에서 지불 |
| AMM v4 → CPMM 마이그레이션 | ~0.04 SOL CPMM 생성 + ~0.000005 SOL 트랜잭션 수수료 | 마이그레이션을 수행하는 LP |
| CLMM 포지션 NFT 이체 | ~0.000005 SOL 트랜잭션 수수료 | 이체자 |

## 일반적인 작업별 네트워크 수수료

이는 Solana 기본 + 우선 순위 수수료이며, Raydium 프로토콜 수수료와는 별개입니다.

| 작업 | CU | 기본 수수료 | 우선순위 수수료 (10k µL/CU) | 총 네트워크 |
|-----------|-----|----------|---------------------------|---------------|
| CPMM 스왑 | ~140,000 | 0.000005 SOL | 0.0014 SOL | ~0.0015 SOL |
| CLMM 스왑 (틱 없음) | ~170,000 | 0.000005 SOL | 0.0017 SOL | ~0.0017 SOL |
| CLMM 스왑 (4개 틱 교차) | ~320,000 | 0.000005 SOL | 0.0032 SOL | ~0.0032 SOL |
| CLMM 포지션 오픈 | ~280,000 | 0.000005 SOL | 0.0028 SOL | ~0.0028 SOL |
| Farm v6 스테이킹 | ~130,000 | 0.000005 SOL | 0.0013 SOL | ~0.0013 SOL |
| 팜 보상 청구 | ~160,000 | 0.000005 SOL | 0.0016 SOL | ~0.0016 SOL |
| v4 → CPMM 마이그레이션 (2개 트랜잭션) | ~200,000 + ~200,000 | 0.00001 SOL | 0.004 SOL | ~0.004 SOL |

우선순위 수수료는 CU 가격에 따라 선형으로 스케일됩니다. [`integration-guides/priority-fee-tuning`](/ko/integration-guides/priority-fee-tuning)을 참조하세요.

## Token-2022 수수료 상호작용

풀의 한쪽 또는 양쪽이 이체 수수료가 있는 Token-2022를 사용하는 경우, 효과적인 수수료가 복합됩니다:

```
net input to pool   = amount_in × (1 − transfer_fee_in)
pool charges swap   = net × swap_fee_rate
net output to user  = out × (1 − transfer_fee_out)
```

예: CPMM 0.25% 풀, 입력측 1% 이체 수수료, 출력측 0.5% 이체 수수료.

```
amount_in        = 1000
net_to_pool      = 1000 × 0.99    = 990
pool swap fee    = 990 × 0.0025   = 2.475 (LP 수수료로 유지)
amount_after_fee = 987.525
amount_out (AMM) = 987.525 × ratio, say 987.525 × 0.98 = 967.77
net_out_to_user  = 967.77 × 0.995 = 962.94
```

효과적인 수수료: **~3.7%** (명목 0.25% 풀 수수료에도 불구하고).

SDK는 `getComputeAmountOut`을 통해 효과적인 요율을 계산합니다 — Token-2022 풀을 견적할 때는 항상 이를 사용하세요.

## LaunchLab 상세 정보

| 단계 | 수수료 |
|-------|-----|
| 본딩 곡선 거래 (구매 또는 판매) | 1% (50/50 풀/재무부 분배) |
| 졸업 (일회성) | 곡선 리저브에서 ~0.04 SOL |
| 사후 졸업 (CPMM 풀) | 0.25% (표준 CPMM 수수료) |

토큰을 런칭하는 프로젝트는 종종 본딩 곡선 단계에서 LP 지분 중에 가져가는 "크리에이터 수수료" 옵션 (0–0.3%)을 포함합니다. 런칭 시 조정 가능합니다. 크리에이터 수수료는 **본딩 곡선 전용**입니다 — 런칭이 CPMM 풀로 졸업되면, 결과 풀은 크리에이터 추가 비용 없이 표준 CPMM 등급에서 실행됩니다. 전체 라이프사이클 및 크리에이터가 LaunchLab NFT 메커니즘을 통해 졸업된 풀에서 계속 수익을 창출하는 방법은 [`products/launchlab/creator-fees`](/ko/products/launchlab/creator-fees)를 참조하세요.

## Farm v6 보상 비용 모델

팜은 엄밀히 말해 수수료를 부과하지 않습니다 — 보상 분배입니다. 하지만 운영 비용이 드는 SOL이 있습니다:

| 비용 | 일반적인 값 |
|------|---------|
| 팜 생성 | 0.1 SOL 스팸 방지 + ~0.02 SOL 렌트 |
| 보상 금고 충전 | 토큰 + ~0.0005 SOL 트랜잭션 수수료만 해당 |
| 미사용 보상 회수 | ~0.0005 SOL 트랜잭션 수수료 |

팜은 또한 팜의 수명 동안 렌트를 충당할 수 있는 충분한 람포트가 보상 금고에 필요합니다. 종료하면 반환됩니다.

## 온체인에서 현재 값을 읽는 방법

### CPMM/CLMM AmmConfig

```ts
const config = await raydium.cpmm.getAmmConfigs();
console.log(config[0]);
// {
//   id: "...",
//   tradeFeeRate: 2500,         // 2500 / 1_000_000 = 0.25%
//   protocolFeeRate: 0,         // 0 / 1_000_000 = 0%
//   fundFeeRate: 0,
//   ...
// }
```

### AMM v4 풀

```ts
const poolInfo = await raydium.liquidity.getPoolInfoFromRpc({ poolId });
console.log(poolInfo.ammFeesNumerator, poolInfo.ammFeesDenominator);
// 25, 10000 → 0.25%
```

### Farm v6 보상 요율

```ts
const farm = await raydium.farm.getFarmInfoFromRpc({ id: farmId });
console.log(farm.farmData.rewardInfos);
// Per-reward: emissionsPerSecond, startTime, endTime
```
