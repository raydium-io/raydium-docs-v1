## 얻을 수 있는 것

- 선택한 목표 토큰과 **당신의 지갑을 레퍼럴 수취자로** 미리 채운 Raydium 스왑 UI의 공유 가능한 URL입니다.
- 누구든지 링크가 세션에서 활성화된 상태에서 스왑을 실행하면 **입력 거래량의 1%이 스왑 거래의 일부로 레퍼러의 지갑으로 라우팅됩니다**.
- X.com에서 공유된 링크는 뷰어의 지갑이 Actions / 블링크를 활성화했을 때 **블링크** (Solana Actions)로 렌더링됩니다. 게시물 내 원클릭 스왑 위젯입니다.

## 링크를 만드는 방법

1. [raydium.io/swap](https://raydium.io/swap)을 열고 지갑을 연결합니다. 레퍼럴 주소는 연결된 지갑에서 파생됩니다. 연결 없이도 링크는 작동하지만 수수료를 얻지 못합니다.
2. 다른 사람들이 스왑할 **목표 토큰**을 선택합니다. 입력 측은 수취자가 선택하도록 둡니다.
3. 스왑 패널의 공유/링크 아이콘을 클릭합니다 (오른쪽 상단).
4. 생성된 URL을 복사합니다. 어디든지 공유하세요.

URL 형식은 안정적입니다. 레퍼럴 링크를 직접 생성하는 앱을 만드는 경우에도 사용할 수 있습니다.

## 블링크의 작동 방식

**블링크 (Blockchain Links)**는 온체인 작업 URL을 소셜 플랫폼 내의 인터랙티브 UI로 렌더링하기 위한 Solana Foundation 표준입니다. 블링크를 지원하는 Solana 지갑이 링크 메타데이터를 읽고 사전에 구성된 거래를 구성하며 사용자에게 서명 프롬프트를 인라인으로 표시합니다.

요구 사항:

- 뷰어의 **지갑이 Actions / 블링크를 활성화**해야 합니다. 이는 일반적으로 지갑 설정의 토글입니다.
- **플랫폼이 표준을 준수**해야 합니다. X.com은 unfurler를 통해 준수하지만, 일부는 그렇지 않아 링크가 일반 하이퍼링크로 폴백됩니다.

레퍼러 관점에서는 블링크 렌더링이 자동입니다. 같은 URL은 블링크를 지원하지 않는 지갑에서 일반 링크로 작동합니다.

## 수수료 계산

1% 레퍼럴 수수료는 **AMM 거래 수수료 위에 부과되며**, 그것에서 공제되지 않습니다. 총 0.25% 수수료의 풀에서 스왑하는 경우:

```
사용자 전송:        1,000 USDC
AMM 거래 수수료:     2.50 USDC   (LP + 프로토콜 분할로)
레퍼럴 수수료:      10.00 USDC   (1% → 레퍼러의 지갑)
스왑 대상:          987.50 USDC  (그 후 곡선을 통해 가격 책정)
```

레퍼럴 수수료는 레퍼럴 링크로 생성된 Raydium 스왑 경로에 적용됩니다. 레퍼럴 경로 외의 직접 풀 상호작용에는 레퍼럴 수수료가 적용되지 않습니다.

## 레퍼럴 비활성화 / 탈퇴

"레퍼럴 제거" 명령어는 없습니다. 레퍼럴 수수료 없이 스왑하려면 레퍼럴 링크를 클릭하는 대신 [raydium.io/swap](https://raydium.io/swap)을 직접 로드하면 됩니다. UI는 URL에 레퍼러 매개변수가 없으면 레퍼러를 사용하지 않습니다.

## FAQ

**레퍼러가 뭔가에 서명해야 하나요?** 아니요. 레퍼러의 지갑 주소만 링크와 함께 전달되며, 레퍼러에 대한 지불은 스왑 실행자의 거래 내 토큰 전송입니다.

**자신을 추천할 수 있나요?** 네, 하지만 소용없습니다. 방금 더 큰 거래에서 지불한 tx 수수료에서 공제된 1%를 자신에게 지불하게 됩니다.

**목표 토큰이 transfer hook을 가진 SPL인 경우 어떻게 되나요?** 레퍼럴 전송은 입력 측 (USDC나 소스 토큰)에서 발생하지, 목표 측에서가 아니므로 목표 민트의 transfer hook은 레퍼럴 지불에 영향을 주지 않습니다. 입력 토큰의 transfer hook은 스왑을 완전히 차단할 수 있습니다. Raydium은 이를 미리 거부합니다.

**스왑이 여러 풀을 통해 라우팅되면 레퍼럴 수수료가 여전히 적용되나요?** 네. 1%는 원래 입력 금액으로 계산되며 경로와 무관합니다.
