## 입력값

모든 스왑에 필요한 것:

- **입금 토큰** (민트 주소 + 수량)
- **출금 토큰** (민트 주소)
- **슬리피지 허용치** (UI 기본값 0.5%; 변동성이 높은 쌍은 1%)
- **우선 수수료** (기본값 자동 설정)

수량은 "exact-in" (지불액을 설정하고 출금액 계산) 또는 "exact-out" (출금액을 설정하고 입금액 계산) 중 하나로 지정합니다. UI는 기본적으로 exact-in을 사용합니다.

## UI 안내

[raydium.io/swap](https://raydium.io/swap)에서:

1. **수량 입력.** 입금 및 출금 토큰을 선택하고 어느 한 상자에 수량을 입력합니다. 다른 쪽은 현재 시세를 사용하여 자동으로 채워집니다.

2. **경로 검토.** 경로 패널은 Raydium이 선택한 풀을 표시합니다:
   - 단일 풀 (가장 일반적): "SOL → USDC via CLMM 0.05%"
   - 멀티홉: "SOL → USDC via CPMM, then USDC → RAY via CLMM"
   - 분할: "SOL → USDC via 60% CLMM 0.05%, 40% CPMM 0.25%"

   각 홉별로: TVL, 수수료, 예상 영향을 표시합니다. 큰 거래의 경우 영향이 1%를 초과할 때 눈에 띄는 경고가 표시됩니다.

3. **슬리피지 확인.** 기본값은 0.5%입니다. 스테이블 쌍은 0.1%; 밈 토큰은 2~5%입니다. 조회 중에 시세가 변동하면 (가격이 변함) UI는 10초마다 재조회하고 깜박입니다.

4. **우선 수수료 설정.** 기본값은 자동; 3가지 레벨:
   - **보통** (50분위수) — 기본값
   - **빠름** (75분위수)
   - **터보** (95분위수)
   고급 사용자는 구체적인 마이크로-램포트/CU 숫자를 입력할 수 있습니다.

5. **확인 및 서명.** 트랜잭션 전송; UI가 확인을 추적합니다. 정상 혼잡도에서 예상 시간은 5~15초입니다.

6. **스왑 후.** UI는 실제 수신 수량을 표시합니다 (시세와 슬리피지만큼 다를 수 있음); 전체 세부 사항을 보려면 tx 탐색기를 클릭합니다.

## 경로 선택 방식

같은 쌍에 대해 여러 풀이 존재할 때, Raydium 라우터는 다음 기준으로 선택합니다:

1. **수수료 후 최고의 `amountOut`.** 각 적격 풀에 대해 시세를 계산하고 가장 많은 출금액을 제공하는 것을 선택합니다.
2. **TVL로 동점 해결.** TVL이 높을수록 다음 거래에 미치는 영향이 작아지므로 조회와 제출 사이의 가격 변동에 강합니다.
3. **단일 풀을 멀티홉보다 선호.** 멀티홉은 수수료와 슬리피지를 누적시키므로, 라우터는 단일 풀을 >5 bps 초과할 때만 멀티홉을 선택합니다.
4. **큰 거래는 분할.** 풀 TVL에 비해 큰 거래는 가격 영향을 줄이기 위해 여러 풀로 분할될 수 있습니다.

일반적인 쌍에서 비교되는 상품:

- **CPMM 풀**은 저 TVL (`<$1M`)에서 지배합니다. 생성이 저렴하고 스왑 수수료 공유가 LP에 직접 자금을 조달하기 때문입니다.
- **CLMM 풀**은 변동성이 높은 쌍에서 중/고 TVL에 지배합니다. 집중된 유동성은 일반적인 가격 범위 내에서 더 나은 시세를 의미합니다.
- **AMM v4** 풀은 깊은 OpenBook 통합이 있는 오래 설립된 쌍에 지배합니다 (SOL-USDC, RAY-USDC). 대부분의 신규 쌍은 AMM v4를 사용하지 않습니다.

## Token-2022 전송 수수료

입금 또는 출금 민트가 전송 수수료가 있는 Token-2022인 경우:

- 시세의 `amountOut`은 수수료 후 사용자가 받는 순액입니다.
- UI의 "수수료" 표시는 "풀 수수료" (LP 수수료) 및 "전송 수수료" (민트 발행자에게 지급)로 분할됩니다.

[`algorithms/token-2022-transfer-fees`](/ko/algorithms/token-2022-transfer-fees)를 참조하세요.

## 슬리피지

슬리피지는 시세와 실행 사이의 불리한 가격 변동으로부터 보호합니다. Raydium은 슬리피지 비율을 받아야 하는 최소 수량으로 변환합니다:

```
minimumAmountOut = expectedAmountOut × (1 - slippage)
```

실제 출금액이 더 적으면 `ExceededSlippage`로 트랜잭션이 되돌려집니다. 일반적인 기본값:

| 쌍 유형 | 슬리피지 |
|---------|--------|
| 스테이블-스테이블 | 0.01–0.1% |
| 메이저-메이저 (SOL/USDC) | 0.1–0.5% |
| 중형 변동성 | 0.5–1% |
| 밈 토큰 | 2–5% |
| 전송 수수료가 있는 Token-2022 | 슬리피지에 전송 수수료 추가 |

너무 낮은 슬리피지는 정상적인 가격 변동 시 빈번한 되돌림을 초래합니다. 너무 높은 슬리피지는 샌드위치 공격을 유발합니다 ([`integration-guides/routing-and-mev`](/ko/integration-guides/routing-and-mev) 참조).

## 우선 수수료

우선 수수료는 혼잡 중에 트랜잭션을 처리하도록 검증자에게 더 많은 비용을 지불합니다. UI는 기본적으로 자동 설정으로 기본값이 지정됩니다. 네트워크가 바쁠 때, 변동성이 높은 쌍을 스왑할 때, 또는 이전 트랜잭션이 확인 전에 만료되었을 때 **빠름** 또는 **터보**를 사용하세요.

우선 수수료가 높을수록 시세가 개선되지 않습니다. 시세가 만료되기 전에 트랜잭션이 착륙할 확률만 개선됩니다.

## 일반적인 실패 모드

### Tx가 `ExceededSlippage`로 되돌려짐

가격이 조회와 실행 사이에 허용치 이상으로 변동했습니다. 해결책:

1. 필요한 만큼만 슬리피지를 늘립니다.
2. 거래 규모를 줄입니다 (영향 감소).
3. 재조회 후 재시도; 가격이 일시적이었을 수 있습니다.
4. 메이저 쌍에서 매우 높은 슬리피지를 피합니다. 더 나쁜 실행에 노출될 수 있기 때문입니다.

### Tx 타임아웃

현재 혼잡도에 비해 우선 수수료가 너무 낮습니다. 해결책: 우선 수수료 인상; `integration-guides/priority-fee-tuning`을 참조하세요.

### CLMM 스왑에서 `InvalidAccountData`

경로는 로드 실패했거나 조회가 열려 있는 동안 변경된 CLMM 계정에 따라 달라질 수 있습니다. 페이지를 새로 고침하고 재조회 후 재시도하세요. 계속 발생하면 UI가 다른 경로를 제공하는 경우 선택하세요.

### "풀을 찾을 수 없음" / 만료된 풀 데이터

풀이 최근에 생성되어 아직 인덱싱되지 않았습니다. ID로 직접 다시 가져오세요:
몇 분 기다렸다가 앱을 새로 고치고 풀 또는 토큰 민트로 다시 검색하세요. 새로 생성된 풀은 인덱싱된 UI 검색에 나타나는 데 짧은 시간이 걸릴 수 있습니다.

### 출금액이 0 / 매우 작음

스왑 규모가 풀 유동성을 초과합니다. 풀이 너무 얕습니다. 거래 규모를 줄이거나, 다른 경로를 시도하거나, 더 많은 유동성을 기다리세요.

### Tx는 착륙했지만 잔액이 변경되지 않음

지갑 및 Raydium 잔액을 새로 고칩니다. 트랜잭션이 탐색기에서 성공했으면 출금 토큰 계정이 지갑 UI에서 숨겨져 있는지 확인하세요.

## 검증

Raydium이 확인 후 표시한 트랜잭션 링크를 사용하세요. Solscan 또는 SolanaFM에서 확인:

- **상태:** 성공.
- **서명자:** 당신의 지갑 주소.
- **토큰 잔액 변경:** 입금 토큰 감소, 출금 토큰 증가.
- **호출된 프로그램:** Raydium 스왑 프로그램과 토큰/시스템 프로그램.
