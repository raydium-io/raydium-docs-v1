## 주문 유형

Raydium Perps는 다음 주문 유형을 지원합니다.

| 주문 유형 | 동작 | 사용 시기 |
|---|---|---|
| **시장가(Market)** | 오더북의 최고 시점가로 즉시 매수 또는 매도합니다. 전체 수량을 채울 수 없으면 미체결 부분은 취소됩니다. | 즉시 체결을 원하고 슬리피지를 감수할 수 있을 때. |
| **지정가(Limit)** | 지정한 가격 이상에서만 매수 또는 매도합니다. 시장이 해당 가격에 도달할 때만 체결됩니다. | 가격 관리를 원하고 체결을 기다릴 수 있을 때. |
| **스탑 시장가(Stop market)** | 트리거 가격에 도달할 때까지 비활성 상태를 유지하다가 시장가 주문으로 전환됩니다. | 실행 트리거를 원하고 트리거 후 진입/청산을 우선시할 때. |
| **스탑 지정가(Stop limit)** | 트리거 가격에 도달할 때까지 비활성 상태를 유지하다가 지정가 주문으로 전환됩니다. | 트리거를 원하면서도 지정가 보호를 원할 때. |
| **스케일(Scale)** | 하나의 큰 주문을 가격 범위에 걸쳐 여러 개의 작은 지정가 주문으로 분할합니다. | 진입 또는 청산을 단계적으로 하고 싶을 때. |

## 주문 플래그

주문 플래그는 지정가 형식의 주문 동작을 변경합니다.

| 플래그 | 동작 |
|---|---|
| **IOC** | Immediate-or-cancel(지정가 또는 취소). 지정가 이상에서 가능한 한 많이 즉시 체결하고, 미체결 부분은 취소합니다. |
| **FOK** | Fill-or-kill(전체 체결 또는 취소). 지정가 이상에서 전체 주문을 즉시 체결하거나, 부분 체결 없이 전체 취소합니다. |
| **포스트온리(Post-only)** | 메이커로서 유동성을 공급합니다. 즉시 체결될 수 있는 경우 대신 취소됩니다. |
| **리듀스온리(Reduce-only)** | 주문이 포지션을 증가시키지 않도록 방지합니다. 해당 시장의 기존 포지션만 감소 또는 청산할 수 있습니다. |

## 주문 제출 전 확인 사항

주문을 제출하기 전에 다음을 확인하세요.

- 시장과 매매 방향이 정확한지 확인.
- 수량과 레버리지가 의도와 일치하는지 확인.
- 스탑 트리거 가격과 지정가가 역순이 아닌지 확인.
- 포지션 감소 또는 청산만 의도할 때 리듀스온리가 활성화되어 있는지 확인.
- 메이커 주문을 원할 때 포스트온리가 활성화되어 있는지 확인.
