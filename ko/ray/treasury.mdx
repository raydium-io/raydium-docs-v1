## 흐름 개요

```text
풀 수수료
  -> 프로그램별 수집 주소
      -> RAY 바이백 경로
      -> USDC 재무 경로 (해당하는 경우)

풀 생성 수수료
  -> 프로그램별 SOL 수수료 주소
```

이 흐름의 모든 주소는 공개되어 있으며 검증 가능합니다.

## 수수료 수집 주소

스왑이 프로토콜 측 수수료를 지불하면 해당 프로그램은 그 수수료를 수집 주소에 축적합니다.

| 출처 | 수집 주소 |
|---|---|
| CLMM | `projjosVCPQH49d5em7VYS7fJZzaqKixqKtus7yk416` |
| CPMM | `ProCXqRcXJjoUd1RNoo28bSizAA6EEqt9wURZYPDc5u` |
| Standard AMM v4 | `PNLCQcVCD26aC7ZWgRyr5ptfaR7bBrWdTFgRWwu2tvF` |

수집 주소는 다양한 SPL 토큰 세트를 보유할 수 있습니다. 예를 들어 SOL/USDC 풀의 수수료는 wSOL 및 USDC 잔액을 생성할 수 있고, SOL/RAY 풀은 wSOL 및 RAY 잔액을 생성할 수 있습니다.

수집 주소를 다음 명령으로 확인할 수 있습니다:

```bash
spl-token accounts --owner projjosVCPQH49d5em7VYS7fJZzaqKixqKtus7yk416
```

## 풀 생성 수수료

CPMM 또는 Standard AMM v4 풀 생성에는 SOL 단위의 풀 생성 수수료가 부과됩니다. 이 수수료는 프로그램별 주소에 축적됩니다.

| 출처 | 주소 |
|---|---|
| CPMM | `DNXgeM9EiiaAbaWvwjHj9fQQLAX5ZsfHyvmYUNRAdNC8` |
| Standard AMM v4 | `7YttLkHDoNj9wyDur5pM1ejNaAvT9X4eqaYcHQqtj2G5` |

CLMM 풀 생성은 여전히 계정 렌트가 필요하지만 동일한 별도 SOL 풀 생성 수수료 경로를 사용하지 않습니다.

## 수수료 흐름 감사 방법

누구나 공개 거래 데이터에서 Raydium의 수수료 흐름을 검증할 수 있습니다:

1. Raydium 풀에서 스왑 거래를 가져옵니다.
2. 풀 프로그램을 식별합니다: CLMM, CPMM, 또는 Standard AMM v4.
3. 거래에서 수수료 회계 및 토큰 전송을 검사합니다.
4. 프로토콜 측 이동을 관련 수집 주소와 일치시킵니다.
5. 수집 주소에서 나가는 거래를 추적합니다.
6. RAY 축적을 `DdHDoz...VEZaz`로 또는 USDC 잔액을 위의 재무 주소로 일치시킵니다.
