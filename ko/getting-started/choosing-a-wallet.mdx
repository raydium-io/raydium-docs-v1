## Raydium 호환 지갑이 해야 할 일

지갑이 Raydium 호환 지갑이 되려면 다음을 모두 수행할 수 있어야 합니다:

- Solana 키페어를 로컬에서 보유하고 개인키를 기기 밖으로 전송하지 **않아야** 합니다.
- **버전이 지정된(V0) 거래에 서명해야 합니다.** Raydium 스왑은 주소 조회 테이블을 사용하며 V0 지원이 필요합니다. 레거시 거래만 서명하는 지갑은 Raydium 스왑을 완료할 수 없습니다.
- **지갑 어댑터 표준을 통해 dApp에 연결해야 합니다.** 이것이 raydium.io의 "지갑 연결" 버튼을 작동시킵니다.
- **승인 전에 명확한 거래 미리보기를 표시해야 합니다**. 호출되는 프로그램과 토큰 이동을 포함합니다.
- **관련 토큰 계정을 지원해야 합니다**(새로운 민트를 처음 받을 때 ATA 생성).

이것들은 기본 요구사항입니다. 모든 최신 Solana 지갑이 이를 충족합니다. 충족하지 않는 것은 Raydium 거래에 서명하기에 안전하지 않습니다.

## 기본 요구사항을 넘어 중요한 기능

지갑이 기본 요구사항을 충족하면, 차이점은 당신의 워크플로우와 위험 프로필에 달려 있습니다. 다음 중 **당신**에게 중요한 것을 결정하고, 이를 처리하는 지갑을 선택하세요:

| 기능 | 중요한 경우 |
|---|---|
| **하드웨어 지갑 페어링** | 상당한 금액을 보유하고 있습니다. 소프트웨어 지갑은 거래 작성자가 되고, 하드웨어 기기는 키를 보유하고 서명합니다. 대부분의 평판이 좋은 Solana 지갑은 최소한 Ledger와 페어링되며, 일부는 Keystone, SafePal 또는 다른 기기와도 페어링됩니다. |
| **모바일 dApp 브라우저** | 주로 휴대폰에서 Raydium을 사용할 예정입니다. 지갑의 앱 내 브라우저가 raydium.io를 로드하고 모바일에서 거래에 서명합니다. |
| **다중 계정 관리** | 자금을 핫/콜드/실험용 지갑으로 나누고 확장 프로그램을 번갈아 사용하지 않고 빠르게 전환하려고 합니다. |
| **Token-2022 확장 인식** | Token-2022 민트(예: 전송 수수료 토큰)를 사용하는 풀과 상호작용합니다. 서명 UI에서 이러한 확장을 표시하는 지갑은 서명하기 전에 예상치 못한 수수료를 찾는 데 도움이 됩니다. |
| **우선순위 수수료 표시** | 혼잡한 시간에 거래하며 승인하기 전에 지갑이 설정한 계산 예산 가격을 확인하려고 합니다. |
| **Solana Actions / Blinks 지원** | Raydium이 소셜 미디어에 게시한 원클릭 스왑 링크를 사용하고 싶습니다. [`user-flows/referrals-and-blinks`](/ko/user-flows/referrals-and-blinks)를 참조하세요. |
| **오픈소스 코드베이스** | 지갑 클라이언트를 감사할 수 있기를 원합니다(또는 커뮤니티가 감사한 것에 의존). |
| **다중 체인 계정** | Solana 외의 다른 체인에 자산을 보유하고 있으며 모든 자산에 대해 하나의 지갑을 선호합니다. 다중 체인 지원이 실제로 사용하는 체인에 대해 완성되었는지 확인하세요. 다양합니다. |

## 인기 있는 Solana 지갑

다음은 활발한 Solana 지원을 하는 잘 알려진 지갑 프로젝트입니다. 알파벳 순서로 나열되어 있습니다. **포함은 정보 제공일 뿐 추천이 아니며**, 이 목록이 완전하지 않습니다. 설치하기 전에 공식 URL을 확인하세요. 피싱 클론이 정기적으로 실제 도메인 이름에 대한 스폰서 광고를 구매합니다.

### 소프트웨어 지갑

| 지갑 | 실행 위치 | 공식 사이트 |
|---|---|---|
| Phantom | 브라우저 확장, iOS, Android | [phantom.app](https://phantom.app) |
| Solflare | 브라우저 확장, iOS, Android, 웹 | [solflare.com](https://solflare.com) |
| OKX Wallet | 브라우저 확장, iOS, Android | [okx.com/wallet](https://www.okx.com/wallet) |
| Backpack | 브라우저 확장, iOS, Android, 데스크톱 | [backpack.app](https://backpack.app) |

네 개 모두 비수탁형이며, 지갑 어댑터를 지원하고, V0 거래에 서명하며, 최소한 하나의 주류 하드웨어 기기와 페어링됩니다. 그 이후로는 차이점이 워크플로우별로 다릅니다. 이전 섹션의 기준을 사용하여 어느 것이 당신에게 맞는지 결정하세요.

### 하드웨어 지갑

하드웨어 지갑은 위의 소프트웨어 지갑 중 하나와 페어링됩니다(소프트웨어 쪽은 거래 작성자가 되고, 하드웨어 기기는 키를 보유하고 서명합니다).

| 기기 | 유형 | 공식 사이트 |
|---|---|---|
| Ledger | USB / Bluetooth | [ledger.com](https://www.ledger.com) |
| Keystone | 에어갭(QR 코드 서명) | [keyst.one](https://keyst.one) |

각 소프트웨어 지갑은 지원하는 하드웨어 기기를 문서화합니다. 구매하기 전에 확인하세요.

## 핫/콜드 지갑 패턴

선택한 지갑에 관계없이 작동하는 일반적인 관행:

- **핫 지갑**은 잃을 수도 있는 적은 양을 보유합니다. 일일 스왑과 실험용입니다.
- **콜드 지갑**(하드웨어 기기와 페어링됨)은 잃고 싶지 않은 모든 금액을 보유합니다. 핫 지갑에 적극적으로 필요할 때만 자금을 이동하세요.

Raydium은 각 지갑을 완전히 독립적으로 취급합니다: 별도의 포지션, 별도의 LP 토큰, 별도의 팜 보상, 별도의 거래 내역. 지갑 전환은 한 번의 클릭입니다.

## 지갑 옵션을 찾을 위치

Raydium UI의 "지갑 연결" 모달은 지갑 어댑터를 통해 Raydium과 통합된 모든 지갑을 나열합니다. 현재 지원되는 것을 확인하려면 이를 살펴보세요. Solana의 공식 지갑 디렉토리([`solana.com/ecosystem/explore?categories=wallet`](https://solana.com/ecosystem/explore?categories=wallet))는 또 다른 중립적인 시작점입니다.

지갑을 평가할 때 이 체크리스트를 확인하세요:

- [ ] 유지관리됨: 최근 릴리스, 활동 중인 이슈 추적 도구, 보안 공개 채널.
- [ ] 검토됨: 평판이 좋은 회사의 공개 감사, 또는 활동 중인 커뮤니티가 있는 오픈소스 코드.
- [ ] V0 거래 지원 확인됨(실제 자금을 전송하기 전에 devnet에서 테스트).
- [ ] 하드웨어 기기를 사용할 계획이 있다면: 지갑이 지원하는 기기와 페어링 방식을 문서화합니다.
- [ ] 프로젝트의 확인된 도메인을 통해 공식 다운로드를 찾았습니다. 스폰서 광고나 전달된 링크가 아닙니다.

## 지갑 보안 기본사항

선택한 지갑에 관계없이:

- **시드 구문을 종이에 적어두세요.** 종이를 물리적으로 떨어진 두 곳에 보관하세요. **절대** 신선한 설치에서 지갑 복구 중을 제외하고는 시드 구문을 어디에도 입력하지 마세요.
- **지갑의 잠금 해제 비밀번호에 비밀번호 관리자를 사용하세요.** 그 비밀번호를 시드와 완전히 분리하세요.
- **암호 활동을 위한 전용 브라우저 프로필을 사용하세요**, 최소한의 확장 프로그램 설치로. 피싱 확장 프로그램은 단일 최대 소모 벡터입니다.
- **모든 지갑 연결 프롬프트를 의심스럽게 취급하세요.** 서명하는 것을 읽으세요. 거래가 특정 금액 대신 토큰의 모든 것을 전송하는 경우 거절하세요.
- **`raydium.io`를 북마크하세요.** 검색 엔진 광고에서가 아닌 북마크에서 항상 탐색하세요.

더 많은 보안 지침은 [`getting-started/trust-and-safety`](/ko/getting-started/trust-and-safety)에 있습니다.
