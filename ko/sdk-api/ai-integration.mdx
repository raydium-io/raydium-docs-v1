## 이용 가능한 리소스

| 리소스 | URL 패턴 | 사용 시기 |
|---|---|---|
| **MCP 서버** | `https://docs.raydium.io/mcp` | AI 편집기(Claude Code, Cursor, Windsurf 등)가 Raydium 문서를 기본적으로 쿼리하고 인용하길 원할 때 |
| **`llms.txt` 인덱스** | `https://docs.raydium.io/llms.txt` | RAG 파이프라인을 구축하고 모든 페이지의 평탄한 인덱스가 필요할 때 |
| **`llms-full.txt` 본문** | `https://docs.raydium.io/llms-full.txt` | 오프라인 인덱싱을 위해 전체 문서를 하나의 연결된 파일로 원할 때 |
| **페이지별 사본 메뉴** | 모든 본문 페이지 상단의 `페이지 복사` 버튼 | 한 페이지를 LLM과의 채팅에 붙여넣을 때 |
| **페이지별 딥 링크** | `마크다운으로 보기`, `ChatGPT에서 열기`, `Claude에서 열기`, `Cursor에서 열기`, `VS Code에서 열기` | 문서 페이지에서 선택한 도구로 한 번에 이동할 때 |

`페이지 복사` 버튼과 딥 링크 메뉴는 모든 페이지의 상단 우측(페이지 제목 옆)에 위치합니다. 둘 다 문서 플랫폼의 상황 메뉴로 동작합니다.

## MCP 서버

MCP(Model Context Protocol) 서버를 사용하면 AI 클라이언트가 도구로서 Raydium 문서를 쿼리할 수 있습니다. 설정 후 에이전트는 "Raydium 문서에서 `addLiquidity` 매개변수를 검색해"라고 요청하면 인용된 답변을 받을 수 있습니다.

### Claude Code

```bash
claude mcp add --transport http raydium-docs https://docs.raydium.io/mcp
```

추가 후 Claude Code에 "Raydium 문서에서 X를 검색해"라고 요청하면 도구를 사용합니다.

### Cursor

Cursor 설정(`Cmd/Ctrl + ,` → MCP)에 추가하세요:

```json
{
  "mcpServers": {
    "raydium-docs": {
      "url": "https://docs.raydium.io/mcp"
    }
  }
}
```

### Windsurf, Continue, 범용 클라이언트

모든 MCP 호환 클라이언트는 `https://docs.raydium.io/mcp`를 가리킬 수 있습니다. 클라이언트가 매니페스트를 요구하는 경우 `https://docs.raydium.io/mcp/.well-known/mcp.json`에 있습니다.

### 서버가 노출하는 것

Raydium 문서 MCP 서버는 하나의 주요 도구 `search_docs(query: string)`을 노출하며, 이는 쿼리와 관련된 가장 높은 순위의 구절과 그들의 정확한 URL을 반환합니다. 에이전트는 사용한 URL을 인용할 책임이 있습니다. 저희는 에이전트별 추적이나 속도 제한을 하지 않습니다.

## RAG용 `llms.txt`

`llms.txt`는 "기계 가독형 문서 인덱스"를 위한 새로운 표준입니다. Raydium은 다음을 게시합니다:

- `https://docs.raydium.io/llms.txt` — 모든 페이지의 평탄한 목록(제목과 한 줄 요약 포함), 장(chapter)별로 정렬됨.
- `https://docs.raydium.io/llms-full.txt` — 모든 페이지의 전체 마크다운 본문, 연결됨, 페이지 경계는 제목으로 유지됨.

`full` 버전은 문서 배포할 때마다 다시 생성됩니다. 스케줄에 따라 가져오거나(매일이면 충분) 필요 시 가져오세요.

```bash
# 전체 문서를 RAG 수집 파이프라인으로 전달합니다.
curl -sSL https://docs.raydium.io/llms-full.txt -o raydium-docs.md
```

## 페이지별 전환 메뉴

모든 본문 페이지에는 상황 메뉴(상단 우측, 제목 옆)가 있으며 다음과 같은 한 번의 클릭으로 실행되는 작업을 제공합니다:

- **복사** — 페이지를 순수 마크다운으로 복사합니다.
- **마크다운으로 보기** — 소스 `.md`를 새 탭에서 열어 저장할 수 있습니다.
- **ChatGPT / Claude / Perplexity에서 열기** — 페이지 내용을 명시된 도구의 채팅 프롬프트에 미리 로드합니다.
- **Cursor / VS Code에서 열기** — 페이지 내용이 있는 버퍼를 명시된 편집기에서 엽니다.

MCP를 설정하지 않고 단일 페이지에 대해 모델에 질문하려 할 때 적합한 리소스입니다.

## 코딩 에이전트를 위한 미리 만들어진 컨텍스트 파일

Raydium을 통합하고 에이전트가 처음부터 올바른 기본 지식을 갖기를 원한다면 이 파일들을 프로젝트에 놓으세요:

### `.cursorrules` / `.windsurfrules` / 에이전트 시스템 프롬프트

```text
You are integrating with Raydium, a Solana DeFi protocol with five product
surfaces: AMM v4, CPMM, CLMM, Farm, and LaunchLab. Authoritative docs live
at docs.raydium.io. The official SDK is `@raydium-io/raydium-sdk-v2` (pin
the version you've verified against). For server-built swaps, prefer the
Trade API at transaction-v1.raydium.io. Always:

- Pass a `Connection` and `cluster` that match.
- Keep all amounts as `BN` instances; never call `.toNumber()` on amounts.
- Pre-fund the user's wallet for ATA creation rent.
- Pass an explicit `computeBudgetConfig` for any tx that may compete in
  high-volume windows.
- Re-fetch `poolInfo` immediately before high-value transactions; cached
  state goes stale.

When uncertain about an instruction's accounts list, defer to
`docs.raydium.io/products/<product>/accounts` and the on-chain IDL.
```

`.cursorrules`(Cursor), `.windsurfrules`(Windsurf) 또는 `CLAUDE.md`(Claude Code)로 저장하거나 에이전트의 시스템 프롬프트에 붙여넣으세요. 정확한 파일명과 위치는 도구 버전에 따라 다르므로, 파일을 자동으로 감지하지 못할 경우 도구의 설정 문서를 확인하세요.

### 고정된 컨텍스트 목록

"항상 포함" 페이지 목록을 받는 코딩 에이전트의 경우, 이 목록이 대부분의 Raydium 통합에 필요한 최소 컨텍스트입니다:

```text
https://docs.raydium.io/sdk-api/typescript-sdk
https://docs.raydium.io/sdk-api/trade-api
https://docs.raydium.io/products/cpmm/instructions
https://docs.raydium.io/products/clmm/instructions
https://docs.raydium.io/integration-guides/priority-fee-tuning
https://docs.raydium.io/reference/program-addresses
https://docs.raydium.io/reference/error-codes
```

통합하는 제품에 따라 조정하세요.

## 인덱싱 레시피

직접 RAG를 구축하면서 `llms-full.txt`를 사용하지 않으려면 정확한 레시피는 다음과 같습니다:

```ts
import { fetch } from "undici";

const INDEX_URL = "https://docs.raydium.io/llms.txt";
const BASE      = "https://docs.raydium.io";

const index    = await (await fetch(INDEX_URL)).text();
const pageUrls = [...index.matchAll(/^- \[(.+?)\]\((.+?)\)/gm)].map(m => m[2]);

for (const path of pageUrls) {
  const md = await (await fetch(`${BASE}${path}.md`)).text();
  // ingest md into your vector store, keyed by path
}
```

모든 페이지는 정확한 경로에 `.md` 접미사가 붙어서 제공됩니다(예: `/sdk-api/typescript-sdk` → `/sdk-api/typescript-sdk.md`).

## 주의사항

- **비공개 상태를 프롬프트에 붙여넣지 마세요.** MCP 서버는 공개 문서의 내용만 알고 있습니다. 지갑 키, RPC 자격증명 등의 비밀을 에이전트 컨텍스트에 입력하면 안 됩니다.
- **AI 출력은 권위적이지 않습니다.** 모델은 프로그램 주소, 명령어 이름, 계정 목록을 놀라운 정도로 자주 착각합니다. 에이전트가 생성한 트랜잭션을 신뢰하기 전에 항상 문서와 IDL에 대비해 검증하세요.
- **버전 드리프트.** SDK v2는 1.0 이전 버전입니다. 이전 릴리스에서 학습된 에이전트는 고정된 버전에 대해 컴파일되지 않는 코드를 생성할 수 있습니다. 에이전트의 시스템 프롬프트에 고정된 SDK 버전을 포함하세요.
