## 등재 기준

도구가 이 페이지에 등재되려면 다음 조건을 충족해야 합니다: (a) 공개적으로 접근 가능, (b) 등재 당시 Raydium의 메인넷 프로그램과 실제로 동작, (c) 유지보수 중 — 즉, 지난 90일 이내에 커밋, 릴리스 또는 눈에 띄는 활동이 있어야 합니다.

아래 카테고리는 초기 분류이며, 어느 카테고리에도 해당하지 않는 도구를 제출하면 새 카테고리를 추가하겠습니다.

Raydium 풀을 통한 애그리게이터 라우팅은 LP 측의 통합이 필요 없습니다 — 풀이 개설되면 애그리게이터가 자동으로 인덱싱합니다.

## 인덱서 및 API (커뮤니티)

| 도구 | 유형 | 설명 |
|---|---|---|
| [Helius](https://helius.dev) | RPC + 인덱싱된 데이터 | 강화된 API를 통해 파싱된 Raydium 이벤트 및 풀 데이터를 제공합니다. |
| [Triton One](https://triton.one) | RPC | Raydium 프로덕션 통합에 일반적으로 사용되는 고성능 RPC. |
| [Solscan API](https://api.solscan.io) | 블록 탐색기 API | 파싱된 Raydium 명령어를 포함한 계정/거래 데이터. |
| [Bitquery](https://bitquery.io) | GraphQL 데이터 | GraphQL을 통해 쿼리 가능한 Raydium 스왑, 유동성 이벤트, 거래 이력. |

공식 Raydium 관리 API는 [`sdk-api/rest-api`](/ko/sdk-api/rest-api)를 참고하세요. 위의 도구들은 공식 API가 제공하지 않는 데이터 형태가 필요할 때 유용합니다.

## SDK 및 언어 클라이언트 (커뮤니티)

| 도구 | 언어 | 설명 |
|---|---|---|
| `solana-py` + 커스텀 Raydium 헬퍼 | Python | Python 통합을 위한 표준 경로입니다. [`sdk-api/python-integration`](/ko/sdk-api/python-integration)에서 우리의 레시피를 참고하세요. |
| Raydium Rust 클라이언트 | Rust | Anchor IDL에서 생성되었습니다 — [`sdk-api/anchor-idl`](/ko/sdk-api/anchor-idl)을 참고하세요. |

Go, Kotlin, Swift, Java 또는 활발한 사용자가 있는 다른 언어로 커뮤니티 SDK를 관리한다면 아래 프로세스를 통해 제출하세요 — 등재하겠습니다.

| MEV / 차익거래 프레임워크 | 일반 솔라나 | 모든 솔라나 차익거래 프레임워크는 Raydium 풀을 대상으로 할 수 있으며, Raydium 특화 기능은 아닙니다. |

## 도구 제출하기

이 페이지에 항목을 추가하거나 업데이트하려면:

1. [도큐 저장소](https://github.com/raydium-io)를 포크하세요 (도큐 소스는 공개 Raydium 도큐 저장소에 있으며 — 정확한 경로는 README에 있습니다).
2. `resources/community-tools.mdx`를 편집하세요.
3. 다음 내용으로 PR을 제출하세요:
   - 도구 이름 및 링크.
   - 한 줄 설명 (도구가 무엇인지, 어느 카테고리인지).
   - 관리자의 GitHub 또는 X 핸들 (검증용).
   - 활동 상태에 대한 설명 (마지막 릴리스 / 의미 있는 마지막 커밋).

등재는 "공개적으로 접근 가능 / 실제로 동작 / 유지보수 중" 기준으로 검토됩니다. 동작하는 소프트웨어 없는 자기홍보 등재, 또는 사용자 안전과 충돌하는 주 목적의 도구 (드레이너, 가짜 풀 생성기)는 병합되지 않습니다.

## 제거 정책

다음 경우 도구는 이 페이지에서 제거됩니다:

- 호스팅된 인스턴스가 30일 이상 오프라인 상태.
- 저장소가 아카이브되었거나 삭제됨.
- 관리자가 180일 이상 침묵하고 도구가 동작 중단.
- 도구가 사용자 피해 행동 (드레이너 스크립트, 기만적 UX 등)을 시작.

제거는 도큐 관리자의 재량이며, 도구가 다시 온라인되고 유지보수되면 복구 신청이 가능합니다.

## 보안 안내

이 페이지에 등재된 도구 중 다음의 행동을 하는 것은 **사용하지 말아야 할 도구**입니다:
- 지갑의 시드 프레이즈를 요청
- 보관 주소로 자금 송금 요구
- 보장된 수익을 약속

여기 등재는 낮은 기준입니다 — 보안을 보장하지 않습니다. 항상 트랜잭션에 서명하기 전에 승인하려는 내용을 검사하세요.
