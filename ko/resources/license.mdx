## 온체인 프로그램

배포된 Raydium 프로그램은 **Apache License 2.0** 하에 완전한 소스 코드와 함께 공개됩니다.

| 저장소 | 라이센스 | 대상 |
|---|---|---|
| [`raydium-amm`](https://github.com/raydium-io/raydium-amm) | Apache-2.0 | AMM v4 (constant-product, OpenBook 통합). |
| [`raydium-cp-swap`](https://github.com/raydium-io/raydium-cp-swap) | Apache-2.0 | CPMM (표준 constant-product, 오더북 없음, Token-2022 지원). |
| [`raydium-clmm`](https://github.com/raydium-io/raydium-clmm) | Apache-2.0 | CLMM (집중 유동성, sqrt-price ticks). |

Apache-2.0은 상용 목적을 포함하여 코드를 사용, 수정, 배포 및 재라이센스할 권리를 부여합니다. 단, 라이센스 약관의 조건을 따라야 하며, 주로 저작권 표시 보존과 특허 양도가 포함됩니다. **GPL 계열 라이센스와 달리** 파생 저작물도 Apache-2.0이어야 한다는 조건은 없습니다.

Farm 및 LaunchLab 프로그램의 라이센스는 [Raydium GitHub 조직](https://github.com/raydium-io) 아래 해당 저장소의 `LICENSE` 파일을 확인하세요. 라이센스는 저장소 소유자의 판단에 따라 업데이트될 수 있으며, `master` (또는 `main`) 브랜치의 `LICENSE` 파일이 공식 정보입니다.

## TypeScript SDK

공식 SDK 패키지 `@raydium-io/raydium-sdk-v2`는 **GPL-3.0** 하에 라이센스됩니다.

| 패키지 | 라이센스 | 저장소 |
|---|---|---|
| `@raydium-io/raydium-sdk-v2` | GPL-3.0 | [`raydium-sdk-V2`](https://github.com/raydium-io/raydium-sdk-V2) |
| `raydium-sdk-V2-demo` (샘플 통합) | 저장소 `LICENSE` 참조 | [`raydium-sdk-V2-demo`](https://github.com/raydium-io/raydium-sdk-V2-demo) |

GPL-3.0은 강력한 카피레프트 라이센스입니다. 프로젝트에 미치는 영향:

- **연결이 중요합니다.** SDK를 포함하여 배포하는 경우 (바이너리, npm 패키지, 호스팅 서비스), 배포된 저작물도 일반적으로 GPL-3.0이어야 합니다.
- **SaaS 예외.** GPL-3.0은 AGPL-3.0과 달리 순수하게 네트워크를 통한 사용에 대해 소스 공개를 요구하지 않습니다. 즉, SDK를 서버 측에서 실행하고 API를 통해서만 결과를 노출하는 경우 자체적으로 소스 공개 의무를 발생시키지 않습니다.
- **특허 양도.** GPL-3.0에는 기여자의 기여에 대한 명시적인 특허 양도가 포함됩니다.

GPL-3.0과 호환되지 않는 사용 사례가 있다면 (예: SDK를 정적으로 연결하여 최종 사용자에게 배포하는 독점 폐쇄형 소프트웨어를 구축하는 경우), 배포하기 전에 Discord (`#dev-support`)를 통해 Raydium 팀에 문의하세요. 다른 방법이 가능할 수도 있습니다. **라이센스 없는 대체 방안이 있다고 가정하지 마세요.** 서면 합의가 없는 한 GPL-3.0이 기본입니다.

## REST 및 Trade API

`api-v3.raydium.io` 및 `transaction-v1.raydium.io`의 HTTP API는 라이센스 코드가 아닌 서비스입니다. 사용은 호스팅 엔드포인트의 암묵적인 서비스 약관에 따릅니다. 주로 합리적인 속도 제한, 악용 금지, 보증 거부입니다. 자세한 내용은 [`sdk-api/rest-api`](/ko/sdk-api/rest-api)를 참조하세요.

이 API는 생태계 통합자를 위한 공공 서비스로 운영됩니다. 대규모 상용 사용은 속도 제한을 받지 않도록 팀과 조율해야 합니다.

## 문서

이 문서 세트는 Raydium 프로젝트에서 공개합니다. 개별 페이지에서 다르게 명시하지 않는 한, 이 사이트의 산문, 코드 샘플 및 예제는 **Creative Commons Attribution 4.0** (CC-BY-4.0) 하에 제공됩니다.

다음을 할 수 있습니다:

- 이 문서에서 콘텐츠를 인용, 발췌 또는 재출판합니다.
- 문서를 다른 언어로 번역합니다.
- 코드 샘플 (기능이 짧은 예시)을 프로젝트에 직접 사용합니다.

다음을 해야 합니다:

- "Raydium documentation"으로 명시적인 저작권 표시를 제공하고 `docs.raydium.io`로 다시 연결합니다.
- 파생 저작물을 공식 Raydium 문서로 표현하지 않습니다.

실수를 발견했거나 변경을 제안하고 싶다면, 모든 페이지에는 "제안 수정" 버튼 (오른쪽 상단)이 있거나, 문서 저장소에 대해 PR 또는 이슈를 열 수 있습니다.

## 상표

다음은 Raydium 프로젝트의 상표 (등록 또는 미등록):

- "Raydium" 이름 및 단어 마크.
- Raydium 로고 및 기호 마크.
- Raydium의 본딩 곡선 장소와 관련된 "LaunchLab" 제품명.

상표권은 **저작권 및 라이센스 권과 별개입니다**. Apache-2.0 및 GPL-3.0은 *코드* 사용 권리를 부여하지만, *Raydium 이름 및 로고* 사용 권리는 부여하지 않습니다. 상표 사용은 "Raydium으로 제공됨"과 같은 저작권 표시는 허용하지만 명시적인 합의 없이 제휴를 암시하지 않는 [Brand kit](/ko/resources/brand-kit)에 의해 관리됩니다.

코드를 포크하여 파생 프로토콜을 배포하는 경우:

- 소스 라이센스 (Apache-2.0 / GPL-3.0)를 준수합니다.
- 자신의 이름과 시각적 정체성을 선택합니다. **"Raydium X"라고 부르거나 Raydium 로고를 사용하지 마세요.**
- README에서 원본 Raydium 코드를 포크했으며 Raydium 프로젝트와 관련이 없음을 명확히 합니다.

## 보증 부인

모든 코드 및 문서는 **명시 또는 암묵적인 모든 종류의 보증 없이 "있는 그대로" 제공**되며, 상업성, 특정 목적에 대한 적합성, 비침해성을 포함하되 이에 한정되지 않습니다. 완전한 보증 부인 텍스트는 각 저장소의 `LICENSE` 파일에 있습니다.

간단히 말해: Raydium 프로젝트는 코드가 버그가 없다는 보장, 온체인 상태가 취약하지 않다는 보장, 자금 손실을 입지 않는다는 보장을 하지 않습니다. 감사 ([`security/audits`](/ko/security/audits)), 버그 바운티 ([`security/disclosure`](/ko/security/disclosure)), 운영 통제 ([`security/admin-and-multisig`](/ko/security/admin-and-multisig))는 완화 수단이지 보장이 아닙니다. 통합하거나 유동성을 제공하기 전에 자신의 위험을 평가하세요.

## 확인 방법

모든 특정 저장소의 경우, 정확한 라이센스 텍스트는 기본 브랜치의 현재 `HEAD`의 `LICENSE` 파일입니다:

```bash
# Apache-2.0 프로그램
curl -s https://raw.githubusercontent.com/raydium-io/raydium-cp-swap/master/LICENSE | head -3
curl -s https://raw.githubusercontent.com/raydium-io/raydium-clmm/master/LICENSE | head -3
curl -s https://raw.githubusercontent.com/raydium-io/raydium-amm/master/LICENSE | head -3

# GPL-3.0 SDK
curl -s https://raw.githubusercontent.com/raydium-io/raydium-sdk-V2/master/LICENSE | head -3
```

이 파일의 헤드가 이 페이지에서 주장하는 라이센스와 일치하지 않으면, 저장소가 우선됩니다. 문서에 대해 이슈를 제출하세요.
