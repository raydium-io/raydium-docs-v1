Temp API는 Raydium의 메인 API에 아직 정식으로 포함되지 않은 엔드포인트를 임시로 제공하는 공간입니다. 이 서비스는 빠른 프로토타이핑, 내부 도구, 활발히 개발 중인 기능을 위해 마련되었습니다.

## 호스트

| 환경 | 호스트 |
|------|--------|
| 메인넷 | `temp-api-v1.raydium.io` |
| 데브넷 | `temp-api-v1-devnet.raydium.io` |

## 안정성 경고

**이 서비스는 API 안정성을 보장하지 않습니다.** 엔드포인트는 사전 예고 없이 변경, 지원 중단, 다른 서비스로 이전, 또는 완전 삭제될 수 있습니다. 이 API는 다음 용도에만 사용하세요:

- 내부 도구 및 스크립트
- 활발히 개발 중인 기능
- 중단되어도 무방한 비핵심 경로
- 테스트 및 프로토타이핑

프로덕션 dApp이나 중요한 연동에는 이 API를 사용하지 마세요.

## 개요

현재 이 서비스에서 제공하는 엔드포인트는 다음과 같습니다:

- **`/cp-creator-fee`** — 특정 지갑의 CPMM 풀 전체에 걸쳐 누적된 크리에이터 수수료를 조회합니다.
- **`/limit-order/order/list`** — 인덱서 캐시에서 제공하는 지갑의 현재 대기 중인 CLMM 지정가 주문 목록(미체결 및 부분 체결 포함)입니다.
- **`/limit-order/history/order/list-by-user`** — 지갑의 과거(종료된) CLMM 지정가 주문 목록으로, `poolId` / `mint1` / `mint2` / `hideCancel` 필터를 선택적으로 적용할 수 있으며 `nextPageId`를 통해 커서 페이지네이션을 지원합니다.
- **`/limit-order/history/event/list-by-pda`** — 하나 이상의 지정가 주문 PDA에 대한 이벤트 로그(`open` / `increase` / `decrease` / `settle` / `close`)로, `nextPageId`를 통해 커서 페이지네이션을 지원합니다.

## 공통 패턴

### 지갑의 크리에이터 수수료 확인

```bash
curl -s "https://temp-api-v1.raydium.io/cp-creator-fee?wallet=<your-wallet-pubkey>"
```

해당 지갑이 크리에이터인 풀 목록과 Token A 및 Token B에 누적되었지만 아직 청구하지 않은 크리에이터 수수료를 반환합니다.

### 지갑의 현재 대기 중인 CLMM 지정가 주문 목록 조회

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/order/list?wallet=<your-wallet-pubkey>"
```

완전 미체결 주문과 부분 체결 주문을 모두 반환합니다. 각 행은 디코딩된 `LimitOrderState`(PDA, 풀, 소유자, 틱, 방향, FIFO 코호트 단계, 전체/체결 수량)와 `SettleLimitOrder` 호출에 필요한 풀 컨텍스트, 사전 계산된 `pendingSettle` 수량을 포함합니다.

활성 주문 엔드포인트는 페이지네이션 없이 지갑의 현재 캐시된 모든 주문을 한 번에 반환합니다.

### 지갑의 종료된 지정가 주문 목록 조회

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/order/list-by-user?wallet=<your-wallet-pubkey>&size=50"
```

과거 주문의 페이지를 반환하며, 각 주문은 `fillStatus`(`NONE` / `PARTIAL` / `FULL`)와 주문이 활성 세트에서 제거된 시점의 입력/출력 수량을 요약해 보여줍니다. 다음 페이지를 가져오려면 `&nextPageId=<이전 응답의 nextPageId>`를 추가하세요.

### 특정 주문의 이벤트 타임라인 조회

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/event/list-by-pda?pda=<limit-order-pda>"
```

`pda`는 단일 키 또는 쉼표로 구분된 목록을 사용할 수 있습니다. 각 행은 온체인 변경 이벤트(`open` / `increase` / `decrease` / `settle` / `close`) 하나를 나타내며, txid, 슬롯, 블록 시간, 소유자 또는 키퍼 서명 여부(`autoRunner`), 그리고 사람이 읽을 수 있는 단위의 입력/출력 델타를 포함합니다.

### 빈 결과

지갑에 캐시된 기록이 없으면 활성 주문 엔드포인트는 다음을 반환합니다:

```json
{
  "id": "...",
  "success": true,
  "data": { "rows": [] }
}
```

히스토리 엔드포인트는 동일한 `{ rows: [], nextPageId?: ... }` 형태로 빈 `rows` 배열을 반환합니다. 기존 `/cp-creator-fee` 엔드포인트는 하위 호환성을 위해 `"data": []`를 반환합니다.
