## 서비스 카탈로그

Raydium은 11개의 공개 HTTP 서비스를 운영합니다. 각각은 왼쪽 네비게이션 트리에서 자신의 그룹으로 문서화되어 있습니다.

| 서비스 | 메인넷 호스트 | 데브넷 호스트 | 기능 |
|---------|--------------|-------------|------|
| [API v3](/ko/api-reference/api-v3/overview) | `api-v3.raydium.io` | `api-v3-devnet.raydium.io` | 풀 / 민트 / 설정 / 체인 정보 — UI와 SDK가 의존하는 정규 읽기 API입니다. |
| [Transaction API](/ko/api-reference/route-api-v2/overview) | `transaction-v1.raydium.io` | `transaction-v1-devnet.raydium.io` | 서버 측 스왑 / 유동성 트랜잭션 구성. 애그리게이터 진입점입니다. |
| [Perps API](/ko/api-reference/perp-api-v1/overview) | `api-perp-v1.raydium.io` | — | Raydium Perps 프론트엔드용 설정, 자산 메타데이터, RPC 선택. |
| [LaunchLab Mint API](/ko/api-reference/launch-mint-v1/overview) | `launch-mint-v1.raydium.io` | `launch-mint-v1-devnet.raydium.io` | 토큰 검색, 인덱스, 리더보드, LaunchLab 런치별 민트 메타데이터. |
| [LaunchLab History API](/ko/api-reference/launch-history-v1/overview) | `launch-history-v1.raydium.io` | `launch-history-v1-devnet.raydium.io` | LaunchLab 풀의 거래 히스토리, K선 집계. |
| [LaunchLab Forum API](/ko/api-reference/launch-forum-v1/overview) | `launch-forum-v1.raydium.io` | `launch-forum-v1-devnet.raydium.io` | LaunchLab 런치와 연결된 댓글, 스레드, IPFS 업로드. 지갑 서명 필요. |
| [LaunchLab Auth API](/ko/api-reference/launch-auth-v1/overview) | `launch-auth-v1.raydium.io` | `launch-auth-v1-devnet.raydium.io` | 지갑 서명 메시지에서 단기 `ray-token` JWT를 발급합니다. 포럼 API 호출 시 필수. |
| [Dynamic IPFS API](/ko/api-reference/dynamic-ipfs-v1/overview) | `dynamic-ipfs.raydium.io` | `dynamic-ipfs-devnet.raydium.io` | NFT 이미지 / 메타데이터 재생성기. CLMM 포지션 아트 및 동적 상태에 따라 외형이 달라지는 온체인 NFT를 구동합니다. |
| [Owner API](/ko/api-reference/owner-api-v1/overview) | `owner-v1.raydium.io` | `owner-v1-devnet.raydium.io` | 지갑별 포지션, 잔액, 청구 가능한 보상. |
| [API v1](/ko/api-reference/api-v1/overview) | `api.raydium.io` | — | 레거시 v1/v2 엔드포인트, API v3로 마이그레이션하지 않은 클라이언트를 위해 유지됨. |
| [Temp API](/ko/api-reference/temp-api-v1/overview) | `temp-api-v1.raydium.io` | `temp-api-v1-devnet.raydium.io` | 단기 맞춤형 엔드포인트용 임시 저장소. 현재 CPMM 크리에이터 수수료 요약을 노출합니다. 예고 없이 변경될 수 있습니다. |

## 인증

대부분의 서비스는 읽기 전용이며 익명 요청을 수용합니다. 두 가지 패턴이 나타납니다:

- **지갑 서명 핸드셰이크** — `launch-auth-v1`이 `ray-token`을 발급하고 `launch-forum-v1`에서 `ray-token` 헤더로 전달하기 위해 필요합니다. `time:<unix-seconds>` 형식의 Solana ed25519 메시지를 지갑으로 서명하고, 서명 + 지갑 주소를 `launch-auth-v1` `/request-token`으로 보낸 다음 JWT를 받아 후속 포럼 호출의 `ray-token` 요청 헤더로 전달합니다.
- **인증 없음** — 다른 모든 서비스. 신뢰하는 호출자(Raydium UI, 통합자)는 HTTPS를 통해 엔드포인트에 직접 접근합니다.

Mintlify 플레이그라운드를 사용하면 포럼 요청을 보내기 전에 인증 패널에 `ray-token`을 붙여넣을 수 있으며, 이 값은 브라우저에만 유지됩니다.

## 요청 한도

모든 호스트는 소스 IP별 점진적 요청 제한이 있는 Cloudflare 뒤에 있습니다.

더 높은 한도가 필요한 통합자는 Raydium 팀에 문의해야 합니다. 공개된 한도를 초과하는 버스트는 `Retry-After` 헤더와 함께 `HTTP 429`로 반환됩니다.

## 응답 엔벨로프

대부분의 서비스는 JSON 페이로드를 통일된 엔벨로프로 감싸습니다:

```json
{
  "id": "uuid-v4-per-request",
  "success": true,
  "data": { ... }
}
```

실패 시:

```json
{
  "id": "uuid-v4-per-request",
  "success": false,
  "msg": "human-readable error string",
  "data": null
}
```

`api-v3`는 엔벨로프를 `{ "id", "success", "data" }`로 반환합니다. pump-* 서비스는 `addBorderSuccess` / `addBorderErr` 헬퍼를 사용하여 동일한 형태를 생성하되 `msg`는 오류 시에만 채워집니다. 각 서비스의 소개 페이지에서 사용하는 정확한 엔벨로프를 확인하세요.

## 이 섹션 사용 방법

왼쪽 네비게이션에서 아무 엔드포인트나 클릭하세요. 다음을 보게 됩니다:

- HTTP 메서드, 전체 URL, 기본 호스트.
- OpenAPI 스펙에서 가져온 타입의 요청 파라미터.
- 파라미터를 설정하고 메인넷(또는 가능한 경우 드롭다운에서 데브넷 서버를 선택)에 실제 요청을 보낼 수 있는 **Try it** 패널.
- 대표 라이브 응답과 소스 코드의 `addBorderSuccess` 호출 사이트에서 가져온 응답 형태.

SDK 레벨 액세스의 경우 — 클라이언트 측 트랜잭션 구성, 스왑 일괄 처리, TypeScript 타입 — [`sdk-api/typescript-sdk`](/ko/sdk-api/typescript-sdk)를 참고하세요.
