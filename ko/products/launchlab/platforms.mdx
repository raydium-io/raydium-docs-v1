<Info>
  LaunchLab의 **플랫폼(Platform)**은 Raydium의 LaunchLab 프로그램 위에 구축된 서드파티 런칭 환경입니다. 플랫폼은 자체 브랜딩을 할 수 있고, 자신의 수수료를 부과할 수 있으며, 자신의 플랫폼에서 런칭된 모든 토큰에 대해 본딩 커브 후 마이그레이션 파라미터를 구성할 수 있습니다. 별도의 온체인 프로그램을 실행할 필요가 없습니다. Raydium의 공식 런칭 UI도 많은 플랫폼 중 하나일 뿐입니다.
</Info>

## 플랫폼이 존재하는 이유

Raydium은 LaunchLab이 **공유 프리미티브(shared primitive)**가 되길 원했으며, 단일 브랜드 제품이 되길 원하지 않았습니다. 플랫폼 PDA 시스템을 통해 생태계, 도구 제공자, 커뮤니티, 밈 팀들이 동일한 기본 LaunchLab 본딩 커브 프로그램과 동일한 CPMM(또는 AMM v4) 후 마이그레이션 풀로 정착하는 큐레이션되거나 브랜드된 런칭 경험을 제공할 수 있습니다.

실제로 볼 수 있는 플랫폼의 예시:

- **Raydium의 공식 LaunchLab UI** (raydium.io/launchlab) — 참조 구현입니다.
- **파트너 런칭패드** — 자신의 플랫폼 PDA를 등록한 서드파티로, 런칭 경험을 브랜딩합니다.
- **특화된 플랫폼** — 예를 들어, 특정 테마와 관련된 밈코인만 런칭하고 자신의 트레저리를 자금 조달하는 커뮤니티 전용 플랫폼.

이 모든 토큰은 동일한 LaunchLab 프로그램에 있고, 동일한 CPMM 프로그램으로 졸업하며, 동일한 애그리게이터로 라우팅할 수 있습니다.

## 플랫폼 PDA

각 플랫폼은 LaunchLab 프로그램에 저장된 플랫폼 PDA로 표현됩니다. 필드는 다음과 같습니다:

| 필드 | 설명 |
|---|---|
| `platform_key` | 플랫폼의 시드/식별자로, 플랫폼 PDA를 유도하는 데 사용됩니다. |
| `platform_fee_destination` | 마이그레이션 전후 거래 수수료의 플랫폼 몫을 받는 계정입니다. |
| `platform_fee_bps` | 마이그레이션 전 스왑에 대한 플랫폼 수수료 비율입니다 (플랫폼마다 다름). |
| `creator_fee_enabled` | 이 플랫폼이 Burn & Earn을 통한 마이그레이션 후 크리에이터 수수료 공유를 활성화하는지 여부입니다. |
| `creator_fee_share_bps` | 활성화된 경우, 크리에이터의 Fee Key NFT로 흐르는 LP 수수료의 비율입니다. |
| `cp_config_id` | 마이그레이션 후 대상이 CPMM인 경우, 사용할 `AmmConfig`입니다 — 수수료 티어를 선택합니다. |
| `name`, `description`, `image_url` | 플랫폼 브랜딩을 지원하는 지갑 및 UI에서 사용되는 메타데이터입니다. |
| `tokens_launched` | 이 플랫폼에서 생성된 토큰의 카운터 또는 온체인 인덱스입니다. |

정확한 필드 이름은 LaunchLab IDL ([`sdk-api/anchor-idl`](/ko/sdk-api/anchor-idl))에 있습니다. 위의 표는 설명적 개요입니다.

## 플랫폼이 제어할 수 있는 것

**수수료 포착:**

- 모든 마이그레이션 전 매수/매도에 대한 **플랫폼 수수료 비율**.
- **마이그레이션 후 비율** — 토큰이 졸업할 때, LP 토큰은 플랫폼으로 정의된 다중 당사자 분할에 민팅됩니다: 소각(Burn & Earn — 소각된 LP가 풀 수수료의 자신의 몫을 계속 청구할 수 있게 하는 유동성 잠금 패턴, [`user-flows/burn-and-earn`](/ko/user-flows/burn-and-earn) 참조), 크리에이터 몫 (Fee Key NFT), 그리고 플랫폼 자체 몫.

**마이그레이션 후 대상:**

- **CPMM (크리에이터 수수료 활성화 시 기본값)** — `cp_config_id`를 통해 선택된 `AmmConfig`. 다른 `AmmConfig`는 서로 다른 수수료 티어를 가집니다. 플랫폼은 자신의 런칭이 높은 거래량의 투기 자산인 경우 더 높은 수수료 티어를 선택할 수 있습니다.
- **AMM v4 (크리에이터 수수료 비활성화 시)** — 원본 LaunchLab 흐름과의 하위 호환성을 위합니다.

**사용자 경험:**

- Raydium 앱과 LaunchLab 토큰을 표시하는 모든 지갑/탐색기에서 사용되는 **브랜딩 메타데이터**.
- **런칭 페이지 커스터마이제이션** — 자신의 프론트엔드를 호스팅하는 플랫폼은 자신의 플랫폼이 런칭한 토큰만 표시하도록 필터링할 수 있습니다.

## 플랫폼이 제어할 수 없는 것

- **본딩 커브 수학**. 곡선 모양(로지스틱, 지수 등)은 LaunchLab 프로그램 자체의 일부입니다. 플랫폼은 지원되는 곡선 중에서 선택할 수 있습니다. 임의의 커스텀 곡선을 추가할 수 없습니다.
- **마이그레이션 임계값 로직**. CPMM/AMM v4로의 졸업을 트리거하는 조건(목표 시가총액 도달)은 프로그램으로 강제됩니다.
- **핵심 보안/감사 태세**. 모든 플랫폼은 LaunchLab의 감사 이력을 상속합니다. 악의적인 플랫폼은 프로그램을 변경하여 이를 우회할 수 없습니다.

## 플랫폼 등록

LaunchLab 프로그램은 `create_platform` 명령어를 노출합니다 (IDL에서의 정확한 이름). 호출자는 다음을 전달합니다:

- 플랫폼 PDA를 유도할 시드.
- 수수료 대상 계정.
- 원하는 수수료 bps (프로그램 수준의 상한선 적용).
- 마이그레이션 후 구성 (해당하는 경우 CPMM `AmmConfig` ID).
- 브랜딩 메타데이터.

플랫폼 PDA의 렌트는 생성자가 지불합니다. 브랜딩 및 일부 수수료 파라미터에 대한 업데이트는 생성 후 플랫폼의 권한으로 허용됩니다. 구조적 파라미터(마이그레이션 대상 프로그램)는 일반적으로 허용되지 않습니다.

현재 LaunchLab IDL의 정확한 명령어 서명은 [`products/launchlab/instructions`](/ko/products/launchlab/instructions)를 참조하세요.

## 경제적 상호작용

LaunchLab 본딩 커브의 단일 스왑은 대략 다음과 같이 가치를 분배합니다 (정확한 비율은 플랫폼으로 설정됨):

```
구매자가 지불:               100 SOL
프로토콜 수수료 (LaunchLab):  – 작은 %
플랫폼 수수료:               – platform_fee_bps%
크리에이터 수수료 (마이그레이션 전): – creator_fee_bps% (활성화된 경우)
나머지 (곡선으로):           ~ 나머지
```

마이그레이션 후, LP 토큰은 플랫폼 구성에 따라 분할됩니다:

```
졸업 시 민팅된 LP 토큰:     100%
  → Burn & Earn (소각됨):    X%  (Raydium 기본값: 90)
  → 크리에이터 Fee Key NFT:  Y%  (Raydium 기본값: 10)
  → 플랫폼:                 Z%  (플랫폼 구성; 0일 수 있음)
```

**Burn & Earn 몫**은 공유 수수료 수입으로 변환되어 Fee Key 소유자에게 분배됩니다 — Raydium의 자체 기본값에서는 토큰 신뢰에 대한 광범위한 시장 수수료 지원을 나타내는 소각 "금고"입니다.

## 통합자 메모

- **플랫폼별 토큰 나열**. API 풀 정보/민트 엔드포인트를 `platform` 필터와 함께 사용합니다 (지원되는 경우). 특정 플랫폼 PDA에서 런칭된 토큰만 나열합니다.
- **브랜딩 표시**. LaunchLab 토큰을 표시하는 지갑이나 탐색기를 빌드하는 경우, 플랫폼 PDA에서 플랫폼의 이름/이미지 URL을 읽는 것이 모든 토큰에 대해 "LaunchLab"을 표시하는 것보다 훨씬 나은 UX입니다.
- **수수료 계산**. 모든 본딩 커브 견적은 프로토콜 및 크리에이터 수수료에 추가하여 플랫폼 수수료를 고려해야 합니다. SDK의 견적 헬퍼는 플랫폼 PDA가 제공될 때 자동으로 이를 수행합니다.
