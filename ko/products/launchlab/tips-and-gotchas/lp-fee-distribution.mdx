LaunchLab 토큰이 CPMM으로 마이그레이션되면, 마이그레이션된 풀이 LP 토큰을 발행합니다. 플랫폼 설정에서 이 LP 토큰에 대한 처리를 제어합니다.

## LP 분할

`migrateCpLockNftScale`는 세 개의 필드를 가집니다. 값의 합은 100%를 나타내는 `1_000_000`이어야 합니다.

| 필드 | 의미 |
|---|---|
| `platformScale` | LP 토큰은 Burn & Earn을 통해 잠깁니다. 플랫폼이 Fee Key NFT를 받습니다. |
| `creatorScale` | LP 토큰은 Burn & Earn을 통해 잠깁니다. 토큰 크리에이터가 Fee Key NFT를 받습니다. |
| `burnScale` | LP 토큰은 영구적으로 소각됩니다. 이 부분에 대해 Fee Key NFT가 생성되지 않습니다. |

이것은 플랫폼 수준의 설정입니다. 같은 플랫폼을 통해 출시된 토큰은 플랫폼이 설정을 업데이트할 때까지 동일한 LP 분배를 상속합니다.

## 예시

```ts
// 40% 플랫폼, 50% 크리에이터, 10% 소각.
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% 크리에이터 Fee Key.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% 소각.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## CPMM 수수료와의 관계

`migrateCpLockNftScale`는 **누가 LP 수수료 권리를 청구할 수 있는지**를 제어합니다. 스왑 수수료율을 설정하지는 않습니다. 마이그레이션된 풀의 스왑 수수료 동작은 `cpConfigId`로 선택된 CPMM `AmmConfig`에서 비롯됩니다.

예를 들어, 마이그레이션된 CPMM 풀이 0.25% 거래 수수료 설정을 사용하고 `creatorScale`이 500,000이면, 크리에이터의 Fee Key는 잠긴 LP 수수료 권리의 50%를 나타냅니다. `platformScale`이 400,000이면, 플랫폼의 Fee Key는 40%를 나타냅니다.

## 분할 업데이트

플랫폼 설정 업데이트는 에포크에 의해 속도 제한됩니다. LP 분배를 중요한 플랫폼 정책으로 취급하고 임의의 UI 설정으로 취급하지 마세요.

업데이트하기 전에:

- 세 개의 스케일 값이 `1_000_000`으로 합산되는지 확인합니다.
- 플랫폼이 향후 마이그레이션 후 누가 Fee Key NFT를 받을지 이해하는지 확인합니다.
- 새로운 출시가 이를 사용하기 전에 크리에이터를 위해 변경 사항을 문서화합니다.
