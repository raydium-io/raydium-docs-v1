## 마이그레이션 전: 직접 수수료 누적

토큰이 본딩 곡선에 있는 동안, 모든 매수와 매도는 토큰별 볼트에 누적되는 크리에이터 수수료 요소를 지불합니다. **크리에이터**(토큰을 출시한 지갑)는 언제든지 볼트에서 청구할 수 있습니다.

특징:

- **인용 토큰**(보통 SOL)으로 표기됩니다.
- **플랫폼**에서 설정합니다(플랫폼 설정이 크리에이터 수수료 활성 여부 및 bps를 제어합니다).
- LaunchLab UI의 Portfolio 페이지에서 언제든지 청구 가능하며, 프로그래매틱하게는 LaunchLab `claim_creator_fees` 명령어를 통해 청구할 수 있습니다(IDL의 정확한 이름).
- 졸업까지 계속되며, 졸업 시 남은 마이그레이션 전 수수료는 청구 가능하지만 새로운 마이그레이션 전 수수료는 누적되지 않습니다.

## 마이그레이션 후: Burn & Earn을 통한 Fee Key NFT

토큰의 본딩 곡선이 목표에 도달하고 유동성이 CPMM(또는 AMM v4) 풀로 마이그레이션될 때, LP 토큰이 발행되고 **플랫폼의 설정에 따라 분배됩니다**. 분배는 항상 100%를 합산하며 항상 다음의 조합을 포함합니다:

- **소각됨** — [Burn & Earn](/ko/user-flows/burn-and-earn)으로 전송되어 영구적으로 잠깁니다. 이 부분의 Fee Key는 플랫폼 제어 또는 소각 지정 계정에서 보유됩니다.
- **크리에이터** — 토큰 크리에이터의 지갑으로 직접 발행되는 Fee Key NFT입니다. 이 Fee Key는 크리에이터의 LP 토큰 쉐어에서 LP 거래 수수료를 수집할 권리를 나타냅니다.
- **플랫폼** — 선택사항입니다. 플랫폼 재무부로 흐르는 쉐어로, 또 다른 Fee Key NFT입니다.

### Raydium 자체 LaunchLab 기본값

| 쉐어 | raydium.io/launchlab의 기본값 |
|---|---|
| 소각됨(Burn & Earn, 영구 잠금) | 90% |
| 크리에이터(Fee Key NFT) | 10% |
| 플랫폼 | 0%(Raydium의 플랫폼은 플랫폼 LP 쉐어를 가지지 않습니다) |

이 기본값에서 **크리에이터만 자신의 쉐어에서 거래 수수료를 청구합니다**. 나머지 90%는 잠겨 있으며 그 수수료는 Burn & Earn으로 흐릅니다(신뢰 신호 메커니즘으로 다시 흐르며 개인의 포켓으로는 흐르지 않습니다).

### 다른 플랫폼

LaunchLab을 기반으로 구축하는 제3자 플랫폼은 자신의 분배를 설정할 수 있습니다. 마이그레이션 후 LP 수수료를 통해 자금을 조달하려는 플랫폼은 70 / 10 / 20(소각 / 크리에이터 / 플랫폼)을 설정할 수 있습니다. 이를 구동하는 Platform PDA 필드는 [`products/launchlab/platforms`](/ko/products/launchlab/platforms)를 참조하세요.

분배와 상관없이, **메커니즘은 동일합니다**: 소각되지 않은 각 쉐어는 Fee Key NFT로 표시되며, 이는 거래, 전송 또는 청구할 수 있습니다.

### 마이그레이션 후 크리에이터 수수료가 비활성화되면 어떻게 되나요?

플랫폼이 `creator_fee_enabled = false`를 설정하면, 졸업된 토큰은 Burn & Earn을 거치지 **않습니다**. 대신, 레거시 LaunchLab 마이그레이션 경로를 사용하여 **AMM v4** 풀로 마이그레이션됩니다. 이 모드에서 크리에이터는 Fee Key NFT를 받지 않으며, 프로그램 수준의 마이그레이션 후 크리에이터 수수료가 없습니다. 이 모드의 일부 크리에이터는 오프체인 CLI 도구를 사용하여 `creatorFeeOn`을 통해 설정된 선택적 마이그레이션 후 수수료를 청구합니다 — 온체인 표면은 [`products/launchlab/instructions`](/ko/products/launchlab/instructions)를 참조하세요.

## 0.1% 토큰 쉐어 레퍼럴

LP 수수료 공유와 별개로, LaunchLab 크리에이터(및 모든 사용자)는 자신의 지갑을 레퍼러로 하는 **토큰 페이지 링크**를 공유할 수 있습니다. 공유된 링크를 통해 해당 토큰(또는 같은 세션의 다른 LaunchLab 토큰)을 거래할 때:

- **모든 거래의 0.1%**이 레퍼러의 지갑으로 에어드롭됩니다.
- SOL 또는 풀의 인용 토큰으로 지불됩니다.
- 모든 거래에서 자동으로 지불되며, 청구 단계가 없습니다.

이는 [`user-flows/referrals-and-blinks`](/ko/user-flows/referrals-and-blinks)에서 다루는 1% 스왑 레퍼럴과는 다른 상품입니다. 1% 레퍼럴은 Raydium의 현물 스왑 UI에 적용되고, 0.1% 토큰 쉐어 레퍼럴은 LaunchLab 토큰 페이지에만 적용됩니다.

## 수수료 청구

### 크리에이터 볼트(마이그레이션 전)

```ts
import { Raydium, TxVersion } from "@raydium-io/raydium-sdk-v2";
const raydium = await Raydium.load({ owner, connection });
const { execute } = await raydium.launchlab.claimCreatorFees({
  mint,           // launched token mint
  txVersion: TxVersion.V0,
});
await execute({ sendAndConfirm: true });
```

### Fee Key NFT(마이그레이션 후)

지갑에 Fee Key를 보유하는 것만으로 충분합니다 — Portfolio 페이지에서 청구하거나, 관련 풀 프로그램의 수수료 청구 명령어를 통해 프로그래매틱하게 청구합니다. 온체인 표면은 [`products/cpmm/instructions`](/ko/products/cpmm/instructions) 또는 해당하는 AMM v4를 참조하세요.

## 통합자 체크리스트

- **두 가지 수수료 흐름을 표시합니다.** "마이그레이션 전 볼트" / "마이그레이션 후 Fee Key" 중 하나만 표시하는 크리에이터 대시보드는 다른 것의 존재를 모르는 사용자에게 기회를 잃게 합니다.
- **Fee Key 소유권 이전을 존중합니다.** 크리에이터가 Fee Key NFT를 판매하거나 전송하면, 이전 소유자는 향후 수수료 청구에 접근할 수 없습니다. UI는 원래 크리에이터 주소를 캐시하지 않고 매번 현재 NFT 소유자를 읽어야 합니다.
- **플랫폼 설정 드리프트.** Raydium의 기본값은 변경될 수 있습니다. Raydium 자체 플랫폼이 아닌 다른 플랫폼의 경우, 항상 Platform PDA를 라이브로 읽어 해당 특정 출시의 실제 소각 / 크리에이터 / 플랫폼 분배를 확인합니다.
- **Token-2022 담보 및 수수료 토큰.** 크리에이터 수수료는 전송 수수료가 있는 Token-2022 토큰으로 표기될 수 있습니다. 표시된 "청구 가능한 수수료" 숫자에 전송 수수료 감소를 반영합니다.
