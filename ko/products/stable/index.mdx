## 개요

Stable AMM은 **Raydium의 독립 프로그램**(AMM v4 프로그램의 모드가 아닌 별도 배포)으로, 알려진 안정적인 가격 관계를 가진 쌍(주로 스테이블코인-스테이블코인 쌍(USDC-USDT, USDH-USDC) 및 담보화된 토큰 쌍)을 위해 설계되었습니다. 상수곱 곡선 대신 각 풀의 `ModelDataInfo` 계정에 내장된 **(x, y, price) 튜플의 조회 테이블**을 사용합니다. 가격 발견은 테이블 내 보간을 통해 이루어지며, 소규모 스왑에서 더 낮은 슬리피지를 제공합니다.

이는 **순수 AMM**입니다. 모든 유동성은 풀의 자체 금고에 있습니다. 초기 OpenBook 마켓 메이킹 경로는 수년간 비활성 상태였으며, [2026-06-22 업그레이드](/ko/reference/changelog)에서 남은 마켓 코드를 제거했습니다. AMM v4와 동일한 `Fees` 구조 및 수수료 모델을 공유하며, 차이점은 순전히 **가격 곡선**입니다. x·y=k 대신 사전에 채워진 모델을 보간합니다.

**프로그램 ID:** [reference/program-addresses](/ko/reference/program-addresses)를 참조하세요.

**Token-2022:** 지원되지 않습니다. 클래식 SPL 토큰만 지원합니다.

**유동성:** 제한적입니다. 대부분의 사용자 대면 통합은 모든 Raydium AMM에서 최고 가격 라우팅을 위해 [AMM Routing 프로그램](/ko/products/routing)을 통해 Stable 풀에 접근합니다. SDK도 Stable 풀을 직접 지원합니다. 풀은 `ApiV3PoolInfoStandardItem`에서 `version: 5`(`pooltype: "StablePool"`)로 표시되며, 표준 `LiquidityModule`(`addLiquidity` / `removeLiquidity` / swap)은 v5를 자동으로 감지하고 올바른 명령어를 내보냅니다. [코드 데모](/ko/products/stable/code-demos)를 참조하세요.

## 읽어야 할 때

- 스테이블코인 또는 자산 쌍을 통합하고 있으며 최고 가격의 Stable AMM 풀을 찾았습니다.
- 라우팅 엔진을 구축 중이며 Stable 풀을 유동성 소스로 지원해야 합니다.
- 기존 Stable 풀에서 유동성을 유지 중입니다. 예치 / 인출 흐름을 배우고 있습니다.
- AMM 가격 책정이 조회 테이블을 통해 프로그래밍 가능하게 만들어지는 방식에 대해 궁금합니다.
