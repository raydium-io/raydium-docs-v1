## 결제 통화 대 담보

- **결제 통화: USDC.** 모든 손익, 펀딩 지불, 청산 회계는 입금한 자산이 무엇이든 상관없이 USDC로 처리됩니다.
- **담보 자산: 여러 개.** USDT, SOL, ETH, BNB, WBTC, YUSD, USD1 (및 USDC 자체)을 입금할 수 있습니다. USDC가 아닌 담보는 현재 시장 가치에 헤어컷을 적용한 후 USDC 상당액으로 증거금에 포함됩니다.

실질적 결과: SOL을 입금했는데 SOL 가격이 절반으로 떨어지면, **변동성 거래 포지션이 움직이지 않았더라도** 증거금 비율이 악화됩니다. 운영 환경에서 이는 USDC를 입금했을 때는 존재하지 않는 의미 있는 위험입니다.

## 글로벌 한도 (모든 사용자가 공유)

이 한도는 **전체 프로토콜**에 적용되며, 계정별이 아닙니다. 자산의 글로벌 한도에 도달하면, 다른 사용자가 출금할 때까지 해당 체인에서 그 자산의 새로운 입금이 거부됩니다.

수량은 USDC 상당액이 아닌 자산 자체의 단위로 표기됩니다(SOL은 SOL, ETH는 ETH, USDT는 USDT 등).

| 담보 | 체인 | 최대 (글로벌, 자산 단위) |
|---|---|---|
| USDT | Arbitrum | 2,000,000 USDT |
| ETH | Arbitrum | 1,000 ETH |
| ETH | Base | 500 ETH |
| BNB | BSC | 10,000 BNB |
| USDT | BSC | 4,000,000 USDT |
| USD1 | BSC | 1,000,000 USD1 |
| YUSD | BSC | 500,000 YUSD |
| ETH | Ethereum | 500 ETH |
| USDT | Ethereum | 3,000,000 USDT |
| USD1 | Ethereum | 1,000,000 USD1 |
| YUSD | Ethereum | 500,000 YUSD |
| WBTC | Ethereum | 80 WBTC |
| SOL | Solana | 40,000 SOL |

## 사용자별 한도

이 한도는 **개별 지갑/변동성 거래 계정별**로 적용됩니다.

| 담보 | 계정별 최대 (자산 단위) |
|---|---|
| USDT | 500,000 USDT |
| ETH | 100 ETH |
| SOL | 2,000 SOL |
| BNB | 500 BNB |
| YUSD | 50,000 YUSD |
| WBTC | 5 WBTC |
| USD1 | 500,000 USD1 |

USDC는 결제 자산이므로 명시된 사용자별 한도가 없습니다. 실질적인 한도는 당시 플랫폼/Orderly 측 한도에서 허용하는 범위입니다.

## 입금 및 출금 흐름

**입금.** 사용자가 원본 체인에서 자산을 Orderly 볼트로 이동하는 온체인 거래에 서명합니다. Orderly의 릴레이어가 입금을 확인하고 Solana 변동성 거래 계정 원장에 크레딧을 입금합니다. 크로스체인 입금(예: Arbitrum의 ETH)은 릴레이어가 원본 체인의 최종성을 기다려야 하므로 네이티브 SOL 입금보다 시간이 오래 걸립니다.

**출금.** 사용자가 오프체인 요청에 서명하면, Orderly가 원장에서 차변을 기록하고 목적지 체인에서 온체인 전송을 발행합니다. **출금당 1 USDC 고정 수수료**(Raydium이 아닌 Orderly에서 설정)가 적용됩니다. 출금으로 인해 증거금 비율이 필수 임계값 이하로 떨어질 경우 출금이 실패합니다. 열린 포지션을 뒷받침하는 담보는 출금할 수 없습니다.

## 증거금 회계

혼합 담보가 있는 크로스마진 계정의 경우:

```
margin_usdc_equivalent
  = sum over collateral assets (
      asset_balance
      × asset_price_usd
      × (1 − haircut_fraction)
    )

free_margin  = margin_usdc_equivalent − initial_margin_used
margin_ratio = margin_usdc_equivalent / maintenance_margin_required
```

- `asset_price_usd`는 Solana 풀이 아닌 Orderly의 가격 오라클에서 옵니다.
- `haircut_fraction`은 담보 가격 폭락으로부터 보호합니다. USDC는 보통 0%이고, SOL이나 ETH 같은 변동성 담보는 상당한 헤어컷을 적용합니다.
- `maintenance_margin_required`는 열린 포지션들에 대해 각 포지션의 유지 증거금 요구사항을 합한 것입니다.

## 통합자를 위한 주의사항

- **헤어컷 없이 담보를 USDC 상당액으로 모델링하지 마세요.** 실제 구매력이 상당히 낮을 수 있습니다.
- **원본 체인 확인을 모니터링하세요.** 크로스체인 입금은 Orderly가 확인할 때까지 사용할 수 없으며, Ethereum에서 방금 ETH를 입금한 사용자는 몇 분 동안 여전히 "증거금 부족"을 볼 수 있습니다.
- **사용자별 한도를 준수하세요.** 한도를 확인하지 않는 자동화된 입금 논리는 사용자가 한도에 가까워질 때 간헐적으로 실패합니다.
- **USDC는 유일한 "고정" 담보입니다.** 담보와 결제 통화 간의 FX와 같은 효과를 처리할 수 없는 포지션 규모 조정 시스템의 경우, USDC 전용 입금을 요구하세요.
