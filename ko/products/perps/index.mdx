## 개요

**Raydium Perpetual**은 수수료 무료 중앙 지정가 주문서(CLOB) 기반 영구선물 프로토콜입니다. 포지션은 Solana의 SPL 지갑에서 자금을 조달하지만, 매칭 엔진과 주문서는 [Orderly Network](https://orderly.network/) 에서 운영합니다. Orderly Network는 다중 체인 거래소 세트 (Solana, Ethereum, Base, Arbitrum, BSC) 에서 유동성을 집계합니다.

주요 특징:

- **수수료 무료 거래.** 주문 생성, 수정, 취소는 Solana의 온체인 거래를 필요로 하지 않습니다. 서명은 로컬에서 무료로 이루어집니다. 입금과 출금만 온체인 처리됩니다.
- **최대 100배 레버리지** (시장마다 다름).
- **다중 체인 담보** — USDC / SOL / USDT / ETH / BNB / WBTC / YUSD / USD1을 각 체인에서 입금할 수 있으며, 모든 손익은 USDC로 정산됩니다.
- **교차 마진만 제공.** 계정의 모든 담보는 모든 오픈 포지션 간에 공유됩니다.
- **일방향 모드.** 동일한 시장에서 롱과 숏을 동시에 보유할 수 없습니다. 반대 방향 주문은 기존 포지션을 감소하거나 뒤집습니다.
- **자기 보관.** 자금은 지갑의 영구선물 계정에 남아 있습니다. Raydium이 보관하는 커스터디 지갑은 없습니다.
- **거래소 수준의 주문 유형.** Market, Limit, Stop (market / limit), Scale, 그리고 IOC / FOK / Post-Only / Reduce-Only 플래그.

## 별개 제품인 이유

Raydium AMM (현물) 과 Raydium Perps (영구선물) 는 브랜드와 UI 진입점을 공유하지만 프로토콜 수준에서는 거의 동일한 것이 없습니다:

| | Raydium 현물 AMM | Raydium Perps |
|---|---|---|
| 거래소 | Solana의 온체인 풀 | Orderly에서 운영하는 오프체인 CLOB |
| 매칭 엔진 | AMM 곡선 (CPMM / CLMM / AMM v4) | 중앙 지정가 주문서 |
| 정산 | 스왑 시 온체인 | 오프체인; 입금/출금은 온체인 |
| 가격 발견 | AMM 곡선 + 차익거래 | 주문서 + 외부 가격 |
| 토큰 범위 | 모든 SPL / Token-2022 민트 | 나열된 영구선물 시장만 (BTC, ETH, SOL 및 기타 주요 자산) |
| [`reference/program-addresses`](/ko/reference/program-addresses) 의 프로그램 ID | 있음 | 없음 — Orderly의 계약 참고 |
| Raydium SDK v2 커버리지 | 전체 | 미포함 |
| Raydium 버그 바운티 | 있음 | Orderly를 통해 라우팅 |

이 문서 사이트의 나머지 부분은 현물 AMM과 Raydium 조직의 GitHub의 프로그램에 거의 전적으로 초점을 맞춥니다. 영구선물은 제품 표면 수준으로 문서화됩니다 (주문 유형, 수수료, 담보, 제한 사항) — 독자들이 그것이 무엇인지와 언제 사용해야 하는지를 이해하도록 하기 위해, 하지만 심층 프로토콜 참조는 Orderly 쪽에 있습니다.

## 제한된 관할권

Raydium Perps는 미국, 아프가니스탄, 벨라루스, 중앙아프리카공화국, 콩고민주공화국, 북한, 우크라이나의 크림반도 / 도네츠크 / 루한스크 지역, 쿠바, 이란, 리비아, 소말리아, 수단, 남수단, 시리아, 예멘, 짐바브웨 또는 Raydium 프로토콜이 금지된 기타 관할권의 거주자에게 **이용할 수 없습니다.**

영구선물 UI 또는 API를 임베드하는 통합자는 이 목록을 준수해야 합니다.

## 통합 참고 사항

- Raydium 현물 SDK (`@raydium-io/raydium-sdk-v2`) 는 영구선물을 **포함하지 않습니다.** 프로그래매틱 접근을 위해 [Orderly의 SDK](https://orderly.network/docs/build-on-omnichain/evm-api/introduction) (TypeScript, REST, WebSocket) 를 사용하세요.
- "스왑을 영구선물 포지션을 통해 라우팅" 하는 기본 요소는 없습니다. 현물과 영구선물 익스포저를 모두 원하는 클라이언트는 두 거래를 독립적으로 구성합니다.
- Orderly의 위험 파라미터 (레버리지 상한, 청산 임계값, 담보 한도) 는 시간에 따라 변합니다. 사용자에게 위험을 인용하기 전에 항상 Orderly의 API에서 라이브 파라미터를 가져오세요.
