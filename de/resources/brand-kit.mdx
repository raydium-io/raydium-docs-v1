## Name und Großschreibung

| Form | Verwendung |
|---|---|
| **Raydium** | Der Protokollname. Immer im Titalkasus. Nie `raydium`, nie `RAYDIUM`. |
| **Raydium Protocol** | Akzeptabel zur Unterscheidung vom Namen Raydium in anderen Kontexten (es ist auch ein chemisches Element, ein Videospiel usw.). |
| **\$RAY** oder **RAY** | Raydiums natives SPL-Token. Mit dem Dollar-Präfix im Marketing-Text, einfach `RAY` in technischen Kontexten oder Tabellen. |
| **Raydium AMM v4** / **Raydium CPMM** / **Raydium CLMM** / **LaunchLab** | Produktnamen. LaunchLab ist ein Wort mit großem L. |
| **Burn & Earn** | Eine Konstruktion mit dem Ampersand, beide Nomen großgeschrieben. |

Schreiben Sie nicht `Raydium DEX`, `Ray protocol` oder `Raydiums` (kein Plural). Das Protokoll ist ein Singularnomen.

## Logos

Das Raydium-Logo gibt es in zwei Formen:

- **Wordmark** — das vollständige „Raydium"-Schrift-Lockup. Verwenden Sie dies in Kopfzeilen, Pressematerialien und überall dort, wo der Name lesbar sein muss.
- **Symbol-Marke** — das eigenständige Monogramm. Verwenden Sie dies für App-Symbole, Favicons und begrenzte Bereiche (Avatare, Social-Media-Profilbilder), in denen das Wordmark bei lesbarer Größe nicht passt.

### Hell- und Dunkelversionen

Zwei Ausführungen pro Form:

| Datei | Hintergrund | Verwendung wenn |
|---|---|---|
| `logo-light.svg` | Dunkle Hintergründe | Primäre Platzierung auf dunklen UI-Oberflächen. |
| `logo-dark.svg` | Helle Hintergründe | Primäre Platzierung auf hellen UI-Oberflächen. |
| `mark-light.svg` | Dunkle Hintergründe | Nur Symbol auf dunklen Oberflächen. |
| `mark-dark.svg` | Helle Hintergründe | Nur Symbol auf hellen Oberflächen. |

Alle Assets sind Vektor-SVGs. Verwenden Sie SVG wann immer möglich. Rastern Sie nur bei der endgültigen Rendergröße zu PNG (vermeiden Sie skalierte PNGs).

### Schutzbereich

Bewahren Sie einen Schutzbereich um das Logo herum, der der Höhe des Raydium-Zeichens „R" entspricht. Crowden Sie das Logo nicht mit Text, Symbolen oder Grafiken in diesem Bereich.

### Minimale Größe

Wordmark: nicht unter **120 px** Breite auf dem Bildschirm oder **20 mm** im Druck rendern.
Symbol: nicht unter **24 px** Breite auf dem Bildschirm oder **6 mm** im Druck rendern.

Unterhalb dieser Schwellwerte verliert das Logo die Lesbarkeit und Partner sollten zur alternativen Form wechseln (z. B. das Wordmark für das Symbol fallen lassen, wenn der Platz begrenzt ist).

## Farbpalette

Die Raydium-Identität ist um ein Violett-Primär mit dunkleren und helleren Varianten für Status, Tiefe und Kontrast aufgebaut.

| Rolle | Hex | RGB | Verwendung |
|---|---|---|---|
| **Primary** | `#8C6EFF` | 140 / 110 / 255 | Hauptmarkenfarbe. CTAs, Links, wichtige Akzente. |
| **Light** | `#A88BFF` | 168 / 139 / 255 | Hover-Zustände, sekundäre Akzente, Hell-Design-Hintergründe. |
| **Dark** | `#5E3BE1` | 94 / 59 / 225 | Aktive/gedrückte Zustände, Dunkel-Design-Akzente, Tiefe. |

Unterstützende Neutraltöne:

| Rolle | Hex | Verwendung |
|---|---|---|
| **Foreground (light)** | `#0A0A0F` | Body-Text auf hellen Oberflächen. |
| **Foreground (dark)** | `#FAFAFA` | Body-Text auf dunklen Oberflächen. |
| **Surface (light)** | `#FFFFFF` | Seitenhintergrund, Hell-Design. |
| **Surface (dark)** | `#0F0F1A` | Seitenhintergrund, Dunkel-Design. |
| **Border** | `#E5E5EB` (light) / `#1F1F2E` (dark) | Karten- und Trennlinien-Ränder. |

Färben Sie das Logo nicht um. Das Wordmark und Symbol werden immer entweder in der hellen oder dunklen Master-Ausführung gerendert — nie getönt, nie Gradient-ersetzt, nie durch Brand-Farben verarbeitet.

## Typografie

Die Docs und Website verwenden **Inter** für UI und Body-Copy. Überschriften verwenden Inter mit starken Schnitten (600–800). Code-Blöcke verwenden **JetBrains Mono** oder einen beliebigen monospaced-Fallback.

| Stil | Familie | Schnitt | Verwendung |
|---|---|---|---|
| Display / H1 | Inter | 800 | Seitentitel. |
| H2–H4 | Inter | 700 | Abschnitts-Header. |
| Body | Inter | 400 | Absatztext. |
| Emphasis | Inter | 600 | Inline-Hervorhebung, starker Text. |
| Code | JetBrains Mono | 400–500 | Inline `code` und Code-Blöcke. |

Inter ist open-lizenziert und verfügbar von [rsms.me/inter](https://rsms.me/inter).

## Logo-Regeln

**Richtig:**

- Verwenden Sie die offiziellen SVG-Master aus dem Docs-Repo im `/logo/`-Verzeichnis.
- Bewahren Sie Schutzbereich und Minimale Größe.
- Paaren Sie das Wordmark mit dem Symbol nur, wenn beide visuell ausgewogen sind (zeigen Sie nicht zwei nebeneinander in konfliktierenden Größen).
- Rendern Sie auf einem Hintergrund, der ausreichend Kontrast hat — verwenden Sie den hellen Master auf dunkel, dunklen Master auf hell.

**Falsch:**

- Färben Sie das Logo um. Kein violettes Wordmark, kein Regenbogen, keine monochromen Varianten außer den bereitgestellten Hell/Dunkel.
- Verzerren Sie das Seitenverhältnis (kein Dehnen, kein Schrägen).
- Fügen Sie Schlagschatten, Umrisse, Glühen oder andere Effekte hinzu.
- Platzieren Sie das Logo auf einem belebten Hintergrundbild ohne einen Kontrast bewahrenden durchgehenden Panel darunter.
- Schneiden Sie das Logo zu (zeigen Sie nicht nur das „R" aus dem Wordmark zugeschnitten — verwenden Sie stattdessen die Symbol-Marke).
- Verwenden Sie das Logo, um die Unterstützung eines Produkts zu implizieren, mit dem sich Raydium nicht formal zusammengetan hat. Verwenden Sie ein „Powered by Raydium"- oder „Integrated with Raydium"-Abzeichen in einem neutralen Kontext, ohne Implikation der Überprüfung.

## „Powered by Raydium"-Abzeichen

Wenn Ihre Integration Raydium als Routing-Quelle oder Liquiditätsvenue verwendet, ist eine einfache Text- oder Badge-Nennung willkommen:

```text
Powered by Raydium
Integrated with Raydium
Liquidity via Raydium
```

Diese Formulierungen sind vorab genehmigt. Platzieren Sie in der Nähe der Preis-, Swap- oder Pool-Oberfläche Ihrer Integration — nicht in Masthead-Positionen, die eine formale Partnerschaft implizieren würden.

Wenn Sie ein Aggregator oder eine Wallet sind, die durch Raydium routet, und eine prominentere Nennung möchten, kontaktieren Sie das Team über Discord (`#partnerships`), bevor Sie veröffentlichen.
