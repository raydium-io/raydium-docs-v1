## Schreibbrief

- Jeder Leitfaden beginnt mit „das ist das Problem" (z. B. „meine Swap-Transaktionen schlagen unter Last fehl") und endet mit einem validierten Rezept.
- Fügen Sie Benchmarks hinzu, wenn sie aussagekräftig sind (z. B. typische CU-Nutzung für einen CPMM-Swap gegenüber einem CLMM-Swap).
- Verlinken Sie zu SDK & API für die zugrunde liegenden Primitives; definieren Sie diese nicht neu.
