Die Owner API bietet Pro-Wallet-Übersichten über Raydium-Positionen und Guthaben. Sie ruft zwischengespeicherte Daten für Positionen, gesperrte Liquidität, erstellte Pools und IDO-Teilnahmen ab und eignet sich daher ideal für Wallet-Oberflächen und Portfolio-Dashboards.

## Hosts

| Umgebung | Host                       |
|-------------|---------------------------|
| Mainnet     | `owner-v1.raydium.io`     |
| Devnet      | `owner-v1-devnet.raydium.io` |

## Übersicht

Die API stellt fünf Hauptendpunkte bereit, die nach Ressourcentyp organisiert sind:

- **`/position/stake/{owner}`** — Stake-Positionen (Standard-AMM / CPMM LP)
- **`/position/clmm-lock/{owner}`** — Gesperrte CLMM-Positionen
- **`/create-pool/{owner}`** — Von dieser Wallet erstellte Pools
- **`/main/ido/{owner}`** — Daten zur IDO-Teilnahme

## Datenfrilichkeit

Dieser Service speichert Positionsdaten zwischen und aktualisiert sie regelmäßig. Sehr aktuelle Positionsänderungen (innerhalb von Sekunden) werden möglicherweise nicht sofort in den Antworten angezeigt.

## Häufige Muster

### Alle Stake-Positionen einer Wallet abrufen

```bash
curl -s "https://owner-v1.raydium.io/position/stake/{owner}"
```

Ersetzen Sie `{owner}` durch den öffentlichen Schlüssel der Wallet im Base58-Format.

### Gesperrte Positionen abrufen

```bash
curl -s "https://owner-v1.raydium.io/position/clmm-lock/{owner}"
```

### IDO-Teilnahme prüfen

```bash
curl -s "https://owner-v1.raydium.io/main/ido/{owner}"
```

## Leere Ergebnisse

Wenn eine Wallet keine Daten für einen bestimmten Endpunkt hat, gibt der Service einen 404-Fehler mit Anfragemetadaten zurück:

```json
{
  "id": "...",
  "success": false
}
```
