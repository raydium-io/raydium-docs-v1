## Was Burn & Earn ist

Burn & Earn ist eine Raydium-Funktion, die es einem Pool-Ersteller (oder jedem LP) ermöglicht, eine Einzahlung **dauerhaft** on-chain zu machen, während das Recht auf Handelsgebühren erhalten bleibt. Eine Position wird nicht mehr zurückzahlbar, aber ein neu geprägtes **Raydium Fee Key** NFT stellt das laufende Recht dar, Gebühren (`collectFees`) von dieser gesperrten Position einzufordern.

Häufige Anwendungsfälle:

- **Launch-Vertrauen.** Projekte sichern ab, nicht rug-zu-pullen, indem sie den LP des Bereitstellers sperren. Inhaber können on-chain überprüfen, dass die Liquidität dauerhaft ist.
- **Gebührentragende Sammelstücke.** Ein handelbares NFT, das Rendite generiert. Verkaufen Sie das NFT; übertragen Sie den Gebührenstrom.
- **Treasury-grade LP.** Protokolle, die Liquidität auf Dauer für Marken- / Erkennungszwecke halten möchten und dabei die Gebührenseite monetarisieren.

## Unterstützte Pool-Typen

| Programm | Burn & Earn Unterstützung |
|---|---|
| CPMM | Ja |
| CLMM | **Nur vollständiger Bereich** (empfohlen) |
| AMM v4 (Hybrid) | Nicht unterstützt |

Bei CLMM sollten nur Positionen, die den gesamten Preisbereich umfassen, gesperrt werden. Nach der Sperrung können Sie nicht neu ausbalancieren – wenn der Preis Ihren gewählten Bereich verlässt, verdienen Sie auf dieser Position dauerhaft keine Gebühren mehr. Eine enge Burn & Earn Position ist fast immer ein Fehler.

## Wie die Sperrung auf Programmebene funktioniert

1. Der ursprüngliche LP-Token (CPMM) oder die Position NFT (CLMM) wird auf ein PDA übertragen, das vom Burn & Earn Programm gesteuert wird.
2. Ein **Fee Key NFT** wird für den Aufrufer geprägt. Seine Metadaten verweisen auf das PDA der gesperrten Position.
3. Burn & Earn stellt `collectFees` (oder den entsprechenden Pool-Typ-spezifischen Aufruf) bereit, der:
   - Überprüft, dass der Unterzeichner das Fee Key NFT für die angegebene gesperrte Position hält.
   - Die Fee-Claim-Anweisung des zugrunde liegenden Pool-Programms über CPI aufruft.
   - Die einbehaltenen Gebühren an die ATAs des Fee Key Inhabers leitet.
4. Es gibt keinen entsprechenden `withdraw` – der zugrunde liegende LP / die Position verlässt den Escrow nie.

Das Fee Key ist ein Standard-SPL-NFT. Es kann auf jedem Solana-NFT-Marktplatz verkauft, zwischen Wallets verschoben, als Sicherheit bei Leihprotokollen verwendet, die NFTs akzeptieren, oder in einen anderen Vertrag eingewickelt werden. Wer es zum Zeitpunkt eines `collectFees` Aufrufs hält, erhält die Gebühren.

## UI-Walkthrough

1. Erstellen Sie einen CPMM-Pool oder einen CLMM-Pool mit einer Vollbereichs-Position wie gewohnt.
2. Gehen Sie zu **Liquidity → Create → Burn & Earn**.
3. Wählen Sie die zu sperrende Position.
4. Überprüfen Sie die Positionsdetails (NFT-Adresse, Größe, Pool) und geben Sie den Bestätigungssatz ein, den die UI anzeigt. Dies ist absichtlich reibungsreich, da der Vorgang irreversibel ist.
5. Klicken Sie auf **Confirm, lock the liquidity permanently**.
6. Ein Fee Key NFT kommt in Ihre Wallet an. Es wird auf der Portfolio-Seite unter gesperrten Positionen angezeigt.

Fordern Sie Gebühren auf die gleiche Weise ein wie bei jeder Position – über die Portfolio-Seite oder über `collectFees`.

## API-Oberfläche

Der Pool-Info-Endpunkt gibt ein `burnPercent` Feld zurück, das den Anteil der Liquidität dieses Pools anzeigt, der über Burn & Earn gesperrt wurde:

```http
GET https://api-v3.raydium.io/pools/info/list
  ?poolType=standard
  &poolSortField=default
  &sortType=desc
  &pageSize=1
  &page=1
```

```json
{
  "data": [
    {
      "id": "...",
      "burnPercent": 99.17,
      ...
    }
  ]
}
```

`burnPercent` ist eine praktische Oberfläche für Vertrauenssignalisierung: ein Token, dessen LP zu 99%+ gebrannt ist, ist dramatisch weniger rug-fähig als einer mit 0% gebranntem LP.

## Fallstricke

- **Irreversibel.** Die LP-Seite kann nie abgerufen werden. Bestätigen Sie, dass der Betrag etwas ist, das Sie nie wieder anfassen möchten.
- **CLMM enge Bereiche sind eine Falle.** Sperren Sie nur Vollbereichs-CLMM-Positionen, es sei denn, Sie verstehen wirklich, dass eine bereichsgebundene gesperrte Position aufhört zu verdienen, sobald der Preis das Band verlässt.
- **Das Fee Key NFT zu verlieren ist, die Gebühren für immer zu verlieren.** Behandeln Sie es wie eine Urkunde. Verbrennen Sie es nicht versehentlich; stecken Sie es nicht in einen feindsligen Vertrag.
- **Token-2022-Mints mit Transfergebühren**: Burn & Earn unterstützt sie, aber die Transfer-Fee-Kürzung gilt bei jedem Fee-Claim – budgetieren Sie dafür.
- **Es gibt keine teilweise Sperrung.** Wenn Sie 50% gesperrt und 50% beweglich haben möchten, teilen Sie die Position zunächst in zwei Positionen auf und sperren dann nur eine.
