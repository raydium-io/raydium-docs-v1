<Info>
  Creator-Gebühren funktionieren nach zwei völlig unterschiedlichen Mechanismen — einer für die Bonding-Curve-Phase und einer für die Post-Graduation-AMM-Phase. Sie werden unterschiedlich finanziert, unterschiedlich verwahrt und unterschiedlich ausgezahlt. Beide sind hier beschrieben.
</Info>

<Warning>

</Warning>

## Phase 1 – Vor der Migration (Bonding Curve)

Während Ihr Token auf der Bonding Curve gehandelt wird, wird eine Creator-Gebührenkomponente von jedem Kauf und Verkauf abgezogen und in einem Token-spezifischen Vault akkumuliert. Dies wird von der **Plattform** konfiguriert, auf der Ihr Token gestartet wird.

### Details

- **Denominiert in** dem Quote-Token (normalerweise SOL).
- **Rate** – wird von der Plattform zum Zeitpunkt des Launches festgelegt. Überprüfen Sie den Gebührenplan der Plattform oder den On-Chain-Status des Tokens.
- **Jederzeit abhebbar** – gehen Sie auf raydium.io zu Ihrer **Portfolio**-Seite und fordern Sie ausstehende Gebühren jederzeit ab. Kein Warten auf Abschluss erforderlich.
- **Endet beim Abschluss** – nach dem Abschluss der Curve werden keine neuen Gebühren vor der Migration akkumuliert. Jeder Restbetrag im Vault bleibt abhebbar.

### So fordern Sie Gebühren ab

1. Verbinden Sie die Creator-Wallet mit [raydium.io](https://raydium.io).
2. Navigieren Sie zu **Portfolio**.
3. Finden Sie den Token unter Ihren erstellten Tokens.
4. Klicken Sie auf **Claim** neben dem Gebührensaldo vor der Migration.

Programmgesteuerte Anforderung: Siehe [`products/launchlab/creator-fees`](/de/products/launchlab/creator-fees) für die SDK-Anweisung.

## Phase 2 – Nach der Migration (Fee Key NFT)

Wenn die Bonding Curve Ihres Tokens die Abschluss-Schwelle erreicht, wird die Liquidität zu einem Raydium CPMM-Pool migriert. LP-Tokens werden gemäß der **Plattformkonfiguration** geprägt und verteilt:

| Empfänger | Raydium-Standard | Beschreibung |
|---|---|---|
| Verbrannt (Burn & Earn) | 90 % | Dauerhaft gesperrt; Handelsgebühren aus dieser Quote fließen in den Burn & Earn-Vertrauensmechanismus. |
| Creator (Fee Key NFT) | 10 % | Ein Fee Key NFT wird in Ihrer Wallet geprägt. Dieses NFT ist Ihr Anspruch auf 10 % der laufenden LP-Handelsgebühren des Pools. |
| Plattform | 0 % (Raydium-Standard) | Einige Drittanbieter-Plattformen nehmen hier einen Anteil. |

Diese Standards gelten für Tokens, die über Raydiums eigene LaunchLab-Benutzeroberfläche gestartet werden. Drittanbieter-Plattformen können unterschiedliche Aufteilungen konfigurieren – überprüfen Sie immer die Plattformseite vor dem Launch.

### Das Fee Key NFT

Das Fee Key NFT ist ein nicht-fungibles Token, das bei Abschluss in Ihre Wallet geprägt wird und Ihr Recht darstellt, LP-Handelsgebühren aus Ihrem Anteil am Post-Graduation-Pool zu erfassen.

<Warning>
  **Verbrennen oder übertragen Sie das Fee Key NFT nicht, es sei denn, Sie möchten Ihre Gebührenanspruchsrechte gezielt aufgeben.** Wenn das NFT verloren geht oder verbrannt wird, gehen die Gebührenanspruchsrechte **dauerhaft verloren** – es gibt keinen Wiederherstellungsweg.
</Warning>

- Das Fee Key kann gehandelt oder übertragen werden; wer es hält, kann Gebühren einfordern.
- Gebühren sammeln sich kontinuierlich aus der LP-Handelsaktivität im CPMM-Pool an.
- Fordern Sie von der Portfolio-Seite ab, oder verwenden Sie programmgesteuert die Gebührenanspruchs-Anweisung des CPMM-Pools.

### So fordern Sie Gebühren nach der Migration ab

1. Verbinden Sie die Wallet, die das Fee Key NFT hält.
2. Navigieren Sie auf raydium.io zu **Portfolio**.
3. Finden Sie den abgeschlossenen Token-Pool unter Ihren Positionen.
4. Klicken Sie auf **Claim**, um akkumulierte Handelsgebühren zu erfassen.

---

## Wenn Gebühren nach der Migration nicht aktiviert sind

Wenn eine Plattform Creator-Gebühren als deaktiviert festlegt, wird Ihr Token zu einem **AMM v4 Pool** (der Legacy-Migrationspfad) statt zu CPMM abgeschlossen. In diesem Modus:
- Es wird kein Fee Key NFT geprägt.
- Es gibt keine Gebühren auf Programmebene nach der Migration für Creators.
- Es können alternative Gebührenoptionen über CLI-Tools vorhanden sein, je nachdem wie der Launch konfiguriert wurde.

## Zusammenfassung der Gebührenfluss

```
Bonding-Curve-Phase:
  Jeder Kauf/Verkauf → kleiner % → Creator-Vault (Quote-Token)
  Jederzeit von Portfolio abheben

Abschluss:
  LP-Tokens geprägt → aufgeteilt pro Plattformkonfiguration
  ├── X% verbrannt (Burn & Earn)
  ├── Y% → Creator als Fee Key NFT      ← Sie halten diesen
  └── Z% → Plattform (falls zutreffend)

Nach dem Abschluss:
  Jeder CPMM-Swap → LP-Gebühren → aufgeteilt nach LP-Anteil
  Ihr Fee Key NFT-Anteil → abhebbar von Portfolio
```
