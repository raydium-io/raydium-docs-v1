## Zwei Erstellungsmodi

LaunchLab bietet zwei Erstellungswege, auf die Sie über [raydium.io/launchlab/create](https://raydium.io/launchlab/create) zugreifen können:

| | **JustSendit** | **LaunchLab** |
|---|---|---|
| Einrichtungszeit | ~1 Minute | ~5 Minuten |
| Token-Angebot | Feste Voreinstellung | Vollständig konfigurierbar |
| SOL-Ziel | 85 SOL (fest) | Minimum 24 SOL, benutzerdefiniert |
| Kurven-% | Voreinstellung | 51–80% auf Kurve |
| Vesting | Keine | Optional |
| Gebühren nach der Migration | Voreinstellung (10% Creator LP-Anteil) | Konfigurierbar |
| Ideal für | Schnelle Community-Starts | Strukturierte Token-Starts |

## JustSendit-Modus

Der schnellste Weg, einen Token zu starten. Minimale Eingabe erforderlich; die Plattform trägt sinnvolle Voreinstellungen ein.

### Was Sie benötigen

- Ein verbundenes Wallet mit ausreichend SOL für Transaktionsgebühren (~0,05–0,1 SOL für die Erstellung).
- Einen Token-Namen und ein Ticker-Symbol.
- Ein Bild oder GIF, das die UI-Anforderungen erfüllt.

### Schritte

1. Gehen Sie zu [raydium.io/launchlab/create](https://raydium.io/launchlab/create) und wählen Sie **JustSendit**.
2. Geben Sie den Token-**Namen** und das **Ticker**-Symbol ein.
3. Fügen Sie optional eine **Beschreibung** und **Social-Links** hinzu (Website, Twitter/X, Telegram).
4. Laden Sie ein **Logo-Bild oder GIF** hoch.
5. Geben Sie optional einen **anfänglichen Kaufbetrag** in SOL ein, um Tokens bei der Erstellung zu kaufen. Dies ändert den anfänglichen Status des On-Curve-Preises und sollte vor der Unterzeichnung überprüft werden.
6. Überprüfen Sie die Zusammenfassung und klicken Sie auf **Create**.
7. Genehmigen Sie die Transaktion in Ihrem Wallet.

Ihr Token wird sofort auf der Bonding Curve gestartet. Er wechselt zu einem Raydium-AMM-Pool, sobald 85 SOL durch die Kurve gesammelt wurden.

## LaunchLab-Modus

Erstellung mit vollständiger Kontrolle mit konfigurierbarem Angebot, Fundraising-Ziel, Kurvenallokation, Vesting und Gebührenparametern.

### Was Sie benötigen

- Ein verbundenes Wallet mit ~0,1–0,5 SOL für die Erstellung und Vault-Rent.
- Alle unten aufgeführten Eingaben vor dem Start bereit (Einstellungen können nach dem Start nicht geändert werden).

### Konfigurationsoptionen

#### Token-Details

- **Name, Ticker, Logo** — wie in JustSendit.
- **Beschreibung und Social-Links** — empfohlen für bessere Auffindbarkeit.

#### Angebot und Kurvenallokation

- **Gesamtangebot** — Gesamtzahl der Tokens, die geprägt werden (z. B. 1.000.000.000).
- **Kurvenallokation (%)** — Der Prozentsatz des Gesamtangebots, der auf der Bonding Curve für öffentliche Käufe platziert wird (51–80%). Der Rest wird in den Post-Migrations-Liquiditätspool eingespielt.

<Warning>
  Die Kurvenallokation und das Gesamtangebot sind bei der Erstellung fest und **können nicht geändert** werden. Treffen Sie Ihre Wahl sorgfältig, bevor Sie unterzeichnen.
</Warning>

#### SOL-Fundraising-Ziel

- Die minimale Graduierungsschwelle beträgt **24 SOL**. Sie können sie höher einstellen.
- Dies ist der SOL-Betrag, den die Kurve sammeln muss, bevor der Token zu einem CPMM-Pool wechselt.
- Wenn Sie es zu hoch einstellen, riskieren Sie, dass die Kurve nie abschliesst, falls die Nachfrage unzureichend ist.

#### Vesting (optional)

Vesting sperrt einen Teil des Token-Angebots und gibt ihn nach einem Zeitplan linear an designierte Wallets frei.

| Feld | Beschreibung |
|---|---|
| Gesamter gesperrter Betrag | Tokens, die für alle Begünstigten gesperrt werden (als % des Gesamtangebots, unterliegt Programmgrenzen). |
| Cliff-Periode | Wartezeit nach der Graduierung, bevor Tokens freigegeben werden (z. B. 180 Tage). |
| Freigabeperiode | Dauer der linearen Freigabe nach der Cliff-Periode (z. B. 365 Tage). |

Gesperrte Tokens treten nicht in die Kurve ein und sind nicht Teil des Graduierungs-Liquiditätspools. Sie werden nach dem obigen Plan linear an das Wallet jedes Begünstigten freigegeben.

**Wichtige Vesting-Regeln:**
- Cliff- und Freigabeperioden sind bei der Erstellung fest — sie können nicht rückwirkend geändert werden.
- Wenn ein Begünstigter seinen Wallet-Schlüssel verliert, sind gesperrte Tokens dauerhaft unerreichbar. Nutzen Sie ein Multisig, wenn eine Wiederherstellung wichtig ist.
- Jeder Teil von `total_locked_amount`, der einem Begünstigten nicht vor der Graduierung zugewiesen wird, bleibt dauerhaft stecken. Weisen Sie den vollständigen Betrag zu.

Für die vollständige On-Chain-Mechanik siehe [LaunchLab Vesting](/de/products/launchlab/vesting).

#### Gebührenaufteilung nach der Migration

Wenn der Token abschliesst:
- LP-Tokens werden im CPMM-Pool geprägt.
- Standard auf Raydium: **90% der LP-Tokens werden verbrannt** (dauerhaft gesperrt über Burn & Earn) und **10% gehen an den Creator** als Fee Key NFT.
- Das Fee Key NFT berechtigt Sie, 10% der LP-Trading-Gebühren aus dem Post-Migrations-Pool zu kassieren — unbegrenzt.

Aktivieren Sie dies, wenn Sie laufende Einnahmen aus der Liquidität Ihres Tokens möchten und die Verwahrungsverantwortung des Fee Key NFT akzeptieren. Siehe [Wie Creator-Gebühren funktionieren](/de/user-flows/how-creator-fees-work) für Details.

### Schritte

1. Wählen Sie den **LaunchLab**-Modus auf der Erstellungsseite.
2. Füllen Sie Token-Details aus (Name, Ticker, Logo, Beschreibung, Socials).
3. Legen Sie **Gesamtangebot** und **Kurvenallokations-Prozentsatz** fest.
4. Setzen Sie Ihr **SOL-Fundraising-Ziel** (≥ 24 SOL).
5. Konfigurieren Sie **Vesting**, falls erforderlich (legen Sie Begünstigte-Wallets und Zeitplan fest).
6. Aktivieren Sie **Gebührenaufteilung nach der Migration**, wenn Sie einen Creator LP-Anteil möchten.
7. Geben Sie optional einen anfänglichen Kaufbetrag ein.
8. Überprüfen Sie die vollständige Zusammenfassung — Kurvenpreisbereich, Graduierungsbedingung, LP-Aufteilung.
9. Klicken Sie auf **Create** und genehmigen Sie die Transaktion.

## Nach dem Start Ihres Tokens

### Kurvenfortschritt überwachen

Die LaunchLab-Seite Ihres Tokens zeigt Echtzeit-Fortschritt:
- % der verkauften Kurve.
- Gesammelte SOL vs. Graduierungsziel.
- Aktueller Preis und Preischart.

LaunchLab greift nicht ein, wenn Starts die Graduierung nicht erreichen. Überwachen Sie den Kurvenfortschritt und kommunizieren Sie klar mit Ihrer Community.

### Graduierung

Die Graduierung wird automatisch durch die erste Transaktion ausgelöst, die die Kurve über die Schwelle bringt — dies kann von jedem Benutzer aufgerufen werden (der Aufrufer verdient einen kleinen Bonus). Sie als Creator müssen nichts tun.

Nach der Graduierung:
- Ein CPMM-Pool wird mit dem gesammelten SOL und den verbleibenden Kurven-Tokens als anfängliche Liquidität erstellt.
- Wenn Sie Post-Migrations-Gebühren aktiviert haben, wird ein **Fee Key NFT** in Ihr Wallet geprägt.
- Beanspruchen Sie Pre-Migrations-Creator-Gebühren von Ihrer Portfolio-Seite.

### Pre-Migrations-Creator-Gebühren beanspruchen

Wenn Ihre Plattform Creator-Gebühren während der Bonding-Curve-Phase aktiviert, sammeln sich diese in einem Vault an. Beanspruchen Sie sie von der **Portfolio**-Seite auf raydium.io zu jeder Zeit.

## Häufige Fehler vermeiden

| Fehler | Was Sie tun sollten |
|---|---|
| SOL-Ziel zu hoch einstellen | Wählen Sie ein Ziel, das der Markt realistisch erreichen kann. |
| Vesting vor Bestätigung der Begünstigten zu stark überallokieren | Nicht zugewiesenes Vesting-Angebot ist nach der Graduierung dauerhaft unerreichbar. |
| Fee Key NFT versehentlich übertragen oder verbrennen | Gebührbeanspruchungsrechte gehen dauerhaft verloren. Halten Sie das NFT in dem Wallet, das den Token erstellt hat, oder einem Wallet, das Sie kontrollieren. |
| Keinen anfänglichen Kauf tätigen | Ohne einen frühen Kauf können Sniping-Bots alle frühen Tokens zu nahezu Nullkosten erfassen, was das Community-Vertrauen untergraben kann. |
