## Was Sie erhalten

- Eine teilbare URL, die die Raydium-Swap-UI mit Ihrem gewählten Zieltoken **und Ihrem Wallet als Empfehlungsempfänger** vorausfüllt.
- Wenn jemand einen Swap über den Link ausführt, während dieser in seiner Sitzung aktiv ist, wird **1% des Eingabevolumens an das Wallet des Empfehlers** als Teil der Swap-Transaktion weitergeleitet.
- Links, die auf X.com geteilt werden, werden als **Blinks** (Solana Actions) dargestellt, wenn das Wallet des Betrachters Actions / Blinks aktiviert hat — ein One-Click-Swap-Widget innerhalb des Posts.

## So erstellen Sie einen Link

1. Öffnen Sie [raydium.io/swap](https://raydium.io/swap) und verbinden Sie Ihr Wallet. Die Empfehlungsadresse wird aus dem verbundenen Wallet abgeleitet; ohne Verbindung funktioniert der Link trotzdem, verdient aber keine Gebühr.
2. Wählen Sie den **Zieltoken**, in den andere swappen sollen. Die Quellseite wird dem Empfänger überlassen.
3. Klicken Sie auf das Share-/Link-Symbol im Swap-Panel (oben rechts).
4. Kopieren Sie die generierte URL. Teilen Sie sie überall.

Das URL-Format ist stabil. Falls Sie eine App entwickeln, die Empfehlungs-Links direkt generiert.

## Wie Blinks funktionieren

**Blinks (Blockchain Links)** sind ein Standard der Solana Foundation zum Rendern von On-Chain-Action-URLs als interaktive UI innerhalb von Social-Media-Plattformen. Ein Blinks-unterstützendes Solana-Wallet liest die Link-Metadaten, erstellt eine vorgefertigte Transaktion und zeigt dem Benutzer inline eine Signiereraufforderung.

Anforderungen:

- Das Wallet des Betrachters **muss Actions / Blinks aktiviert haben**. Dies ist normalerweise ein Schalter in den Wallet-Einstellungen.
- Die **Plattform muss den Standard befolgen**. X.com tut dies über seinen Unfurler; einige andere nicht, in diesem Fall fällt der Link auf einen einfachen Hyperlink zurück.

Aus der Perspektive des Empfehlers ist das Blink-Rendern automatisch — die gleiche URL funktioniert als einfacher Link in Wallets, die Blinks nicht unterstützen.

## Gebührenabrechnung

Die 1%-Empfehlungsgebühr wird **zusätzlich zur AMM-Handelsgebühr** berechnet, nicht aus ihr herausgeschnitten. Bei einem Swap in einem Pool mit insgesamt 0,25% Gebühr:

```
Benutzer sendet:   1.000 USDC
AMM-Handelsgebühr: 2,50 USDC   (für LP + Protokoll-Split)
Empfehlungsgebühr: 10,00 USDC  (1% → Wallet des Empfehlers)
Netto zum Swap:    987,50 USDC (dann über die Kurve bepreist)
```

Empfehlungsgebühren gelten für die Raydium-Swap-Route, die durch den Empfehlungs-Link erstellt wird. Direkte Pool-Interaktionen außerhalb der Empfehlungs-Route unterliegen keinen Empfehlungsgebühren.

## Empfehlung deaktivieren / beenden

Es gibt keinen „Empfehlung entfernen"-Befehl. Um einen Swap ohne Empfehlungsgebühr durchzuführen, laden Sie einfach [raydium.io/swap](https://raydium.io/swap) direkt, statt auf einen Empfehlungs-Link zu klicken. Die UI verwendet keinen Empfehler, wenn die URL keinen Empfehler-Parameter enthält.

## FAQ

**Muss der Empfehler etwas unterzeichnen?** Nein. Die Wallet-Adresse des Empfehlers sind die einzigen Daten, die mit dem Link mitgeführt werden; die Zahlung an den Empfehler ist eine Token-Überweisung innerhalb der Transaktion des Swappers.

**Kann ich mich selbst empfehlen?** Ja, aber sinnlos — Sie zahlen sich selbst 1%, abzüglich der Transaktionsgebühr, die Sie gerade für eine größere Transaktion bezahlt haben.

**Was passiert, wenn der Zieltoken ein SPL mit einem Transfer-Hook ist?** Die Empfehlungsüberweisung erfolgt auf der Eingabeseite (in USDC oder dem Quelltoken), nicht dem Zieltoken, daher beeinflussen Transfer-Hooks für die Ziel-Mint nicht die Empfehlungszahlung. Transfer-Hooks auf dem Eingabetoken können den Swap vollständig blockieren; Raydium lehnt diese im Voraus ab.

**Gilt die Empfehlungsgebühr immer noch, wenn der Swap über mehrere Pools geleitet wird?** Ja — die 1% wird auf der ursprünglichen Eingabemenge berechnet und ist unabhängig von der Route.
