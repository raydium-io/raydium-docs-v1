<Info>
  Eine **Platform** in LaunchLab ist eine Drittanbieter-Launch-Umgebung, die auf Raydiums LaunchLab-Programm aufsetzt. Plattformen können sich selbst branden, eigene Gebühren erheben und die Parameter für die Migration nach der Bonding-Curve für jeden unter ihnen gestarteten Token konfigurieren — ohne ein separates On-Chain-Programm zu betreiben. Raydiums eigene Launch-UI ist nur eine Plattform unter vielen.
</Info>

## Warum Plattformen existieren

Raydium wollte, dass LaunchLab ein **gemeinsames Primitive** ist, nicht ein einzelnes Branded-Produkt. Das Platform-PDA-System ermöglicht es Ökosystemen, Tooling-Anbietern, Communities und Meme-Teams, kuratierte oder gebrandete Launch-Erfahrungen anzubieten, die alle in dasselbe zugrunde liegende LaunchLab-Bonding-Curve-Programm und denselben CPMM- (oder AMM v4) Pool nach der Migration abwickeln.

Beispiele für Plattformen, die Sie in freier Wildbahn sehen könnten:

- **Raydiums eigene LaunchLab-UI** unter raydium.io/launchlab — die Referenzimplementierung.
- **Partner-Launchpads** — Dritte, die ihre eigene Platform PDA registriert haben, um die Launch-Erfahrung zu branden.
- **Nischen-Plattformen** — z. B. eine Community-spezifische Plattform, die nur Memecoins mit einem bestimmten Thema startet und über eine benutzerdefinierte Gebührenaufteilung ihre Kasse finanziert.

All diese Token befinden sich immer noch auf demselben LaunchLab-Programm, graduieren zum gleichen CPMM-Programm und können von denselben Aggregatoren geroutet werden.

## Die Platform PDA

Jede Plattform wird durch eine Platform PDA dargestellt, die im LaunchLab-Programm gespeichert ist. Die Felder umfassen:

| Feld | Beschreibung |
|---|---|
| `platform_key` | Seed / Identifikator für die Plattform, wird zur Ableitung der Platform PDA verwendet. |
| `platform_fee_destination` | Konto, das den Anteil der Plattform an den Handelsgebühren vor und nach der Migration erhält. |
| `platform_fee_bps` | Plattformgebührenanteil auf Swaps vor der Migration (variiert je nach Plattform). |
| `creator_fee_enabled` | Ob diese Plattform das Teilen von Creator-Gebühren nach der Migration über Burn & Earn ermöglicht. |
| `creator_fee_share_bps` | Falls aktiviert, Anteil der LP-Gebühren, der zum Fee-Key-NFT des Creators fließt. |
| `cp_config_id` | Wenn das Migrationsziel CPMM ist, welche `AmmConfig` verwendet werden soll — wählt die Gebührenebene aus. |
| `name`, `description`, `image_url` | Anzeigende Metadaten, die von Wallets und UIs verwendet werden, die Plattform-Branding unterstützen. |
| `tokens_launched` | Zähler oder On-Chain-Index der unter dieser Plattform erstellten Token. |

Genaue Feldnamen befinden sich in der LaunchLab IDL ([`sdk-api/anchor-idl`](/de/sdk-api/anchor-idl)). Die obige Tabelle ist eine beschreibende Übersicht.

## Was eine Plattform kontrollieren kann

**Gebührenerfassung:**

- **Plattformgebührenanteil** auf jedem Kauf/Verkauf vor der Migration.
- **Anteil nach der Migration** — wenn der Token graduiert, werden die LP-Token an eine von der Plattform definierte Mehrbeteiligungs-Aufteilung geprägt: ein Anteil zum Verbrennen (Burn & Earn — ein Liquiditätssperr-Muster, das es dem gesperrten LP ermöglicht, weiterhin seinen Anteil an den Pool-Gebühren zu behaupten, siehe [`user-flows/burn-and-earn`](/de/user-flows/burn-and-earn)), ein Anteil zum Creator (Fee-Key-NFT) und ein Anteil zur Plattform selbst.

**Post-Migration-Ziel:**

- **CPMM (Standard, wenn Creator-Gebühren aktiviert sind)** mit einer über `cp_config_id` gewählten `AmmConfig`. Unterschiedliche `AmmConfig`s haben unterschiedliche Gebührenebenen; eine Plattform kann eine höhere Gebührenebene wählen, wenn ihre Launches High-Volume-Spekulations-Assets sind.
- **AMM v4 (wenn Creator-Gebühren deaktiviert sind)** — für Rückwärtskompatibilität mit dem ursprünglichen LaunchLab-Flow.

**Benutzerfreundlichkeit:**

- **Branding-Metadaten**, die von der Raydium-App und jedem Wallet / Explorer verwendet werden, das LaunchLab-Token darstellt.
- **Launch-Seiten-Anpassung** — Plattformen, die ihr eigenes Frontend hosten, können Token filtern, um nur die Launch-Seiten ihrer eigenen Plattform anzuzeigen.

## Was eine Plattform nicht kontrollieren kann

- **Bonding-Curve-Mathematik.** Die Kurvenform (logistisch, exponentiell usw.) ist Teil des LaunchLab-Programms selbst. Plattformen wählen aus den unterstützten Kurven; sie können keine beliebigen benutzerdefinierten Kurven hinzufügen.
- **Logik des Migrations-Schwellenwerts.** Die Bedingung, die die Graduierung zu CPMM / AMM v4 auslöst (Zielmarktkapitalisierung erreicht), wird programmatisch erzwungen.
- **Kernige Sicherheits- / Audit-Lage.** Jede Plattform erbt LaunchLabs Audit-Historie; eine böswillige Plattform kann sie nicht umgehen, indem sie das Programm ändert.

## Eine Plattform registrieren

Das LaunchLab-Programm stellt eine `create_platform`-Anweisung bereit (genauer Name aus der IDL). Aufrufer übermitteln:

- Den Seed, der die Platform PDA ableiten wird.
- Das Konto der Gebührenzielstelle.
- Gewünschte Gebühren bps (unterliegen programmgesteuerten Obergrenzen).
- Post-Migration-Konfiguration (CPMM `AmmConfig` ID, falls zutreffend).
- Branding-Metadaten.

Die Miete für die Platform PDA wird vom Creator bezahlt. Updates für Branding und einige Gebührenparameter sind nach der Erstellung durch die Behörde der Plattform zulässig; strukturelle Parameter (Migrations-Zielprogramm) typischerweise nicht.

Siehe [`products/launchlab/instructions`](/de/products/launchlab/instructions) für die genaue Anweisungssignatur in der aktuellen LaunchLab IDL.

## Ökonomische Wechselwirkung

Ein einzelner Swap auf einer LaunchLab-Bonding-Curve verteilt den Wert ungefähr so (genaue Verhältnisse werden von der Plattform festgelegt):

```
Käufer zahlt:                    100 SOL
Protokollgebühr (LaunchLab):     – kleiner %
Plattformgebühr:                 – platform_fee_bps%
Creator-Gebühr (vor Migration):  – creator_fee_bps% (falls aktiviert)
Rest (zu Curve):                 ~ Rest
```

Nach der Migration werden LP-Token pro Plattform-Konfiguration aufgeteilt:

```
LP-Token geprägt bei Graduierung:  100%
  → Burn & Earn (verbrannt):       X%  (Standard bei Raydium: 90)
  → Creator Fee-Key-NFT:           Y%  (Standard bei Raydium: 10)
  → Plattform:                     Z%  (plattformkonfiguriert; kann 0 sein)
```

Der **Burn & Earn-Anteil** wird in gemeinsame Gebühreneinkommen umgewandelt, die dem Fee-Key-Inhaber verteilt werden — was unter Raydiums eigenen Standards der Verbrennung „Tresor" ist, der die breite Markt-Gebührenunterstützung für Token-Vertrauen repräsentiert.

## Integrator-Notizen

- **Tokens nach Plattform auflisten.** Verwenden Sie die API-Pool-Info / Mint-Endpunkte mit einem `platform`-Filter (falls unterstützt), um nur Token aufzulisten, die unter einer bestimmten Platform PDA gestartet wurden.
- **Branding-Anzeige.** Wenn Sie ein Wallet oder einen Explorer erstellen, der LaunchLab-Token darstellt, ist das Lesen des Plattformnamens / der Bild-URL aus der Platform PDA eine viel bessere UX, als „LaunchLab" für jeden Token anzuzeigen.
- **Gebühren-Mathematik.** Jedes Bonding-Curve-Angebot muss die Plattformgebühr zusätzlich zu den Protokoll- und Creator-Gebühren berücksichtigen. Die Quote-Helfer des SDK führen dies automatisch durch, wenn die Platform PDA bereitgestellt wird.
