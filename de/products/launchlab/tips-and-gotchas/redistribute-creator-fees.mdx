Standardmäßig werden CPMM-Creator-Gebühren aus einem migrierten LaunchLab-Pool dem ursprünglichen Token-Creator zugeordnet. Plattformen können dies für zukünftige Migrationen überschreiben, indem sie `platformCpCreator` setzen.

## Gründe für die Umleitung von Creator-Gebühren

Plattformen nutzen die Umleitung von Creator-Gebühren, wenn sie benötigen:

- Benutzerdefinierte Verteilungslogik zwischen Creators, Communities und Plattform-Einnahmen.
- Ein PDA-gesteuertes Gebühren-Sammelkonto für On-Chain-Umverteilung.
- Einheitliche Abrechnung über alle migrierten Pools einer Plattform hinweg.

<Warning>
  Die Umleitung von Creator-Gebühren ändert, wer diese einziehen kann. Token-Creator verlieren den direkten Zugriff für Pools, die nach Aktivierung dieser Einstellung migrieren, es sei denn, die Plattform verteilt die Gebühren an sie weiter.
</Warning>

## So funktioniert es

1. Der Plattform-Admin ruft `updatePlatformConfig` auf.
2. `updateInfo.type` wird auf `updatePlatformCpCreator` gesetzt.
3. Der Wert ist eine Wallet oder PDA, die zum CPMM-Pool-Creator für zukünftig migrierte Pools wird.
4. Zukünftige LaunchLab-Migrationen unter dieser Plattform weisen CPMM-Creator-Gebühren der konfigurierten Adresse zu.
5. Die Plattform sammelt Creator-Gebühren und verteilt sie nach ihren eigenen Regeln um.

```ts
await raydium.launchpad.updatePlatformConfig({
  programId,
  platformAdmin: owner,
  updateInfo: {
    type: "updatePlatformCpCreator",
    value: new PublicKey("<PLATFORM_WALLET_OR_PDA>"),
  },
  txVersion: TxVersion.V0,
});
```

## Fallstricke

- **Alles-oder-nichts pro Plattform.** Die Einstellung gilt für jede zukünftige Migration unter dieser Plattform.
- **Nur zukunftsgerichtet.** Bereits migrierte Pools behalten ihren bestehenden Pool-Creator.
- **Auf Epoch-Basis begrenzte Updates.** Plattform-Konfigurations-Updates sind durch die Solana-Epoch begrenzt.
- **Creator-Gebühren müssen vorhanden sein.** Wenn die gewählte CPMM `AmmConfig` `creator_fee_rate = 0` hat, gibt es keine CPMM-Creator-Gebühren zum Einziehen.
- **WSOL-Handling ist wichtig.** Wenn eingezogene Gebühren WSOL sind und Sie natives SOL benötigen, schließen Sie das WSOL-Token-Konto nach der Einziehung.

## Checkliste zur Umverteilung

Wenn Sie an ursprüngliche Creators umverteilen:

1. Lösen Sie jeden ursprünglichen Creator aus dem LaunchLab-Pool-Konto auf, das an die CPMM-Pool-Mints gebunden ist.
2. Sammeln Sie ausstehende Creator-Gebühren für die Plattform-Wallet oder PDA.
3. Berechnen Sie den vereinbarten Anteil jedes Creators.
4. Überweisen Sie SPL-Token oder SOL an Creators, erstellen Sie erforderlichenfalls ATAs.
5. Veröffentlichen Sie ausreichend Buchhaltungsdetails, damit Creators Verteilungen überprüfen können.
