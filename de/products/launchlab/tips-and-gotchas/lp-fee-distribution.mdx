Wenn ein LaunchLab-Token zu CPMM migriert, prägt der migrierte Pool LP-Token. Die Plattformkonfiguration steuert, was mit diesen LP-Token geschieht.

## Die LP-Aufteilung

`migrateCpLockNftScale` hat drei Felder. Die Werte müssen sich zu `1_000_000` summieren, was 100% repräsentiert.

| Feld | Bedeutung |
|---|---|
| `platformScale` | LP-Token werden durch Burn & Earn gesperrt. Die Plattform erhält die Gebührenschlüssel-NFT. |
| `creatorScale` | LP-Token werden durch Burn & Earn gesperrt. Der Token-Creator erhält die Gebührenschlüssel-NFT. |
| `burnScale` | LP-Token werden dauerhaft verbrannt. Für diesen Anteil wird keine Gebührenschlüssel-NFT erstellt. |

Dies ist eine Einstellung auf Plattformebene. Token, die über dieselbe Plattform gestartet werden, erben die gleiche LP-Verteilung, bis die Plattform die Einstellung aktualisiert.

## Beispiele

```ts
// 40% Plattform, 50% Creator, 10% verbrannt.
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% Creator-Gebührenschlüssel.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% verbrannt.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## Beziehung zu CPMM-Gebühren

`migrateCpLockNftScale` steuert, **wer LP-Gebührenrechte geltend machen kann**. Es legt nicht die Swap-Gebührenrate fest. Das Swap-Gebührenverhalten des migrierten Pools stammt aus der CPMM `AmmConfig`, die von `cpConfigId` ausgewählt wird.

Wenn beispielsweise ein migrierter CPMM-Pool eine 0,25%-Handelsgebühren-Config verwendet und `creatorScale` 500.000 ist, repräsentiert die Gebührenschlüssel-NFT des Creators 50% der gesperrten LP-Gebührenrechte. Wenn `platformScale` 400.000 ist, repräsentiert die Gebührenschlüssel-NFT der Plattform 40%.

## Aufteilung aktualisieren

Plattformkonfigurationsaktualisierungen sind durch die Epoch begrenzt. Betrachten Sie die LP-Verteilung als eine wichtige Plattformrichtlinie, nicht als eine beiläufige UI-Einstellung.

Vor einer Aktualisierung:

- Bestätigen Sie, dass die drei Skalenwerte sich zu `1_000_000` summieren.
- Bestätigen Sie, dass die Plattform versteht, wer nach zukünftigen Migrationen Gebührenschlüssel-NFTs erhält.
- Dokumentieren Sie die Änderung für Creators, bevor neue Launches diese verwenden.
