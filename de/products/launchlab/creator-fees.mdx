## Pre-Migration: direkte Gebührenabgrenzung

Während sich ein Token noch auf der Bonding Curve befindet, zahlen alle Käufe und alle Verkäufe einen Ersteller-Gebührenanteil, der in einem Pro-Token-Tresor angesammelt wird. Der **Ersteller** (die Wallet, die das Token gestartet hat) kann jederzeit aus dem Tresor abheben.

Charakteristiken:

- Nominiert im **Quote-Token** (normalerweise SOL).
- Konfiguriert durch die **Plattform** (die Plattformkonfiguration steuert, ob Ersteller-Gebühren aktiv sind und zu welchem Bps-Satz).
- Jederzeit von der Portfolio-Seite in der LaunchLab-Benutzeroberfläche abrufbar; programmgesteuert über die LaunchLab-Anweisung `claim_creator_fees` (exakter Name in der IDL).
- Wird fortgesetzt bis zur Graduation; bei Graduation können restliche Pre-Migration-Gebühren eingezogen werden, aber es werden keine neuen Pre-Migration-Gebühren abgegrenzt.

## Post-Migration: Fee Key NFT über Burn & Earn

Wenn die Bonding Curve eines Tokens das Ziel erreicht und die Liquidität zu einem CPMM- (oder AMM v4-)Pool migriert, werden LP-Tokens gemünzt und **gemäß der Konfiguration der Plattform aufgeteilt**. Die Aufteilung summiert sich immer auf 100% und beinhaltet immer eine Kombination aus:

- **Verbrannt** — in [Burn & Earn](/de/user-flows/burn-and-earn) für permanente Sperrung übertragen. Der Fee Key für diesen Anteil wird von einem von der Plattform kontrollierten oder Burn-designierten Konto gehalten.
- **Ersteller** — ein Fee Key NFT, der direkt in die Wallet des Token-Erstellers gemünzt wird. Dieser Fee Key repräsentiert das Recht, LP-Handelsgebühren aus dem Anteil des Erstellers an LP-Tokens einzuziehen.
- **Plattform** — optional; ein Anteil, der als weiterer Fee Key NFT in die Plattform-Schatzkammer fließt.

### Rayidums eigene LaunchLab-Standards

| Anteil | Standard auf raydium.io/launchlab |
|---|---|
| Verbrannt (Burn & Earn, dauerhaft gesperrt) | 90% |
| Ersteller (Fee Key NFT) | 10% |
| Plattform | 0% (Rayidums Plattform nimmt keinen Plattform-LP-Anteil) |

Bei diesen Standards **zieht nur der Ersteller Handelsgebühren aus seinem Anteil ein**; die anderen 90% sind gesperrt und ihre Gebühren fließen zu Burn & Earn (zurück zum Trust-Signalisierungsmechanismus statt in irgendeine einzelne Tasche).

### Andere Plattformen

Drittanbieter-Plattformen, die auf LaunchLab aufbauen, können ihre eigene Aufteilung konfigurieren. Eine Plattform, die sich über Post-Migration-LP-Gebühren finanzieren möchte, kann (zum Beispiel) 70 / 10 / 20 (burn / creator / platform) einstellen. Weitere Informationen finden Sie in [`products/launchlab/platforms`](/de/products/launchlab/platforms) für die Platform-PDA-Felder, die dies bestimmen.

Unabhängig von der Aufteilung ist der **Mechanismus gleich**: Jeder nicht verbrannte Anteil wird als Fee Key NFT dargestellt, die gehandelt, übertragen oder eingezogen werden kann.

### Was passiert, wenn Post-Migration-Ersteller-Gebühren deaktiviert sind

Wenn eine Plattform `creator_fee_enabled = false` setzt, durchlaufen die graduierten Token **nicht** Burn & Earn. Stattdessen migrieren sie zu einem **AMM v4**-Pool mit dem Legacy-LaunchLab-Migrationspfad. In diesem Modus empfängt der Ersteller keinen Fee Key NFT und es gibt keine Programm-Level-Post-Migration-Ersteller-Gebühr. Einige Ersteller in diesem Modus verwenden ein Off-Chain-CLI-Tool, um optionale Post-Migration-Gebühren zu beanspruchen, die über `creatorFeeOn` konfiguriert sind — weitere Informationen finden Sie in [`products/launchlab/instructions`](/de/products/launchlab/instructions) für die On-Chain-Oberfläche.

## Das 0,1%-Token-Share-Referral

Getrennt von der LP-Gebührenfreigabe können LaunchLab-Ersteller (und jeder Benutzer wirklich) einen **Token-Seiten-Link** mit ihrer Wallet als Referrer freigeben. Wenn jemand dieses Token (oder andere LaunchLab-Tokens in derselben Sitzung) über den freigegebenen Link handelt:

- **0,1% jedes Handels** wird an die Wallet des Referrers luftgedropt.
- Bezahlt in SOL oder dem Quote-Token des Pools.
- Automatisch bei jedem Handel bezahlt; kein Einzugsschritt.

Dies ist ein anderes Produkt als das 1%-Swap-Referral in [`user-flows/referrals-and-blinks`](/de/user-flows/referrals-and-blinks). Das 1%-Referral gilt für Rayidums Spot-Swap-Benutzeroberfläche; das 0,1%-Token-Share-Referral gilt spezifisch für LaunchLab-Token-Seiten.

## Gebühren einziehen

### Ersteller-Tresor (Pre-Migration)

```ts
import { Raydium, TxVersion } from "@raydium-io/raydium-sdk-v2";
const raydium = await Raydium.load({ owner, connection });
const { execute } = await raydium.launchlab.claimCreatorFees({
  mint,           // launched token mint
  txVersion: TxVersion.V0,
});
await execute({ sendAndConfirm: true });
```

### Fee Key NFT (Post-Migration)

Das Halten des Fee Key in Ihrer Wallet ist ausreichend — zieht ihn von der Portfolio-Seite ein, oder programmgesteuert über die Fee-Claim-Anweisung des relevanten Pool-Programms. Weitere Informationen finden Sie in [`products/cpmm/instructions`](/de/products/cpmm/instructions) oder dem AMM v4-Äquivalent für die On-Chain-Oberfläche.

## Integrator-Checkliste

- **Zeigen Sie beide Gebührenströme an.** Ein Ersteller-Dashboard, das nur einen von „Pre-Migration-Tresor" / „Post-Migration Fee Key" anzeigt, lässt Geld auf dem Tisch liegen für Benutzer, die von dem anderen nicht wissen.
- **Respektieren Sie Fee Key-Eigentumsübertragungen.** Wenn ein Ersteller seinen Fee Key NFT verkauft oder überträgt, verliert der bisherige Eigentümer den Zugriff auf zukünftige Gebühreneinzüge. Ihre Benutzeroberfläche sollte jedes Mal den aktuellen NFT-Inhaber lesen, nicht die ursprüngliche Ersteller-Adresse cachen.
- **Plattformkonfiguration-Drift.** Rayidums Standards können sich ändern. Für jede Plattform außer Rayidums eigener sollten Sie die Platform-PDA immer live lesen, um die tatsächliche Burn / Creator / Platform-Aufteilung für diesen spezifischen Launch zu sehen.
- **Token-2022-Sicherheiten und Gebühren-Tokens.** Ersteller-Gebühren können in Token-2022-Tokens mit Transfergebühren nominiert sein; berücksichtigen Sie den Transfer-Gebühren-Schnitt in einer angezeigten „einziehbaren Gebühren"-Zahl.
