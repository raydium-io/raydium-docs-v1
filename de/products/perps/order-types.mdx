## Die fünf Ordertypen

**Market.** Kauft oder verkauft sofort zu den besten verfügbaren Preisen im Order-Buch. Wird gegen ruhende Orders ausgeführt, bis Ihre gewünschte Menge gefüllt ist. Falls das Buch nicht die vollständige Menge füllen kann, wird der Rest storniert (nicht ins Buch aufgenommen). Verwenden Sie diesen Typ für sofortige Ausführung, akzeptieren aber Slippage in Kauf.

**Limit.** Kauft oder verkauft zu einem bestimmten Preis, den Sie wählen (der **Limitpreis**), oder besser. Eine Limitorder bleibt im Buch, bis sie ausgeführt oder storniert wird. Verwenden Sie diesen Typ, wenn der Preis wichtiger ist als Geschwindigkeit.

**Stop Market.** Inaktiv, bis der Markt Ihren **Trigger-Preis (Stop-Preis)** erreicht. Bei Auslösung wird die Order als Market-Order eingereicht und wird zum nächsten verfügbaren Preis gefüllt, falls ausreichend Liquidität vorhanden ist. Verwenden Sie diesen Typ, wenn sofortige Ausführung wichtiger ist als Preiskontrolle.

**Stop Limit.** Inaktiv, bis der Markt Ihren Trigger-Preis erreicht. Bei Auslösung wird die Order als Limitorder zu Ihrem angegebenen Limitpreis eingereicht und wird nur gefüllt, wenn das Buch diesen Preis erreicht. Verwenden Sie diesen Typ, wenn Sie einen Stop möchten, der niemals schlechter als ein Mindest-/Höchstpreis ausgeführt wird, wobei Sie akzeptieren, dass er möglicherweise gar nicht gefüllt wird, wenn ein Gap auftritt.

**Scale.** Teilt eine einzelne größere Order in mehrere kleinere Limitorders auf, die über eine Preisspanne verteilt sind. Verwenden Sie diesen Typ für schrittweise Ein- oder Ausstieg statt die gesamte Größe auf einmal zu platzieren.

## Die vier Order-Flags

Diese können Limit- oder Stop-Limit-Orders angehängt werden, um die Ausführungssemantik zu ändern:

**IOC — Immediate-or-Cancel.** Füllt so viel wie möglich sofort zum Limitpreis (oder besser). Jeder ungefüllte Anteil wird storniert statt im Buch zu bleiben. Verwenden Sie diesen Typ, wenn Sie sofortige Ausführung mit einer Preisobergrenze möchten.

**FOK — Fill-or-Kill.** Füllt entweder die **gesamte** Order sofort zum Limitpreis (oder besser), oder storniert sie ohne Teilfüllung. Verwenden Sie diesen Typ, wenn Teilfüllungen schlechter sind als gar keine Füllung — z. B., wenn die Position nur in voller Größe sinnvoll ist.

**Post-Only.** Stellt sicher, dass die Order Liquidität hinzufügt (Maker-Seite). Falls sie bei Platzierung gegen eine ruhende Order matchen würde (d. h., sie wäre ein Taker), wird sie stattdessen storniert. Verwenden Sie diesen Typ, wenn Sie Maker-Ausführung statt Taker-Ausführung möchten.

**Reduce-Only.** Verhindert, dass die Order Ihre Position *vergrößert*. Sie kann nur eine bestehende Position in diesem Markt schließen oder reduzieren. Falls Sie keine Position haben oder die Order diese vergrößern würde, wird die Order nicht ausgeführt. Verwenden Sie diesen Typ für automatisierte Take-Profit-/Stop-Loss-Logik, bei der Sie niemals versehentlich in die entgegengesetzte Richtung wechseln möchten.

## Typ + Flags kombinieren

Nicht jede Kombination ist zulässig. Die gängigen sind:

| Order | Flag | Typischer Anwendungsfall |
|---|---|---|
| Limit | — | Ruhende Maker-Order |
| Limit | Post-Only | Strikte Maker-Order (kein versehentlicher Take) |
| Limit | IOC | Aggressiver Take mit Preisobergrenze |
| Limit | FOK | Alles-oder-Nichts-Take |
| Limit | Reduce-Only | Ausgang zu einem Preis |
| Market | Reduce-Only | Notfallaustritt zu beliebigem Preis |
| Stop Market | Reduce-Only | Stop-Loss, der Ausgang priorisiert ohne Richtungswechsel |
| Stop Limit | Reduce-Only | Stop-Loss mit Preisuntergrenze, kein Richtungswechsel |
| Scale | Post-Only | Passive abgestufte Einstiegsstrategie |

Post-Only und Reduce-Only können auf derselben Order kombiniert werden.

## Gebührenauswirkungen

Ordertyp und Flags interagieren mit der Gebührenstruktur:

- **Market-Orders zahlen immer Taker-Gebühren.** Es gibt keine Möglichkeit, Maker-Rabatte auf eine Market-Order zu erhalten.
- **Limitorders** zahlen Maker-Gebühren, wenn sie im Buch ruhen und später gemacht werden; zahlen Taker-Gebühren, wenn sie sofort bei Platzierung gemacht werden.
- **Post-Only** zielt auf Maker-Gebühren ab, indem jede Platzierung abgelehnt wird, die das Buch kreuzen würde.
- **Stop-Orders** zahlen die Gebührenklasse, in die sie bei Auslösung konvertiert werden (Stop-Market → Taker; Stop-Limit → Maker, es sei denn, sie kreuzen).

Volumenbasierte Taker-Stufen und die pauschale Maker-Gebühr von 0 bps sind unter [Gebühren](/de/products/perps/fees) dokumentiert.
