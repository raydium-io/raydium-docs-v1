## Was es ist

**Raydium Perpetual** ist ein gebührenfreies Central-Limit-Order-Book-(CLOB-)Perpetuals-Protokoll. Positionen werden aus einer SPL-Wallet auf Solana finanziert, aber die Matching-Engine und das Order Book werden von [Orderly Network](https://orderly.network/) betrieben, das Liquidität über mehrere Chains (Solana, Ethereum, Base, Arbitrum, BSC) bündelt.

Wichtige Eigenschaften:

- **Gebührenfreies Trading.** Orderplatzierung, -änderung und -stornierung erfordern keine On-Chain-Transaktionen auf Solana – das Signieren erfolgt lokal und kostenlos. Nur Einzahlungen und Auszahlungen berühren die Chain.
- **Bis zu 100× Hebel**, je nach Markt.
- **Multi-Chain-Collateral** – Sie können USDC / SOL / USDT / ETH / BNB / WBTC / YUSD / USD1 von ihrer nativen Chain einzahlen; alle Gewinne/Verluste werden in USDC abgerechnet.
- **Nur Cross-Margin.** Alle Sicherheiten in einem Konto sind über alle offenen Positionen verteilt.
- **One-Way-Modus.** Sie können nicht gleichzeitig eine Long- und eine Short-Position auf demselben Markt halten; eine gegenseitige Order reduziert oder flippt die bestehende Position.
- **Selbstverwahrung.** Gelder bleiben im Perps-Konto Ihrer Wallet; es gibt keine von Raydium gehaltene Custodian-Wallet.
- **CEX-Grade-Order-Typen.** Market, Limit, Stop (Market / Limit), Scale, plus IOC / FOK / Post-Only / Reduce-Only-Flags.

## Warum ein eigenständiges Produkt

Die Raydium-AMMs (Spot) und Raydium Perps (Perpetual Futures) teilen eine Marke und einen UI-Einstiegspunkt, aber fast nichts auf Protokollebene:

| | Raydium Spot-AMMs | Raydium Perps |
|---|---|---|
| Venue | On-Chain-Pools auf Solana | Off-Chain-CLOB von Orderly |
| Matching-Engine | AMM-Kurve (CPMM / CLMM / AMM v4) | Central-Limit-Order-Book |
| Settlement | On-Chain bei Swap | Off-Chain; Einzahlungen/Auszahlungen On-Chain |
| Preisermittlung | AMM-Kurve + Arbitrage | Order Book + externe Preise |
| Token-Universum | Jede SPL / Token-2022-Mint | Nur gelistete Perp-Märkte (BTC, ETH, SOL und andere Major) |
| Program IDs in [`reference/program-addresses`](/de/reference/program-addresses) | Ja | Nein – siehe Orderly's Contracts |
| Raydium SDK v2 Coverage | Vollständig | Nicht enthalten |
| Raydium Bug Bounty | Ja | Über Orderly geleitet |

Der Rest dieser Dokumentation konzentriert sich fast ausschließlich auf die Spot-AMMs und die Programme in der GitHub-Organisation von Raydium. Perps ist auf Produktoberflächen-Ebene dokumentiert – Order-Typen, Gebühren, Collateral, Beschränkungen – damit Leser verstehen, worum es geht und wann es zu nutzen ist, aber die tiefe Protokollreferenz liegt auf Orderly's Seite.

## Integrations-Hinweise

- Das Spot-Raydium-SDK (`@raydium-io/raydium-sdk-v2`) deckt Perps **nicht** ab. Für programmatischen Zugriff verwenden Sie [Orderly's SDKs](https://orderly.network/docs/build-on-omnichain/evm-api/introduction) (TypeScript, REST, WebSocket).
- Es gibt kein „Swap durch eine Perp-Position routen"-Primitiv. Ein Client, der Spot- und Perp-Exposure möchte, erstellt die beiden Transaktionen unabhängig.
- Orderlys Risiko-Parameter (Hebel-Caps, Liquidationsschwellen, Collateral-Limits) ändern sich im Laufe der Zeit. Holen Sie sich immer die aktuellen Parameter von Orderly's API, bevor Sie Risiken an einen Benutzer kommunizieren.
