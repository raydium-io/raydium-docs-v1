<Info>
  LaunchLab 中的**平台**是構築在 Raydium 的 LaunchLab 程式上的第三方上線環境。平台可以自有品牌、收取自己的費用，以及設定在它們之下上線的每個代幣的遷移後參數 — 無需執行獨立的鏈上程式。Raydium 自己的上線 UI 只是眾多平台之一。
</Info>

## 為什麼平台存在

Raydium 希望 LaunchLab 成為一個**共享的基礎設施**，而不是單一的品牌產品。平台 PDA 系統讓生態、工具提供者、社群和迷因團隊能夠提供具策展或品牌特色的上線體驗，所有這些都在同一個底層 LaunchLab 聯合曲線程式和同一個 CPMM（或 AMM v4）遷移後池中結算。

你可能在野外看到的平台例子包括：

- **Raydium 自己的 LaunchLab UI** —— raydium.io/launchlab，參考實現。
- **合作夥伴上線平台** —— 已註冊自有平台 PDA 來品牌化上線體驗的第三方。
- **利基平台** —— 例如社群特定的平台，只上線與特定主題相關的迷因幣，並用自訂費用分享來資助其金庫。

所有這些代幣仍然存在於同一個 LaunchLab 程式上，畢業進入同一個 CPMM 程式，並可由相同的聚合器路由。

## 平台 PDA

每個平台由存儲在 LaunchLab 程式中的平台 PDA 表示。欄位包括：

| 欄位 | 描述 |
|---|---|
| `platform_key` | 平台的種子/識別符，用於推導平台 PDA。 |
| `platform_fee_destination` | 接收平台在遷移前後交易費用分成的帳戶。 |
| `platform_fee_bps` | 平台在遷移前交換上的費用分成（因平台而異）。 |
| `creator_fee_enabled` | 此平台是否啟用遷移後通過「燒毀與賺取」的創作者費用分享。 |
| `creator_fee_share_bps` | 若啟用，流向創作者的費用密鑰 NFT 的 LP 費用分成。 |
| `cp_config_id` | 若遷移後目標是 CPMM，使用哪個 `AmmConfig` —— 選擇費用層級。 |
| `name`, `description`, `image_url` | 由錢包和支援平台品牌的 UI 使用的顯示元資料。 |
| `tokens_launched` | 在此平台下建立的代幣計數器或鏈上索引。 |

精確的欄位名稱在 LaunchLab IDL 中（[`sdk-api/anchor-idl`](/zh-Hant/sdk-api/anchor-idl)）。上表是描述性概覽。

## 平台可以控制什麼

**費用捕捉：**

- 每次遷移前買賣上的**平台費用分成**。
- **遷移後分成** —— 當代幣畢業時，LP 代幣被鑄造到由平台定義的多方分割：一部分用於燒毀（「燒毀與賺取」—— 流動性鎖定模式，讓被鎖定的 LP 繼續領取其池費用分享，見 [`user-flows/burn-and-earn`](/zh-Hant/user-flows/burn-and-earn)），一部分給創作者（費用密鑰 NFT），一部分給平台自身。

**遷移後目標：**

- **CPMM（若啟用創作者費用則預設）**，通過 `cp_config_id` 選擇的 `AmmConfig`。不同的 `AmmConfig` 有不同的費用層級；如果平台的上線是高成交量的投機資產，可以選擇更高的費用層級。
- **AMM v4（若禁用創作者費用）** —— 與原始 LaunchLab 流程的向後相容。

**使用者體驗：**

- **品牌元資料**，由 Raydium 應用程式和任何顯示 LaunchLab 代幣的錢包/瀏覽器使用。
- **上線頁面自訂** —— 託管自己前端的平台可以篩選代幣以只顯示自己平台的上線。

## 平台無法控制什麼

- **聯合曲線數學。** 曲線形狀（邏輯、指數等）是 LaunchLab 程式本身的一部分。平台從支援的曲線中選擇；它們無法添加任意自訂曲線。
- **遷移閾值邏輯。** 觸發畢業到 CPMM/AMM v4 的條件（達到目標市值）由程式強制。
- **核心安全/審計姿態。** 每個平台都繼承 LaunchLab 的審計歷史；惡意平台無法通過改變程式來規避它。

## 註冊平台

LaunchLab 程式公開一個 `create_platform` 指令（精確名稱來自 IDL）。呼叫者傳遞：

- 將推導平台 PDA 的種子。
- 費用目標帳戶。
- 所需的費用 bps（受程式級上限限制）。
- 遷移後設定（若適用，為 CPMM `AmmConfig` ID）。
- 品牌元資料。

平台 PDA 的租金由創作者支付。品牌和某些費用參數的更新在建立後由平台的權限允許；結構參數（遷移目標程式）通常不允許。

見 [`products/launchlab/instructions`](/zh-Hant/products/launchlab/instructions) 以了解當前 LaunchLab IDL 中的精確指令簽名。

## 經濟互動

LaunchLab 聯合曲線上的單次交換大致分配價值如下（精確比率由平台設定）：

```
買方支付：                  100 SOL
協議費用（LaunchLab）：    – 小 %
平台費用：                – platform_fee_bps%
創作者費用（遷移前）：    – creator_fee_bps%（若啟用）
剩餘（到曲線）：          ~ 其他
```

遷移後，LP 代幣按平台設定分割：

```
畢業時鑄造的 LP 代幣：  100%
  → 燒毀與賺取（已燒毀）： X%  (Raydium 預設: 90)
  → 創作者費用密鑰 NFT：   Y%  (Raydium 預設: 10)
  → 平台：                Z%  (平台設定; 可能為 0)
```

**「燒毀與賺取」分成**轉換為分配給費用密鑰持有者的共享費用收入 —— 在 Raydium 的預設下，是代表廣泛市場費用支持代幣信任的燒毀「金庫」。

## 整合商備註

- **按平台列出代幣。** 使用帶有 `platform` 篩選器的 API 池資訊/mint 端點（若支援）來只列出在特定平台 PDA 下上線的代幣。
- **品牌顯示。** 如果你建置顯示 LaunchLab 代幣的錢包或瀏覽器，從平台 PDA 讀取平台的名稱/圖片 URL 遠比為每個代幣顯示「LaunchLab」的使用者體驗更好。
- **費用數學。** 每個聯合曲線報價必須除了協議和創作者費用外還要計算平台費用。當提供平台 PDA 時，SDK 的報價幫助器會自動執行此操作。
