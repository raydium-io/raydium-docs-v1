## 遷移前：直接費用累積

當代幣仍在結合曲線上時，每次買入和賣出都會產生創作者費用，並累積在該代幣專用的保管庫中。**創作者**（啟動代幣的錢包）可隨時從保管庫領取費用。

特點：

- 以**報價代幣**計價（通常是 SOL）。
- 由**平台**設定（平台配置控制是否啟用創作者費用及費率）。
- 可隨時從 LaunchLab UI 的投資組合頁面領取；或透過 LaunchLab `claim_creator_fees` 指令以程式方式領取（IDL 中的確切名稱）。
- 持續到畢業；畢業時，剩餘的遷移前費用仍可領取，但不會產生新的遷移前費用。

## 遷移後：透過 Burn & Earn 的 Fee Key NFT

當代幣的結合曲線達到目標且流動性遷移到 CPMM（或 AMM v4）資金池時，LP 代幣被鑄造並**根據平台的配置進行分配**。分配總和始終為 100%，且始終涉及以下某種組合：

- **銷毀** — 轉入 [Burn & Earn](/zh-Hant/user-flows/burn-and-earn) 進行永久鎖定。此部分的 Fee Key 由平台控制的或銷毀指定的帳戶持有。
- **創作者** — 直接鑄造到代幣創作者錢包的 Fee Key NFT。此 Fee Key 代表向創作者的 LP 代幣份額收取流動性池交易費用的權利。
- **平台** — 選填；流向平台金庫的份額，以另一個 Fee Key NFT 的形式。

### Raydium 自有 LaunchLab 預設值

| 份額 | raydium.io/launchlab 上的預設 |
|---|---|
| 銷毀（Burn & Earn，永久鎖定） | 90% |
| 創作者（Fee Key NFT） | 10% |
| 平台 | 0%（Raydium 平台不收取平台 LP 份額） |

在這些預設值下，**只有創作者從他們的份額中領取交易費用**；其他 90% 被鎖定，其費用流向 Burn & Earn（回流到信任信號機制而非任何個人帳戶）。

### 其他平台

在 LaunchLab 基礎上構建的第三方平台可配置自己的分配。想透過遷移後 LP 費用為自身融資的平台可設定（例如）70 / 10 / 20（銷毀 / 創作者 / 平台）。請參閱 [`products/launchlab/platforms`](/zh-Hant/products/launchlab/platforms) 瞭解驅動此功能的平台 PDA 欄位。

無論分配如何，**機制都相同**：每個未銷毀的份額都表示為 Fee Key NFT，可以交易、轉讓或支付。

### 遷移後創作者費用被禁用時會發生什麼

如果平台設定 `creator_fee_enabled = false`，畢業代幣**不會**經過 Burn & Earn。相反，它會使用遺留 LaunchLab 遷移路徑遷移到 **AMM v4** 資金池。在該模式下，創作者不會收到 Fee Key NFT，且不存在程式級遷移後創作者費用。某些處於此模式的創作者使用離鏈 CLI 工具領取透過 `creatorFeeOn` 配置的可選遷移後費用 — 請參閱 [`products/launchlab/instructions`](/zh-Hant/products/launchlab/instructions) 瞭解鏈上表面。

## 0.1% 代幣分享推薦

除了 LP 費用分享外，LaunchLab 創作者（實際上任何使用者都可以）可分享包含其錢包作為推薦者的**代幣頁面連結**。當任何人透過共享連結交易該代幣（或同一個會話中的其他 LaunchLab 代幣）時：

- **每筆交易的 0.1%** 被空投到推薦者的錢包。
- 以 SOL 或資金池的報價代幣支付。
- 自動在每筆交易上支付；無需領取步驟。

這是與 [`user-flows/referrals-and-blinks`](/zh-Hant/user-flows/referrals-and-blinks) 中涵蓋的 1% 掉期推薦不同的產品。1% 推薦適用於 Raydium 的現貨掉期 UI；0.1% 代幣分享推薦專門適用於 LaunchLab 代幣頁面。

## 領取費用

### 創作者保管庫（遷移前）

```ts
import { Raydium, TxVersion } from "@raydium-io/raydium-sdk-v2";
const raydium = await Raydium.load({ owner, connection });
const { execute } = await raydium.launchlab.claimCreatorFees({
  mint,           // launched token mint
  txVersion: TxVersion.V0,
});
await execute({ sendAndConfirm: true });
```

### Fee Key NFT（遷移後）

在錢包中持有 Fee Key 就足夠了 — 從投資組合頁面領取，或透過相關資金池程式的費用領取指令以程式方式領取。請參閱 [`products/cpmm/instructions`](/zh-Hant/products/cpmm/instructions) 或 AMM v4 等效項瞭解鏈上表面。

## 整合商檢查清單

- **展示兩條費用流。** 只顯示「遷移前保管庫」或「遷移後 Fee Key」之一的創作者儀表板會讓不知道另一個存在的使用者損失金錢。
- **尊重 Fee Key 所有權轉讓。** 如果創作者銷售或轉讓其 Fee Key NFT，前任所有者將喪失未來費用領取的訪問權限。你的 UI 應每次讀取當前 NFT 持有者，而不是快取原始創作者地址。
- **平台配置漂移。** Raydium 的預設值可能會改變。對於 Raydium 自有平台以外的任何平台，請始終動態讀取平台 PDA 以查看該特定啟動的實際銷毀 / 創作者 / 平台分配。
- **Token-2022 抵押品和費用代幣。** 創作者費用可以以具有轉讓費用的 Token-2022 代幣計價；將轉讓費用的稅收納入任何顯示的「可領取費用」數字中。
