根據預設，來自 LaunchLab 遷移集區的 CPMM 建立者費用會歸屬於原始代幣建立者。平台可以透過設定 `platformCpCreator` 來覆寫此行為，以適用於後續的遷移。

## 為什麼要重新導向建立者費用

平台在以下情況下會使用建立者費用重新導向：

- 在建立者、社群和平台收益之間實施自訂分配邏輯。
- 使用 PDA 控制的費用收集器進行鏈上重新分配。
- 統一核算單一平台下所有遷移集區的帳目。

<Warning>
  重新導向建立者費用會改變誰可以收集它們。除非平台將費用重新分配給代幣建立者，否則在設定啟動後遷移的集區，代幣建立者將失去直接收集存取權。
</Warning>

## 運作原理

1. 平台管理員呼叫 `updatePlatformConfig`。
2. `updateInfo.type` 設定為 `updatePlatformCpCreator`。
3. 該值是一個錢包或 PDA，將成為未來遷移集區的 CPMM 集區建立者。
4. 該平台下的後續 LaunchLab 遷移會將 CPMM 建立者費用分配給已設定的地址。
5. 平台根據自己的規則收集並重新分配建立者費用。

```ts
await raydium.launchpad.updatePlatformConfig({
  programId,
  platformAdmin: owner,
  updateInfo: {
    type: "updatePlatformCpCreator",
    value: new PublicKey("<PLATFORM_WALLET_OR_PDA>"),
  },
  txVersion: TxVersion.V0,
});
```

## 注意事項

- **單一平台的全有或全無。** 該設定適用於該平台下的每一次未來遷移。
- **僅面向未來。** 已遷移的集區保留其現有的集區建立者。
- **Epoch 限制的更新。** 平台配置更新受 Solana epoch 速率限制。
- **建立者費用必須存在。** 如果選定的 CPMM `AmmConfig` 的 `creator_fee_rate = 0`，則沒有 CPMM 建立者費用可收集。
- **WSOL 處理很重要。** 如果收集的費用是 WSOL，且你需要原生 SOL，請在收集後關閉 WSOL 代幣帳戶。

## 重新分配檢查清單

如果你重新分配給原始建立者：

1. 從與 CPMM 集區 mint 相關的 LaunchLab 集區帳戶解析每個原始建立者。
2. 收集平台錢包或 PDA 的待領建立者費用。
3. 計算每個建立者的協議份額。
4. 將 SPL 代幣或 SOL 轉帳給建立者，視需要建立 ATA。
5. 發布足夠的核算詳情供建立者驗證分配。
