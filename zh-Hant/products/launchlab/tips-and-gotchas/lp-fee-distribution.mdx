當 LaunchLab 代幣遷移到 CPMM 時，遷移後的池會鑄造 LP 代幣。平台設定控制這些 LP 代幣的處理方式。

## LP 分配

`migrateCpLockNftScale` 有三個欄位。這些值的總和必須是 `1_000_000`，代表 100%。

| 欄位 | 說明 |
|---|---|
| `platformScale` | LP 代幣透過 Burn & Earn 鎖定。平台獲得費用金鑰 NFT。 |
| `creatorScale` | LP 代幣透過 Burn & Earn 鎖定。代幣建立者獲得費用金鑰 NFT。 |
| `burnScale` | LP 代幣被永久銷毀。此份額不會建立費用金鑰 NFT。 |

這是平台級設定。透過同一平台啟動的代幣會繼承相同的 LP 分配，直到平台更新設定為止。

## 範例

```ts
// 40% 平台, 50% 建立者, 10% 銷毀。
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% 建立者費用金鑰。
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% 銷毀。
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## 與 CPMM 費用的關係

`migrateCpLockNftScale` 控制**誰可以申領 LP 費用權利**。它不設定交換費率。遷移後池的交換費用行為來自 `cpConfigId` 選定的 CPMM `AmmConfig`。

例如，如果遷移的 CPMM 池使用 0.25% 交易費設定且 `creatorScale` 為 500,000，建立者的費用金鑰代表鎖定 LP 費用權利的 50%。如果 `platformScale` 為 400,000，平台的費用金鑰代表 40%。

## 更新分配

平台設定更新受到時期速率限制。將 LP 分配視為重要的平台政策，而非隨意的 UI 設定。

更新前：

- 確認三個比例值的總和為 `1_000_000`。
- 確認平台理解未來遷移後誰將接收費用金鑰 NFT。
- 在新啟動使用它之前，為建立者記錄變更。
