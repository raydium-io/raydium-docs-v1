## 五種訂單類型

**市價訂單。** 以訂單簿中最佳可用價格立即買進或賣出。會逐筆配對待成交訂單，直到你的數量成交。若訂單簿無法填滿全部數量，未成交部分會被取消（不會掛單）。當你優先考慮立即成交而願意接受滑點時使用。

**限價訂單。** 以你選擇的特定價格（**限價**）或更好的價格買進或賣出。限價訂單會掛在訂單簿上，直到成交或被取消。當你優先考慮價格而非成交速度時使用。

**停損市價訂單。** 在市場達到你的**觸發（停損）價格**之前保持無效狀態。觸發後，會以市價訂單提交，並在有足夠流動性時以下一個可用價格成交。當你優先考慮立即成交而非價格控制時使用。

**停損限價訂單。** 在市場達到你的觸發價格之前保持無效狀態。觸發後，會以你指定的限價訂單提交，且只有在訂單簿達到該價格時才會成交。當你希望停損訂單永遠不會以比底線/上限更差的價格成交時使用，但要接受在價格跳空時可能無法成交的情況。

**分層訂單。** 將單一較大訂單分割成多個較小的限價訂單，分散在一個價格區間內。當你想要漸進式進場或出場，而不是一次性全部推進時使用。

## 四種訂單標記

這些標記可以附加到限價或停損限價訂單上，以改變執行語義：

**IOC — 立即成交或取消。** 以限價（或更優價格）盡快成交。任何未成交部分會被取消，而不是掛在訂單簿上。當你想要立即成交但有價格上限時使用。

**FOK — 全部成交或取消。** 以限價（或更優價格）成交**整個**訂單，或者完全取消且不允許部分成交。當部分成交比沒有成交更差的情況使用——例如，該交易只有在全部數量成交時才有意義。

**Post-Only。** 確保訂單提供流動性（做市商方）。如果訂單在提交時會配對到待成交訂單（即會成為吃單方），訂單會被取消而不是越過。當你希望以做市商身份成交而非吃單方身份時使用。

**Reduce-Only。** 防止訂單*增加*你的倉位。它只能在該市場關閉或減少現有倉位。如果你沒有倉位，或訂單會增加倉位，該訂單不會執行。當用於自動止盈/停損邏輯，且你永遠不想意外翻轉到相反方向時使用。

## 結合訂單類型與標記

並非所有組合都被允許。常見的組合包括：

| 訂單 | 標記 | 典型使用場景 |
|---|---|---|
| 限價 | — | 掛單做市商報價 |
| 限價 | Post-Only | 嚴格做市商報價（無意外吃單） |
| 限價 | IOC | 具有價格上限的激進吃單 |
| 限價 | FOK | 全部成交或沒有成交 |
| 限價 | Reduce-Only | 以特定價格出場 |
| 市價 | Reduce-Only | 以任何價格緊急平倉 |
| 停損市價 | Reduce-Only | 優先考慮出場、不翻轉的停損 |
| 停損限價 | Reduce-Only | 帶價格下限、不翻轉的停損 |
| 分層 | Post-Only | 被動分層進場 |

Post-Only 和 Reduce-Only 可以在同一訂單上組合使用。

## 手續費影響

訂單類型和標記會影響手續費等級：

- **市價訂單總是支付吃單費用。** 市價訂單無法獲得做市商返傭。
- **限價訂單**如果掛在訂單簿上並在之後成交，則支付做市商費用；如果在提交時立即成交，則支付吃單費用。
- **Post-Only** 藉由拒絕任何會越過的提交來針對做市商費用。
- **停損訂單**支付觸發時轉換成的訂單類別的費用（停損市價 → 吃單；停損限價 → 做市商，除非它會越過）。

以成交量為基礎的吃單等級和固定的 0 bps 做市商費用記錄在[手續費](/zh-Hant/products/perps/fees)中。
