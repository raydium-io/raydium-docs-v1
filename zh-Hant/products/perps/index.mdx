## 產品介紹

**Raydium Perpetual** 是一個無 Gas 中央限單簿 (CLOB) 永續合約協議。部位由 Solana 上的 SPL 錢包提供資金，但配對引擎和訂單簿由 [Orderly Network](https://orderly.network/) 運營，該網路在多鏈場所集合中（Solana、Ethereum、Base、Arbitrum、BSC）匯集流動性。

主要特性：

- **無 Gas 交易。** 訂單建立、修改和取消不需要在 Solana 上進行鏈上交易——簽署是本地進行且免費的。只有存款和提款涉及鏈上操作。
- **最高 100 倍槓桿**，取決於市場。
- **多鏈抵押品** ——你可以從原生鏈上存入 USDC / SOL / USDT / ETH / BNB / WBTC / YUSD / USD1；所有盈虧均以 USDC 結算。
- **僅限跨保證金。** 帳戶中的所有抵押品在所有開放部位之間共用。
- **單向模式。** 你不能在同一市場上同時持有多頭和空頭；相反方向的訂單會減少或翻轉現有部位。
- **自託管。** 資金保留在你的錢包的 Perps 帳戶中；沒有 Raydium 持有的託管錢包。
- **CEX 級訂單類型。** 市價單、限價單、止損（市價 / 限價）、Scale 訂單，以及 IOC / FOK / Post-Only / Reduce-Only 標記。

## 為何是獨立產品

Raydium AMM（現貨）和 Raydium Perps（永續期貨）共享品牌和 UI 入口點，但在協議層面上幾乎沒有任何共同之處：

| | Raydium 現貨 AMM | Raydium Perps |
|---|---|---|
| 場所 | Solana 上的鏈上池 | Orderly 運營的鏈下 CLOB |
| 配對引擎 | AMM 曲線（CPMM / CLMM / AMM v4） | 中央限單簿 |
| 結算 | 交換時鏈上結算 | 鏈下；存款/提款在鏈上 |
| 價格發現 | AMM 曲線 + 套利 | 訂單簿 + 外部價格 |
| 代幣生態 | 任何 SPL / Token-2022 鑄幣 | 僅限上市的 Perp 市場（BTC、ETH、SOL 及其他主要代幣） |
| 位於 [`reference/program-addresses`](/zh-Hant/reference/program-addresses) 的程式 ID | 是 | 否——參見 Orderly 的合約 |
| Raydium SDK v2 涵蓋範圍 | 完整 | 未涵蓋 |
| Raydium 漏洞賞金計畫 | 是 | 透過 Orderly 路由 |

本文檔網站的其餘部分幾乎完全聚焦於現貨 AMM 和 Raydium 組織 GitHub 中的程式。Perps 在產品層面上有文檔——訂單類型、費用、抵押品、限制——因此讀者可以了解它是什麼以及何時使用它，但深入的協議參考文檔位於 Orderly 一方。

## 限制司法管轄區

Raydium Perps **不可用**於美國、阿富汗、白俄羅斯、中非共和國、剛果民主共和國、朝鮮民主主義人民共和國（北韓）、烏克蘭克里米亞 / 頓涅茨克 / 盧甘斯克地區、古巴、伊朗、利比亞、索馬里、蘇丹、南蘇丹、敘利亞、葉門、津巴布韋或任何其他禁止 Raydium Protocol 的司法管轄區的居民。

嵌入 Perps UI 或 API 的整合商必須遵守此清單。

## 整合說明

- 現貨 Raydium SDK（`@raydium-io/raydium-sdk-v2`）**不**涵蓋 Perps。如需程式化存取，請使用 [Orderly 的 SDK](https://orderly.network/docs/build-on-omnichain/evm-api/introduction)（TypeScript、REST、WebSocket）。
- 沒有「透過 Perp 部位路由交換」的基本操作。想同時獲得現貨和 Perp 敞口的客戶端應獨立構建這兩筆交易。
- Orderly 的風險參數（槓桿上限、清算閾值、抵押品限制）會隨時間變化。在向用戶引用風險之前，請始終從 Orderly 的 API 取得即時參數。
