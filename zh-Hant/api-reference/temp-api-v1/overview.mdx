Temp API 是一個臨時存放區，用於存放尚未在 Raydium 主要 API 中取得永久位置的端點。此服務主要用於快速原型開發、內部工具，以及正在積極開發中的功能。

## 主機

| 環境 | 主機 |
|-------------|----------------------------------|
| 主網     | `temp-api-v1.raydium.io`         |
| 開發網      | `temp-api-v1-devnet.raydium.io`  |

## 穩定性警告

**此服務不提供任何 API 穩定性保證。** 端點可能在未事先通知的情況下變更、被棄用、移至其他服務或完全移除。請僅在以下情況使用此 API：

- 內部工具與腳本
- 正在積極開發中的功能
- 可以承受中斷的非關鍵路徑
- 測試與原型開發

請勿在正式環境的 dApp 或關鍵整合中依賴此 API。

## 概覽

此服務目前提供以下端點：

- **`/cp-creator-fee`** — 取得某個錢包在所有 CPMM 池中累積的創建者費用。
- **`/limit-order/order/list`** — 從索引器快取中取得錢包目前掛單的 CLMM 限價單（包含未成交與部分成交）。
- **`/limit-order/history/order/list-by-user`** — 取得錢包的歷史（已關閉）CLMM 限價單，支援 `poolId` / `mint1` / `mint2` / `hideCancel` 等可選篩選條件；透過 `nextPageId` 進行游標分頁。
- **`/limit-order/history/event/list-by-pda`** — 取得一個或多個限價單 PDA 的逐筆事件記錄（`open` / `increase` / `decrease` / `settle` / `close`）；透過 `nextPageId` 進行游標分頁。

## 常見使用模式

### 查詢錢包的創建者費用

```bash
curl -s "https://temp-api-v1.raydium.io/cp-creator-fee?wallet=<your-wallet-pubkey>"
```

回傳該錢包作為創建者的池子清單，以及 Token A 和 Token B 中已累積但尚未提領的創建者費用。

### 列出錢包目前掛單的 CLMM 限價單

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/order/list?wallet=<your-wallet-pubkey>"
```

同時回傳完全未成交與部分成交的訂單。每筆資料均為已解碼的 `LimitOrderState`（包含 PDA、池子、擁有者、tick、方向、FIFO 同批次階段、總量/已成交量），以及呼叫 `SettleLimitOrder` 所需的池子上下文，包含預先計算的 `pendingSettle` 金額。

活躍訂單端點不分頁——一次回傳錢包所有已快取的訂單。

### 列出錢包的已關閉限價單

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/order/list-by-user?wallet=<your-wallet-pubkey>&size=50"
```

回傳一頁歷史訂單，每筆訂單包含 `fillStatus`（`NONE` / `PARTIAL` / `FULL`）摘要，以及訂單離開活躍集合時的輸入／輸出金額。在請求中附加 `&nextPageId=<上一頁回應的 nextPageId>` 即可取得下一頁。

### 查詢特定訂單的事件時間軸

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/event/list-by-pda?pda=<limit-order-pda>"
```

`pda` 可為單一公鑰或以逗號分隔的多個公鑰。每筆資料代表一次鏈上狀態變更（`open` / `increase` / `decrease` / `settle` / `close`），並包含 txid、slot、區塊時間、由擁有者或 keeper 簽署的旗標（`autoRunner`），以及以人類可讀單位表示的輸入／輸出變化量。

### 空結果

若錢包沒有任何已快取的資料，活躍訂單端點將回傳：

```json
{
  "id": "...",
  "success": true,
  "data": { "rows": [] }
}
```

歷史端點同樣回傳 `{ rows: [], nextPageId?: ... }` 結構，但 `rows` 陣列為空。舊版 `/cp-creator-fee` 端點則為了向下相容，回傳 `"data": []`。
