## 選擇你的整合路徑

有五種路徑可供選擇。選擇哪一條取決於呼叫 Raydium 的對象是什麼，以及你需要多少控制權。

| 路徑 | 最適用於 | 延遲 | 自訂邏輯 | 語言 |
|---|---|---|---|---|
| [Trade API](/zh-Hant/sdk-api/trade-api) | 後端、機器人、Blinks — 任何需要伺服器建構交易的地方 | 最低 | 受限（預設路由） | 任何 |
| [TypeScript SDK](/zh-Hant/sdk-api/typescript-sdk) | 需要完整建構器存取權限的 Node/TS 應用和前端 | 中等 | 完整 | TS / JS |
| [Python 整合](/zh-Hant/sdk-api/python-integration) | Python 後端和 ML 流程 | 中等 | 完整（手動指令建構） | Python |
| [Rust CPI](/zh-Hant/sdk-api/rust-cpi) | 組合從鏈上呼叫 Raydium 的程式 | N/A（鏈上） | 完整 + 可組合 | Rust |
| [Anchor IDL](/zh-Hant/sdk-api/anchor-idl) | 為沒有原生支援的語言產生程式碼（Go、Kotlin、Swift 等） | 變動 | 完整 | 任何使用 Anchor 產生碼工具的語言 |

如果你不確定：後端和機器人 → **Trade API**；前端和完整控制應用 → **TypeScript SDK**；鏈上組合 → **Rust CPI**。
