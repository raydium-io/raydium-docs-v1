## 遷移的原因

| 面向 | AMM v4 | CPMM |
|-----------|--------|------|
| 費用流向 | 協議與 LP 分配 | LP 獲得預設的交換費分配 |
| OpenBook 依賴性 | 有 | 無 |
| 建立成本 | 較高 | 較低 |
| Token-2022 支援 | 否 | 是 |
| 新掛牌路由至 | 罕見 | 預設 |

如果你的交易對交易量很大，LP 費用提升可能能夠證明遷移的合理性。如果舊資金池仍保持良好流動性，且項目尚未認可目標 CPMM 資金池，遷移可能不值得協調成本。

## 遷移前檢查清單

針對資金池運營者和項目國庫：

- [ ] 向 LP 公告遷移時間表。
- [ ] 盡可能給予 LP 至少 48 小時的準備時間。
- [ ] 確認目標 CPMM 資金池和費率級別。
- [ ] 準備遷移後公告，包含新的資金池 ID 和 LP 代幣。
- [ ] 決定是否需要建立新的農場。

針對個人 LP：

- [ ] 確認你追蹤的是項目認可的遷移。
- [ ] 保留足夠的 SOL 以支付多筆交易。
- [ ] 記下你的 AMM v4 LP 餘額和預期的基礎代幣數量。
- [ ] 檢查你的 AMM v4 LP 代幣是否質押在農場中。

## UI 遷移：個人 LP

遷移涉及兩個操作：從 AMM v4 移除流動性，然後向 CPMM 添加流動性。

### 第 1 步：從 AMM v4 提取

1. 開啟 AMM v4 資金池或你的投資組合位置。
2. 如果你的 LP 代幣質押在農場中，先取消質押。
3. 點擊**移除流動性**。
4. 選擇要提取的百分比。
5. 審查你將收到的代幣金額。
6. 確認並批准。

### 第 2 步：存入 CPMM

1. 開啟目標 CPMM 資金池。
2. 點擊**添加流動性**。
3. 輸入一種代幣的金額。
4. 審查配對代幣金額和任何剩餘餘額。
5. 確認並批准。

在提取和存入之間，資金池價格可能會變動。如果兩個資金池的比率不同，某些代幣可能在你的錢包中留下灰塵。

## UI 遷移：項目國庫

對於項目國庫，流程相同但需要協調：

1. **提前公告**遷移計畫。
2. **在要求用戶遷移之前為 CPMM 資金池提供流動性種子**。
3. **先遷移國庫 LP**並發佈交易鏈接。
4. **建立新的農場**（如果激勵應繼續在 CPMM LP 代幣上）。
5. **監控個人 LP 遷移**（在接下來的幾天或幾週內）。
6. **棄用舊資金池**（僅在用戶有足夠時間移動之後）。

## 回滾考慮

從 CPMM 回滾到 AMM v4 意味著從 CPMM 提取並存入 AMM v4。通常除非有特定的產品原因，否則不值得這樣做。AMM v4 資金池建立成本更高且需要 OpenBook 市場設定。

## 常見問題

### 我的 AMM v4 LP 質押在農場中。我可以遷移嗎？

可以。先取消質押，從 AMM v4 提取 LP，存入 CPMM，如果新農場存在，則質押新的 CPMM LP 代幣。

### CPMM 資金池還不存在

大多數 LP 應等待項目建立並認可目標資金池。自己建立一個意味著你承擔初始價格和流動性風險。

### 我有大額頭寸。滑點如何？

如果你的頭寸相對於資金池 TVL 較大，分批次提取和存入。審查每個報價並避免在收到的金額明顯低於預期時簽名。

### 我忘記遷移的 LP 代幣會怎樣？

它們保留在舊資金池中並按照 AMM v4 規則繼續賺取收益。沒有任何東西強制遷移。

## 故障排除

### 目標 LP 代幣帳戶是新的

第一次 CPMM 存入可能需要建立新的 LP 代幣帳戶。在你的錢包中保留額外的 SOL 以支付租金和費用。

### 第二筆交易過期

如果提取成功但存入過期，你的代幣保留在你的錢包中。重新開啟 CPMM 資金池並使用更高的優先費設定重試。

### 錯誤的 CPMM 資金池

同一交易對可以存在多個 CPMM 資金池。使用項目認可的資金池或具有預期費率級別和流動性的資金池。

### 稅務影響

在某些司法管轄區，LP 提取可能是應稅事件。在遷移大額頭寸之前請諮詢當地規則。
