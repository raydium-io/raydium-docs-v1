## Burn & Earn 是什麼

Burn & Earn 是一個 Raydium 功能，讓池創建者（或任何 LP）在保留領取交易費用權利的同時，將存款在鏈上**永久化**。頭寸變為不可贖回，但新鑄造的 **Raydium Fee Key** NFT 代表從該鎖定頭寸領取 `collectFees` 的持續權利。

常見用途：

- **啟動信任。** 專案透過鎖定部署者的 LP 來承諾不地毯。持有人可以在鏈上驗證流動性是永久的。
- **產生費用的收藏品。** 可交易的 NFT，產生收益。出售 NFT；轉移費用流。
- **金庫級別的 LP。** 想要永久保留流動性以用於品牌和發現目的的協議，同時仍然從費用方面獲利。

## 支援的池類型

| 程式 | Burn & Earn 支援 |
|---|---|
| CPMM | 是 |
| CLMM | **僅全範圍**（推薦） |
| AMM v4（混合） | 不支援 |

對於 CLMM，只有跨越完整價格範圍的頭寸應該被鎖定。一旦鎖定，你無法重新平衡——如果價格移出你選擇的範圍，你將永久停止從該頭寸賺取費用。緊範圍的 Burn & Earn 頭寸幾乎總是一個錯誤。

## 鎖定在程式層級的工作原理

1. 原始 LP token（CPMM）或頭寸 NFT（CLMM）轉移到由 Burn & Earn 程式所有的程式所有 PDA。
2. **Fee Key NFT** 被鑄造給呼叫者。其元資料指回鎖定頭寸的 PDA。
3. Burn & Earn 公開 `collectFees`（或每種池類型的等效呼叫），它會：
   - 驗證簽署者持有給定鎖定頭寸的 Fee Key NFT。
   - 透過 CPI 呼叫基礎池程式的費用領取指令。
   - 將收集的費用路由到 Fee Key 持有人的 ATA。
4. 沒有對應的 `withdraw`——基礎 LP 或頭寸永遠不會離開代管帳戶。

Fee Key 是一個標準 SPL NFT。它可以在任何 Solana NFT 市集上出售、在錢包之間移動、用作接受 NFT 的借貸協議的抵押品，或包裝到另一個合約中。無論誰在 `collectFees` 呼叫時持有它，都會收到費用。

## UI 演練

1. 照常建立一個 CPMM 池或具有全範圍頭寸的 CLMM 池。
2. 前往**流動性 → 建立 → Burn & Earn**。
3. 選擇要鎖定的頭寸。
4. 驗證頭寸詳細資訊（NFT 地址、大小、池）並輸入 UI 顯示的確認句子。這是故意設置摩擦，因為該操作是不可逆的。
5. 點擊**確認，永久鎖定流動性**。
6. Fee Key NFT 抵達你的錢包。它出現在投資組合頁面的鎖定頭寸下。

以相同的方式領取費用——透過投資組合頁面或 `collectFees`。

## API 表面

池資訊端點返回一個 `burnPercent` 欄位，表示該池中透過 Burn & Earn 鎖定的流動性的份額：

```http
GET https://api-v3.raydium.io/pools/info/list
  ?poolType=standard
  &poolSortField=default
  &sortType=desc
  &pageSize=1
  &page=1
```

```json
{
  "data": [
    {
      "id": "...",
      "burnPercent": 99.17,
      ...
    }
  ]
}
```

`burnPercent` 是一個方便的信任信號表面：LP 被 99%+ 燒毀的代幣遠不如 LP 被 0% 燒毀的代幣容易地毯。

## 陷阱

- **不可逆。** LP 方永遠無法取回。確認該金額是你願意永遠不再接觸的東西。
- **CLMM 緊範圍是一個陷阱。** 只鎖定全範圍的 CLMM 頭寸，除非你真正理解範圍限制的鎖定頭寸在價格超出波段後停止賺取費用。
- **丟失 Fee Key NFT 就是永遠丟失費用。** 像對待契約一樣對待它。不要意外銷毀它；不要將其質押在惡意合約中。
- **具有轉移費用的 Token-2022 mint**：Burn & Earn 支援它們，但轉移費用扣減適用於每次費用領取——預算它。
- **沒有部分鎖定。** 如果你想要 50% 鎖定和 50% 可移動，先將頭寸拆分為兩個頭寸，然後只鎖定其中一個。
