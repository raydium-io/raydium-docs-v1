兌換是最常見的用戶操作。本頁將介紹 Raydium UI 的流程、解釋你可以控制的設定，以及列出你可能遇到的故障模式。

## 輸入參數

每次兌換都需要：

- **輸入代幣**（Mint 地址 + 數量）
- **輸出代幣**（Mint 地址）
- **滑點容差**（UI 預設 0.5%；波動性交易對為 1%）
- **優先費用**（預設自動計算）

數量可以指定為「固定輸入」（你決定支付多少；輸出經由計算得出）或「固定輸出」（你決定獲得多少；輸入經由計算得出）。UI 預設使用固定輸入。

## UI 操作流程

在 [raydium.io/swap](https://raydium.io/swap)：

1. **輸入數量。** 選擇輸入和輸出代幣；在任一個欄位輸入數量。另一個欄位會自動填入當前報價。

2. **檢查路由。** 路由面板顯示 Raydium 選擇的流動性池：
   - 單池（最常見）：「SOL → USDC via CLMM 0.05%」
   - 多跳：「SOL → USDC via CPMM，然後 USDC → RAY via CLMM」
   - 分割：「SOL → USDC via 60% CLMM 0.05%，40% CPMM 0.25%」

   對於每個跳轉：顯示 TVL、費用和預期影響。較大的交易在影響超過 1% 時會顯示警告。

3. **檢查滑點。** 預設 0.5%。穩定交易對為 0.1%；迷因代幣為 2–5%。如果你看報價時行情波動（價格變動），UI 每 10 秒重新報價一次並閃爍顯示。

4. **設定優先費用。** 預設自動；三個等級：
   - **一般**（50 百分位）— 預設
   - **快速**（75 百分位）
   - **超快速**（95 百分位）
   進階用戶可輸入特定的微 lamport/CU 值。

5. **確認並簽署。** 交易發出；UI 追蹤確認。在正常擁塞情況下預期時間 5–15 秒。

6. **兌換後。** UI 顯示實際收到的數量（可能與報價相差至多滑點）；點擊進入交易瀏覽器查看完整詳情。

## 路由如何選擇流動性池

當同一交易對存在多個流動性池時，Raydium 的路由器依據以下條件選擇：

1. **費用後最高 `amountOut`。** 計算每個符合條件的流動性池的報價；選擇輸出最多的那個。
2. **TVL 作為平手情況下的決定因素。** 更高的 TVL 意味著對下一筆交易的影響更小 — 對報價和提交之間的價格波動更有抵抗力。
3. **偏好單池而非多跳。** 多跳會複合費用和滑點；路由器只有在多跳超過單池 >5 bps 時才選擇多跳。
4. **大額交易時分割。** 對於相對於流動性池 TVL 較大的交易，路由可能在多個流動性池之間分割，以減少價格影響。

典型交易對的產品比較：

- **CPMM 流動性池**在低 TVL（`<$1M`）時佔主導，因為建立成本低，且兌換費用分享直接資助流動性提供者。
- **CLMM 流動性池**在中高 TVL 的波動性交易對上佔主導 — 集中流動性在常見價格範圍內提供更好的報價。
- **AMM v4 流動性池**在與 OpenBook 深度整合的歷史悠久交易對上佔主導（SOL-USDC、RAY-USDC）。大多數新交易對從不涉及 AMM v4。

## Token-2022 轉移費用

如果輸入或輸出 Mint 是具有轉移費用的 Token-2022：

- 報價中的 `amountOut` 是用戶在費用扣除後的淨收到金額。
- UI 中顯示的「費用」分為「流動性池費用」（LP 費用）和「轉移費用」（支付給 Mint 發行人）。

參閱 [`algorithms/token-2022-transfer-fees`](/zh-Hant/algorithms/token-2022-transfer-fees)。

## 滑點

滑點保護你免受報價和執行之間的不利價格變動。Raydium 將你的滑點百分比轉換為你必須接收的最小數量：

```
minimumAmountOut = expectedAmountOut × (1 - slippage)
```

如果實際輸出會較少，交易會以 `ExceededSlippage` 回復。常見預設值：

| 交易對類型 | 滑點 |
|-----------|------|
| 穩定對穩定 | 0.01–0.1% |
| 主流對主流（SOL/USDC） | 0.1–0.5% |
| 中市值波動性 | 0.5–1% |
| 迷因代幣 | 2–5% |
| Token-2022 with transfer fee | 加上轉移費用至滑點 |

滑點過緊會在正常價格變動下頻繁回復。滑點過寬會招致三明治攻擊（參閱 [`integration-guides/routing-and-mev`](/zh-Hant/integration-guides/routing-and-mev)）。

## 優先費用

優先費用向驗證人支付更多以在擁塞時期處理你的交易。UI 預設為自動設定。當網路繁忙、你兌換波動性交易對，或先前的交易在確認前過期時，使用**快速**或**超快速**。

更高的優先費用不會改進報價的價格。它們只是提高你的交易在報價變陳舊之前著陸的機會。

## 常見故障模式

### 交易以 `ExceededSlippage` 回復

價格在報價和執行之間變動超過你的容差。修復：

1. 只根據需要增加滑點。
2. 減少交易規模（減少影響）。
3. 重新報價並重試；價格可能只是暫時波動。
4. 避免在主流交易對上設置非常高的滑點，因為它可能會讓你面臨更差的執行。

### 交易逾時

當前擁塞下優先費用過低。修復：提高優先費用；參閱 `integration-guides/priority-fee-tuning`。

### CLMM 兌換上的 `InvalidAccountData`

路由可能依賴於無法載入或在報價開放時更改的 CLMM 帳戶。重新整理頁面、重新報價並重試。如果問題持續，如果 UI 提供，選擇另一路由。

### 「流動性池未找到」/ 陳舊的流動性池數據

流動性池最近建立且尚未索引。按 ID 直接重新取得：等幾分鐘、重新整理應用程式，然後再次按流動性池或代幣 Mint 搜尋。新建立的流動性池可能需要一段時間才能在索引 UI 搜尋中顯示。

### 輸出為零 / 非常小

兌換規模超過流動性池流動性。流動性池太淺；減少交易規模、嘗試另一路由，或等待更多流動性。

### 交易著陸但餘額未變

重新整理你的錢包和 Raydium 餘額。如果交易在區塊瀏覽器上成功，檢查輸出代幣帳戶是否在你的錢包 UI 中隱藏。

## 驗證

使用 Raydium 確認後顯示的交易連結。在 Solscan 或 SolanaFM 上，檢查：

- **狀態：** 成功。
- **簽署人：** 你的錢包地址。
- **代幣餘額變化：** 輸入代幣減少，輸出代幣增加。
- **程式已調用：** Raydium 兌換程式加上代幣/系統程式。
