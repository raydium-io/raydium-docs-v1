## 訂單類型

Raydium 永續合約支援以下訂單類型：

| 訂單類型 | 功能 | 適用情況 |
|---|---|---|
| **市價** | 立即按訂單簿上最優價格買進或賣出。如果訂單簿無法完整成交，未成交部分會被取消。 | 你需要立即成交，並願意接受潛在滑價。 |
| **限價** | 按你設定的限價或更優價格買進或賣出。訂單只在市場達到該價格時才會成交。 | 你想控制成交價格，並且可以等待成交。 |
| **止損市價** | 保持待命狀態直到觸發價格被達到，然後轉變為市價訂單。 | 你想設定執行觸發條件，並優先在觸發後立即進場或出場。 |
| **止損限價** | 保持待命狀態直到觸發價格被達到，然後轉變為限價訂單。 | 你想設定觸發條件，但同時想保留限價。 |
| **規模化** | 將一筆大訂單分拆成多筆較小的限價訂單，分散在一個價格區間內。 | 你想逐步進場或出場。 |

## 訂單旗標

訂單旗標會改變限價類訂單的行為方式。

| 旗標 | 行為 |
|---|---|
| **IOC** | 立即成交或取消。以你設定的限價或更優價格立即成交盡可能多的量，然後取消未成交部分。 |
| **FOK** | 全部成交或全部取消。以你設定的限價或更優價格完整成交整筆訂單，否則全部取消且不接受部分成交。 |
| **只掛單** | 作為流動性提供者增加流動性。如果會立即成交，訂單反而會被取消。 |
| **只減倉** | 防止訂單增加你在該市場的部位。訂單只能減少或平倉現有部位。 |

## 提交訂單前的檢查

在提交訂單前，請確認：

- 市場與方向正確。
- 數量與槓桿符合你的意圖。
- 止損觸發價與限價沒有顛倒。
- 當你只想平倉或減少敞口時，已啟用只減倉。
- 當你想採用掛單者行為時，已啟用只掛單。
