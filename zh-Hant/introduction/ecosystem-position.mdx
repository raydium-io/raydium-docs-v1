Raydium 並非獨立運作。它從其他 Solana 協議網絡中消耗輸入，同時產生輸出。本頁映射這些關係，幫助你理解 Raydium 責任的邊界以及其他協議的行為如何影響你作為使用者或集成者的體驗。

## 上游依賴

### OpenBook（前身為 Serum）

- **角色**：中央限價訂單簿。AMM v4 池原本對 OpenBook 採取混合模式 — 交換可以針對池的 AMM 曲線或匹配的 OpenBook 市場填充 — 但 **OpenBook 集成已隨後被停用**。AMM v4 池不再在 OpenBook 上發佈或維護訂單。
- **相關之處**：歷史上僅影響 AMM v4 池；目前，Raydium 沒有任何產品對 OpenBook 有即時依賴。
- **風險面**：以前，OpenBook 升級或中斷可能影響 AMM v4 行為（2023 年 1 月發生過一起這樣的事件）。既然集成已關閉，那種耦合也已移除。CPMM 和 CLMM 從未對 OpenBook 有依賴。
- **狀態**：OpenBook 繼續在 Solana 上運作；Raydium 對任何當前交換流量都不依賴它。現有 AMM v4 池引用的 OpenBook 市場帳戶保留在鏈上作為向後相容的惰性狀態。

### Solana 執行時

- **角色**：所有 Raydium 程序的執行基礎。
- **相關之處**：處處相關。Solana 的終性時間（約 1 秒）、計算預算限制（每個交易 140 萬計算單元）和地址查找表都直接影響 Raydium 使用者體驗。
- **狀態**：Solana 定期升級；Raydium 跟蹤驗證者客戶端版本（Agave、Firedancer）並在行為改變時進行調整。

### 代幣程序（SPL Token 和 Token-2022）

- **角色**：所有 Raydium 池持有的代幣基礎。
- **SPL Token**：舊版程序；在所有產品上運作。
- **Token-2022**：啟用擴展的程序；在 CPMM、CLMM、Farm v6 上運作，不支持 AMM v4。
- **風險面**：擴展行為（轉帳費用、掛鉤等）可能影響池功能。參見 [`security/oracle-and-token-risks`](/zh-Hant/security/oracle-and-token-risks)。

## 下游集成

### Jupiter 及其他聚合器

- **角色**：在多個 Solana DEX 之間路由交易，包括 Raydium。
- **集成方式**：Jupiter 從 Raydium 的 REST API 和直接鏈上呼叫讀取池狀態，然後通過 Raydium SDK（有時直接）組成交換指令。
- **交易量占比**：大約 60% 的 Jupiter 交易量至少接觸一次 Raydium 池。
- **集成者路徑**：一般聚合器模式記錄在 [`integration-guides/aggregator`](/zh-Hant/integration-guides/aggregator)。

### 錢包

- **角色**：讓使用者無需離開錢包即可交換、追蹤 LP 部位並參與農場。
- **集成方式**：
  - 交換：通過 Jupiter 或直接通過 Raydium SDK 路由。
  - 部位顯示：列舉使用者的 ATA 和 NFT 持有，與 Raydium 的池列表交叉引用（[`integration-guides/wallet-integration`](/zh-Hant/integration-guides/wallet-integration)）。
  - 農場顯示：呼叫 Raydium 的 `/positions/staking` 端點。
- **使用者影響**：任何具有應用內交換的錢包都可能通過 Raydium 路由，而你不會察覺。

### 借貸協議（Kamino、Marginfi、Drift）

- **角色**：需要預言機價格和清算執行場所的借貸協議。
- **集成方式**：使用 CLMM ObservationState TWAP 作為預言機；通過 Raydium 直接交換以實現清算抵押品轉換。
- **風險**：如果借貸協議使用現貨 Raydium 價格而不是 TWAP，可能被閃電貸式操縱攻擊（參見 [`security/attack-vectors`](/zh-Hant/security/attack-vectors)）。

### MEV 搜索者和套利者

- **角色**：使 Raydium、Orca、Phoenix 和中心化交易所之間的價格保持一致。
- **對使用者的影響**：大多數時間價格緊密；對保護較差的交換造成三明治攻擊。參見 [`integration-guides/routing-and-mev`](/zh-Hant/integration-guides/routing-and-mev)。

### 機器人（做市商、DCA 執行者、清算人）

- **角色**：提供流動性（在淺層池上做 LP）、執行計劃交易、平倉部位。
- **集成路徑**：通常使用 [`sdk-api/python-integration`](/zh-Hant/sdk-api/python-integration) 的 Python 或 Rust 機器人或 Rust CPI 封裝。

## 聚合交易量中的位置

截至 2026 年 4 月：

- Raydium 每天處理約 35–45% 的 Solana 範圍內 DEX 交易量。
- 當模因幣活動集中在 LaunchLab 和畢業池時，峰值市場占有率約為 55%。
- 在大機構流向 Phoenix 的重度訂單簿驅動活動期間，低端約為 25%。

這些數字會波動；重點是 Raydium 始終是交易量最高的兩大場所之一。

## Token-2022 生態

Raydium 是首批原生支持 Token-2022 的主要 DEX 之一。影響包括：

- 具有轉帳費用的新鑄幣（例如，版稅、手續費轉向發行者的代幣）可以做 LP。
- 轉帳掛鉤鑄幣可以存在（儘管存在風險警告）。
- 不可轉帳和永久委託鑄幣被阻止。

隨著越來越多項目在 Token-2022 下啟動 — 特別是使用受監管鑄幣的「機構」群體 — Raydium 的支持將其定位為該類流動性的場所。

## 前瞻性依賴

可能改變 Raydium 生態位置的監控項：

- **Solana 驗證者多樣化**：如果 Firedancer 實現主導，性能特性可能轉變。
- **跨鏈橋接**：Raydium 一直在研究向其他 L1/L2 的擴展。暫未部署。
- **Jito 的演進**：區塊引擎與 DeFi 之間的持續集成可能改變 MEV 動態。
- **鏈上訂單簿（Phoenix）**：持續成熟可能使某些 AMM 式流向訂單簿轉移。
