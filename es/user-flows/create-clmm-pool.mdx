<Info>
  Un pool CLMM requiere dos decisiones permanentes: una **capa de comisión** y un **precio inicial**. También necesitas abrir una posición inicial para que el pool tenga liquidez. Ten especial cuidado antes de firmar.
</Info>

## Lo que necesitas

- Una billetera Solana que contenga:
  - Los dos *mint* de token que quieres emparejar.
  - Aproximadamente 0,3 SOL para la renta de creación del pool y posición, más comisiones de prioridad.
- Una decisión sobre la capa de comisión.
- Un precio inicial de mercado claro y un primer rango.

CLMM admite tanto *mints* de SPL Token como de Token-2022.

## Capas de comisión

| Capa de comisión | Mejor para |
|----------|----------|
| 0.01% | Pares stablecoin-stablecoin |
| 0.05% | Pares correlacionados |
| 0.25% | La mayoría de pares principales volátiles |
| 1% | Pares long-tail o exóticos |

Las capas de comisión más bajas requieren precios más ajustados y mayor volumen. Las capas más altas toleran mayor volatilidad pero pueden reducir el volumen.

## Recorrido por la interfaz

En [raydium.io/clmm/create-pool](https://raydium.io/clmm/create-pool):

1. **Elige el par de tokens.** Pega ambas direcciones de *mint*.
2. **Selecciona la capa de comisión.** Usa la recomendación de la interfaz como punto de partida.
3. **Establece el precio inicial.** Usa un precio de mercado existente. Si ya existe un pool para el par, compáralo con él.
4. **Elige el primer rango.** Ingresa el precio inferior y superior para la posición inicial.
5. **Ingresa el monto a depositar.** La interfaz calcula la proporción de token requerida a partir de tu rango.
6. **Revisa y firma.** Confirma el par, la capa de comisión, el precio inicial, el rango, los montos a depositar y la renta.

## Establecer el precio inicial correctamente

Si existen otros pools CLMM o CPMM para el mismo par, usa su precio actual como punto de partida. Si no existe ningún pool de Raydium, usa una fuente de mercado confiable.

Si estableces el precio inicial incorrectamente, es probable que la primera operación realice un arbitraje del pool y agote el lado subvaluado. No hay un botón de reinicio simple. Normalmente necesitas retirar y recrear el pool.

## Primera posición: selección del rango

- **Rango ajustado:** mejor para pares correlacionados, pero requiere gestión activa.
- **Rango medio:** común para pares principales volátiles como SOL/USDC.
- **Rango amplio:** menor eficiencia de capital, pero más fácil de dejar sin atender.
- **Rango completo:** se comporta más como un CPMM, pero pierde gran parte del beneficio de la liquidez concentrada.

Si el precio actual está fuera de tu rango, la posición inicial está fuera de rango y no genera comisiones hasta que el precio se mueva dentro del rango.

## Lista de verificación posterior a la creación

1. **La página del pool se carga.** Confirma la dirección del pool, el par y la capa de comisión.
2. **La posición aparece.** Verifica tu cartera de Raydium para el NFT de posición.
3. **El rango es correcto.** Confirma el precio inferior y superior.
4. **La liquidez es distinta de cero.** El pool debe mostrar liquidez activa para tu rango seleccionado.
5. **Prueba un swap pequeño.** Verifica que la ruta funciona antes de compartir el pool públicamente.

## Solución de problemas

### El costo de creación es mayor de lo esperado

Los rangos amplios y las capas de comisión pequeñas pueden requerir más renta de matriz de *tick*. Reduce el rango inicial o añade más SOL antes de reintentar.

### Orden de token incorrecto

La interfaz canonicaliza el orden de tokens, por lo que el token A/token B mostrado puede diferir del orden que ingresaste. Revisa cuidadosamente el presupuesto de precio antes de firmar.

### Capa de comisión incorrecta

No puedes cambiar la capa de comisión después de crear el pool. Si es incorrecta, cierra tu liquidez y crea un nuevo pool en la capa correcta.

### Pool sin liquidez

Crear el estado del pool sin una posición deja el pool inutilizable para operaciones. Abre una posición inicial antes de compartir el pool.

### Posición inicial fuera de rango

Si tu primer rango no incluye el precio inicial, la posición no genera comisiones. Reabre una posición alrededor del precio actual.
