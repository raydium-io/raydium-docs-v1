La API de propietario proporciona vistas por billetera de posiciones y saldos en Raydium. Obtiene datos en caché de posiciones, liquidez bloqueada, pools creados y participación en IDO, lo que la hace ideal para interfaces de billetera y paneles de cartera.

## Hosts

| Entorno   | Host                       |
|-----------|---------------------------|
| Mainnet   | `owner-v1.raydium.io`     |
| Devnet    | `owner-v1-devnet.raydium.io` |

## Descripción general

La API expone cinco puntos finales principales organizados por tipo de recurso:

- **`/position/stake/{owner}`** — Posiciones de stake (AMM estándar / CPMM LP)
- **`/position/clmm-lock/{owner}`** — Posiciones CLMM bloqueadas
- **`/create-pool/{owner}`** — Pools creados por esta billetera
- **`/main/ido/{owner}`** — Datos de participación en IDO

## Actualización de datos

Este servicio almacena en caché los datos de posiciones y los actualiza periódicamente. Los cambios de posición muy recientes (dentro de segundos) pueden no aparecer inmediatamente en las respuestas.

## Patrones comunes

### Obtener todas las posiciones de stake de una billetera

```bash
curl -s "https://owner-v1.raydium.io/position/stake/{owner}"
```

Reemplaza `{owner}` con la clave pública de la billetera en formato base58.

### Obtener posiciones bloqueadas

```bash
curl -s "https://owner-v1.raydium.io/position/clmm-lock/{owner}"
```

### Verificar participación en IDO

```bash
curl -s "https://owner-v1.raydium.io/main/ido/{owner}"
```

## Resultados vacíos

Si una billetera no tiene datos para un punto final particular, el servicio devuelve un error 404 con metadatos de solicitud:

```json
{
  "id": "...",
  "success": false
}
```
