La API de autenticación gestiona la autenticación basada en billetera para los servicios de LaunchLab. Intercambia un mensaje firmado por un token JWT y, a continuación, utiliza ese token para autenticar solicitudes a las APIs de Foro y creación de acuñación.

## Puntos finales de API

**Mainnet:** https://launch-auth-v1.raydium.io  
**Devnet:** https://launch-auth-v1-devnet.raydium.io

## Flujo de autenticación

No se requieren credenciales preexistentes. La autenticación se basa en billetera:

1. **Obtén la marca de tiempo actual** (segundos Unix)
2. **Firma el mensaje** con tu billetera: `"Sign in to raydium.io: " + <timestamp>`
3. **Envía la firma** al punto final `/request-token`
4. **Recibe un token JWT** válido por la duración especificada
5. **Usa el token** en el encabezado `ray-token` para los puntos finales protegidos

## Formato del mensaje

El mensaje que firmes debe seguir este formato exacto:

```
Sign in to raydium.io: <UNIX_TIMESTAMP_SECONDS>
```

**Ejemplo:**
```
Sign in to raydium.io: 1704067200
```

- Utiliza tu marca de tiempo Unix actual en segundos (no milisegundos)
- Firma Ed25519 (firma nativa de Solana)
- Codifica la firma como una cadena base58
- Pasa la marca de tiempo y la firma a `/request-token`

## Validación del token

Utiliza `/check-token` para verificar que tu token sigue siendo válido. El punto final también extiende automáticamente la duración del token si está próximo a expirar.

La duración del token se puede configurar en el servidor; el valor predeterminado suele ser de 24 horas.

## Revocación del token

Revoca un token con el punto final `/del-token`.

## Métodos de firma admitidos

- **Billetera estándar:** firma Ed25519 con `/request-token`
- **Billetera Ledger:** transacción firmada con `/request-token-ledger`

## Casos de uso

- **Publicación en foro** — auténticate antes de publicar comentarios
- **Creación de acuñación** — firma la propiedad de la billetera antes de enviar nuevos tokens
- **Sesiones de usuario** — mantén el estado autenticado en las llamadas de API
- **Soporte multibilletera** — emite tokens separados por dirección de billetera
