La Temp API es un área de alojamiento temporal para endpoints que aún no tienen un lugar permanente en las APIs principales de Raydium. Este servicio está pensado para la creación rápida de prototipos, herramientas internas y funcionalidades en desarrollo activo.

## Hosts

| Entorno  | Host                              |
|----------|----------------------------------|
| Mainnet  | `temp-api-v1.raydium.io`         |
| Devnet   | `temp-api-v1-devnet.raydium.io`  |

## Aviso de estabilidad

**Este servicio no ofrece garantías de estabilidad de API.** Los endpoints pueden cambiar, quedar obsoletos, trasladarse a otro servicio o eliminarse por completo sin previo aviso. Usa esta API únicamente para:

- Herramientas y scripts internos
- Funcionalidades en desarrollo activo
- Rutas no críticas que puedan tolerar interrupciones
- Pruebas y prototipos

No dependas de esta API para dApps en producción ni para integraciones críticas.

## Resumen

Este servicio expone actualmente:

- **`/cp-creator-fee`** — Recupera las comisiones de creador acumuladas para una wallet en todos los pools CPMM.
- **`/limit-order/order/list`** — Las órdenes límite CLMM actualmente en espera de una wallet (abiertas y parcialmente ejecutadas), servidas desde la caché del indexador.
- **`/limit-order/history/order/list-by-user`** — Las órdenes límite CLMM históricas (cerradas) de una wallet, con filtros opcionales `poolId` / `mint1` / `mint2` / `hideCancel`; paginación por cursor mediante `nextPageId`.
- **`/limit-order/history/event/list-by-pda`** — Registro de eventos por PDA (`open` / `increase` / `decrease` / `settle` / `close`) para una o más PDAs de órdenes límite; paginación por cursor mediante `nextPageId`.

## Patrones comunes

### Consultar las comisiones de creador de una wallet

```bash
curl -s "https://temp-api-v1.raydium.io/cp-creator-fee?wallet=<your-wallet-pubkey>"
```

Devuelve una lista de pools en los que la wallet es la creadora, junto con las comisiones de creador acumuladas y pendientes de reclamar en Token A y Token B.

### Listar las órdenes límite CLMM en espera de una wallet

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/order/list?wallet=<your-wallet-pubkey>"
```

Devuelve tanto las órdenes totalmente sin ejecutar como las ejecutadas parcialmente. Cada fila es un `LimitOrderState` decodificado (PDA, pool, propietario, tick, lado, fase de cohorte FIFO, cantidades totales/ejecutadas) más el contexto del pool necesario para llamar a `SettleLimitOrder`, incluido un importe `pendingSettle` precalculado.

El endpoint de órdenes activas no está paginado: devuelve todas las órdenes en caché de la wallet en una sola respuesta.

### Listar las órdenes límite cerradas de una wallet

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/order/list-by-user?wallet=<your-wallet-pubkey>&size=50"
```

Devuelve una página de órdenes históricas, cada una resumida con `fillStatus` (`NONE` / `PARTIAL` / `FULL`) y los importes de entrada/salida en el momento en que la orden salió del conjunto activo. Añade `&nextPageId=<nextPageId de la respuesta anterior>` para obtener la página siguiente.

### Consultar la línea de tiempo de eventos de una orden específica

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/event/list-by-pda?pda=<limit-order-pda>"
```

`pda` puede ser una clave única o una lista separada por comas. Cada fila representa una mutación on-chain (`open` / `increase` / `decrease` / `settle` / `close`) con el txid, slot, tiempo de bloque, indicador de firmado por el propietario o el keeper (`autoRunner`), y los deltas de entrada/salida en unidades legibles.

### Resultado vacío

Si la wallet no tiene registros en caché, el endpoint de órdenes activas devuelve:

```json
{
  "id": "...",
  "success": true,
  "data": { "rows": [] }
}
```

Los endpoints de historial devuelven la misma estructura `{ rows: [], nextPageId?: ... }` con un array `rows` vacío. El endpoint heredado `/cp-creator-fee` devuelve `"data": []` por compatibilidad con versiones anteriores.
