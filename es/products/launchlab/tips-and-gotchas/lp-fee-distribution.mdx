Cuando un token de LaunchLab migra a CPMM, el pool migrado acuña tokens LP. La configuración de plataforma controla qué sucede con esos tokens LP.

## División de LP

`migrateCpLockNftScale` tiene tres campos. Los valores deben sumar `1_000_000`, que representa el 100%.

| Campo | Significado |
|---|---|
| `platformScale` | Los tokens LP se bloquean a través de Burn & Earn. La plataforma recibe el NFT de Fee Key. |
| `creatorScale` | Los tokens LP se bloquean a través de Burn & Earn. El creador del token recibe el NFT de Fee Key. |
| `burnScale` | Los tokens LP se queman permanentemente. No se crea ningún NFT de Fee Key para esta parte. |

Esta es una configuración a nivel de plataforma. Los tokens lanzados a través de la misma plataforma heredan la misma distribución de LP hasta que la plataforma actualice la configuración.

## Ejemplos

```ts
// 40% plataforma, 50% creador, 10% quemado.
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% Fee Key de creador.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% quemado.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## Relación con las comisiones de CPMM

`migrateCpLockNftScale` controla **quién puede reclamar derechos de comisiones de LP**. No establece la tasa de comisión de swap. El comportamiento de comisión de swap del pool migrado proviene del `AmmConfig` de CPMM seleccionado por `cpConfigId`.

Por ejemplo, si un pool CPMM migrado usa una configuración de comisión comercial del 0.25% y `creatorScale` es 500,000, el Fee Key del creador representa el 50% de los derechos de comisión de LP bloqueados. Si `platformScale` es 400,000, el Fee Key de la plataforma representa el 40%.

## Actualizar la división

Las actualizaciones de configuración de plataforma están limitadas por época. Trata la distribución de LP como una política de plataforma importante, no como un parámetro casual de interfaz.

Antes de actualizar:

- Confirma que los tres valores de escala sumen `1_000_000`.
- Confirma que la plataforma entiende quién recibirá NFTs de Fee Key después de futuras migraciones.
- Documenta el cambio para los creadores antes de que nuevos lanzamientos lo utilicen.
