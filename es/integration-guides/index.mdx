## Directrices de redacción

- Cada guía comienza con "este es el problema" (por ejemplo, "mis transacciones de swap fallan bajo carga") y termina con una receta validada.
- Incluye referencias de rendimiento donde sean significativas (por ejemplo, uso típico de CU para un swap de CPMM frente a un swap de CLMM).
- Enlaza con SDK y API para las primitivas subyacentes; no las redefinas.
