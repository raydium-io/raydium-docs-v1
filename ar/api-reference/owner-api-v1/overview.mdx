يوفر Owner API طرق عرض منفصلة لكل محفظة تتضمن مراكز وأرصدة Raydium. يقوم بجلب البيانات المخزنة مؤقتًا للمراكز والسيولة المُقفلة والمجمعات المُنشأة والمشاركة في IDO، مما يجعله مثاليًا لواجهات المحفظ ولوحات التحكم للمحافظ.

## الخوادم

| البيئة | الخادم |
|-------------|---------------------------|
| Mainnet     | `owner-v1.raydium.io`     |
| Devnet      | `owner-v1-devnet.raydium.io` |

## نظرة عامة

تُعرّض الـ API خمس نقاط نهاية رئيسية منظمة حسب نوع المورد:

- **`/position/stake/{owner}`** — مراكز الرهن (AMM / CPMM LP الأساسي)
- **`/position/clmm-lock/{owner}`** — مراكز CLMM المُقفلة
- **`/create-pool/{owner}`** — المجمعات المُنشأة بواسطة هذه المحفظة
- **`/main/ido/{owner}`** — بيانات المشاركة في IDO

## حداثة البيانات

تقوم هذه الخدمة بتخزين بيانات المراكز مؤقتًا وتحديثها بشكل دوري. قد لا تظهر التغييرات الحديثة جدًا للمراكز (خلال ثوانٍ) على الفور في الاستجابات.

## الأنماط الشائعة

### جلب جميع مراكز الرهن للمحفظة

```bash
curl -s "https://owner-v1.raydium.io/position/stake/{owner}"
```

استبدل `{owner}` بالمفتاح العام للمحفظة بصيغة base58.

### جلب المراكز المُقفلة

```bash
curl -s "https://owner-v1.raydium.io/position/clmm-lock/{owner}"
```

### التحقق من المشاركة في IDO

```bash
curl -s "https://owner-v1.raydium.io/main/ido/{owner}"
```

## النتائج الفارغة

إذا لم تكن لدى المحفظة بيانات لنقطة نهاية معينة، تعيد الخدمة خطأ 404 مع بيانات وصفية للطلب:

```json
{
  "id": "...",
  "success": false
}
```

## المستندات ذات الصلة

راجع [CLMM](/ar/products/clmm) و [CPMM](/ar/products/cpmm) للاطلاع على هياكل حساب المراكز والمصطلحات المستخدمة.
