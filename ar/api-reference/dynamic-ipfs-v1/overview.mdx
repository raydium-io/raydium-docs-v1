توفر Dynamic IPFS API بيانات وصور معاد توليدها لمراكز السيولة CLMM و CPMM على Solana. تم تصميم هذه الخدمة لتطبيقات Web3 التي تعرض NFTs ديناميكية—مثل بطاقات مراكز السيولة المركزة—من خلال توفير معلومات البركة في الوقت الفعلي وتقييم المركز والرسوم المتراكمة غير المطالب بها دون الحاجة إلى عمليات بحث على السلسلة.

## الخوادم

| البيئة | الخادم |
|--------|--------|
| Mainnet     | `dynamic-ipfs.raydium.io`         |
| Devnet      | `dynamic-ipfs-devnet.raydium.io`  |

## نظرة عامة

تعرض API ثلاث نقاط نهاية، كل منها تُرجع كائن واجهة موضع كامل:

- **`/clmm/position`** — جلب البيانات الوصفية لمركز CLMM نشط
- **`/lock/clmm/position`** — جلب البيانات الوصفية لمركز CLMM مقفول
- **`/lock/cpmm/position`** — جلب البيانات الوصفية لمركز CPMM مقفول

تتضمن كل استجابة تفاصيل البركة الكاملة (الرموز والعناصر العشرية والقيمة الإجمالية المقفلة)، ومبالغ المركز، والتقييم بالدولار الأمريكي، ونسبة القيمة الإجمالية المقفلة داخل البركة، والرسوم والمكافآت المتراكمة (لكن غير المطالب بها).

## حالات الاستخدام

توليد NFTs عرض موضع ديناميكي بطلب البيانات الوصفية لموضع معين. تحسب API حالة الموضع في الوقت الفعلي من البيانات على السلسلة وتخزن النتائج مؤقتًا، مما يجعلها مثالية لـ:

- عرض بطاقات الموضع مع القيمة بالدولار الأمريكي المباشرة وتراكم الرسوم
- عرض نسبة القيمة الإجمالية المقفلة ومبالغ الرموز في البركة
- إظهار رسوم التداول غير المطالب بها والرموز المكافآت
- بناء لوحات معلومات المحفظة التي تتابع أداء الموضع

## المصادقة

لا تُطلب مصادقة. جميع النقاط النهاية عامة.

## حد المعدل

لا يتم تطبيق أي حد. خدمة بأفضل جهد.

## الأنماط الشائعة

### جلب موضع CLMM

```bash
curl -s "https://dynamic-ipfs.raydium.io/clmm/position?id=4Ygp92zRvl4kCr97V3Zs5xg4k3qX1tN6m8pR2vL9jA1"
```

يجب أن تكون معاملة `id` هي المفتاح العام للموضع على السلسلة (عنوان Solana بصيغة base58).

### جلب موضع مقفول

```bash
curl -s "https://dynamic-ipfs.raydium.io/lock/clmm/position?id=<position_pda>"
```

## الوثائق ذات الصلة

اطّلع على [CLMM](/ar/products/clmm) لمعرفة بنية حساب الموضع وكيفية اشتقاق معرفات الموضع.
