## كتالوج الخدمات

تدير Raydium أحدى عشرة خدمات HTTP عامة. يتم توثيق كل واحدة منها كمجموعة خاصة بها في شجرة التنقل على اليسار.

| الخدمة | مضيف Mainnet | مضيف Devnet | ما الذي تفعله |
|---------|--------------|-------------|--------------|
| [API v3](/ar/api-reference/api-v3/overview) | `api-v3.raydium.io` | `api-v3-devnet.raydium.io` | المجموعات / النعت / الإعدادات / معلومات السلسلة — API القراءة الكنسي الذي يعتمد عليه الواجهة والـ SDK. |
| [Transaction API](/ar/api-reference/route-api-v2/overview) | `transaction-v1.raydium.io` | `transaction-v1-devnet.raydium.io` | إنشاء معاملات المبادلة والسيولة من جانب الخادم. نقطة دخول المجمع. |
| [Perps API](/ar/api-reference/perp-api-v1/overview) | `api-perp-v1.raydium.io` | — | الإعدادات، البيانات الوصفية للأصول، اختيار RPC لواجهة Raydium Perps الأمامية. |
| [LaunchLab Mint API](/ar/api-reference/launch-mint-v1/overview) | `launch-mint-v1.raydium.io` | `launch-mint-v1-devnet.raydium.io` | بحث الرموز، الفهارس، لوحات المتصدرين، البيانات الوصفية لكل نعت لإطلاقات LaunchLab. |
| [LaunchLab History API](/ar/api-reference/launch-history-v1/overview) | `launch-history-v1.raydium.io` | `launch-history-v1-devnet.raydium.io` | سجل التداول، تجميعات K-line لمجموعات LaunchLab. |
| [LaunchLab Forum API](/ar/api-reference/launch-forum-v1/overview) | `launch-forum-v1.raydium.io` | `launch-forum-v1-devnet.raydium.io` | التعليقات، المواضيع، تحميلات IPFS المرتبطة بإطلاق LaunchLab. موقعة بالمحفظة. |
| [LaunchLab Auth API](/ar/api-reference/launch-auth-v1/overview) | `launch-auth-v1.raydium.io` | `launch-auth-v1-devnet.raydium.io` | يصدر JWTs `ray-token` قصيرة الأجل من رسالة موقعة بالمحفظة. مطلوبة لاستدعاء forum API. |

| [API v1](/ar/api-reference/api-v1/overview) | `api.raydium.io` | — | نقاط نهائية v1/v2 الموروثة، تبقى نشطة للعملاء الذين لم ينتقلوا إلى API v3. |
| [Temp API](/ar/api-reference/temp-api-v1/overview) | `temp-api-v1.raydium.io` | `temp-api-v1-devnet.raydium.io` | قلم انتظار لنقاط نهائية مخصصة قصيرة الأجل. حاليًا يكشف ملخصات رسوم منشئ CPMM. قد يتغير بدون إشعار. |

## المصادقة

معظم الخدمات للقراءة فقط وتقبل الطلبات المجهولة. يظهر نمطان:

- **مصافحة موقعة بالمحفظة** — مطلوبة بواسطة `launch-auth-v1` لسك `ray-token`، ثم يتم نقلها كرأس `ray-token` بواسطة `launch-forum-v1`. وقّع رسالة Solana ed25519 بالشكل `time:<unix-seconds>` بمحفظتك، أرسل التوقيع وعنوان المحفظة إلى `/request-token` في `launch-auth-v1`، استقبل JWT وأعده كرأس طلب `ray-token` في استدعاءات منتدى لاحقة.
- **لا توجد مصادقة** — كل خدمة أخرى. يضرب المتصلون الموثوقون (واجهة Raydium، المدمجون) النقاط النهائية مباشرة عبر HTTPS.

تتيح لك ساحة لعب Mintlify لصق `ray-token` في لوحة المصادقة قبل إرسال طلبات منتدى؛ يتم الاحتفاظ بالقيمة في متصفحك فقط.

## حدود المعدل

جميع المضيفين خلف Cloudflare مع تحديد معدل تدريجي لكل IP مصدر.

يجب على المدمجين الذين يحتاجون إلى حدود أعلى الاتصال بفريق Raydium. يتم إرجاع الانفجارات التي تتجاوز الحدود المنشورة باعتبارها `HTTP 429` برأس `Retry-After`.

## غلاف الاستجابة

تقوم معظم الخدمات بتغليف حمولات JSON الخاصة بها في غلاف موحد:

```json
{
  "id": "uuid-v4-per-request",
  "success": true,
  "data": { ... }
}
```

عند الفشل:

```json
{
  "id": "uuid-v4-per-request",
  "success": false,
  "msg": "human-readable error string",
  "data": null
}
```

`api-v3` تعيد الغلاف باسم `{ "id", "success", "data" }`. تستخدم خدمات pump-* مساعدات `addBorderSuccess` / `addBorderErr` التي تنتج نفس الشكل مع ملء `msg` فقط عند الأخطاء. تحقق من صفحة المقدمة لكل خدمة للحصول على الغلاف الدقيق الذي تستخدمه.

## كيفية استخدام هذا القسم

انقر فوق أي نقطة نهائية في التنقل الأيسر. سترى:

- طريقة HTTP والعنوان الكامل والمضيف الأساسي.
- معاملات الطلب مع الأنواع المسحوبة من مواصفات OpenAPI.
- لوحة **جرّب الآن** التي تتيح لك تعيين المعاملات وإرسال طلب حقيقي إلى mainnet (أو اختيار خادم devnet من القائمة المنسدلة عند توفره).
- شكل الاستجابة، المستمد من استجابة مباشرة تمثيلية وموقع الاستدعاء `addBorderSuccess` في الكود المصدري.

للوصول على مستوى SDK — بناء المعاملات من جانب العميل، معاملات الدفعات، أنواع TypeScript — انظر [`sdk-api/typescript-sdk`](/ar/sdk-api/typescript-sdk).
