## ما هو الحال

**Raydium Perpetual** هو بروتوكول عقود دائمة بدون رسوم غاز مع دفتر أوامر مركزي (CLOB). يتم تمويل المراكز من محفظة SPL على Solana لكن محرك المطابقة ودفتر الأوامر يتم تشغيلهما بواسطة [Orderly Network](https://orderly.network/)، والذي يجمع السيولة عبر مجموعة متعددة السلاسل (Solana و Ethereum و Base و Arbitrum و BSC).

الخصائص الرئيسية:

- **تداول بدون رسوم غاز.** إنشاء الأوامر والتعديل عليها والإلغاء لا تتطلب معاملات على السلسلة على Solana — التوقيع محلي ومجاني. فقط الإيداعات والسحب تلمس السلسلة.
- **رافعة مالية تصل إلى 100×**، حسب السوق.
- **ضمانات متعددة السلاسل** — يمكنك إيداع USDC / SOL / USDT / ETH / BNB / WBTC / YUSD / USD1 من سلسلتهم الأصلية؛ جميع الأرباح والخسائر تسويتها في USDC.
- **هامش متقاطع فقط.** جميع الضمانات في الحساب مشتركة عبر كل مركز مفتوح.
- **وضع أحادي الاتجاه.** لا يمكنك الاحتفاظ بمركز طويل وقصير على نفس السوق في نفس الوقت؛ أمر الجانب المعاكس يقلل أو يقلب المركز الموجود.
- **حفظ النفس.** تبقى الأموال في حساب perps الخاص بمحفظتك؛ لا توجد محفظة حفظ مركزية يحتفظ بها Raydium.
- **أنواع أوامر بدرجة سوق الصرافة.** السوق والحد والإيقاف (سوق / حد)، والحجم، بالإضافة إلى أعلام IOC / FOK / Post-Only / Reduce-Only.

## لماذا منتج منفصل

يشترك AMMs Raydium (فوري) و Raydium Perps (عقود دائمة) في علامة تجارية ونقطة دخول واجهة المستخدم ولكن لا يشتركان في أي شيء تقريباً على مستوى البروتوكول:

| | AMMs فورية في Raydium | Raydium Perps |
|---|---|---|
| الموقع | حمامات على السلسلة على Solana | CLOB خارج السلسلة يتم تشغيله بواسطة Orderly |
| محرك المطابقة | منحنى AMM (CPMM / CLMM / AMM v4) | دفتر أوامر مركزي |
| التسوية | على السلسلة وقت المبادلة | خارج السلسلة؛ الإيداعات والسحب على السلسلة |
| اكتشاف السعر | منحنى AMM + التحكيم | دفتر الأوامر + الأسعار الخارجية |
| عالم الرموز | أي رمز SPL / Token-2022 | أسواق perps المدرجة فقط (BTC و ETH و SOL والعملات الرئيسية الأخرى) |
| معرفات البرنامج في [`reference/program-addresses`](/ar/reference/program-addresses) | نعم | لا — انظر عقود Orderly |
| تغطية Raydium SDK v2 | كاملة | غير مغطاة |
| مكافأة الأخطاء في Raydium | نعم | موجهة عبر Orderly |

يركز بقية موقع التوثيق هذا بشكل شبه حصري على AMMs الفورية والبرامج في منظمة GitHub لـ Raydium. يتم توثيق Perps على مستوى سطح المنتج — أنواع الأوامر والرسوم والضمانات والقيود — حتى يفهم القراء ما هو عليه ومتى يتم استخدامه، لكن مرجع البروتوكول العميق يقع على جانب Orderly.

## الولايات القضائية المقيدة

Raydium Perps **غير متاح** لسكان الولايات المتحدة وأفغانستان وبيلاروس والجمهورية الأفريقية الوسطى والكونغو الديمقراطية وكوريا الشمالية (DPRK) ومناطق Crimea و Donetsk و Luhansk في أوكرانيا وكوبا وإيران وليبيا والصومال والسودان وجنوب السودان وسوريا واليمن وزيمبابوي أو أي ولاية قضائية أخرى يُحظر فيها بروتوكول Raydium.

يجب على المدمجين الذين يدمجون واجهة المستخدم أو API للعقود الدائمة احترام هذه القائمة.

## ملاحظات الدمج

- Raydium SDK الفورية (`@raydium-io/raydium-sdk-v2`) **لا تغطي** العقود الدائمة. للوصول البرمجي استخدم [Orderly's SDKs](https://orderly.network/docs/build-on-omnichain/evm-api/introduction) (TypeScript و REST و WebSocket).
- لا توجد بدائية "توجيه مبادلة عبر مركز perp". العميل الذي يريد كل من التعرض الفوري والدائم يبني المعاملتين بشكل مستقل.
- معاملات مخاطر Orderly (حدود الرافعة المالية وعتبات التصفية وحدود الضمانات) تتغير بمرور الوقت. قم دائماً بجلب المعاملات المباشرة من API Orderly قبل تقديم المخاطر للمستخدم.
