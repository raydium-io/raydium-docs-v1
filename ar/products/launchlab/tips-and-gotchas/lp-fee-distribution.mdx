عندما يهاجر رمز LaunchLab إلى CPMM، يقوم مجمع الهجرة بسك رموز LP. يتحكم تكوين المنصة في ما يحدث لتلك الرموز.

## تقسيم LP

حقل `migrateCpLockNftScale` يحتوي على ثلاثة حقول. يجب أن تجمع القيم إلى `1_000_000`، وهو يمثل 100%.

| الحقل | المعنى |
|---|---|
| `platformScale` | يتم قفل رموز LP من خلال Burn & Earn. تستقبل المنصة NFT مفتاح الرسوم. |
| `creatorScale` | يتم قفل رموز LP من خلال Burn & Earn. يستقبل منشئ الرمز NFT مفتاح الرسوم. |
| `burnScale` | يتم حرق رموز LP بشكل دائم. لا يتم إنشاء NFT مفتاح رسوم لهذا الجزء. |

هذا إعداد على مستوى المنصة. الرموز المطروحة من خلال نفس المنصة ترث نفس توزيع LP حتى تقوم المنصة بتحديث الإعداد.

## أمثلة

```ts
// 40% منصة، 50% منشئ، 10% محروق.
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% مفتاح رسوم المنشئ.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% محروق.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## العلاقة برسوم CPMM

`migrateCpLockNftScale` يتحكم في **من يمكنه المطالبة بحقوق رسوم LP**. لا يحدد معدل رسوم المبادلة. يأتي سلوك رسوم المبادلة في مجمع CPMM المهاجر من `AmmConfig` المختار بواسطة `cpConfigId`.

على سبيل المثال، إذا استخدم مجمع CPMM المهاجر إعداد رسوم تجارية بنسبة 0.25% وكان `creatorScale` بقيمة 500,000، فإن مفتاح الرسوم الخاص بالمنشئ يمثل 50% من حقوق رسوم LP المقفولة. إذا كان `platformScale` بقيمة 400,000، فإن مفتاح الرسوم الخاص بالمنصة يمثل 40%.

## تحديث التقسيم

تحديثات تكوين المنصة محدودة بمعدل حسب الحقبة. تعامل مع توزيع LP كسياسة منصة مهمة، وليس إعدادًا عابرًا للواجهة.

قبل التحديث:

- تأكد من أن قيم المقياس الثلاث تجمع إلى `1_000_000`.
- تأكد من أن المنصة تفهم من سيستقبل NFTs مفتاح الرسوم بعد الهجرات المستقبلية.
- وثّق التغيير للمنشئين قبل أن تستخدمه الإطلاقات الجديدة.
