## رسوم المبادلة حسب المنتج

قيم الرسوم هي نسب من المبلغ المدخل، مطبقة قبل حسابات Pool.

| المنتج | المستويات المتاحة | الافتراضي / الأكثر استخداماً | التقسيم |
|---------|-----------------|---------------------|-------|
| **AMM v4** | 0.25% (فقط) | 0.25% | 88% LP، 12% بروتوكول (22/3 من 25) |
| **CPMM** | 0.01%، 0.25%، 1% | 0.25% | 100% LP (حصة البروتوكول قابلة للتكوين حسب AmmConfig، حالياً 0%) |
| **CLMM** | 0.01%، 0.05%، 0.25%، 1% | 0.25% | 88% LP، 12% بروتوكول |
| **LaunchLab (قبل التخرج)** | 1% (افتراضي) | 1% | 50% لـ pool seed، 50% لـ LaunchLab treasury |
| **LaunchLab (بعد التخرج CPMM)** | 0.25% | 0.25% | يرث تقسيم CPMM |

### اتفاقية مقسوم الرسوم

- **AMM v4**: يتم ترميز الرسوم كـ x/10,000. "25" تعني 0.25%.
- **CPMM/CLMM**: يتم ترميز الرسوم كـ x/1,000,000. "2500" تعني 0.25%.

عند قراءة البيانات على السلسلة مباشرة، استخدم المقسوم الصحيح لـ program الـ pool.

## تقسيمات رسوم البروتوكول

ضمن حصة "البروتوكول" من كل رسم، تنطبق تقسيمات إضافية:

| المنتج | حصة البروتوكول من رسم المبادلة | التقسيم ضمن البروتوكول |
|---------|----------------------------|------------------------|
| AMM v4 | 12% (3/25) | 100% للخزينة |
| CPMM | قابل للتكوين حسب AmmConfig (حالياً 0%) | N/A |
| CLMM | 12% من رسم المبادلة | 100% للخزينة |
| LaunchLab | 50% من رسم المبادلة قبل التخرج | 100% لـ LaunchLab treasury |

تتدفق **حصة LP** إلى حاملي رموز LP تلقائياً:

- AMM v4 / CPMM: متراكمة في احتياطيات pool، موزعة نسبياً عند السحب.
- CLMM: متراكمة في `fee_growth_global_X/Y`، قابلة للمطالبة عبر `collectFee`.

## تكاليف الإنشاء

تكاليف لمرة واحدة مدفوعة بـ SOL لنشر pool.

| المنتج | تكلفة Rent | رسم الإنشاء | الإجمالي النموذجي |
|---------|-----------|--------------|---------------|
| AMM v4 | ~0.15 SOL (حسابات متعددة) | 0.2 SOL | ~0.35 SOL |
| CPMM | ~0.04 SOL (pool + vaults + observation) | 0.15 SOL | ~0.19 SOL |
| CLMM | ~0.075 SOL (pool + observation + initial tick arrays) | 0 SOL | ~0.075 SOL |
| Farm v6 | ~0.02 SOL (farm state + reward vaults) | 0.1 SOL (مكافحة البريد العشوائي) | ~0.12 SOL |
| LaunchLab launch | ~0.015 SOL (bonding curve account) | 0.1 SOL (رسم LaunchLab) | ~0.115 SOL |

كل Rent قابل للاسترداد — إغلاق الـ pool / position / farm يعيد lamports.

## تفاعل رسوم Token-2022

عندما تستخدم أحد جانبي الـ pool أو كلاهما Token-2022 برسم تحويل، يتم ترتيب الرسم الفعلي:

```
net input to pool   = amount_in × (1 − transfer_fee_in)
pool charges swap   = net × swap_fee_rate
net output to user  = out × (1 − transfer_fee_out)
```

مثال: pool CPMM بنسبة 0.25%، الجانب المدخل يحتوي على رسم تحويل 1%، الجانب المخرج يحتوي على رسم تحويل 0.5%.

```
amount_in        = 1000
net_to_pool      = 1000 × 0.99    = 990
pool swap fee    = 990 × 0.0025   = 2.475 (kept as LP fee)
amount_after_fee = 987.525
amount_out (AMM) = 987.525 × ratio, say 987.525 × 0.98 = 967.77
net_out_to_user  = 967.77 × 0.995 = 962.94
```

الرسم الفعلي: **~3.7%** رغم أن رسم pool الاسمي 0.25%.

يحسب SDK المعدل الفعلي عبر `getComputeAmountOut` — استخدمه دائماً عند الحصول على عرض أسعار لـ pools Token-2022.

## تفاصيل LaunchLab

| المرحلة | الرسم |
|-------|-----|
| تداولات bonding curve (شراء أو بيع) | 1% (مقسمة 50/50 pool/treasury) |
| التخرج (لمرة واحدة) | ~0.04 SOL من احتياطيات curve |
| بعد التخرج (CPMM pool) | 0.25% (رسوم CPMM القياسية) |

المشاريع التي تطلق رموز غالباً ما تتضمن خيار "رسم المبتكر" (0–0.3%) مأخوذ من حصة LP أثناء مرحلة bonding-curve؛ قابل للتكوين عند الإطلاق. رسم المبتكر **bonding-curve فقط** — بمجرد تخرج الإطلاق إلى pool CPMM، يعمل الـ pool الناتج بـ tier CPMM القياسي بدون رسم إضافي للمبتكر. انظر إلى [`products/launchlab/creator-fees`](/ar/products/launchlab/creator-fees) للدورة الحياة الكاملة وكيف يستمر المبتكرون في الكسب من pools المتخرجة عبر آلية LaunchLab NFT.

## نموذج تكلفة مكافآت Farm v6

لا تفرض المزارع رسوماً بالمعنى الحقيقي — فهي توزيع مكافآت. لكنها تكلف SOL للتشغيل:

| التكلفة | النموذجية |
|------|---------|
| إنشاء مزرعة | 0.1 SOL مكافحة البريد العشوائي + ~0.02 SOL rent |
| ملء vault المكافآت | فقط الرموز + ~0.0005 SOL tx |
| استرجاع المكافآت غير المستخدمة | ~0.0005 SOL tx |

تتطلب المزارع أيضاً عدداً كافياً من lamports في vault المكافآت لتغطية rent لمدى حياة المزرعة؛ الإغلاق يعيدها.

## كيفية قراءة القيم الحالية على السلسلة

### CPMM/CLMM AmmConfig

```ts
const config = await raydium.cpmm.getAmmConfigs();
console.log(config[0]);
// {
//   id: "...",
//   tradeFeeRate: 2500,         // 2500 / 1_000_000 = 0.25%
//   protocolFeeRate: 0,         // 0 / 1_000_000 = 0%
//   fundFeeRate: 0,
//   ...
// }
```

### AMM v4 pool

```ts
const poolInfo = await raydium.liquidity.getPoolInfoFromRpc({ poolId });
console.log(poolInfo.ammFeesNumerator, poolInfo.ammFeesDenominator);
// 25, 10000 → 0.25%
```

### معدل مكافآت Farm v6

```ts
const farm = await raydium.farm.getFarmInfoFromRpc({ id: farmId });
console.log(farm.farmData.rewardInfos);
// Per-reward: emissionsPerSecond, startTime, endTime
```
