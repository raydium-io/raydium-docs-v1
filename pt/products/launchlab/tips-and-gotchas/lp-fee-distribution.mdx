Quando um token LaunchLab migra para CPMM, o pool migrado cria tokens LP. A configuração da plataforma controla o que acontece com esses tokens LP.

## A divisão de LP

`migrateCpLockNftScale` tem três campos. Os valores devem somar `1_000_000`, que representa 100%.

| Campo | Significado |
|---|---|
| `platformScale` | Tokens LP são bloqueados através de Burn & Earn. A plataforma recebe o NFT Fee Key. |
| `creatorScale` | Tokens LP são bloqueados através de Burn & Earn. O criador do token recebe o NFT Fee Key. |
| `burnScale` | Tokens LP são queimados permanentemente. Nenhum NFT Fee Key é criado para essa parcela. |

Esta é uma configuração em nível de plataforma. Tokens lançados através da mesma plataforma herdam a mesma distribuição de LP até que a plataforma atualize a configuração.

## Exemplos

```ts
// 40% plataforma, 50% criador, 10% queimado.
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% Fee Key do criador.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% queimado.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## Relação com taxas de CPMM

`migrateCpLockNftScale` controla **quem pode reivindicar direitos de taxa LP**. Ele não define a taxa de swap. O comportamento de taxa de swap do pool migrado vem da `AmmConfig` de CPMM selecionada por `cpConfigId`.

Por exemplo, se um pool CPMM migrado usa uma configuração de taxa de negociação de 0,25% e `creatorScale` é 500.000, o Fee Key do criador representa 50% dos direitos de taxa LP bloqueados. Se `platformScale` é 400.000, o Fee Key da plataforma representa 40%.

## Atualizando a divisão

Atualizações de configuração de plataforma são limitadas por epoch. Trate a distribuição de LP como uma política de plataforma importante, não como uma configuração casual de UI.

Antes de atualizar:

- Confirme que os três valores de scale somam `1_000_000`.
- Confirme que a plataforma entende quem receberá NFTs Fee Key após futuras migrações.
- Documente a alteração para criadores antes que novos lançamentos a utilizem.
