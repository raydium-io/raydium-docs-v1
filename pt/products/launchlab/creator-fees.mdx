## Pré-migração: acúmulo direto de taxas

Enquanto um token ainda está na curva de ligação, cada compra e cada venda paga um componente de taxa de criador que se acumula em um cofre por token. O **criador** (a carteira que lançou o token) pode reivindicar do cofre a qualquer momento.

Características:

- Denominada no **token de cotação** (geralmente SOL).
- Configurada pela **plataforma** (a configuração da plataforma controla se as taxas de criador estão ativadas e em quantos bps).
- Reivindicável a qualquer momento pela página Portfolio na UI do LaunchLab; programaticamente através da instrução `claim_creator_fees` do LaunchLab (nome exato no IDL).
- Continua até a graduação; na graduação, taxas residuais pré-migração são reivindicáveis, mas nenhuma nova taxa pré-migração será acumulada.

## Pós-migração: Fee Key NFT via Burn & Earn

Quando a curva de ligação de um token atinge seu alvo e a liquidez é migrada para um pool CPMM (ou AMM v4), tokens LP são cunhados e **divididos de acordo com a configuração da plataforma**. A divisão sempre soma 100% e sempre envolve alguma combinação de:

- **Queimado** — transferido para [Burn & Earn](/pt/user-flows/burn-and-earn) para bloqueio permanente. O Fee Key para esta porção é mantido por uma conta controlada pela plataforma ou designada para queima.
- **Criador** — um Fee Key NFT cunhado diretamente para a carteira do criador do token. Este Fee Key representa o direito de coletar taxas de trading LP da participação LP do criador.
- **Plataforma** — opcional; uma participação que flui para o tesouro da plataforma como outro Fee Key NFT.

### Padrões próprios do LaunchLab do Raydium

| Participação | Padrão em raydium.io/launchlab |
|---|---|
| Queimado (Burn & Earn, bloqueado permanentemente) | 90% |
| Criador (Fee Key NFT) | 10% |
| Plataforma | 0% (O LaunchLab do Raydium não leva uma participação de LP da plataforma) |

Nestes padrões, **apenas o criador reclama taxas de trading de sua participação**; os outros 90% são bloqueados e suas taxas vão para Burn & Earn (fluindo de volta para a mecânica de sinalização de confiança em vez de qualquer bolso individual).

### Outras plataformas

Plataformas de terceiros que constroem no LaunchLab podem configurar sua própria divisão. Uma plataforma que queira se financiar através de taxas LP pós-migração pode definir (digamos) 70 / 10 / 20 (burn / criador / plataforma). Veja [`products/launchlab/platforms`](/pt/products/launchlab/platforms) para os campos Platform PDA que impulsionam isso.

Independentemente da divisão, o **mecanismo é o mesmo**: cada participação não queimada é representada como um Fee Key NFT, que pode ser negociado, transferido ou coletado.

### O que acontece se as taxas de criador pós-migração forem desativadas

Se uma plataforma define `creator_fee_enabled = false`, o token graduado **não** passa por Burn & Earn. Em vez disso, ele é migrado para um pool **AMM v4** usando o caminho de migração legado do LaunchLab. Nesse modo, o criador não recebe um Fee Key NFT, e não há taxa de criador pós-migração em nível de programa. Alguns criadores neste modo usam uma ferramenta CLI off-chain para reivindicar taxas pós-migração opcionais configuradas via `creatorFeeOn` — veja [`products/launchlab/instructions`](/pt/products/launchlab/instructions) para a superfície on-chain.

## O referral de 0,1% de token-share

Separadamente do compartilhamento de taxa LP, criadores do LaunchLab (e qualquer usuário, realmente) podem compartilhar um **link de página de token** com sua carteira como referenciador. Quando qualquer pessoa negocia esse token (ou outros tokens do LaunchLab na mesma sessão) através do link compartilhado:

- **0,1% de cada negociação** é despachado para a carteira do referenciador.
- Pago em SOL ou no token de cotação do pool.
- Pago automaticamente em cada negociação; nenhuma etapa de reivindicação.

Este é um produto distinto do referral de 1% coberto em [`user-flows/referrals-and-blinks`](/pt/user-flows/referrals-and-blinks). O referral de 1% se aplica à UI de swap spot do Raydium; o referral de token share de 0,1% se aplica especificamente às páginas de token do LaunchLab.

## Reivindicando taxas

### Cofre de criador (pré-migração)

```ts
import { Raydium, TxVersion } from "@raydium-io/raydium-sdk-v2";
const raydium = await Raydium.load({ owner, connection });
const { execute } = await raydium.launchlab.claimCreatorFees({
  mint,           // launched token mint
  txVersion: TxVersion.V0,
});
await execute({ sendAndConfirm: true });
```

### Fee Key NFT (pós-migração)

Manter o Fee Key em sua carteira é suficiente — reivindicar pela página Portfolio, ou programaticamente através da instrução de reivindicação de taxa do programa de pool relevante. Veja [`products/cpmm/instructions`](/pt/products/cpmm/instructions) ou o equivalente AMM v4 para a superfície on-chain.

## Checklist do integrador

- **Mostrar ambos os fluxos de taxas.** Um painel de criador que mostra apenas um de "cofre pré-migração" / "Fee Key pós-migração" deixa dinheiro na mesa para usuários que não sabem que o outro existe.
- **Respeite transferências de propriedade de Fee Key.** Se um criador vender ou transferir seu Fee Key NFT, o proprietário anterior perde acesso a futuras reivindicações de taxa. Sua UI deve ler o detentor de NFT atual cada vez, não armazenar em cache o endereço de criador original.
- **Desvio de configuração de plataforma.** Os padrões do Raydium podem mudar. Para qualquer plataforma que não seja a própria do Raydium, sempre leia a Platform PDA ao vivo para ver a divisão real de burn / criador / plataforma para esse lançamento específico.
- **Colateral e tokens de taxa do Token-2022.** As taxas de criador podem ser denominadas em tokens Token-2022 com taxas de transferência; fatore o desconto da taxa de transferência em qualquer número de taxa "reivindicável" exibido.
