<Info>
  Uma **Platform** no LaunchLab é um ambiente de lançamento de terceiros construído sobre o programa LaunchLab do Raydium. As plataformas podem se marcar, cobrar suas próprias taxas e configurar os parâmetros de migração pós-curva de vinculação para cada token lançado sob elas — sem executar um programa on-chain separado. A própria UI de lançamento do Raydium é apenas uma plataforma entre muitas.
</Info>

## Por que as plataformas existem

O Raydium queria que o LaunchLab fosse uma **primitiva compartilhada**, não um único produto com marca. O sistema Platform PDA permite que ecossistemas, provedores de ferramentas, comunidades e equipes de memes ofereçam experiências de lançamento com curadoria ou marca própria que todas se liquidam no mesmo programa de curva de vinculação LaunchLab subjacente e no mesmo pool pós-migração CPMM (ou AMM v4).

Exemplos de plataformas que você pode ver na prática:

- **UI próprio do LaunchLab do Raydium** em raydium.io/launchlab — a implementação de referência.
- **Launchpads de parceiros** — terceiros que registraram sua própria Platform PDA para marcar a experiência de lançamento.
- **Plataformas especializadas** — por exemplo, uma plataforma específica da comunidade que lança apenas memecoins vinculadas a um tema particular, com uma participação de taxa customizada financiando seu tesouro.

Todos esses tokens ainda residem no mesmo programa LaunchLab, se formam no mesmo programa CPMM e podem ser roteados pelos mesmos agregadores.

## O PDA da Platform

Cada plataforma é representada por uma Platform PDA armazenada no programa LaunchLab. Os campos incluem:

| Campo | Descrição |
|---|---|
| `platform_key` | Seed / identificador para a plataforma, usado para derivar a Platform PDA. |
| `platform_fee_destination` | Conta que recebe a parte da plataforma nas taxas de negociação pré e pós-migração. |
| `platform_fee_bps` | Participação de taxa da plataforma em swaps pré-migração (varia por plataforma). |
| `creator_fee_enabled` | Se esta plataforma habilita o compartilhamento de taxa de criador pós-migração via Burn & Earn. |
| `creator_fee_share_bps` | Se habilitado, participação das taxas LP que fluem para o NFT da Fee Key do criador. |
| `cp_config_id` | Se o alvo pós-migração é CPMM, qual `AmmConfig` usar — escolhe o nível de taxa. |
| `name`, `description`, `image_url` | Metadados de exibição usados por carteiras e UIs que suportam marca de plataforma. |
| `tokens_launched` | Contador ou índice on-chain de tokens criados sob esta plataforma. |

Os nomes exatos dos campos estão no IDL do LaunchLab ([`sdk-api/anchor-idl`](/pt/sdk-api/anchor-idl)). A tabela acima é uma visão geral descritiva.

## O que uma plataforma pode controlar

**Captura de taxas:**

- **Participação de taxa da plataforma** em cada compra/venda pré-migração.
- **Participação pós-migração** — quando o token se forma, os tokens LP são cunhados para uma divisão de múltiplas partes definida pela plataforma: uma parte para queimar (Burn & Earn — um padrão de bloqueio de liquidez que permite que o LP bloqueado continue reivindicando sua participação nas taxas do pool, veja [`user-flows/burn-and-earn`](/pt/user-flows/burn-and-earn)), uma parte para o criador (Fee Key NFT) e uma parte para a própria plataforma.

**Destino pós-migração:**

- **CPMM (padrão se taxas de criador habilitadas)** com um `AmmConfig` escolhido via `cp_config_id`. Diferentes `AmmConfig`s têm diferentes níveis de taxa; uma plataforma pode escolher um nível de taxa mais alto se seus lançamentos forem ativos especulativos de alto volume.
- **AMM v4 (se taxas de criador desabilitadas)** — para compatibilidade com o fluxo original do LaunchLab.

**Experiência do usuário:**

- **Metadados de marca** usados pelo app Raydium e qualquer carteira / explorador que exiba tokens LaunchLab.
- **Customização da página de lançamento** — plataformas que hospedam seu próprio frontend podem filtrar tokens para mostrar apenas os lançamentos de sua própria plataforma.

## O que uma plataforma não pode controlar

- **Matemática da curva de vinculação.** A forma da curva (logística, exponencial, etc.) é parte do próprio programa LaunchLab. As plataformas escolhem entre as curvas suportadas; elas não podem adicionar curvas customizadas arbitrárias.
- **Lógica de limite de migração.** A condição que dispara a formação para CPMM / AMM v4 (capitalização de mercado alvo atingida) é aplicada pelo programa.
- **Postura de segurança/auditoria central.** Cada plataforma herda o histórico de auditoria do LaunchLab; uma plataforma maliciosa não pode contorná-lo alterando o programa.

## Registrando uma plataforma

O programa LaunchLab expõe uma instrução `create_platform` (nome exato do IDL). Os chamadores passam:

- A seed que vai derivar a Platform PDA.
- A conta de destino da taxa.
- Bps de taxa desejados (sujeito a limites de nível de programa).
- Config pós-migração (ID `AmmConfig` de CPMM, se aplicável).
- Metadados de marca.

O aluguel da Platform PDA é pago pelo criador. Atualizações de marca e alguns parâmetros de taxa são permitidas pós-criação pela autoridade da plataforma; parâmetros estruturais (programa de destino de migração) normalmente não são.

Veja [`products/launchlab/instructions`](/pt/products/launchlab/instructions) para a assinatura exata da instrução no IDL LaunchLab atual.

## Interação econômica

Um único swap em uma curva de vinculação LaunchLab distribui valor aproximadamente assim (proporções exatas definidas pela plataforma):

```
Comprador paga:              100 SOL
Taxa de protocolo (LaunchLab): – pequeno %
Taxa de plataforma:          – platform_fee_bps%
Taxa de criador (pré-migração): – creator_fee_bps% (se habilitado)
Resto (para a curva):        ~ resto
```

Pós-migração, tokens LP são divididos conforme configuração da plataforma:

```
Tokens LP cunhados na formação:  100%
  → Burn & Earn (queimados):       X%  (padrão no Raydium: 90)
  → Creator Fee Key NFT:           Y%  (padrão no Raydium: 10)
  → Plataforma:                    Z%  (configurado pela plataforma; pode ser 0)
```

A **participação do Burn & Earn** converte em renda de taxa compartilhada distribuída ao detentor da Fee Key — que, sob os padrões próprios do Raydium, é o "vault" de queima representando respaldo de taxa de mercado amplo para confiança do token.

## Notas para integradores

- **Listando tokens por plataforma.** Use os endpoints de pool-info / mint da API com um filtro `platform` (quando suportado) para listar apenas tokens lançados sob uma Platform PDA específica.
- **Exibição de marca.** Se você construir uma carteira ou explorador que exiba tokens LaunchLab, ler o nome / URL de imagem da plataforma a partir da Platform PDA é muito melhor UX do que mostrar "LaunchLab" para cada token.
- **Matemática de taxa.** Cada cotação de curva de vinculação deve levar em conta a taxa de plataforma além das taxas de protocolo e criador. Os ajudantes de cotação do SDK fazem isso automaticamente quando a Platform PDA é fornecida.
