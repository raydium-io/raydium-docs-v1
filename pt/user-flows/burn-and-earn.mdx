## O que é Burn & Earn

Burn & Earn é um recurso Raydium que permite a um criador de pool (ou qualquer LP) fazer um depósito **permanente** na blockchain, mantendo o direito de coletar taxas de negociação. Uma posição se torna não-resgatável, mas um **NFT de Chave de Taxa Raydium** recém-cunhado representa o direito contínuo de `collectFees` dessa posição bloqueada.

Casos de uso comuns:

- **Confiança no lançamento.** Projetos se comprometem a não fazer rug-pull bloqueando o LP do deployer. Os detentores podem verificar na blockchain que a liquidez é permanente.
- **Colecionável gerador de taxa.** Um NFT negociável que gera rendimento. Venda o NFT; transfira o fluxo de taxas.
- **LP em nível de tesouro.** Protocolos que desejam manter liquidez permanentemente para fins de marca / descoberta, enquanto ainda monetizam o lado das taxas.

## Tipos de pool suportados

| Programa | Suporte a Burn & Earn |
|---|---|
| CPMM | Sim |
| CLMM | **Apenas intervalo completo** (recomendado) |
| AMM v4 (Híbrido) | Não suportado |

Para CLMM, apenas posições que abrangem toda a faixa de preço devem ser bloqueadas. Uma vez bloqueado, você não pode rebalancear — se o preço sair do seu intervalo escolhido, você deixa de ganhar taxas permanentemente nessa posição. Uma posição Burn & Earn de intervalo restrito é quase sempre um erro.

## Como o bloqueio funciona no nível do programa

1. O token de LP original (CPMM) ou NFT de posição (CLMM) é transferido para um PDA de propriedade do programa, pertencente ao programa Burn & Earn.
2. Um **NFT de Chave de Taxa** é cunhado para o chamador. Seus metadados apontam de volta para o PDA da posição bloqueada.
3. Burn & Earn expõe `collectFees` (ou a chamada equivalente por tipo de pool) que:
   - Verifica se o signatário detém o NFT de Chave de Taxa para a posição bloqueada fornecida.
   - Chama a instrução de reivindicação de taxa do programa de pool subjacente via CPI.
   - Roteia as taxas coletadas para os ATAs do detentor da Chave de Taxa.
4. Não há um `withdraw` correspondente — o LP / posição subjacente nunca sai do escrow.

A Chave de Taxa é um NFT SPL padrão. Pode ser vendida em qualquer marketplace de NFT Solana, movida entre carteiras, usada como garantia em protocolos de empréstimo que aceitam NFTs, ou encapsulada em outro contrato. Quem a detiver no momento de uma chamada `collectFees` recebe as taxas.

## Guia da interface

1. Crie um pool CPMM ou um pool CLMM com uma posição de intervalo completo, como de costume.
2. Vá para **Liquidez → Criar → Burn & Earn**.
3. Selecione a posição a bloquear.
4. Verifique os detalhes da posição (endereço NFT, tamanho, pool) e digite a frase de confirmação que a interface mostra. Isso é intencionalmente pesado em fricção porque a operação é irreversível.
5. Clique em **Confirmar, bloquear a liquidez permanentemente**.
6. Um NFT de Chave de Taxa chega à sua carteira. Aparece na página Portfolio sob posições bloqueadas.

Reivindicações de taxas da mesma forma que você reclama de qualquer posição — através da página Portfolio ou via `collectFees`.

## Superfície da API

O endpoint de informações de pool retorna um campo `burnPercent` indicando a parcela da liquidez desse pool que foi bloqueada via Burn & Earn:

```http
GET https://api-v3.raydium.io/pools/info/list
  ?poolType=standard
  &poolSortField=default
  &sortType=desc
  &pageSize=1
  &page=1
```

```json
{
  "data": [
    {
      "id": "...",
      "burnPercent": 99.17,
      ...
    }
  ]
}
```

`burnPercent` é uma superfície útil para sinalização de confiança: um token cujo LP é 99%+ queimado é dramaticamente menos suscetível a rug-pull do que um cujo LP é 0% queimado.

## Armadilhas

- **Irreversível.** O lado LP nunca pode ser recuperado. Confirme que o valor é algo que você está disposto a nunca tocar novamente.
- **Intervalos restritivos CLMM são uma armadilha.** Bloqueie apenas posições CLMM de intervalo completo, a menos que você realmente entenda que uma posição bloqueada por intervalo para de ganhar assim que o preço sai da faixa.
- **Perder o NFT de Chave de Taxa é perder as taxas para sempre.** Trate-o como um título de propriedade. Não o queime acidentalmente; não o coloque em stake em um contrato hostil.
- **Mints Token-2022 com taxas de transferência**: Burn & Earn os suporta, mas o corte de taxa de transferência se aplica em cada reivindicação de taxa — orçamento para isso.
- **Não há bloqueio parcial.** Se você deseja 50% bloqueado e 50% móvel, divida a posição em duas posições primeiro, depois bloqueie apenas uma.

## LaunchLab usa Burn & Earn internamente

Tokens formados no [LaunchLab](/pt/products/launchlab) usam Burn & Earn para LP pós-migração. A divisão padrão no próprio LaunchLab da Raydium: **90% dos tokens de LP são queimados** (bloqueados em Burn & Earn) e **10% vão para o criador como um NFT de Chave de Taxa**. Outras plataformas que constroem em LaunchLab podem configurar diferentes divisões de burn / criador / plataforma. Veja [`products/launchlab/platforms`](/pt/products/launchlab/platforms) e [`products/launchlab/creator-fees`](/pt/products/launchlab/creator-fees).
