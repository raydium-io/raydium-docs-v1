<Info>
  Um pool CLMM exige duas escolhas permanentes: uma **camada de taxa** e um **preço inicial**. Você também precisa abrir uma posição inicial para que o pool tenha liquidez. Tenha cuidado extra antes de assinar.
</Info>

## O que você precisa

- Uma carteira Solana contendo:
  - Os dois mints de token que você deseja parear.
  - Aproximadamente 0,3 SOL para criar o pool e a posição (rent) mais taxas de prioridade.
- Uma decisão sobre a camada de taxa.
- Um preço de mercado inicial claro e primeiro intervalo.

CLMM suporta tanto mints SPL Token quanto Token-2022.

## Camadas de taxa

| Camada de taxa | Melhor para |
|----------|----------|
| 0,01% | Pares stablecoin-stablecoin |
| 0,05% | Pares correlacionados |
| 0,25% | Maioria dos principais pares voláteis |
| 1% | Pares de cauda longa ou exóticos |

Camadas de taxa mais baixas exigem preços mais ajustados e maior volume. Camadas de taxa mais altas toleram mais volatilidade, mas podem reduzir volume.

## Guia passo a passo da UI

Em [raydium.io/clmm/create-pool](https://raydium.io/clmm/create-pool):

1. **Escolha o par de tokens.** Cole os dois endereços de mint.
2. **Escolha a camada de taxa.** Use a recomendação da UI como ponto de partida.
3. **Defina o preço inicial.** Use um preço de mercado existente. Se um pool já existe para o par, compare com ele.
4. **Escolha o primeiro intervalo.** Digite o preço inferior e superior para a posição inicial.
5. **Digite o valor do depósito.** A UI calcula a taxa de token necessária a partir do seu intervalo.
6. **Revise e assine.** Confirme o par, a camada de taxa, o preço inicial, o intervalo, os valores do depósito e a rent.

## Definindo o preço inicial corretamente

Se outros pools CLMM ou CPMM existem para o mesmo par, use seu preço atual como ponto de partida. Se nenhum pool Raydium existe, use uma fonte de mercado confiável.

Se você definir o preço inicial errado, a primeira negociação provavelmente fará arbitragem do pool e drenará o lado subavaliado. Não existe um botão de reset simples. Normalmente você precisa sacar e recriar o pool.

## Primeira posição: seleção do intervalo

- **Intervalo ajustado:** melhor para pares correlacionados, mas precisa de gerenciamento ativo.
- **Intervalo médio:** comum para principais voláteis como SOL/USDC.
- **Intervalo amplo:** eficiência de capital menor, mas mais fácil deixar sem gerenciamento.
- **Intervalo completo:** se comporta mais como um CPMM, mas perde muito do benefício da liquidez concentrada.

Se o preço atual estiver fora do seu intervalo, a posição inicial estará fora de intervalo e não ganhará taxas até o preço se mover para dentro do intervalo.

## Checklist pós-criação

1. **Página do pool carrega.** Confirme o endereço do pool, o par e a camada de taxa.
2. **Posição aparece.** Verifique seu portfólio Raydium para o NFT da posição.
3. **Intervalo está correto.** Confirme o preço inferior e superior.
4. **Liquidez é diferente de zero.** O pool deve mostrar liquidez ativa para seu intervalo selecionado.
5. **Teste um pequeno swap.** Verifique se a rota funciona antes de compartilhar o pool publicamente.

## Solução de problemas

### O custo de criação é maior que o esperado

Intervalos amplos e camadas de taxa pequenas podem exigir mais rent de tick-array. Estreite o intervalo de início ou adicione mais SOL antes de tentar novamente.

### Ordem de token incorreta

A UI canonicaliza a ordem de token, então o token A/token B exibido pode diferir da ordem que você digitou. Revise a cotação de preço cuidadosamente antes de assinar.

### Camada de taxa incorreta

Você não pode mudar a camada de taxa após a criação do pool. Se estiver errada, feche sua liquidez e crie um novo pool na camada correta.

### Pool com liquidez zero

Criar estado do pool sem uma posição deixa o pool inutilizável para swaps. Abra uma posição inicial antes de compartilhar o pool.

### Posição inicial fora do intervalo

Se seu primeiro intervalo não incluir o preço inicial, a posição não ganha taxas. Reabra uma posição em torno do preço atual.
