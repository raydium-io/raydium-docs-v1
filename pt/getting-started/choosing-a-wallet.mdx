## O que uma carteira compatível com Raydium deve fazer

Uma carteira é compatível com Raydium se conseguir fazer tudo o seguinte:

- Manter um keypair Solana localmente e **nunca** transmitir a chave privada para fora do dispositivo.
- **Assinar transações versionadas (V0).** Os swaps do Raydium usam Address Lookup Tables e exigem suporte a V0. Carteiras que assinam apenas transações legacy não conseguem completar um swap no Raydium.
- **Conectar a dApps via o padrão Wallet Adapter.** Isso é o que faz o botão "Connect Wallet" em raydium.io funcionar.
- **Mostrar uma prévia clara da transação** antes de você aprovar, incluindo o(s) programa(s) sendo chamado(s) e os movimentos de token.
- **Suportar contas de token associadas** (criando uma ATA na primeira vez que você recebe um novo mint).

Estes são requisitos básicos. Qualquer carteira Solana moderna atende a eles. Qualquer coisa que não atenda é insegura para assinar transações do Raydium — período.

## Funcionalidades que importam além do básico

Quando uma carteira passa pelo básico, as diferenças dependem do seu fluxo de trabalho e perfil de risco. Decida qual das seguintes funcionalidades importa para **você**, e escolha uma carteira que as gerencie:

| Funcionalidade | Quando importa |
|---|---|
| **Emparelhamento com carteira de hardware** | Você está segurando mais que uma quantia casual. A carteira de software se torna um construtor de transações; o dispositivo de hardware mantém a chave e assina. A maioria das carteiras Solana confiáveis se emparelha com pelo menos Ledger; algumas se emparelham com Keystone, SafePal ou outros dispositivos. |
| **Navegador dApp mobile** | Você espera usar Raydium principalmente de um telefone. O navegador dentro da carteira é o que carrega raydium.io e assina transações no mobile. |
| **Gerenciamento de múltiplas contas** | Você separa fundos entre carteiras quentes / frias / experimentais e quer mudar rapidamente sem malabarismos com extensões. |
| **Consciência da extensão Token-2022** | Você vai interagir com pools que usam mints Token-2022 (por ex. tokens com transfer-fee). Uma carteira que mostra essas extensões na UI de assinatura ajuda você a detectar taxas inesperadas antes de assinar. |
| **Exibição de taxa de prioridade** | Você negocia durante congestionamento e quer ver o que a carteira definiu para o preço do compute-budget antes de aprovar. |
| **Suporte a Solana Actions / Blinks** | Você quer usar os links de swap com um clique que Raydium publica em mídia social. Veja [`user-flows/referrals-and-blinks`](/pt/user-flows/referrals-and-blinks). |
| **Código-fonte aberto** | Você quer ser capaz de auditar o cliente da carteira (ou contar com a comunidade tendo feito isso). |
| **Contas multi-chain** | Você mantém ativos em chains além de Solana e prefere uma carteira para todos eles. Verifique se o suporte multi-chain é maduro para as chains que você realmente usa; isso varia. |

## Carteiras Solana populares

Os seguintes são projetos de carteira bem conhecidos com suporte ativo a Solana. Eles estão listados alfabeticamente; **a inclusão é informativa, não uma aprovação**, e a lista não é exaustiva. Verifique a URL oficial antes de instalar — clones de phishing costumam comprar anúncios patrocinados contra os nomes de domínio reais.

### Carteiras de software

| Carteira | Onde funciona | Site oficial |
|---|---|---|
| Phantom | Extensão de navegador, iOS, Android | [phantom.app](https://phantom.app) |
| Solflare | Extensão de navegador, iOS, Android, web | [solflare.com](https://solflare.com) |
| OKX Wallet | Extensão de navegador, iOS, Android | [okx.com/wallet](https://www.okx.com/wallet) |
| Backpack | Extensão de navegador, iOS, Android, desktop | [backpack.app](https://backpack.app) |

Todas as quatro são não-custodiais, suportam Wallet Adapter, assinam transações V0 e se emparelham com pelo menos um dispositivo de hardware mainstream. Além disso, as diferenças são específicas do fluxo de trabalho — use os critérios da seção anterior para decidir qual funciona para você.

### Carteiras de hardware

Uma carteira de hardware se emparelha com uma das carteiras de software acima (o lado do software se torna o construtor de transações; o dispositivo de hardware mantém a chave e assina).

| Dispositivo | Tipo | Site oficial |
|---|---|---|
| Ledger | USB / Bluetooth | [ledger.com](https://www.ledger.com) |
| Keystone | Air-gapped (assinatura por código QR) | [keyst.one](https://keyst.one) |

Cada carteira de software documenta quais dispositivos de hardware ela suporta; verifique antes de comprar.

## Padrão de carteira quente / fria

Uma prática comum que funciona independentemente de qual carteira você escolha:

- Uma **carteira quente** mantém uma pequena quantia que você aceita que pode ser perdida — para swaps diários e experimentação.
- Uma **carteira fria** (emparelhada com um dispositivo de hardware) mantém tudo que você não gostaria de perder. Mova fundos apenas quando você realmente precisar deles na carteira quente.

Raydium trata cada carteira como completamente independente: posições separadas, tokens LP separados, recompensas de farm separadas, histórico separado. Mudar de carteira é um clique.

## Onde encontrar opções de carteira

O modal "Connect Wallet" da UI do Raydium lista cada carteira que se integrou ao Raydium via Wallet Adapter. Navegue nele para ver o que é atualmente suportado. O diretório oficial de carteiras do Solana ([`solana.com/ecosystem/explore?categories=wallet`](https://solana.com/ecosystem/explore?categories=wallet)) é outro ponto de partida neutro.

Quando você avaliar uma carteira, passe por esta lista de verificação:

- [ ] Mantida: lançamentos recentes, um rastreador de issues ativo, um canal de divulgação de segurança.
- [ ] Revisada: uma auditoria publicada por uma empresa confiável, ou código aberto com uma comunidade ativa.
- [ ] Suporte a transações V0 confirmado (teste na devnet antes de transferir fundos reais).
- [ ] Se você planeja usar um dispositivo de hardware: a carteira documenta quais dispositivos ela suporta e como o emparelhamento funciona.
- [ ] Você encontrou o download oficial via domínio verificado do projeto — **não** via um anúncio patrocinado ou um link encaminhado.

## Básicos de segurança de carteira

Independentemente de qual carteira você escolha:

- **Escreva a frase-semente em papel.** Armazene o papel em dois locais fisicamente separados. **Nunca** digite a frase-semente em lugar nenhum exceto durante uma recuperação de carteira em uma instalação nova.
- **Use um gerenciador de senhas** para a senha de desbloqueio da carteira — e mantenha essa senha completamente separada da frase-semente.
- **Use um perfil de navegador dedicado** para atividade cripto, com o número mínimo de extensões instaladas. Extensões de phishing são o maior vetor de vazamento.
- **Trate cada prompt de wallet-connect com suspeita.** Leia o que você está assinando. Se uma transação transferir tudo de um token em vez de uma quantia específica, rejeite-a.
- **Coloque um marcador em `raydium.io`.** Sempre navegue pelo marcador, nunca por um anúncio de mecanismo de busca.

Mais orientação de segurança está em [`getting-started/trust-and-safety`](/pt/getting-started/trust-and-safety).
