A Auth API gerencia a autenticação baseada em carteira para os serviços do LaunchLab. Troque uma mensagem assinada por um token JWT e, em seguida, utilize esse token para autenticar requisições nas APIs de Forum e criação de Mint.

## Endpoints da API

**Mainnet:** https://launch-auth-v1.raydium.io  
**Devnet:** https://launch-auth-v1-devnet.raydium.io

## Fluxo de Autenticação

Nenhuma credencial prévia é necessária. A autenticação é baseada em carteira:

1. **Obtenha o timestamp atual** (segundos Unix)
2. **Assine a mensagem** com sua carteira: `"Sign in to raydium.io: " + <timestamp>`
3. **Envie a assinatura** para o endpoint `/request-token`
4. **Receba o token JWT** válido pelo período especificado
5. **Utilize o token** no cabeçalho `ray-token` para os endpoints protegidos

## Formato da Mensagem

A mensagem a ser assinada deve seguir exatamente este formato:

```
Sign in to raydium.io: <UNIX_TIMESTAMP_SECONDS>
```

**Exemplo:**
```
Sign in to raydium.io: 1704067200
```

- Utilize o timestamp Unix atual em segundos (não em milissegundos)
- Assinatura Ed25519 (assinatura nativa do Solana)
- Codifique a assinatura como string base58
- Passe o timestamp e a assinatura para `/request-token`

## Validação do Token

Utilize `/check-token` para verificar se o seu token ainda é válido. O endpoint também estende automaticamente o tempo de vida do token quando ele estiver próximo do vencimento.

O tempo de vida do token pode ser configurado no servidor; o padrão é tipicamente 24 horas.

## Revogação do Token

Revogue um token pelo endpoint `/del-token`.

## Métodos de Assinatura Suportados

- **Carteira padrão:** Assinatura Ed25519 com `/request-token`
- **Ledger wallet:** Transação assinada com `/request-token-ledger`

## Casos de Uso

- **Publicação no Forum** — autentique-se antes de postar comentários
- **Criação de Mint** — confirme a propriedade da carteira antes de submeter novos tokens
- **Sessões de usuário** — mantenha o estado autenticado entre chamadas à API
- **Suporte a múltiplas carteiras** — emita tokens separados por endereço de carteira
