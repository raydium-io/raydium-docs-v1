A Owner API fornece visualizações por carteira de posições e saldos do Raydium. Ela busca dados em cache para posições, liquidez bloqueada, pools criados e participação em IDO, tornando-a ideal para UIs de carteira e painéis de portfólio.

## Hosts

| Ambiente | Host                       |
|----------|---------------------------|
| Mainnet  | `owner-v1.raydium.io`     |
| Devnet   | `owner-v1-devnet.raydium.io` |

## Visão geral

A API expõe cinco endpoints principais organizados por tipo de recurso:

- **`/position/stake/{owner}`** — Posições de stake (LP AMM / CPMM padrão)
- **`/position/clmm-lock/{owner}`** — Posições CLMM bloqueadas
- **`/create-pool/{owner}`** — Pools criados por esta carteira
- **`/main/ido/{owner}`** — Dados de participação em IDO

## Atualização de dados

Este serviço armazena dados de posição em cache e os atualiza periodicamente. Mudanças de posição muito recentes (dentro de segundos) podem não aparecer imediatamente nas respostas.

## Padrões comuns

### Buscar todas as posições de stake para uma carteira

```bash
curl -s "https://owner-v1.raydium.io/position/stake/{owner}"
```

Substitua `{owner}` pela chave pública da carteira em formato base58.

### Buscar posições bloqueadas

```bash
curl -s "https://owner-v1.raydium.io/position/clmm-lock/{owner}"
```

### Verificar participação em IDO

```bash
curl -s "https://owner-v1.raydium.io/main/ido/{owner}"
```

## Resultados vazios

Se uma carteira não tiver dados para um endpoint particular, o serviço retorna um erro 404 com metadados da solicitação:

```json
{
  "id": "...",
  "success": false
}
```
