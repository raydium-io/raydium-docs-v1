A Temp API é uma área temporária para endpoints que ainda não possuem um lugar definitivo nas APIs principais do Raydium. Este serviço é destinado a prototipagem rápida, ferramentas internas e funcionalidades em desenvolvimento ativo.

## Hosts

| Ambiente | Host                              |
|----------|----------------------------------|
| Mainnet  | `temp-api-v1.raydium.io`         |
| Devnet   | `temp-api-v1-devnet.raydium.io`  |

## Aviso de estabilidade

**Este serviço não oferece garantias de estabilidade de API.** Os endpoints podem ser alterados, descontinuados, movidos para outro serviço ou removidos completamente sem aviso prévio. Utilize esta API apenas para:

- Ferramentas e scripts internos
- Funcionalidades em desenvolvimento ativo
- Caminhos não críticos que toleram falhas
- Testes e prototipagem

Não dependa desta API para dApps em produção ou integrações críticas.

## Visão geral

Este serviço expõe atualmente:

- **`/cp-creator-fee`** — Recupera as taxas de criador acumuladas para uma carteira em pools CPMM.
- **`/limit-order/order/list`** — Ordens limitadas CLMM atualmente pendentes de uma carteira (abertas e parcialmente preenchidas), servidas a partir do cache do indexador.
- **`/limit-order/history/order/list-by-user`** — Ordens limitadas CLMM históricas (fechadas) de uma carteira, com filtros opcionais `poolId` / `mint1` / `mint2` / `hideCancel`; paginadas via cursor por `nextPageId`.
- **`/limit-order/history/event/list-by-pda`** — Log de eventos por PDA (`open` / `increase` / `decrease` / `settle` / `close`) para um ou mais PDAs de ordens limitadas; paginados via cursor por `nextPageId`.

## Padrões comuns

### Verificar taxas de criador de uma carteira

```bash
curl -s "https://temp-api-v1.raydium.io/cp-creator-fee?wallet=<your-wallet-pubkey>"
```

Retorna uma lista de pools em que a carteira é a criadora, juntamente com as taxas de criador acumuladas e ainda não resgatadas em Token A e Token B.

### Listar ordens limitadas CLMM pendentes de uma carteira

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/order/list?wallet=<your-wallet-pubkey>"
```

Retorna ordens completamente não preenchidas e parcialmente preenchidas. Cada linha é um `LimitOrderState` decodificado (PDA, pool, proprietário, tick, lado, fase de cohort FIFO, quantidades totais/preenchidas) mais o contexto do pool necessário para chamar `SettleLimitOrder`, incluindo um valor `pendingSettle` pré-calculado.

O endpoint de ordens ativas não é paginado — ele retorna todas as ordens atualmente em cache da carteira em um único payload.

### Listar ordens limitadas fechadas de uma carteira

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/order/list-by-user?wallet=<your-wallet-pubkey>&size=50"
```

Retorna uma página de ordens históricas, cada uma resumida com `fillStatus` (`NONE` / `PARTIAL` / `FULL`) e os valores de entrada/saída no momento em que a ordem saiu do conjunto ativo. Acrescente `&nextPageId=<nextPageId da resposta anterior>` para buscar a próxima página.

### Obter a linha do tempo de eventos de uma ordem específica

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/event/list-by-pda?pda=<limit-order-pda>"
```

`pda` pode ser uma chave única ou uma lista separada por vírgulas. Cada linha representa uma mutação on-chain (`open` / `increase` / `decrease` / `settle` / `close`) com txid, slot, horário do bloco, indicador de assinatura por proprietário ou keeper (`autoRunner`), e os deltas de entrada/saída em unidades legíveis por humanos.

### Resultado vazio

Se a carteira não tiver registros em cache, o endpoint de ordens ativas retorna:

```json
{
  "id": "...",
  "success": true,
  "data": { "rows": [] }
}
```

Os endpoints de histórico retornam o mesmo formato `{ rows: [], nextPageId?: ... }` com um array `rows` vazio. O endpoint legado `/cp-creator-fee` retorna `"data": []` por compatibilidade com versões anteriores.
