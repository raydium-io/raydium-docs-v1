## À qui s'adresse ce chapitre

Aux développeurs qui intègrent Raydium hors-chaîne (TS/JS, Python, backends) ou à partir d'un autre programme Solana (Rust CPI). Également aux agents IA et aux intégrateurs d'outils de codage — voir [Intégration IA](/fr/sdk-api/ai-integration).

</Info>

## Choisissez votre chemin d'intégration

Cinq chemins parmi lesquels choisir. Le bon dépend de ce qui appelle Raydium et du niveau de contrôle dont vous avez besoin.

| Chemin | Idéal pour | Latence | Logique personnalisée | Langages |
|---|---|---|---|---|
| [Trade API](/fr/sdk-api/trade-api) | Backends, bots, Blinks — partout où vous voulez une tx construite par serveur | La plus faible | Limitée (routes prédéfinies) | N'importe lequel |
| [TypeScript SDK](/fr/sdk-api/typescript-sdk) | Apps Node/TS et frontends nécessitant un accès complet au builder | Moyenne | Complète | TS / JS |
| [Intégration Python](/fr/sdk-api/python-integration) | Backends Python et pipelines ML | Moyenne | Complète (construction manuelle d'instructions) | Python |
| [Rust CPI](/fr/sdk-api/rust-cpi) | Composition de programmes qui appellent Raydium depuis la chaîne | N/A (on-chain) | Complète + composable | Rust |
| [Anchor IDL](/fr/sdk-api/anchor-idl) | Codegen pour les langages sans support natif (Go, Kotlin, Swift, ...) | Variable | Complète | N'importe quel langage avec un outil de codegen Anchor |

Si vous n'êtes pas sûr : backends et bots → **Trade API** ; frontends et apps à contrôle total → **TypeScript SDK** ; composition on-chain → **Rust CPI**.

## Discipline de versioning

Chaque exemple de code sur chaque page de ce chapitre comporte une bannière en haut :

```
SDK:       @raydium-io/raydium-sdk-v2 vX.Y.Z
Program:   see /reference/program-addresses
Cluster:   mainnet-beta
Verified:  YYYY-MM-DD
```

Si vous modifiez un exemple de code, mettez à jour la bannière.
