## Pré-migration : accrual direct des frais

Tant qu'un token se trouve sur la courbe de liaison, chaque achat et chaque vente paient une composante de frais créateur qui s'accumule dans un vault dédié par token. Le **créateur** (le portefeuille qui a lancé le token) peut réclamer du vault à tout moment.

Caractéristiques :

- Dénominés dans le **quote token** (généralement SOL).
- Configurés par la **plateforme** (la configuration de la plateforme contrôle si les frais créateur sont activés et à combien de bps).
- Réclamables à tout moment depuis la page Portfolio de l'interface LaunchLab ; programmatiquement via l'instruction LaunchLab `claim_creator_fees` (nom exact dans l'IDL).
- Se poursuivent jusqu'à la graduation ; à la graduation, les frais pré-migration résiduels sont réclamables mais aucun nouveau frais pré-migration ne s'accumulera.

## Post-migration : Fee Key NFT via Burn & Earn

Quand la courbe de liaison d'un token atteint sa cible et que la liquidité migre vers un pool CPMM (ou AMM v4), les LP tokens sont émis et **répartis selon la configuration de la plateforme**. La répartition totalise toujours 100 % et implique toujours une combinaison de :

- **Burned** — transféré dans [Burn & Earn](/fr/user-flows/burn-and-earn) pour verrouillage permanent. La Fee Key pour cette portion est détenue par un compte contrôlé par la plateforme ou désigné pour le burn.
- **Creator** — une Fee Key NFT émise directement au portefeuille du créateur du token. Cette Fee Key représente le droit de collecter les frais de trading LP à partir de la part du créateur des LP tokens.
- **Platform** — optionnel ; une part qui va à la trésorerie de la plateforme sous forme d'une autre Fee Key NFT.

### Valeurs par défaut de LaunchLab de Raydium

| Part | Par défaut sur raydium.io/launchlab |
|---|---|
| Burned (Burn & Earn, verrouillé perpétuellement) | 90% |
| Creator (Fee Key NFT) | 10% |
| Platform | 0% (la plateforme Raydium ne prend pas de part LP plateforme) |

À ces valeurs par défaut, **seul le créateur réclame les frais de trading à partir de sa part** ; l'autre 90% est verrouillé et ses frais vont à Burn & Earn (reflétant le mécanisme de signalisation de confiance plutôt que tout portefeuille individuel).

### Autres plateformes

Les plateformes tierces construisant sur LaunchLab peuvent configurer leur propre répartition. Une plateforme souhaitant se financer via les frais LP post-migration peut définir (par exemple) 70 / 10 / 20 (burn / creator / platform). Consultez [`products/launchlab/platforms`](/fr/products/launchlab/platforms) pour les champs Platform PDA qui pilotent cela.

Indépendamment de la répartition, le **mécanisme est le même** : chaque part non brûlée est représentée comme une Fee Key NFT, qui peut être échangée, transférée ou collectée.

### Que se passe-t-il si les frais créateur post-migration sont désactivés

Si une plateforme définit `creator_fee_enabled = false`, le token gradué **ne** passe pas par Burn & Earn. À la place, il migre vers un pool **AMM v4** en utilisant le chemin de migration LaunchLab hérité. Dans ce mode, le créateur ne reçoit pas de Fee Key NFT, et il n'y a pas de frais créateur post-migration au niveau du programme. Certains créateurs dans ce mode utilisent un outil CLI hors chaîne pour réclamer des frais post-migration optionnels configurés via `creatorFeeOn` — consultez [`products/launchlab/instructions`](/fr/products/launchlab/instructions) pour la surface on-chain.

## Le referral 0,1% token-share

Séparé du partage des frais LP, les créateurs LaunchLab (et tout utilisateur, vraiment) peuvent partager un **lien de page token** avec leur portefeuille en tant que référent. Quand quelqu'un échange ce token (ou d'autres tokens LaunchLab dans la même session) via le lien partagé :

- **0,1% de chaque trade** est airdroppé au portefeuille du référent.
- Payé en SOL ou dans le quote token du pool.
- Payé automatiquement à chaque trade ; aucune étape de réclamation.

C'est un produit distinct du referral 1% couvert dans [`user-flows/referrals-and-blinks`](/fr/user-flows/referrals-and-blinks). Le referral 1% s'applique à l'interface spot swap de Raydium ; le referral token-share 0,1% s'applique spécifiquement aux pages token LaunchLab.

## Réclamation des frais

### Vault créateur (pré-migration)

```ts
import { Raydium, TxVersion } from "@raydium-io/raydium-sdk-v2";
const raydium = await Raydium.load({ owner, connection });
const { execute } = await raydium.launchlab.claimCreatorFees({
  mint,           // launched token mint
  txVersion: TxVersion.V0,
});
await execute({ sendAndConfirm: true });
```

### Fee Key NFT (post-migration)

Détenir la Fee Key dans votre portefeuille suffit — réclamez depuis la page Portfolio, ou programmatiquement via l'instruction de réclamation de frais du programme de pool pertinent. Consultez [`products/cpmm/instructions`](/fr/products/cpmm/instructions) ou l'équivalent AMM v4 pour la surface on-chain.

## Checklist pour les intégrateurs

- **Affichez les deux flux de frais.** Un tableau de bord créateur qui n'affiche que « vault pré-migration » ou « Fee Key post-migration » laisse de l'argent sur la table pour les utilisateurs qui ne savent pas que l'autre existe.
- **Respectez les transferts de propriété de Fee Key.** Si un créateur vend ou transfère sa Fee Key NFT, le propriétaire précédent perd l'accès aux réclamations futures. Votre interface doit lire le détenteur actuel de NFT à chaque fois, pas mettre en cache l'adresse créateur d'origine.
- **Dérive de la config plateforme.** Les valeurs par défaut de Raydium peuvent changer. Pour toute plateforme autre que celle de Raydium, lisez toujours le Platform PDA en direct pour voir la répartition burn / creator / platform actuelle pour ce lancement spécifique.
- **Token-2022 collatéral et tokens de frais.** Les frais créateur peuvent être dénominés en tokens Token-2022 avec frais de transfert ; prenez en compte la réduction des frais de transfert dans tout nombre « frais réclamables » affiché.
