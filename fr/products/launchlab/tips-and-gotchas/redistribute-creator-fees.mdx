Par défaut, les frais de créateur CPMM d'un pool migré depuis LaunchLab sont attribués au créateur de token original. Les plateformes peuvent remplacer ce comportement pour les futures migrations en définissant `platformCpCreator`.

## Pourquoi rediriger les frais de créateur

Les plateformes utilisent la redirection des frais de créateur quand elles ont besoin :

- D'une logique de distribution personnalisée entre les créateurs, les communautés et les revenus de plateforme.
- D'un collecteur de frais contrôlé par PDA pour la redistribution on-chain.
- D'une comptabilité unifiée pour tous les pools migrés sous une plateforme.

<Warning>
  Rediriger les frais de créateur change qui peut les collecter. Les créateurs de token perdent l'accès à la collecte directe pour les pools qui migrent après l'activation du paramètre, à moins que la plateforme ne leur redistribue les frais.
</Warning>

## Comment cela fonctionne

1. L'administrateur de plateforme appelle `updatePlatformConfig`.
2. `updateInfo.type` est défini à `updatePlatformCpCreator`.
3. La valeur est un portefeuille ou un PDA qui deviendra le créateur du pool CPMM pour les futures migrations.
4. Les futures migrations LaunchLab sous cette plateforme attribuent les frais de créateur CPMM à l'adresse configurée.
5. La plateforme collecte les frais de créateur et les redistribue selon ses propres règles.

```ts
await raydium.launchpad.updatePlatformConfig({
  programId,
  platformAdmin: owner,
  updateInfo: {
    type: "updatePlatformCpCreator",
    value: new PublicKey("<PLATFORM_WALLET_OR_PDA>"),
  },
  txVersion: TxVersion.V0,
});
```

## Points d'attention

- **Tout ou rien par plateforme.** Le paramètre s'applique à chaque future migration sous cette plateforme.
- **Prospectif uniquement.** Les pools déjà migrés conservent leur créateur de pool existant.
- **Mises à jour limitées par époque.** Les mises à jour de configuration de plateforme sont limitées en débit par l'époque Solana.
- **Les frais de créateur doivent exister.** Si le `AmmConfig` CPMM sélectionné a `creator_fee_rate = 0`, il n'y a pas de frais de créateur CPMM à collecter.
- **La gestion WSOL importe.** Si les frais collectés sont en WSOL et vous avez besoin de SOL natif, fermez le compte de token WSOL après la collecte.

## Liste de vérification de redistribution

Si vous redistribuez aux créateurs originaux :

1. Résolvez chaque créateur original depuis le compte du pool LaunchLab lié aux mints du pool CPMM.
2. Collectez les frais de créateur en attente pour le portefeuille ou le PDA de plateforme.
3. Calculez la part convenue de chaque créateur.
4. Transférez les tokens SPL ou SOL aux créateurs, en créant les ATA si nécessaire.
5. Publiez suffisamment de détails comptables pour que les créateurs puissent vérifier les distributions.
