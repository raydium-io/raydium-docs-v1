Les tokens LaunchLab ont deux phases de trading, chacune avec son propre modèle de frais. Ne regroupez pas ces phases en un seul « frais créateur » dans les textes de l'interface ou la comptabilité.

## Cycle de vie des frais

### Phase 1 : Courbe de liaison

Chaque échange sur la courbe de liaison peut inclure quatre frais cumulatifs :

| Frais | Défini par | Bénéficiaire |
|---|---|---|
| Frais protocole | Configuration globale LaunchLab, `tradeFeeRate` | Raydium |
| Frais plateforme | Configuration plateforme, `feeRate` | Portefeuille de réclamation plateforme |
| Frais créateur | Configuration plateforme, `creatorFeeRate` | Vault créateur token |
| Frais parrainage | Par transaction, `shareFeeRate` | Portefeuille parrain |

Les frais accumulés dans les vaults plateforme et créateur doivent être réclamés. Les frais de parrainage sont transférés directement lors de l'échange.

## Migration

Quand la courbe de liaison atteint la cible de graduation :

1. La liquidité migre vers le pool Raydium configuré.
2. Pour la migration CPMM, des tokens LP sont créés.
3. `migrateCpLockNftScale` détermine si les tokens LP sont brûlés ou verrouillés via Burn & Earn pour la plateforme et le créateur.
4. `cpConfigId` définit quel `AmmConfig` CPMM le pool migré utilise.

## Phase 2 : Pool CPMM

Après la migration, les swaps CPMM peuvent inclure :

| Frais | Fonctionnement |
|---|---|
| Frais de trade | Déduits de l'entrée du swap et répartis entre les LPs, le protocole et le trésor. |
| Frais créateur CPMM | Frais supplémentaires déduits de l'entrée du swap et attribués au créateur du pool. |

Les détenteurs de NFT Fee Key gagnent de la part LP des frais de trade. Les frais créateur CPMM sont séparés et sont réclamés via la collecte des frais créateur.

## Flux de revenu créateur

Un créateur peut avoir jusqu'à trois flux de revenus distincts :

| Source | Mécanisme | Chemin de réclamation typique |
|---|---|---|
| Frais créateur courbe de liaison | Vault pré-migration | `claimCreatorFee()` |
| Frais créateur CPMM | `creator_fee_rate` sur la config CPMM | `collectCreatorFee()` |
| Part frais LP | NFT Fee Key issus des tokens LP verrouillés | Flux de récolte Burn & Earn |

Ces sources ne se remplacent pas. Si toutes sont activées, elles doivent être suivies et réclamées séparément.

## Pièges courants

- Les frais créateur CPMM ne sont pas prélevés sur les frais de trade. Ce sont des frais supplémentaires.
- Les droits NFT Fee Key sont des droits de frais LP, pas des droits de frais créateur CPMM.
- Les pools CPMM standard créés en dehors du chemin de migration LaunchLab n'ont pas automatiquement de frais créateur CPMM.
- Les tokens LP brûlés laissent la liquidité dans le pool de façon permanente, mais aucun portefeuille ne peut réclamer les frais représentés par la part brûlée.
