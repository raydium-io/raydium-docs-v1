Quand un jeton LaunchLab migre vers CPMM, le pool migré génère des jetons LP. La configuration de la plateforme contrôle ce qu'il advient de ces jetons LP.

## La répartition du LP

`migrateCpLockNftScale` possède trois champs. Les valeurs doivent totaliser `1_000_000`, ce qui représente 100 %.

| Champ | Signification |
|---|---|
| `platformScale` | Les jetons LP sont verrouillés via Burn & Earn. La plateforme reçoit le NFT de clé de frais. |
| `creatorScale` | Les jetons LP sont verrouillés via Burn & Earn. Le créateur du jeton reçoit le NFT de clé de frais. |
| `burnScale` | Les jetons LP sont brûlés de façon permanente. Aucun NFT de clé de frais n'est créé pour cette part. |

Ceci est un paramètre au niveau de la plateforme. Les jetons lancés via la même plateforme héritent de la même distribution LP jusqu'à ce que la plateforme mette à jour le paramètre.

## Exemples

```ts
// 40% plateforme, 50% créateur, 10% brûlé.
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% clé de frais du créateur.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% brûlé.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## Relation avec les frais CPMM

`migrateCpLockNftScale` contrôle **qui peut réclamer les droits de frais LP**. Il ne définit pas le taux de frais de swap. Le comportement des frais de swap du pool migré provient de l'`AmmConfig` CPMM sélectionné par `cpConfigId`.

Par exemple, si un pool CPMM migré utilise une config de frais commerciaux de 0,25 % et que `creatorScale` est 500 000, la clé de frais du créateur représente 50 % des droits de frais LP verrouillés. Si `platformScale` est 400 000, la clé de frais de la plateforme représente 40 %.

## Mise à jour de la répartition

Les mises à jour de la configuration de la plateforme sont limitées en débit par époque. Considérez la distribution LP comme une politique importante de la plateforme, non comme un paramètre interface occasionnel.

Avant de mettre à jour :

- Vérifiez que les trois valeurs de scale totalisent `1_000_000`.
- Confirmez que la plateforme sait qui recevra les NFT de clé de frais après les migrations futures.
- Documentez le changement pour les créateurs avant que de nouveaux lancements l'utilisent.
