## A

**AcceleRaytor** — Le launchpad de tokens original de Raydium (2021-2023). Retiré quand LaunchLab l'a remplacé. Voir [`introduction/history-and-milestones`](/fr/introduction/history-and-milestones).

**Account** — Une ligne de données sur Solana, identifiée par une clé publique, possédée par un programme, contenant des octets de données et des lamports. Voir [`solana-fundamentals/account-model`](/fr/solana-fundamentals/account-model).

**Address Lookup Table (ALT)** — Un compte publié permettant aux transactions de référencer les comptes par un index d'1 octet, économisant des octets. Le SDK Raydium utilise les ALT pour les routes multi-sauts. Voir [`solana-fundamentals/transactions-and-fees`](/fr/solana-fundamentals/transactions-and-fees).

**AmmConfig** — Compte de configuration pour CPMM/CLMM définissant le niveau de frais et la part des frais de protocole. Plusieurs configurations peuvent coexister ; les pools en référencent une à la création. Voir [`ray/protocol-fees`](/fr/ray/protocol-fees).

**AMM v4** — Le pool de produits constant-product AMM original de Raydium (2021). Initialement un design hybride qui reflétait la courbe sur OpenBook ; l'intégration avec OpenBook a été désactivée et les pools fonctionnent maintenant comme des AMM constants purs. Toujours entièrement opérationnel ; n'est plus le défaut recommandé pour les nouveaux pools. Voir [`products/amm-v4/overview`](/fr/products/amm-v4/overview).

**Anchor** — Un framework Rust pour les programmes Solana qui gère la validation des comptes, la gestion des erreurs et génère les IDL. Utilisé par CPMM, CLMM, Farm v6, LaunchLab. Voir [`solana-fundamentals/programs-and-anchor`](/fr/solana-fundamentals/programs-and-anchor).

**Anchor discriminator** — Préfixe de 8 octets sur les comptes et instructions gérés par Anchor, dérivé du nom du compte/instruction. Identifie les types de comptes. Voir [`solana-fundamentals/account-model`](/fr/solana-fundamentals/account-model).

**Arbitrage** — Exploiter les différences de prix entre les venues (Raydium ↔ Orca ↔ CEX) pour aligner les prix. Fondamental à la découverte de prix par AMM.

**Associated Token Account (ATA)** — Compte de tokens conventionnel dérivé de PDA pour une paire (utilisateur, mint). Voir [`solana-fundamentals/spl-token-and-token-2022`](/fr/solana-fundamentals/spl-token-and-token-2022).

## B

**Bonding curve** — Une fonction de tarification pour un token qui fixe déterministe le prix en fonction de l'offre. Utilisé dans la pré-graduation LaunchLab. Voir [`products/launchlab/bonding-curve`](/fr/products/launchlab/bonding-curve).

**BPF Loader (Upgradable)** — Le programme chargeur Solana qui déploie et met à niveau les programmes. Voir [`solana-fundamentals/programs-and-anchor`](/fr/solana-fundamentals/programs-and-anchor).

**Bump** — Octet unique ajouté aux seeds pour dériver une PDA hors-courbe. Le bump canonique est la valeur la plus élevée qui produit une adresse hors-courbe. Voir [`solana-fundamentals/pdas-and-cpis`](/fr/solana-fundamentals/pdas-and-cpis).

## C

**Canonical bump** — Le bump trouvé en décrémentant à partir de 255. Les programmes Raydium utilisent toujours le bump canonique. Voir [`solana-fundamentals/pdas-and-cpis`](/fr/solana-fundamentals/pdas-and-cpis).

**CLMM** — Concentrated Liquidity Market Maker, le produit Raydium 2022 de style Uniswap-V3. Voir [`products/clmm/overview`](/fr/products/clmm/overview).

**Compute Budget** — Limite par transaction sur les unités de calcul. Par défaut 200k, max 1,4M. Défini via `ComputeBudget::SetComputeUnitLimit`. Voir [`solana-fundamentals/transactions-and-fees`](/fr/solana-fundamentals/transactions-and-fees).

**Compute Unit (CU)** — Unité de coût de calcul de Solana. Les frais de priorité sont facturés par CU. Voir [`integration-guides/priority-fee-tuning`](/fr/integration-guides/priority-fee-tuning).

**Constant product** — L'invariant AMM `x × y = k`. Voir [`algorithms/constant-product`](/fr/algorithms/constant-product).

**CPI (Cross-Program Invocation)** — Un programme Solana en appelant un autre dans une seule transaction. Voir [`solana-fundamentals/pdas-and-cpis`](/fr/solana-fundamentals/pdas-and-cpis).

**CPMM** — Le pool de produits constant-product AMM moderne de Raydium (2024). Remplace AMM v4 pour les nouveaux pools. Voir [`products/cpmm/overview`](/fr/products/cpmm/overview).

**Creator fee (LaunchLab)** — Pourcentage optionnel des frais de swap acheminés vers le créateur du token pendant la phase de bonding-curve. Voir [`products/launchlab/overview`](/fr/products/launchlab/overview).

## D

**DLMM** — Dynamic Liquidity Market Maker. Variante CLMM de Meteora. Pas Raydium.

**Discriminator** — Voir « Anchor discriminator ».

## E

**Emissions (Farm)** — Taux auquel les récompenses de la farm sont distribuées, en tokens par seconde. Voir [`products/farm-staking/overview`](/fr/products/farm-staking/overview).

**Escrow** — Un compte de tokens détenu par PDA contenant des fonds jusqu'à ce qu'une condition soit remplie. Motif d'intégration courant. Voir [`integration-guides/cpi-integration`](/fr/integration-guides/cpi-integration).

**Executable account** — Compte Solana avec `executable = true`, contenant le bytecode du programme. Voir [`solana-fundamentals/account-model`](/fr/solana-fundamentals/account-model).

## F

**Farm v3 / v5 / v6** — Générations successives du programme de farming des récompenses de Raydium. v6 est actuel ; v3/v5 hérité. Voir [`products/farm-staking/overview`](/fr/products/farm-staking/overview).

**Fee growth (CLMM)** — Compteur global de frais par unité de liquidité, suivi dans `fee_growth_global_X` et `fee_growth_global_Y`. Voir [`products/clmm/math`](/fr/products/clmm/math).

**Fee tier** — Un pourcentage de frais de swap spécifique offert par un pool. Par ex., CLMM 0,05 % tier. Voir [`reference/fee-comparison`](/fr/reference/fee-comparison).

**Finalization** — Le niveau d'engagement le plus solide de Solana, ~13 secondes après l'arrivée de la tx. Voir [`solana-fundamentals/transactions-and-fees`](/fr/solana-fundamentals/transactions-and-fees).

**Freeze authority** — Pouvoir de geler (verrouiller) les comptes de tokens pour un mint donné. Risque pour les utilisateurs LP-ing dans des tokens avec freeze authority actif.

## G

**Graduation (LaunchLab)** — Événement où un lancement de bonding-curve atteint un seuil et migre automatiquement vers un pool CPMM. Voir [`products/launchlab/overview`](/fr/products/launchlab/overview).

## H

**Hybrid AMM** — Un AMM qui s'intègre avec un carnet d'ordres externe pour acheminer les trades via celui qui a les meilleurs prix. AMM v4 était à l'origine hybride (contre OpenBook), mais son côté carnet d'ordres a depuis été désactivé, donc AMM v4 fonctionne maintenant comme un pur AMM. CPMM n'est pas hybride.

## I

**IDL** — Interface Description Language. Description JSON d'Anchor des instructions, comptes, événements et erreurs d'un programme. Voir [`sdk-api/anchor-idl`](/fr/sdk-api/anchor-idl).

**Impermanent loss (IL)** — La différence entre la valeur du token du LP et la possession des tokens sous-jacents, due à la divergence des prix. Fondamental pour les AMM. Voir [`algorithms/constant-product`](/fr/algorithms/constant-product).

**Invoke / invoke_signed** — Appels système Solana pour CPI. `invoke_signed` est utilisé pour les CPI autorisés par PDA. Voir [`solana-fundamentals/pdas-and-cpis`](/fr/solana-fundamentals/pdas-and-cpis).

## J

**Jito** — Moteur de bloc Solana qui prend en charge les transactions groupées pour la protection MEV. Voir [`integration-guides/routing-and-mev`](/fr/integration-guides/routing-and-mev).

**Jupiter** — L'agrégateur DEX Solana dominant. Achemine ~60 % de son volume via Raydium. Voir [`introduction/ecosystem-position`](/fr/introduction/ecosystem-position).

## K

**Kudelski** — Cabinet d'audit qui a audité les programmes Raydium. Voir [`security/audits`](/fr/security/audits).

## L

**Lamport** — La plus petite unité de SOL. 1 SOL = 1 000 000 000 lamports.

**LaunchLab** — La plateforme de lancement de tokens de Raydium (2024). Lancements de bonding-curve qui se graduent en CPMM. Voir [`products/launchlab/overview`](/fr/products/launchlab/overview).

**Liquidity L (CLMM)** — La contribution d'une position à la tarification du pool, dans les unités définies par les mathématiques Uniswap V3. Voir [`algorithms/clmm-math`](/fr/algorithms/clmm-math).

**LP (Liquidity Provider)** — Un utilisateur qui dépose des tokens dans un pool en échange de tokens LP (ou d'un NFT pour CLMM) et d'une réclamation sur les frais futurs.

**LP token** — Token SPL représentant la part d'un LP dans un pool CPMM ou AMM v4. CLMM utilise des NFT à la place.

## M

**MadShield** — Cabinet d'audit qui a audité les programmes Raydium. Voir [`security/audits`](/fr/security/audits).

**Master-chef pattern** — Design de suivi des récompenses basé sur le contrat MasterChef de SushiSwap. Style d'accumulateur `reward_per_share`. Utilisé par Farm v6.

**Memecoin** — Terme informel pour un token piloté par la communauté/spéculation. Beaucoup se lancent via LaunchLab.

**MEV (Maximal Extractable Value)** — Valeur extractible par ordonnancement de transactions (front-running, sandwich, back-running). Voir [`integration-guides/routing-and-mev`](/fr/integration-guides/routing-and-mev).

**Mint** — Un compte Token Mint définissant un token. Ne confondez pas avec « minting » le verbe (émission de nouvelle offre).

**Multisig** — Compte multi-signature nécessitant N-sur-M signataires. Raydium utilise les multisigs Squads pour les autorités. Voir [`security/admin-and-multisig`](/fr/security/admin-and-multisig).

## N

**Non-transferable (Token-2022)** — Extension empêchant les transferts de tokens. Bloqué par la création de pool Raydium.

**NFT** — Non-Fungible Token. Les positions CLMM sont des NFT ; de même certains artefacts LaunchLab.

## O

**Observation / ObservationState** — Le compte d'historique des prix on-chain de CLMM. Stocke le prix cumulatif pondéré par le temps, permettant les oracles TWAP. Voir [`products/clmm/accounts`](/fr/products/clmm/accounts).

**OpenBook** — Anciennement Serum. Le carnet d'ordres limite central de Solana. AMM v4 *s'intégrait à l'origine* avec OpenBook, mais cette intégration a été désactivée ; CPMM et CLMM ne se sont jamais intégrés avec. Voir [`products/amm-v4/overview`](/fr/products/amm-v4/overview).

**Oracle** — Flux de prix externe. Pyth, Jupiter, et (pour usage interne) TWAP ObservationState de CLMM. Voir [`security/oracle-and-token-risks`](/fr/security/oracle-and-token-risks).

**Orca** — DEX Solana rival. Whirlpools est le produit CLMM d'Orca.

## P

**PDA (Program-Derived Address)** — Une adresse dérivée de manière déterministe d'un programme + seeds. Pas de clé privée ; le programme propriétaire signe via `invoke_signed`. Voir [`solana-fundamentals/pdas-and-cpis`](/fr/solana-fundamentals/pdas-and-cpis).

**Permanent delegate (Token-2022)** — Extension accordant à un portefeuille désigné un pouvoir de transfert/brûlage inconditionnel. Bloqué par la création de pool Raydium.

**Phoenix** — DEX carnet d'ordres natif Solana. Pair, pas concurrent direct.

**PnL (AMM v4)** — Historiquement : Profit et Perte des remplissages OpenBook, accumulés dans les pools AMM v4 et réglés via cranks `MonitorStep`. Avec l'intégration OpenBook désactivée, aucun nouveau PnL de carnet d'ordres ne s'accumule — les compteurs `state_data.total_pnl_*` portent des valeurs historiques uniquement. Le chemin de frais de protocole 0,03 % qui utilise `need_take_pnl_*` n'est pas affecté et reste actif.

**Pool authority** — PDA qui signe pour les transferts de vault de pool. Un par programme (pas par pool). Voir [`solana-fundamentals/pdas-and-cpis`](/fr/solana-fundamentals/pdas-and-cpis).

**Pool state** — Compte on-chain contenant les réserves, frais et configuration d'un pool. Structure de données centrale par produit.

**Priority fee** — Frais par CU payés aux validateurs pour prioriser une transaction dans un bloc. Voir [`integration-guides/priority-fee-tuning`](/fr/integration-guides/priority-fee-tuning).

**Program account** — Compte contenant les métadonnées/bytecode d'un programme. Voir [`solana-fundamentals/programs-and-anchor`](/fr/solana-fundamentals/programs-and-anchor).

**Program Derived Address** — Voir PDA.

**Protocol fee** — Portion des frais de swap acheminée vers les rachats RAY ou la trésorerie plutôt que vers les LP. Voir [`ray/protocol-fees`](/fr/ray/protocol-fees).

**Pyth** — Réseau d'oracle Solana. Utilisé par les frontends pour l'affichage USD ; pas par les mathématiques du pool Raydium.

## Q

**Q64.64 (sqrt_price_x64)** — Représentation en virgule fixe : 64 bits entiers + 64 bits fractionnaires. Utilisé par CLMM pour le sqrt-price afin d'éviter la dérive en virgule flottante. Voir [`algorithms/clmm-math`](/fr/algorithms/clmm-math).

## R

**\$RAY** — Le token SPL natif de Raydium. Voir [`ray/index`](/fr/ray).

**Realized fee (CLMM)** — Frais accumulés à une position depuis la dernière `collectFee`. Réclamés via `collectFee`. Voir [`products/clmm/math`](/fr/products/clmm/math).

**Remaining accounts** — Comptes supplémentaires ajoutés à la liste des comptes d'une instruction, interprétés par position. Utilisé pour les listes de comptes de longueur variable (arrays de ticks, extras de transfer-hook). Voir [`solana-fundamentals/pdas-and-cpis`](/fr/solana-fundamentals/pdas-and-cpis).

**Rent** — Solana facture le loyer du stockage, mais tous les comptes modernes doivent être exempt de loyer (contenir suffisamment de lamports pour prépayer le loyer à jamais). Voir [`solana-fundamentals/account-model`](/fr/solana-fundamentals/account-model).

## S

**Sandwich attack** — Tactique MEV : faire une front-run sur le swap d'un utilisateur, puis faire une back-run pour extraire l'impact de prix. Voir [`integration-guides/routing-and-mev`](/fr/integration-guides/routing-and-mev).

**Serum** — Le prédécesseur d'OpenBook. Pas actuellement actif ; voir « OpenBook ».

**Slippage** — Différence entre la production cotée et la production réelle, à partir de l'impact des prix + mouvement des prix. Contrôlé via le paramètre `min_out` sur les swaps.

**SOL** — Le token natif de Solana.

**SPL Token** — Le programme de token Solana legacy. Adresse `Tokenkeg...`. Voir [`solana-fundamentals/spl-token-and-token-2022`](/fr/solana-fundamentals/spl-token-and-token-2022).

**sqrt_price_x64** — Voir Q64.64.

**Squads** — Plateforme multisig que Raydium utilise pour les autorités opérationnelles et de mise à niveau. Voir [`security/admin-and-multisig`](/fr/security/admin-and-multisig).

**Swap / SwapV2** — Variantes d'instructions Raydium. `SwapV2` supporte Token-2022 ; le legacy `Swap` ne le fait pas. Voir [`solana-fundamentals/spl-token-and-token-2022`](/fr/solana-fundamentals/spl-token-and-token-2022).

## T

**Tick (CLMM)** — Niveau de prix discret. Les positions couvrent une plage de ticks. Voir [`algorithms/clmm-math`](/fr/algorithms/clmm-math).

**Tick array** — PDA contenant l'état de 60 ticks consécutifs. Permet le stockage par chunks de la lattice de ticks. Voir [`products/clmm/ticks-and-positions`](/fr/products/clmm/ticks-and-positions).

**Tick spacing** — L'écart entier entre les ticks utilisables dans un pool. Lié au niveau de frais (0,01 % → 1, 0,25 % → 60, etc.). Voir [`algorithms/clmm-math`](/fr/algorithms/clmm-math).

**Timelock** — Délai entre l'approbation et l'exécution d'une action sensible. Les mises à niveau Raydium ont un délai d'attente de 24 heures. Voir [`security/admin-and-multisig`](/fr/security/admin-and-multisig).

**Token-2022** — Le programme de token plus récent avec extensions. Adresse `TokenzQ...`. Voir [`solana-fundamentals/spl-token-and-token-2022`](/fr/solana-fundamentals/spl-token-and-token-2022).

**Transfer fee (Token-2022)** — Extension facturant un pourcentage sur chaque transfert de token. Comptabilisé dans CPMM/CLMM Raydium via `SwapV2`.

**Transfer hook (Token-2022)** — Extension invoquant un programme sur chaque transfert. Supporté par Raydium avec des réserves.

**TVL (Total Value Locked)** — Somme des soldes pool/farm en USD. TVL Raydium en avril 2026 : ~1,8 B\$.

**TWAP (Time-Weighted Average Price)** — Prix moyen pondéré par le temps, dérivé de CLMM ObservationState. Voir [`products/clmm/accounts`](/fr/products/clmm/accounts).

## U

**UserLedger (Farm v6)** — PDA stockant le solde de mise et la dette de récompense d'un utilisateur dans une farm.

## V

**Vault** — Un Token Account possédé par l'autorité PDA d'un programme de pool, contenant les réserves du pool. Chaque pool a deux vaults (un par côté).

**Versioned transaction (v0)** — Format de transaction Solana actuel supportant les ALT. Les transactions legacy sont en version 0xff. Voir [`solana-fundamentals/transactions-and-fees`](/fr/solana-fundamentals/transactions-and-fees).

## W

**Whirlpools** — Le produit CLMM d'Orca. Design similaire à Raydium CLMM ; codebase différente.

**Wrapped SOL (wSOL)** — SOL enrobé en tant que Token SPL pour une utilisation dans les contextes de comptes tokens uniquement. Créé et fermé par transaction. Voir [`user-flows/swap`](/fr/user-flows/swap).

## X

**x × y = k** — L'invariant constant-product. Voir [`algorithms/constant-product`](/fr/algorithms/constant-product).
