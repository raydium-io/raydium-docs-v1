L'échange est l'opération utilisateur la plus courante. Cette page vous guide à travers le flux de l'interface Raydium, explique les paramètres que vous pouvez contrôler et répertorie les modes de défaillance que vous risquez de rencontrer.

## Entrées

Chaque échange nécessite :

- **Token entrant** (adresse mint + montant)
- **Token sortant** (adresse mint)
- **Tolérance de slippage** (0,5% par défaut dans l'interface ; 1% sur les paires volatiles)
- **Frais prioritaires** (par défaut redimensionnés automatiquement)

Le montant est spécifié soit en « exact-in » (vous définissez ce que vous payez ; la sortie est calculée), soit en « exact-out » (vous définissez ce que vous recevez ; l'entrée est calculée). L'interface utilise par défaut exact-in.

## Guide de l'interface

Sur [raydium.io/swap](https://raydium.io/swap) :

1. **Entrez les montants.** Choisissez les tokens d'entrée et de sortie ; tapez un montant dans l'une ou l'autre des cases. L'autre se remplit automatiquement en utilisant la cotation actuelle.

2. **Examinez l'itinéraire.** Le panneau d'itinéraire montre quel(s) pool(s) Raydium a sélectionné :
   - Pool unique (le plus courant) : « SOL → USDC via CLMM 0,05% »
   - Sauts multiples : « SOL → USDC via CPMM, puis USDC → RAY via CLMM »
   - Fractionné : « SOL → USDC via 60% CLMM 0,05%, 40% CPMM 0,25% »

   Pour chaque saut : la TVL, les frais et l'impact attendu. Les transactions plus importantes affichent un avertissement bien visible quand l'impact dépasse 1%.

3. **Vérifiez le slippage.** Le défaut est 0,5%. Pour les paires stables, c'est 0,1% ; pour les meme tokens 2–5%. Si la cotation change pendant que vous la regardez (les prix fluctuent), l'interface renouvelle la cotation toutes les 10 secondes et clignote.

4. **Définissez les frais prioritaires.** Automatique par défaut ; trois niveaux :
   - **Normal** (50e percentile) — par défaut
   - **Rapide** (75e percentile)
   - **Turbo** (95e percentile)
   Les utilisateurs avancés peuvent entrer un nombre spécifique de micro-lamports/CU.

5. **Confirmez et signez.** La transaction est envoyée ; l'interface assure le suivi de la confirmation. Temps attendu 5–15 secondes en congestion normale.

6. **Après l'échange.** L'interface affiche le montant réellement reçu (peut différer de la cotation d'environ le slippage) ; cliquez pour accéder à l'explorateur de transactions pour plus de détails.

## Comment le routage sélectionne un pool

Quand plusieurs pools existent pour la même paire, le routeur de Raydium choisit par :

1. **`amountOut` le plus élevé après frais.** Calculez la cotation pour chaque pool admissible ; choisissez celui qui donne le plus de sortie.
2. **Départage par TVL.** Une TVL plus élevée signifie un impact plus faible sur le prochain échange — résistant aux fluctuations de prix entre la cotation et la soumission.
3. **Préférer pool unique aux sauts multiples.** Les sauts multiples composent les frais et le slippage ; le routeur ne choisit les sauts multiples que s'ils surpassent le pool unique de plus de 5 bps.
4. **Fractionner si volumineux.** Pour les échanges volumineux par rapport à la TVL du pool, l'itinéraire peut se diviser entre les pools pour réduire l'impact des prix.

Produits comparés sur les paires typiques :

- **Pools CPMM** dominent à faible TVL (`<1M $`) car ils sont bon marché à créer et les frais d'échange financent directement les LP.
- **Pools CLMM** dominent à TVL moyen/élevé pour les paires volatiles — la liquidité concentrée signifie de meilleures cotations dans la gamme de prix commune.
- **Pools AMM v4** dominent sur les paires établies de longue date avec intégration OpenBook profonde (SOL-USDC, RAY-USDC). La plupart des nouvelles paires ne touchent jamais AMM v4.

## Frais de transfert Token-2022

Si le mint d'entrée ou de sortie est Token-2022 avec frais de transfert :

- L'`amountOut` de la cotation est le montant net reçu par l'utilisateur après frais.
- Les « frais » affichés dans l'interface se divisent en « Frais de pool » (frais LP) et « Frais de transfert » (payés à l'émetteur du mint).

Voir [`algorithms/token-2022-transfer-fees`](/fr/algorithms/token-2022-transfer-fees).

## Slippage

Le slippage protège contre les mouvements de prix défavorables entre la cotation et l'exécution. Raydium convertit votre pourcentage de slippage en montant minimum que vous devez recevoir :

```
minimumAmountOut = expectedAmountOut × (1 - slippage)
```

Si la sortie réelle était inférieure, la tx revient avec `ExceededSlippage`. Valeurs par défaut courantes :

| Type de paire | Slippage |
|-----------|----------|
| Stable-stable | 0,01–0,1% |
| Majeur-majeur (SOL/USDC) | 0,1–0,5% |
| Volatiles mid-cap | 0,5–1% |
| Meme tokens | 2–5% |
| Token-2022 avec frais de transfert | ajouter frais de transfert au slippage |

Un slippage trop serré provoque des annulations fréquentes lors du mouvement normal des prix. Un slippage trop lâche invite les attaques sandwich (voir [`integration-guides/routing-and-mev`](/fr/integration-guides/routing-and-mev)).

## Frais prioritaires

Les frais prioritaires paient les validateurs plus cher pour traiter votre transaction pendant la congestion. L'interface utilise par défaut un paramètre automatique. Utilisez **Rapide** ou **Turbo** quand le réseau est encombré, quand vous échangez une paire volatile, ou quand une transaction précédente a expiré avant confirmation.

Les frais prioritaires plus élevés n'améliorent pas le prix coté. Ils n'améliorent que la probabilité que votre transaction se termine avant que la cotation ne devienne obsolète.

## Modes de défaillance courants

### Tx revient avec `ExceededSlippage`

Le prix a bougé plus que votre tolérance entre la cotation et l'exécution. Solutions :

1. N'augmentez le slippage que si nécessaire.
2. Réduisez la taille de l'échange (moins d'impact).
3. Re-cotez et recommencez ; le prix peut avoir été passager.
4. Évitez un slippage très élevé sur les paires majeures, car cela peut vous exposer à une exécution pire.

### Tx expire

Frais prioritaires trop bas pour la congestion actuelle. Solution : augmentez les frais prioritaires ; voir `integration-guides/priority-fee-tuning`.

### `InvalidAccountData` sur échange CLMM

L'itinéraire peut dépendre d'un compte CLMM qui n'a pas pu se charger ou qui a changé pendant que la cotation était ouverte. Actualisez la page, re-cotez et recommencez. Si cela persiste, choisissez un autre itinéraire si l'interface en propose un.

### « Pool non trouvé » / données de pool obsolètes

Le pool a été créé récemment et n'a pas encore été indexé. Relancez par ID directement :
Attendez quelques minutes, actualisez l'application et recherchez à nouveau le pool ou le mint du token. Les pools récemment créés peuvent mettre un peu de temps à apparaître dans la recherche d'interface indexée.

### La sortie est nulle / très petite

La taille de l'échange dépasse la liquidité du pool. Le pool est trop peu profond ; réduisez la taille de l'échange, essayez un autre itinéraire ou attendez plus de liquidité.

### Tx se conclut mais le solde n'a pas changé

Actualisez vos soldes de portefeuille et Raydium. Si la transaction a réussi sur l'explorateur, vérifiez si le compte du token de sortie est masqué dans votre interface de portefeuille.

## Vérification

Utilisez le lien de transaction affiché par Raydium après confirmation. Sur Solscan ou SolanaFM, vérifiez :

- **Statut :** Succès.
- **Signataire :** l'adresse de votre portefeuille.
- **Changements de solde des tokens :** token d'entrée diminué, token de sortie augmenté.
- **Programmes invoqués :** un programme d'échange Raydium plus les programmes de tokens/système.
