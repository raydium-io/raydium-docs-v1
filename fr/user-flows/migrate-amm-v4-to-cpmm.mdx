<Info>
  AMM v4 est entièrement opérationnel et aucune date de fermeture n'a été annoncée. La migration est un choix, non une obligation. CPMM offre un meilleur partage des frais LP par défaut, des coûts de création plus faibles, le support Token-2022, et n'a pas de dépendance OpenBook.
</Info>

## Pourquoi migrer

| Dimension | AMM v4 | CPMM |
|-----------|--------|------|
| Destination des frais | Protocole + LP partagés | Les LP reçoivent la part swap-fee par défaut |
| Dépendance OpenBook | Oui | Aucune |
| Coût de création | Plus élevé | Plus faible |
| Support Token-2022 | Non | Oui |
| Destination pour les nouvelles cotations | Rare | Par défaut |

Si votre paire a un volume élevé, l'augmentation des frais LP peut justifier la migration. Si l'ancien pool a encore une bonne liquidité et le projet n'a pas endorsé un pool CPMM spécifique, la migration peut ne pas valoir le coût de coordination.

## Checklist pré-migration

Pour les opérateurs de pool et trésoreries de projets :

- [ ] Annoncez le calendrier de migration aux LP.
- [ ] Accordez aux LP au moins 48 heures si possible.
- [ ] Confirmez le pool CPMM de destination et son tier de frais.
- [ ] Préparez les annonces post-migration avec le nouvel ID de pool et le mint LP.
- [ ] Décidez si une nouvelle ferme est nécessaire.

Pour les LP individuels :

- [ ] Confirmez que vous suivez la migration endorsée par le projet.
- [ ] Gardez assez de SOL pour plusieurs transactions.
- [ ] Notez votre solde de LP AMM v4 et les montants sous-jacents attendus.
- [ ] Vérifiez si vos tokens LP AMM v4 sont en staking dans une ferme.

## Migration UI : LP individuel

La migration se fait en deux actions : retirer la liquidité de AMM v4, puis l'ajouter à CPMM.

### Étape 1 : retirer de AMM v4

1. Ouvrez le pool AMM v4 ou votre position de portefeuille.
2. Si vos tokens LP sont en staking dans une ferme, déstakez-les d'abord.
3. Cliquez sur **Remove liquidity**.
4. Choisissez le pourcentage à retirer.
5. Examinez les montants de tokens que vous recevrez.
6. Confirmez et approuvez.

### Étape 2 : déposer dans CPMM

1. Ouvrez le pool CPMM de destination.
2. Cliquez sur **Add liquidity**.
3. Entrez le montant d'un token.
4. Examinez le montant du token associé et tout solde restant.
5. Confirmez et approuvez.

Entre le retrait et le dépôt, les prix du pool peuvent se déplacer. Si les deux pools ont des ratios différents, certains tokens peuvent rester comme poussière dans votre portefeuille.

## Migration UI : trésorerie de projet

Pour une trésorerie de projet, le processus est le même mais nécessite une coordination :

1. **Annoncez** la migration à l'avance.
2. **Initialisez le pool CPMM** avant de demander aux utilisateurs de migrer.
3. **Migrez d'abord la liquidité de la trésorerie** et publiez les liens de transaction.
4. **Créez une nouvelle ferme** si les incitations doivent continuer sur le mint LP CPMM.
5. **Surveillez les migrations des LP individuels** au cours des jours ou semaines suivants.
6. **Dépréciez l'ancien pool** seulement après avoir donné aux utilisateurs assez de temps pour se déplacer.

## Vérification

### Après retrait

- Le solde du token LP AMM v4 devrait diminuer ou atteindre zéro.
- Les deux tokens sous-jacents devraient apparaître dans votre portefeuille.
- La transaction devrait afficher **Success** sur un explorateur Solana.

### Après dépôt

- Le solde du token LP CPMM devrait apparaître dans votre portefeuille.
- Le portefeuille de Raydium devrait afficher la nouvelle position.
- La page du pool CPMM devrait afficher votre part de liquidité.

### Preuve de migration

Pour la transparence, publiez les deux liens de transaction :

- Retrait de AMM v4.
- Dépôt dans CPMM.

## Considérations de retour en arrière

Revenir de CPMM à AMM v4 signifie retirer de CPMM et déposer dans AMM v4. C'est généralement peu intéressant sauf s'il y a une raison produit spécifique. La création de pool AMM v4 est plus chère et nécessite la configuration du marché OpenBook.

## Préoccupations courantes

### Mon LP AMM v4 est en staking dans une ferme. Puis-je migrer ?

Oui. Déstakez d'abord, retirez le LP de AMM v4, déposez dans CPMM, puis stakez les nouveaux tokens LP CPMM s'il existe une nouvelle ferme.

### Le pool CPMM n'existe pas encore

La plupart des LP doivent attendre que le projet crée et endorsse un pool de destination. En créer un vous-même signifie que vous prenez le risque du prix initial et de la liquidité.

### J'ai une position importante. Qu'en est-il du slippage ?

Retirez et déposez par tranches plus petites si votre position est importante par rapport à la TVL du pool. Examinez chaque devis et évitez de signer si les montants reçus sont notablement pires que prévu.

### Qu'advient-il des tokens LP que j'oublie de migrer ?

Ils restent dans l'ancien pool et continuent à générer des revenus selon les règles d'AMM v4. Rien ne force la migration.

## Dépannage

### Le compte du token LP de destination est nouveau

Le premier dépôt CPMM peut avoir besoin de créer un nouveau compte de token LP. Gardez du SOL supplémentaire dans votre portefeuille pour le loyer et les frais.

### La deuxième transaction expire

Si le retrait réussit mais que le dépôt expire, vos tokens restent dans votre portefeuille. Réouvrez le pool CPMM et réessayez avec un paramètre de priority-fee plus élevé.

### Mauvais pool CPMM

Plusieurs pools CPMM peuvent exister pour la même paire. Utilisez le pool endorsé par le projet ou le pool avec le tier de frais et la liquidité prévus.

### Implications fiscales

Dans certaines juridictions, le retrait de LP peut être un événement imposable. Consultez les règles locales avant de migrer des positions importantes.
