## Ce dont vous avez besoin

- Un portefeuille Solana contenant :
  - Assez de chaque mint pour alimenter la liquidité initiale.
  - Environ 0,2 SOL pour le loyer de création du pool, la création de compte token et les frais de priorité.
- Les deux adresses de mint de token.
- Un palier de frais. Le palier par défaut de 0,25 % convient à la plupart des paires volatiles.

## Présentation de l'interface

Sur [raydium.io/liquidity/create-pool](https://raydium.io/liquidity/create-pool) :

1. **Sélectionnez le type de pool.** Choisissez **Standard AMM (CPMM)**. N'utilisez CLMM que si vous comprenez la gestion des plages.
2. **Entrez les deux tokens.** Collez les adresses de mint ou recherchez par symbole. Pour les mints Token-2022, examinez les avertissements.
3. **Définissez le palier de frais.** Les options courantes sont 0,01 %, 0,25 % et 1 %. Les frais plus élevés conviennent aux paires plus volatiles ou de queue longue.
4. **Définissez le prix initial et le dépôt.** Entrez d'un côté et le prix initial. L'interface calcule l'autre côté.
5. **Vérifiez.** Vérifiez les adresses de mint, le palier de frais, le prix initial, les tokens LP, la part du pool et le loyer unique.
6. **Confirmez et signez.** Votre portefeuille signe la transaction de création du pool.
7. **Attendez l'indexation.** Le pool est actif on-chain immédiatement, mais l'interface Raydium peut prendre quelques minutes pour l'afficher dans la recherche.

## Choisir le prix initial

Le premier prix du pool provient de votre ratio de dépôt initial. Si vous déposez 10 SOL et 1000 USDC, le prix initial de SOL est 100 USDC.

Choisissez un prix proche du marché. Si le prix est trop élevé ou trop bas, les traders arbitragers rééquilibreront le pool et vous paierez cette différence à partir de votre inventaire initial.

## Liste de vérification post-création

1. **Ouvrez la page du pool.** Confirmez la paire, le palier de frais et l'adresse du pool.
2. **Vérifiez les tokens LP.** Votre portefeuille ou le portefeuille Raydium devrait afficher le nouveau token LP.
3. **Effectuez un petit swap de test.** Utilisez un petit montant pour confirmer que le pool s'achemine comme prévu.
4. **Attendez la découverte.** Si le pool n'apparaît pas immédiatement dans la recherche, actualisez après quelques minutes et recherchez par mint ou ID du pool.

## Considérations Token-2022

Si l'un des mints est Token-2022 avec des frais de transfert, le coût utilisateur effectif est le frais du pool plus le frais de transfert du mint. L'interface affiche les avertissements Token-2022 avant de signer.

Si l'un des mints possède un hook de transfert, les swaps dépendent de ce programme de hook. Si le programme de hook change ou bloque les transferts, le pool peut devenir difficile à utiliser. Traitez les mints avec hook de transfert inconnu comme un risque plus élevé.

Les mints non-transférables ne peuvent pas être utilisés dans les pools CPMM.

## Dépannage

### Prix initial incorrect

Si vous détectez l'erreur avant de signer, revenez en arrière et corrigez le ratio de dépôt. Si vous avez déjà créé le pool, vous devez généralement ajouter de la liquidité au prix corrigé, laisser l'arbitrage le stabiliser ou créer un nouveau pool.

### Frais de priorité trop bas

La création du pool peut expirer lors de la congestion. Réessayez avec un réglage de frais de priorité plus élevé dans le portefeuille ou l'interface Raydium.

### La liquidité initiale est trop petite

Les très petits pools produisent un mauvais impact de prix et une mauvaise expérience utilisateur. Ajoutez plus de liquidité avant de partager le pool publiquement.

### Pools en doublon

Plusieurs pools CPMM peuvent exister pour la même paire de tokens à différents paliers de frais. Avant d'en créer un, recherchez sur Raydium par les deux mints de tokens et vérifiez si le projet approuve déjà un pool.

### Mints autorisés

Un mint avec une autorité de gel peut geler les comptes de tokens. Pour les mints inconnus, vérifiez l'autorité de gel et les contrôles du projet avant de fournir une liquidité significative.
