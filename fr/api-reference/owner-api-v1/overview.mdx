L'API Owner fournit une vue par portefeuille des positions et des soldes de Raydium. Elle récupère des données mises en cache pour les positions, les liquidités verrouillées, les pools créés et la participation aux IDO, ce qui la rend idéale pour les interfaces de portefeuille et les tableaux de bord de portefeuille.

## Serveurs

| Environnement | Serveur                      |
|---------------|------------------------------|
| Mainnet       | `owner-v1.raydium.io`        |
| Devnet        | `owner-v1-devnet.raydium.io` |

## Aperçu

L'API expose cinq points de terminaison principaux organisés par type de ressource :

- **`/position/stake/{owner}`** — Positions de jalonnement (AMM standard / LP CPMM)
- **`/position/clmm-lock/{owner}`** — Positions CLMM verrouillées
- **`/create-pool/{owner}`** — Pools créés par ce portefeuille
- **`/main/ido/{owner}`** — Données de participation aux IDO

## Fraîcheur des données

Ce service met en cache les données de position et les met à jour périodiquement. Les changements très récents de position (en quelques secondes) peuvent ne pas apparaître immédiatement dans les réponses.

## Modèles courants

### Récupérer toutes les positions de jalonnement pour un portefeuille

```bash
curl -s "https://owner-v1.raydium.io/position/stake/{owner}"
```

Remplacez `{owner}` par la clé publique du portefeuille au format base58.

### Récupérer les positions verrouillées

```bash
curl -s "https://owner-v1.raydium.io/position/clmm-lock/{owner}"
```

### Vérifier la participation aux IDO

```bash
curl -s "https://owner-v1.raydium.io/main/ido/{owner}"
```

## Résultats vides

Si un portefeuille n'a aucune donnée pour un point de terminaison particulier, le service retourne une erreur 404 avec les métadonnées de la requête :

```json
{
  "id": "...",
  "success": false
}
```
