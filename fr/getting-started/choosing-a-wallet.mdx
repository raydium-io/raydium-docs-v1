## Ce qu'un portefeuille compatible Raydium doit faire

Un portefeuille est compatible Raydium s'il peut faire tout ce qui suit :

- Conserver une paire de clés Solana localement et **jamais** transmettre la clé privée hors du dispositif.
- **Signer les transactions versionnées (V0).** Les swaps Raydium utilisent les Address Lookup Tables et nécessitent la prise en charge de V0. Les portefeuilles qui ne signent que les transactions legacy ne peuvent pas exécuter un swap Raydium.
- **Se connecter aux dApps via la norme Wallet Adapter.** C'est ce qui rend possible le bouton « Connecter le portefeuille » sur raydium.io.
- **Afficher un aperçu clair de la transaction** avant que vous l'approuviez, y compris le(s) programme(s) appelé(s) et les mouvements de jetons.
- **Prendre en charge les comptes de jetons associés** (créer une ATA la première fois que vous recevez un nouveau mint).

Ce sont les exigences minimales. Tout portefeuille Solana moderne les remplit. Tout ce qui ne les remplit pas est dangereux pour signer les transactions Raydium — point final.

## Fonctionnalités qui comptent au-delà du minimum

Une fois qu'un portefeuille satisfait aux exigences minimales, les différences dépendent de votre flux de travail et de votre profil de risque. Décidez laquelle des fonctionnalités suivantes vous importe, et choisissez un portefeuille qui les gère :

| Fonctionnalité | Quand c'est important |
|---|---|
| **Appairage avec portefeuille matériel** | Vous détenez une montant plus que casual. Le portefeuille logiciel devient un constructeur de transactions ; le dispositif matériel conserve la clé et signe. La plupart des portefeuilles Solana réputés s'appairent au moins avec Ledger ; certains s'appairent avec Keystone, SafePal ou d'autres dispositifs. |
| **Navigateur dApp mobile** | Vous prévoyez d'utiliser Raydium principalement depuis un téléphone. Le navigateur intégré du portefeuille est ce qui charge raydium.io et signe les transactions sur mobile. |
| **Gestion de plusieurs comptes** | Vous séparez les fonds entre portefeuilles chauds / froids / expérimentaux et souhaitez basculer rapidement sans jongler avec les extensions. |
| **Sensibilisation à l'extension Token-2022** | Vous interagirez avec des pools qui utilisent les mints Token-2022 (p. ex. jetons avec frais de transfert). Un portefeuille qui affiche ces extensions dans l'interface de signature vous aide à repérer les frais inattendus avant de signer. |
| **Affichage des frais prioritaires** | Vous tradez pendant la congestion et souhaitez voir ce que le portefeuille a défini pour le prix du budget de calcul avant d'approuver. |
| **Prise en charge de Solana Actions / Blinks** | Vous souhaitez utiliser les liens de swap en un clic que Raydium publie sur les réseaux sociaux. Voir [`user-flows/referrals-and-blinks`](/fr/user-flows/referrals-and-blinks). |
| **Base de code open-source** | Vous souhaitez pouvoir auditer le client du portefeuille (ou vous appuyer sur la communauté pour l'avoir fait). |
| **Comptes multi-chaînes** | Vous détenez des actifs sur des chaînes autres que Solana et préférez un portefeuille pour tous. Vérifiez que la prise en charge multi-chaînes est mature pour les chaînes que vous utilisez réellement ; elle varie. |

## Portefeuilles Solana populaires

Les éléments suivants sont des projets de portefeuille bien connus avec un support Solana actif. Ils sont listés alphabétiquement ; **l'inclusion est informationnelle, non pas une approbation**, et la liste n'est pas exhaustive. Vérifiez l'URL officielle avant d'installer — les clones de phishing achètent régulièrement des annonces sponsorisées contre les noms de domaine réels.

### Portefeuilles logiciels

| Portefeuille | Où il s'exécute | Site officiel |
|---|---|---|
| Phantom | Extension navigateur, iOS, Android | [phantom.app](https://phantom.app) |
| Solflare | Extension navigateur, iOS, Android, web | [solflare.com](https://solflare.com) |
| OKX Wallet | Extension navigateur, iOS, Android | [okx.com/wallet](https://www.okx.com/wallet) |
| Backpack | Extension navigateur, iOS, Android, bureau | [backpack.app](https://backpack.app) |

Les quatre sont non-custodial, prennent en charge Wallet Adapter, signent les transactions V0 et s'appairent avec au moins un dispositif matériel courant. Au-delà de cela, les différences sont spécifiques au flux de travail — utilisez les critères de la section précédente pour décider lequel vous convient.

### Portefeuilles matériels

Un portefeuille matériel s'appaire avec l'un des portefeuilles logiciels ci-dessus (le côté logiciel devient le constructeur de transactions ; le dispositif matériel conserve la clé et signe).

| Dispositif | Type | Site officiel |
|---|---|---|
| Ledger | USB / Bluetooth | [ledger.com](https://www.ledger.com) |
| Keystone | Air-gapped (signature par code QR) | [keyst.one](https://keyst.one) |

Chaque portefeuille logiciel documente les dispositifs matériels qu'il prend en charge ; vérifiez avant d'acheter.

## Modèle de portefeuille chaud / froid

Une pratique courante qui fonctionne indépendamment du portefeuille que vous choisissez :

- Un **portefeuille chaud** conserve une petite montant que vous acceptez pourrait être perdue — pour les swaps quotidiens et essayer les choses.
- Un **portefeuille froid** (associé à un dispositif matériel) conserve tout ce que vous seriez malheureux de perdre. Déplacez les fonds que lorsque vous en avez activement besoin dans le portefeuille chaud.

Raydium traite chaque portefeuille comme complètement indépendant : positions séparées, jetons LP séparés, récompenses de ferme séparées, historique séparé. Le changement de portefeuille est un clic.

## Où trouver des options de portefeuille

La modal « Connecter le portefeuille » de l'interface Raydium répertorie tous les portefeuilles qui se sont intégrés à Raydium via Wallet Adapter. Parcourez-la pour voir ce qui est actuellement pris en charge. Le répertoire officiel des portefeuilles Solana ([`solana.com/ecosystem/explore?categories=wallet`](https://solana.com/ecosystem/explore?categories=wallet)) est un autre point de départ neutre.

Lorsque vous évaluez un portefeuille, passez en revue cette liste de contrôle :

- [ ] Maintenu : versions récentes, un suivi des problèmes actif, un canal de divulgation de sécurité.
- [ ] Examiné : un audit publié par une entreprise réputée, ou code open-source avec une communauté active.
- [ ] Prise en charge des transactions V0 confirmée (testez sur devnet avant de transférer des fonds réels).
- [ ] Si vous prévoyez d'utiliser un dispositif matériel : le portefeuille documente les dispositifs qu'il prend en charge et comment l'appairage fonctionne.
- [ ] Vous avez trouvé le téléchargement officiel via le domaine vérifié du projet — **pas** via une annonce sponsorisée ou un lien redirigé.

## Principes fondamentaux de la sécurité des portefeuilles

Quel que soit le portefeuille que vous choisissez :

- **Écrivez la phrase de graine sur papier.** Stockez le papier dans deux emplacements physiquement séparés. **Ne jamais** taper la phrase de graine nulle part sauf lors d'une récupération de portefeuille sur une installation fraîche.
- **Utilisez un gestionnaire de mots de passe** pour le mot de passe de déverrouillage du portefeuille — et gardez ce mot de passe complètement séparé de la graine.
- **Utilisez un profil de navigateur dédié** pour l'activité crypto, avec le nombre minimum d'extensions installées. Les extensions de phishing sont le plus grand vecteur de drainage.
- **Traitez chaque invite de connexion au portefeuille avec suspicion.** Lisez ce que vous signez. Si une transaction transfère la totalité d'un jeton au lieu d'un montant spécifique, rejetez-la.
- **Mettez raydium.io en signet.** Naviguez toujours à partir du signet, jamais à partir d'une annonce de moteur de recherche.

Plus de conseils de sécurité se trouvent dans [`getting-started/trust-and-safety`](/fr/getting-started/trust-and-safety).
