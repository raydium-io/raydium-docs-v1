## Rilis

<Update label="2026-06-22" description="Stable AMM: hapus kode OpenBook (market) yang tidak aktif">
  Stable AMM menghapus akun dan kode market-making OpenBook yang telah lama tidak digunakan. Layout yang lebih kecil untuk `SwapBaseIn` / `SwapBaseOut` (18 → 9), `Deposit` (14 → 12), dan `Withdraw` (21/22 → 12) (layout lama masih kompatibel); perubahan `WithdrawPnl` yang **hard-breaking** (16 → 10, tanpa kompatibilitas); biaya referral dihapus; dan formula aset pool yang disederhanakan hanya untuk vault. Sebagian besar instruksi Stable lainnya tidak lagi dapat dipanggil.

  [Baca entri lengkapnya →](/id/reference/changelog/2026-06-22-stable-amm-openbook-cleanup)
</Update>

<Update label="2026-05-18" description="CLMM: limit orders, single-sided fee, dynamic fee">
  Tiga kemampuan CLMM yang opt-in dan kompatibel ke belakang: limit orders kelas satu (dengan `limit_order_admin` settle keeper), pengumpulan biaya single-sided (`CollectFeeOn`), dan dynamic fee yang melacak volatilitas. Menambahkan `CreateCustomizablePool`, reshape `PoolState` (**perubahan breaking untuk indexer**), field `TickState` baru, sebelas kode error baru (dengan pergeseran numerik), dan penambahan SDK / API yang sesuai.

  [Baca entri lengkapnya →](/id/reference/changelog/2026-05-18-clmm-limit-orders)
</Update>

## Konvensi dokumentasi

- **Versioning**: dokumentasi ini menggunakan versioning berbasis kalender (YYYY-MM-DD). Setiap pembaruan menambahkan halaman entri baru dan baris baru di bagian atas garis waktu di atas.
- **Satu halaman per rilis**: setiap ringkasan rilis berada di halaman tersendiri di bawah `reference/changelog/`, sehingga indeks ini tetap singkat dan setiap entri dapat ditautkan secara independen.
- **Tanggal verifikasi**: setiap entri mencatat kapan konten terakhir kali diperiksa silang terhadap status on-chain / API dan sumber program. Jika tidak dinyatakan, asumsikan tanggal utama entri.
- **Breaking changes**: ditandai dalam peringatan kotak pada halaman yang terpengaruh dan diberi tag dalam entri.
- **Cakupan**: changelog ini mencakup dokumentasi itu sendiri. Garis waktu historis protokol itu sendiri berada di [`introduction/history-and-milestones`](/id/introduction/history-and-milestones) dan merupakan sumber kebenaran untuk "kapan X terjadi di Raydium".
