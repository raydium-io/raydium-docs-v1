## Alur sekilas

```text
Biaya pool
  -> Alamat pengumpulan khusus program
      -> Rute buyback RAY
      -> Rute treasury USDC, jika berlaku

Biaya pembuatan pool
  -> Alamat biaya SOL khusus program
```

Setiap alamat dalam alur ini bersifat publik dan dapat diperiksa.

## Alamat pengumpulan biaya

Ketika swap membayar biaya sisi protokol, program yang relevan mengumpulkan biaya tersebut di alamat pengumpulan.

| Sumber | Alamat pengumpulan |
|---|---|
| CLMM | `projjosVCPQH49d5em7VYS7fJZzaqKixqKtus7yk416` |
| CPMM | `ProCXqRcXJjoUd1RNoo28bSizAA6EEqt9wURZYPDc5u` |
| Standard AMM v4 | `PNLCQcVCD26aC7ZWgRyr5ptfaR7bBrWdTFgRWwu2tvF` |

Alamat pengumpulan dapat menyimpan kumpulan token SPL yang bercampur. Misalnya, biaya dari pool SOL/USDC dapat menghasilkan saldo wSOL dan USDC, sedangkan pool SOL/RAY dapat menghasilkan saldo wSOL dan RAY.

Periksa alamat pengumpulan dengan:

```bash
spl-token accounts --owner projjosVCPQH49d5em7VYS7fJZzaqKixqKtus7yk416
```

## Cara mengaudit alur biaya

Siapa pun dapat memverifikasi alur biaya Raydium dari data transaksi publik:

1. Tarik transaksi swap dari pool Raydium.
2. Identifikasi program pool: CLMM, CPMM, atau Standard AMM v4.
3. Periksa akuntansi biaya dan transfer token dalam transaksi.
4. Cocokkan pergerakan sisi protokol ke alamat pengumpulan yang relevan.
5. Lacak transaksi keluar dari alamat pengumpulan.
6. Cocokkan akumulasi RAY ke `DdHDoz...VEZaz` atau saldo USDC ke alamat treasury di atas.
