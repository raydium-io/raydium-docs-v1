## Setup

Pastikan Anda telah membaca [Prasyarat Quick start](/id/quick-start) dan memiliki `RPC_URL`, `KEYPAIR`, serta dependensi yang terinstal.

Anda juga perlu **mendanai dompet dengan jumlah awal** dari kedua mint, ditambah cukup SOL untuk menutupi biaya pembuatan pool sekali jalan (~0,15 SOL di mainnet, lihat [`reference/program-addresses`](/id/reference/program-addresses) untuk nilai terbaru).

## Skrip

Simpan sebagai `create-cpmm.mjs`:

```ts
// create-cpmm.mjs
import { Connection, Keypair, PublicKey } from "@solana/web3.js";
import {
  Raydium,
  TxVersion,
  CREATE_CPMM_POOL_PROGRAM,
  CREATE_CPMM_POOL_FEE_ACC,
} from "@raydium-io/raydium-sdk-v2";
import BN from "bn.js";
import fs from "node:fs";

// ── Config from env ──────────────────────────────────────────────
const RPC_URL  = process.env.RPC_URL  ?? "https://api.mainnet-beta.solana.com";
const KEYPAIR  = process.env.KEYPAIR  ?? `${process.env.HOME}/.config/solana/id.json`;
const MINT_A   = process.env.MINT_A;   // required, base58
const MINT_B   = process.env.MINT_B;   // required, base58
const AMOUNT_A = process.env.AMOUNT_A; // required, integer raw units
const AMOUNT_B = process.env.AMOUNT_B; // required, integer raw units

if (!MINT_A || !MINT_B || !AMOUNT_A || !AMOUNT_B) {
  console.error("Set MINT_A, MINT_B, AMOUNT_A, AMOUNT_B env vars.");
  process.exit(1);
}

// ── Setup ────────────────────────────────────────────────────────
const connection = new Connection(RPC_URL, "confirmed");
const owner = Keypair.fromSecretKey(
  new Uint8Array(JSON.parse(fs.readFileSync(KEYPAIR, "utf8"))),
);
const raydium = await Raydium.load({
  owner,
  connection,
  cluster: "mainnet",
  disableFeatureCheck: true,
  blockhashCommitment: "finalized",
});

// ── Pick the 0.25% fee tier ─────────────────────────────────────
const feeConfigs = await raydium.api.getCpmmConfigs();
const feeConfig  = feeConfigs.find((c) => c.index === 0);
if (!feeConfig) throw new Error("0.25% fee tier not found in CPMM configs.");

// ── Resolve mint metadata (Token-2022-aware) ────────────────────
const mintA = await raydium.token.getTokenInfo(new PublicKey(MINT_A));
const mintB = await raydium.token.getTokenInfo(new PublicKey(MINT_B));

// ── Build and execute ───────────────────────────────────────────
const { execute, extInfo } = await raydium.cpmm.createPool({
  programId:       CREATE_CPMM_POOL_PROGRAM,
  poolFeeAccount:  CREATE_CPMM_POOL_FEE_ACC,
  mintA,
  mintB,
  mintAAmount:     new BN(AMOUNT_A),
  mintBAmount:     new BN(AMOUNT_B),
  startTime:       new BN(0), // open immediately
  feeConfig,
  associatedOnly:  false,
  ownerInfo:       { useSOLBalance: true },
  txVersion:       TxVersion.V0,
  computeBudgetConfig: {
    units: 600_000,
    microLamports: 100_000,
  },
});

const { txId } = await execute({ sendAndConfirm: true });

console.log(`Pool ID:  ${extInfo.address.poolId.toBase58()}`);
console.log(`LP mint:  ${extInfo.address.lpMint.toBase58()}`);
console.log(`Vault A:  ${extInfo.address.vaultA.toBase58()}`);
console.log(`Vault B:  ${extInfo.address.vaultB.toBase58()}`);
console.log(`Tx:       https://solscan.io/tx/${txId}`);
```

## Jalankan

Contoh: membuat pool SOL/USDC dengan seed 1 SOL dan 160 USDC:

```bash
export MINT_A="So11111111111111111111111111111111111111112"   # wSOL
export MINT_B="EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"   # USDC
export AMOUNT_A="1000000000"   # 1 SOL (9 decimals)
export AMOUNT_B="160000000"    # 160 USDC (6 decimals)

node create-cpmm.mjs
```

Output yang diharapkan:

```
Pool ID:  HgC5...3kXb
LP mint:  4ZAS...9rkV
Vault A:  J9Mu...mP2k
Vault B:  AaJq...8wxx
Tx:       https://solscan.io/tx/5dQ...
```

## Apa yang baru saja terjadi

1. **`getCpmmConfigs`** mengambil daftar tier biaya langsung dari `api-v3.raydium.io` dan memilih indeks 0 (tier 0,25% — lihat [`reference/fee-comparison`](/id/reference/fee-comparison) untuk set lengkap).
2. **`getTokenInfo`** menyelesaikan metadata setiap mint, termasuk program token mana yang memilikinya. CPMM menerima mint SPL Token dan Token-2022; SDK merutekan secara otomatis.
3. **`createPool`** membangun satu transaksi yang:
   - mengurutkan mint dalam urutan kanonis,
   - menurunkan pool PDA, vault, LP mint, dan otoritas,
   - membayar `create_pool_fee` sekali jalan ke `CREATE_CPMM_POOL_FEE_ACC`,
   - membuat ATA pemanggil jika hilang,
   - menabur vault dengan `AMOUNT_A` dan `AMOUNT_B`.
4. **Harga awal** ditetapkan oleh rasio seed: `price = AMOUNT_B / AMOUNT_A` setelah penyesuaian desimal. Pilih ini dengan hati-hati — bot akan arbitrase setiap kesalahan harga dalam hitungan detik setelah pool dibuka.
5. **`startTime: new BN(0)`** membuka perdagangan segera. Untuk menyelenggarakan likuiditas sebelum dibuka untuk publik, atur timestamp Unix masa depan.

## Kesalahan umum

- **`pool already exists`** — Pool sudah ada untuk pasangan mint ini pada tier biaya ini. Cari terlebih dahulu sebelum membuat.
- **`insufficient funds`** — Dompet Anda tidak memiliki cukup `MINT_A`, `MINT_B`, atau SOL (untuk biaya pembuatan pool + sewa).
- **`Token-2022 extension not supported`** — Salah satu mint Anda menggunakan ekstensi yang tidak diterima CPMM. Lihat [`reference/token-2022-support`](/id/reference/token-2022-support).

## Setelah deploy

Anda dapat langsung melakukan swap terhadap pool baru — skrip [Swap from CLI](/id/quick-start/swap-from-cli) menerima `POOL_ID` baru Anda secara langsung. Agregator (Jupiter, dll.) akan mengindeks pool baru dalam hitungan menit.
