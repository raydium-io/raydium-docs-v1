## Input

Setiap swap memerlukan:

- **Token in** (alamat mint + jumlah)
- **Token out** (alamat mint)
- **Toleransi slippage** (default 0.5% di UI; 1% pada pair volatil)
- **Biaya prioritas** (default ukuran otomatis)

Jumlah ditentukan sebagai "exact-in" (Anda menetapkan apa yang Anda bayar; output dihitung) atau "exact-out" (Anda menetapkan apa yang Anda dapatkan; input dihitung). UI default ke exact-in.

## Panduan walkthrough UI

Di [raydium.io/swap](https://raydium.io/swap):

1. **Masukkan jumlah.** Pilih token input dan output; ketik jumlah di salah satu kotak. Kotak lainnya otomatis terisi menggunakan quote saat ini.

2. **Tinjau rute.** Panel rute menunjukkan pool mana yang dipilih Raydium:
   - Single-pool (paling umum): "SOL → USDC via CLMM 0.05%"
   - Multi-hop: "SOL → USDC via CPMM, lalu USDC → RAY via CLMM"
   - Split: "SOL → USDC via 60% CLMM 0.05%, 40% CPMM 0.25%"

   Untuk setiap hop: TVL, biaya, dan dampak yang diharapkan. Trade yang lebih besar menampilkan peringatan menonjol ketika dampak melebihi 1%.

3. **Periksa slippage.** Default adalah 0.5%. Untuk pair stabil adalah 0.1%; untuk meme token 2–5%. Jika quote bergerak saat Anda memandangnya (harga berfluktuasi), UI akan me-request kembali setiap 10 detik dan berkilau.

4. **Atur biaya prioritas.** Otomatis secara default; tiga tingkatan:
   - **Normal** (persentil ke-50) — default
   - **Fast** (persentil ke-75)
   - **Turbo** (persentil ke-95)
   Pengguna lanjutan dapat memasukkan nomor micro-lamports/CU spesifik.

5. **Konfirmasi dan tanda tangani.** Transaksi keluar; UI melacak konfirmasi. Waktu yang diharapkan 5–15 detik pada kemacetan normal.

6. **Pasca-swap.** UI menampilkan jumlah aktual yang diterima (mungkin berbeda dari quote hingga slippage); klik untuk melihat detail lengkap di tx explorer.

## Bagaimana routing memilih pool

Ketika ada beberapa pool untuk pair yang sama, router Raydium memilih berdasarkan:

1. **`amountOut` tertinggi setelah biaya.** Hitung quote untuk setiap pool yang memenuhi syarat; pilih yang memberikan output terbanyak.
2. **Tie-break menurut TVL.** TVL lebih tinggi berarti dampak lebih kecil pada trade berikutnya — tahan terhadap fluktuasi harga antara quote dan submission.
3. **Lebih suka single-pool daripada multi-hop.** Multi-hop menggabungkan biaya dan slippage; router hanya memilih multi-hop jika mengalahkan single-pool lebih dari 5 bps.
4. **Split jika besar.** Untuk trade yang besar relatif terhadap TVL pool, rute dapat memisah di beberapa pool untuk mengurangi dampak harga.

Produk dibandingkan pada pair khas:

- **Pool CPMM** mendominasi pada TVL rendah (`<$1M`) karena murah dibuat dan share biaya swap mendanai LP secara langsung.
- **Pool CLMM** mendominasi pada TVL medium/tinggi untuk pair volatil — likuiditas terkonsentrasi berarti quote lebih baik dalam range harga umum.
- **Pool AMM v4** mendominasi pada pair yang sudah mapan dengan integrasi OpenBook dalam (SOL-USDC, RAY-USDC). Sebagian besar pair baru tidak pernah menyentuh AMM v4.

## Biaya transfer Token-2022

Jika mint input atau output adalah Token-2022 dengan biaya transfer:

- `amountOut` dari quote adalah jumlah netto yang diterima pengguna setelah biaya.
- "Biaya" yang ditampilkan di UI terbagi menjadi "Pool fee" (biaya LP) dan "Transfer fee" (dibayarkan ke penerbit mint).

Lihat [`algorithms/token-2022-transfer-fees`](/id/algorithms/token-2022-transfer-fees).

## Slippage

Slippage melindungi terhadap pergerakan harga yang tidak menguntungkan antara quote dan eksekusi. Raydium mengonversi persentase slippage Anda menjadi jumlah minimum yang harus Anda terima:

```
minimumAmountOut = expectedAmountOut × (1 - slippage)
```

Jika output aktual lebih kecil, tx akan dikembalikan dengan `ExceededSlippage`. Default umum:

| Tipe Pair | Slippage |
|-----------|----------|
| Stable-stable | 0.01–0.1% |
| Major-major (SOL/USDC) | 0.1–0.5% |
| Volatiles mid-cap | 0.5–1% |
| Meme tokens | 2–5% |
| Token-2022 dengan biaya transfer | tambahkan biaya transfer ke slippage |

Slippage yang terlalu ketat menyebabkan reverts sering di bawah pergerakan harga normal. Slippage yang terlalu longgar mengundang sandwich attacks (lihat [`integration-guides/routing-and-mev`](/id/integration-guides/routing-and-mev)).

## Biaya prioritas

Biaya prioritas membayar validator lebih banyak untuk memproses transaksi Anda selama kemacetan. UI default ke pengaturan otomatis. Gunakan **Fast** atau **Turbo** ketika jaringan sibuk, ketika Anda menukar pair volatil, atau ketika transaksi sebelumnya kedaluwarsa sebelum konfirmasi.

Biaya prioritas yang lebih tinggi tidak meningkatkan harga yang dikutip. Mereka hanya meningkatkan kemungkinan transaksi Anda mendarat sebelum quote menjadi ketinggalan zaman.

## Mode kegagalan umum

### Tx kembali dengan `ExceededSlippage`

Harga bergerak lebih dari toleransi Anda antara quote dan eksekusi. Perbaikan:

1. Tingkatkan slippage hanya sebanyak yang diperlukan.
2. Kurangi ukuran trade (dampak lebih kecil).
3. Re-quote dan coba lagi; harga mungkin sementara.
4. Hindari slippage yang sangat tinggi pada pair utama, karena dapat membuat Anda terbuka pada eksekusi yang lebih buruk.

### Tx habis waktu

Biaya prioritas terlalu rendah untuk kemacetan saat ini. Perbaikan: naikkan biaya prioritas; lihat `integration-guides/priority-fee-tuning`.

### `InvalidAccountData` pada swap CLMM

Rute mungkin bergantung pada akun CLMM yang gagal dimuat atau berubah saat quote terbuka. Segarkan halaman, re-quote, dan coba lagi. Jika berlanjut, pilih rute lain jika UI menawarkannya.

### "Pool not found" / data pool ketinggalan zaman

Pool baru dibuat dan belum diindeks. Re-fetch menurut ID secara langsung:
Tunggu beberapa menit, segarkan aplikasi, dan cari lagi menurut pool atau mint token. Pool yang baru dibuat dapat memakan waktu sebentar untuk muncul di pencarian UI yang diindeks.

### Output adalah nol / sangat kecil

Ukuran swap melebihi likuiditas pool. Pool terlalu dangkal; kurangi ukuran trade, coba rute lain, atau tunggu lebih banyak likuiditas.

### Tx mendarat tetapi saldo tidak berubah

Segarkan saldo dompet dan Raydium Anda. Jika transaksi berhasil di explorer, periksa apakah akun token output tersembunyi di UI dompet Anda.

## Verifikasi

Gunakan tautan transaksi yang ditampilkan Raydium setelah konfirmasi. Di Solscan atau SolanaFM, periksa:

- **Status:** Success.
- **Signer:** alamat dompet Anda.
- **Perubahan saldo token:** token input berkurang, token output meningkat.
- **Program yang diinvokan:** program swap Raydium ditambah program token/sistem.
