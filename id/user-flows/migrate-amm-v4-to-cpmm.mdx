## Mengapa bermigrasi

| Aspek | AMM v4 | CPMM |
|-----------|--------|------|
| Tujuan biaya | Protokol + pembagian LP | LP menerima bagian biaya swap default |
| Ketergantungan OpenBook | Ya | Tidak ada |
| Biaya pembuatan | Lebih tinggi | Lebih rendah |
| Dukungan Token-2022 | Tidak | Ya |
| Listing baru dialihkan ke | Jarang | Default |

Jika pasangan Anda diperdagangkan dengan berat, peningkatan biaya LP dapat membenarkan migrasi. Jika pool lama masih memiliki likuiditas kuat dan proyek belum menunjuk pool CPMM tujuan, migrasi mungkin tidak sebanding dengan biaya koordinasi.

## Daftar periksa pra-migrasi

Untuk operator pool dan kas proyek:

- [ ] Umumkan garis waktu migrasi kepada LP.
- [ ] Berikan LP setidaknya 48 jam jika memungkinkan.
- [ ] Konfirmasi pool CPMM tujuan dan tingkat biaya.
- [ ] Siapkan pengumuman pasca-migrasi dengan ID pool baru dan LP mint.
- [ ] Tentukan apakah farm baru diperlukan.

Untuk LP individu:

- [ ] Konfirmasi Anda mengikuti migrasi yang disetujui proyek.
- [ ] Simpan cukup SOL untuk beberapa transaksi.
- [ ] Catat saldo LP AMM v4 Anda dan jumlah dasar yang diharapkan.
- [ ] Periksa apakah token LP AMM v4 Anda di-stake di farm.

## Migrasi UI: LP individu

Migrasi terdiri dari dua tindakan: menghapus likuiditas dari AMM v4, kemudian menambahkan likuiditas ke CPMM.

### Langkah 1: tarik dari AMM v4

1. Buka pool AMM v4 atau posisi portofolio Anda.
2. Jika token LP Anda di-stake di farm, lepas stake-nya terlebih dahulu.
3. Klik **Remove liquidity**.
4. Pilih persentase untuk ditarik.
5. Tinjau jumlah token yang akan Anda terima.
6. Konfirmasi dan setujui.

### Langkah 2: setorkan ke CPMM

1. Buka pool CPMM tujuan.
2. Klik **Add liquidity**.
3. Masukkan jumlah satu token.
4. Tinjau jumlah token berpasangan dan saldo sisa apa pun.
5. Konfirmasi dan setujui.

Antara penarikan dan penyetoran, harga pool dapat bergerak. Jika kedua pool memiliki rasio berbeda, beberapa token mungkin tertinggal sebagai debu di dompet Anda.

## Migrasi UI: kas proyek

Untuk kas proyek, prosesnya sama tetapi memerlukan koordinasi:

1. **Umumkan** migrasi sebelumnya.
2. **Benih pool CPMM** sebelum meminta pengguna bermigrasi.
3. **Migrasi LP kas** terlebih dahulu dan publikasikan tautan transaksi.
4. **Buat farm baru** jika insentif harus berlanjut pada LP mint CPMM.
5. **Pantau migrasi LP individu** selama hari atau minggu berikutnya.
6. **Hentikan pool lama** hanya setelah pengguna memiliki cukup waktu untuk berpindah.

## Verifikasi

### Setelah penarikan

- Saldo token LP AMM v4 harus berkurang atau mencapai nol.
- Dua token dasar harus muncul di dompet Anda.
- Transaksi harus menunjukkan **Success** di penjelajah Solana.

### Setelah penyetoran

- Saldo token LP CPMM harus muncul di dompet Anda.
- Portofolio Raydium harus menampilkan posisi baru.
- Halaman pool CPMM harus menampilkan bagian likuiditas Anda.

### Bukti migrasi

Untuk transparansi, publikasikan kedua tautan transaksi:

- Tarik dari AMM v4.
- Setorkan ke CPMM.

## Pertimbangan rollback

Rollback dari CPMM ke AMM v4 berarti menarik dari CPMM dan menyetor ke AMM v4. Biasanya tidak sebanding untuk dilakukan kecuali ada alasan produk spesifik. Pembuatan pool AMM v4 lebih mahal dan memerlukan pengaturan pasar OpenBook.

## Pertanyaan umum

### LP AMM v4 saya di-stake di farm. Dapatkah saya bermigrasi?

Ya. Lepas stake terlebih dahulu, tarik LP dari AMM v4, setorkan ke CPMM, kemudian stake token LP CPMM baru jika farm baru ada.

### Pool CPMM belum ada

Sebagian besar LP harus menunggu proyek untuk membuat dan menyetujui pool tujuan. Membuat satu sendiri berarti Anda mengambil risiko harga dan likuiditas awal.

### Saya memiliki posisi besar. Bagaimana dengan slippage?

Tarik dan setorkan dalam irisan yang lebih kecil jika posisi Anda besar relatif terhadap TVL pool. Tinjau setiap kuotasi dan hindari menandatangani jika jumlah yang diterima secara material lebih buruk dari yang diharapkan.

### Apa yang terjadi pada token LP yang lupa saya migrasi?

Mereka tinggal di pool lama dan terus menghasilkan sesuai dengan aturan AMM v4. Tidak ada yang memaksa migrasi.

## Penyelesaian masalah

### Rekening token LP tujuan baru

Penyetoran CPMM pertama mungkin perlu membuat rekening token LP baru. Simpan SOL ekstra di dompet Anda untuk sewa dan biaya.

### Transaksi kedua kedaluwarsa

Jika penarikan berhasil tetapi penyetoran kedaluwarsa, token Anda tetap berada di dompet Anda. Buka kembali pool CPMM dan coba lagi dengan pengaturan priority-fee yang lebih tinggi.

### Pool CPMM salah

Beberapa pool CPMM dapat ada untuk pasangan yang sama. Gunakan pool yang disetujui proyek atau pool dengan tingkat biaya dan likuiditas yang dimaksudkan.

### Implikasi pajak

Di beberapa yurisdiksi, penarikan LP dapat menjadi peristiwa yang dapat dikenakan pajak. Konsultasikan dengan aturan lokal sebelum bermigrasi posisi besar.
