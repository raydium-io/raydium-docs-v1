## Apa itu Raydium Perps

**Raydium Perpetual** adalah protokol perpetual dengan central-limit-order-book (CLOB) tanpa gas. Posisi didanai dari dompet SPL di Solana, tetapi mesin matching dan order book dioperasikan oleh [Orderly Network](https://orderly.network/), yang mengumpulkan likuiditas di seluruh venue multi-chain (Solana, Ethereum, Base, Arbitrum, BSC).

Properti utama:

- **Trading tanpa gas.** Pembuatan, modifikasi, dan pembatalan order tidak memerlukan transaksi on-chain di Solana — penandatanganan bersifat lokal dan gratis. Hanya deposit dan penarikan yang menyentuh chain.
- **Leverage hingga 100×**, tergantung market.
- **Collateral multi-chain** — Anda dapat melakukan deposit USDC / SOL / USDT / ETH / BNB / WBTC / YUSD / USD1 dari chain native mereka; semua P&L diselesaikan dalam USDC.
- **Cross-margin saja.** Semua collateral di akun dibagikan di seluruh setiap posisi yang terbuka.
- **One-way mode.** Anda tidak dapat menahan posisi long dan short pada market yang sama secara bersamaan; order sisi berlawanan mengurangi atau membalik posisi yang ada.
- **Self-custody.** Dana tetap berada di perps account dompet Anda; tidak ada dompet custodian yang dipegang Raydium.
- **Order type tingkat CEX.** Market, Limit, Stop (market / limit), Scale, plus flag IOC / FOK / Post-Only / Reduce-Only.

## Mengapa produk terpisah

AMM Raydium (spot) dan Raydium Perps (perpetual futures) berbagi brand dan titik masuk UI tetapi hampir tidak ada yang sama di level protokol:

| | Raydium AMM spot | Raydium Perps |
|---|---|---|
| Venue | Pool on-chain di Solana | CLOB off-chain yang dioperasikan Orderly |
| Mesin matching | Kurva AMM (CPMM / CLMM / AMM v4) | Central limit order book |
| Settlement | On-chain saat swap | Off-chain; deposit/penarikan on-chain |
| Price discovery | Kurva AMM + arbitrase | Order book + harga eksternal |
| Universum token | Mint SPL / Token-2022 apa pun | Hanya perp market terdaftar (BTC, ETH, SOL, dan major lainnya) |
| Program ID di [`reference/program-addresses`](/id/reference/program-addresses) | Ya | Tidak — lihat kontrak Orderly |
| Cakupan Raydium SDK v2 | Lengkap | Tidak tercakup |
| Bug bounty Raydium | Ya | Dirutekan melalui Orderly |

Perps didokumentasikan di tingkat surface produk — order types, fee, collateral, batasan — sehingga pembaca memahami apa itu dan kapan menggunakannya, tetapi referensi protokol mendalam berada di sisi Orderly.

## Catatan integrasi

- SDK Raydium spot (`@raydium-io/raydium-sdk-v2`) **tidak** mencakup perps. Untuk akses terprogram gunakan [SDK Orderly](https://orderly.network/docs/build-on-omnichain/evm-api/introduction) (TypeScript, REST, WebSocket).
- Tidak ada primitif "route swap melalui posisi perp". Client yang menginginkan eksposur spot dan perp membuat dua transaksi secara independen.
- Parameter risiko Orderly (leverage caps, liquidation thresholds, collateral limits) berubah seiring waktu. Selalu ambil parameter live dari API Orderly sebelum memberikan risk quote kepada user.
