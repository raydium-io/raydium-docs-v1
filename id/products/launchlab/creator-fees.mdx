## Pra-migrasi: akrual biaya langsung

Saat token masih berada di bonding curve, setiap pembelian dan penjualan membayar komponen biaya kreator yang terakumulasi dalam vault per-token. **Kreator** (wallet yang meluncurkan token) dapat mengklaim dari vault kapan saja.

Karakteristik:

- Didenominasikan dalam **token kutipan** (biasanya SOL).
- Dikonfigurasi oleh **platform** (konfigurasi platform mengontrol apakah biaya kreator aktif dan pada bps berapa).
- Dapat diklaim kapan saja dari halaman Portfolio di UI LaunchLab; secara programatik melalui instruksi LaunchLab `claim_creator_fees` (nama sebenarnya di IDL).
- Berlanjut hingga lulus; saat lulus, biaya pra-migrasi yang tersisa dapat diklaim tetapi biaya pra-migrasi baru tidak akan terakumulasi.

## Pasca-migrasi: Fee Key NFT melalui Burn & Earn

Ketika bonding curve token mencapai target dan likuiditas bermigrasi ke pool CPMM (atau AMM v4), token LP dimintai dan **dibagi sesuai konfigurasi platform**. Pembagian selalu berjumlah 100% dan selalu melibatkan kombinasi:

- **Burned** — ditransfer ke [Burn & Earn](/id/user-flows/burn-and-earn) untuk penguncian permanen. Fee Key untuk porsi ini dipegang oleh akun yang dikendalikan platform atau akun yang ditunjuk untuk pembakaran.
- **Creator** — Fee Key NFT yang dimintai langsung ke wallet kreator token. Fee Key ini mewakili hak untuk mengumpulkan biaya trading LP dari bagian token LP kreator.
- **Platform** — opsional; bagian yang mengalir ke treasury platform sebagai Fee Key NFT lain.

### Default LaunchLab Raydium sendiri

| Bagian | Default di raydium.io/launchlab |
|---|---|
| Burned (Burn & Earn, terkunci selamanya) | 90% |
| Creator (Fee Key NFT) | 10% |
| Platform | 0% (Platform Raydium tidak mengambil bagian LP platform) |

Dengan default ini, **hanya kreator yang mengklaim biaya trading dari bagian mereka**; 90% lainnya terkunci dan biayanya mengalir ke Burn & Earn (kembali ke mekanik sinyal kepercayaan daripada ke saku individu mana pun).

### Platform lain

Platform pihak ketiga yang dibangun di atas LaunchLab dapat mengonfigurasi pembagian mereka sendiri. Platform yang ingin mendanai dirinya melalui biaya LP pasca-migrasi dapat mengatur (misalnya) 70 / 10 / 20 (burn / kreator / platform). Lihat [`products/launchlab/platforms`](/id/products/launchlab/platforms) untuk field Platform PDA yang mendorong ini.

Terlepas dari pembagian, **mekanismenya sama**: setiap bagian yang tidak dibakar diwakili sebagai Fee Key NFT, yang dapat diperdagangkan, ditransfer, atau diklaim.

### Apa yang terjadi jika biaya kreator pasca-migrasi dinonaktifkan

Jika platform menetapkan `creator_fee_enabled = false`, token yang lulus **tidak** melalui Burn & Earn. Sebaliknya, token bermigrasi ke pool **AMM v4** menggunakan jalur migrasi LaunchLab legacy. Dalam mode itu, kreator tidak menerima Fee Key NFT, dan tidak ada biaya kreator pasca-migrasi level program. Beberapa kreator dalam mode ini menggunakan alat CLI offline untuk mengklaim biaya pasca-migrasi opsional yang dikonfigurasi melalui `creatorFeeOn` — lihat [`products/launchlab/instructions`](/id/products/launchlab/instructions) untuk permukaan on-chain.

## Referral token-share 0,1%

Terpisah dari pembagian biaya LP, kreator LaunchLab (dan pengguna mana pun, sebenarnya) dapat membagikan **tautan halaman token** dengan wallet mereka sebagai referrer. Ketika siapa pun memperdagangkan token itu (atau token LaunchLab lain dalam sesi yang sama) melalui tautan bersama:

- **0,1% dari setiap perdagangan** dialokasikan ke wallet referrer.
- Dibayar dalam SOL atau token kutipan pool.
- Dibayar secara otomatis pada setiap perdagangan; tidak ada langkah klaim.

Ini adalah produk yang berbeda dari referral swap 1% yang tercakup dalam [`user-flows/referrals-and-blinks`](/id/user-flows/referrals-and-blinks). Referral 1% berlaku untuk UI swap spot Raydium; referral token share 0,1% berlaku untuk halaman token LaunchLab secara spesifik.

## Mengklaim biaya

### Vault kreator (pra-migrasi)

```ts
import { Raydium, TxVersion } from "@raydium-io/raydium-sdk-v2";
const raydium = await Raydium.load({ owner, connection });
const { execute } = await raydium.launchlab.claimCreatorFees({
  mint,           // launched token mint
  txVersion: TxVersion.V0,
});
await execute({ sendAndConfirm: true });
```

### Fee Key NFT (pasca-migrasi)

Menahan Fee Key di wallet Anda sudah cukup — klaim dari halaman Portfolio, atau secara programatik melalui instruksi klaim biaya program pool yang relevan. Lihat [`products/cpmm/instructions`](/id/products/cpmm/instructions) atau setara AMM v4 untuk permukaan on-chain.

## Daftar periksa integrator

- **Tampilkan kedua aliran biaya.** Dashboard kreator yang hanya menampilkan salah satu dari "vault pra-migrasi" / "Fee Key pasca-migrasi" meninggalkan uang di atas meja untuk pengguna yang tidak tahu yang lain ada.
- **Hormati transfer kepemilikan Fee Key.** Jika kreator menjual atau mentransfer Fee Key NFT mereka, pemilik sebelumnya kehilangan akses ke klaim biaya di masa depan. UI Anda harus membaca pemegang NFT saat ini setiap kali, bukan cache alamat kreator asli.
- **Drift konfigurasi platform.** Default Raydium mungkin berubah. Untuk platform apa pun selain milik Raydium sendiri, selalu baca Platform PDA secara langsung untuk melihat pembagian burn / kreator / platform aktual untuk peluncuran spesifik itu.
- **Token-2022 jaminan dan token biaya.** Biaya kreator dapat didenominasikan dalam token Token-2022 dengan biaya transfer; faktor potongan biaya transfer ke dalam nomor "biaya yang dapat diklaim" apa pun yang ditampilkan.
