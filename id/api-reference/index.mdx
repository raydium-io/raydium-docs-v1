## Autentikasi

Sebagian besar layanan hanya-baca dan menerima permintaan anonim. Dua pola muncul:

- **Handshake tertanda dompet** — diperlukan oleh `launch-auth-v1` untuk mencetak `ray-token`, kemudian dibawa sebagai header `ray-token` oleh `launch-forum-v1`. Tandatangani pesan Solana ed25519 dengan bentuk `time:<unix-seconds>` menggunakan dompet Anda, kirim signature + alamat dompet ke `/request-token` `launch-auth-v1`, terima JWT sebagai balasan, dan teruskan sebagai header permintaan `ray-token` pada panggilan forum berikutnya.
- **Tanpa auth** — setiap layanan lainnya. Pemanggil terpercaya (UI Raydium, integrator) menghubungi endpoint langsung melalui HTTPS.

Playground Mintlify memungkinkan Anda menempel `ray-token` di panel auth sebelum mengirim permintaan forum; nilainya hanya disimpan di browser Anda.

## Batas kecepatan

Semua host berada di belakang Cloudflare dengan pembatasan laju progresif per IP sumber.

Integrator yang membutuhkan batas lebih tinggi harus menghubungi tim Raydium. Lonjakan di atas batas yang dipublikasikan dikembalikan sebagai `HTTP 429` dengan header `Retry-After`.

## Sampul respons

Sebagian besar layanan membungkus payload JSON mereka dalam sampul seragam:

```json
{
  "id": "uuid-v4-per-request",
  "success": true,
  "data": { ... }
}
```

Pada kegagalan:

```json
{
  "id": "uuid-v4-per-request",
  "success": false,
  "msg": "human-readable error string",
  "data": null
}
```

`api-v3` mengembalikan sampul sebagai `{ "id", "success", "data" }`. Layanan pump-* menggunakan helper `addBorderSuccess` / `addBorderErr` yang menghasilkan bentuk yang sama dengan `msg` hanya diisi pada error. Periksa halaman pengantar setiap layanan untuk sampul yang tepat yang digunakannya.

## Cara menggunakan bagian ini

Klik endpoint apa pun di navigasi sebelah kiri. Anda akan melihat:

- Metode HTTP, URL lengkap, dan host dasar.
- Parameter permintaan dengan tipe yang ditarik dari spek OpenAPI.
- Panel **Try it** yang memungkinkan Anda menetapkan parameter dan mengirim permintaan nyata ke mainnet (atau pilih server devnet dari dropdown jika tersedia).
- Bentuk respons, bersumber dari respons langsung perwakilan dan situs `addBorderSuccess` kode sumber.

Untuk akses level SDK — membangun transaksi sisi klien, transaksi swap batch, tipe TypeScript — lihat [`sdk-api/typescript-sdk`](/id/sdk-api/typescript-sdk).
