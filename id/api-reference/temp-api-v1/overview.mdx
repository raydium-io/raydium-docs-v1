Temp API adalah area penampungan sementara untuk endpoint yang belum memiliki tempat permanen di Raydium API utama. Layanan ini ditujukan untuk prototyping cepat, alat internal, dan fitur yang sedang dalam pengembangan aktif.

## Host

| Lingkungan | Host                              |
|------------|----------------------------------|
| Mainnet    | `temp-api-v1.raydium.io`         |
| Devnet     | `temp-api-v1-devnet.raydium.io`  |

## Peringatan stabilitas

**Layanan ini tidak memberikan jaminan stabilitas API.** Endpoint dapat berubah, dihapus, dipindahkan ke layanan lain, atau dihentikan sepenuhnya tanpa pemberitahuan. Gunakan API ini hanya untuk:

- Alat dan skrip internal
- Fitur yang sedang dalam pengembangan aktif
- Jalur non-kritis yang dapat mentoleransi gangguan
- Pengujian dan prototyping

Jangan mengandalkan API ini untuk dApps produksi atau integrasi yang kritis.

## Ikhtisar

Layanan ini saat ini mengekspos:

- **`/cp-creator-fee`** — Mengambil biaya kreator yang terakumulasi untuk suatu wallet di seluruh pool CPMM.
- **`/limit-order/order/list`** — Limit order CLMM wallet yang sedang aktif (terbuka dan terisi sebagian), disajikan dari cache indexer.
- **`/limit-order/history/order/list-by-user`** — Limit order CLMM wallet yang sudah ditutup (historis), dengan filter opsional `poolId` / `mint1` / `mint2` / `hideCancel`; dipaginasi menggunakan `nextPageId`.
- **`/limit-order/history/event/list-by-pda`** — Log event per PDA (`open` / `increase` / `decrease` / `settle` / `close`) untuk satu atau lebih PDA limit order; dipaginasi menggunakan `nextPageId`.

## Pola umum

### Cek biaya kreator untuk suatu wallet

```bash
curl -s "https://temp-api-v1.raydium.io/cp-creator-fee?wallet=<your-wallet-pubkey>"
```

Mengembalikan daftar pool di mana wallet tersebut adalah kreator, beserta biaya kreator yang telah terakumulasi namun belum diklaim dalam Token A dan Token B.

### Daftar limit order CLMM wallet yang sedang aktif

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/order/list?wallet=<your-wallet-pubkey>"
```

Mengembalikan order yang belum terisi sama sekali maupun yang terisi sebagian. Setiap baris adalah `LimitOrderState` yang sudah didekode (PDA, pool, owner, tick, side, fase FIFO cohort, jumlah total/terisi) beserta konteks pool yang diperlukan untuk memanggil `SettleLimitOrder`, termasuk jumlah `pendingSettle` yang sudah dihitung sebelumnya.

Endpoint order aktif tidak dipaginasi — endpoint ini mengembalikan semua order wallet yang sedang di-cache dalam satu payload.

### Daftar limit order wallet yang sudah ditutup

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/order/list-by-user?wallet=<your-wallet-pubkey>&size=50"
```

Mengembalikan satu halaman order historis, masing-masing diringkas dengan `fillStatus` (`NONE` / `PARTIAL` / `FULL`) dan jumlah input/output pada saat order keluar dari set aktif. Tambahkan `&nextPageId=<nextPageId dari respons sebelumnya>` untuk mengambil halaman berikutnya.

### Ambil timeline event untuk order tertentu

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/event/list-by-pda?pda=<limit-order-pda>"
```

`pda` dapat berupa satu key atau daftar yang dipisahkan koma. Setiap baris adalah satu mutasi on-chain (`open` / `increase` / `decrease` / `settle` / `close`) lengkap dengan txid, slot, waktu blok, tanda ditandatangani oleh owner atau keeper (`autoRunner`), serta delta input/output dalam satuan yang dapat dibaca manusia.

### Hasil kosong

Jika wallet tidak memiliki catatan yang di-cache, endpoint order aktif akan mengembalikan:

```json
{
  "id": "...",
  "success": true,
  "data": { "rows": [] }
}
```

Endpoint history mengembalikan bentuk `{ rows: [], nextPageId?: ... }` yang sama dengan array `rows` kosong. Endpoint lama `/cp-creator-fee` justru mengembalikan `"data": []` untuk kompatibilitas mundur.
