Bir LaunchLab tokenı CPMM'ye geçiş yaptığında, geçiş yapılan pool LP tokenları bastırır. Platform yapılandırması bu LP tokenlarına ne olacağını kontrol eder.

## LP bölümlemesi

`migrateCpLockNftScale` üç alana sahiptir. Değerler %100'ü temsil eden `1_000_000` değerine toplanmalıdır.

| Alan | Anlamı |
|---|---|
| `platformScale` | LP tokenları Burn & Earn aracılığıyla kilitlenir. Platform, Ücret Anahtarı NFT'sini alır. |
| `creatorScale` | LP tokenları Burn & Earn aracılığıyla kilitlenir. Token oluşturucu, Ücret Anahtarı NFT'sini alır. |
| `burnScale` | LP tokenları kalıcı olarak yakılır. Bu pay için Ücret Anahtarı NFT'si oluşturulmaz. |

Bu platform düzeyinde bir ayardır. Aynı platform aracılığıyla başlatılan tokenlar, platform ayarı güncellenene kadar aynı LP dağıtımını devralır.

## Örnekler

```ts
// %40 platform, %50 oluşturucu, %10 yakıldı.
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// %100 oluşturucu Ücret Anahtarı.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// %100 yakıldı.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## CPMM ücretleriyle ilişkisi

`migrateCpLockNftScale` **LP ücret haklarını kimin talep edebileceğini** kontrol eder. Swap ücret oranını belirlemez. Geçiş yapılan poolun swap ücret davranışı, `cpConfigId` tarafından seçilen CPMM `AmmConfig`'den gelir.

Örneğin, geçiş yapılan bir CPMM pool'u %0,25 işlem ücreti yapılandırması kullanıyorsa ve `creatorScale` 500.000 ise, oluşturucunun Ücret Anahtarı kilitli LP ücret haklarının %50'sini temsil eder. `platformScale` 400.000 ise, platformun Ücret Anahtarı %40'ını temsil eder.

## Bölümlemesini güncelleme

Platform yapılandırması güncellemeleri epoch'a göre hız sınırlandırılır. LP dağıtımını önemli bir platform politikası olarak değerlendirin, hafif bir UI ayarı olarak değil.

Güncelleme yapmadan önce:

- Üç ölçek değerinin `1_000_000` toplam olduğunu doğrulayın.
- Platformun gelecekteki geçişlerden sonra kimin Ücret Anahtarı NFT'leri alacağını anladığını doğrulayın.
- Yeni başlatmalar bunu kullanmadan önce değişikliği oluşturucular için belgeleyin.
