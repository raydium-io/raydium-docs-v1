## Ürün başına swap ücretleri

Ücret değerleri giriş miktarının yüzdesidir ve havuz matematiği uygulanmadan önce alınır.

| Ürün | Mevcut katmanlar | Varsayılan / en çok kullanılan | Dağılım |
|---------|-----------------|---------------------|-------|
| **AMM v4** | 0.25% (yalnızca) | 0.25% | %88 LP, %12 protokol (25'in 22/3'ü) |
| **CPMM** | 0.01%, 0.25%, 1% | 0.25% | %100 LP (protokol payı AmmConfig başına yapılandırılabilir, şu anda %0) |
| **CLMM** | 0.01%, 0.05%, 0.25%, 1% | 0.25% | %88 LP, %12 protokol |
| **LaunchLab (mezuniyet öncesi)** | 1% (varsayılan) | 1% | Havuza %50, LaunchLab hazinesine %50 |
| **LaunchLab (mezuniyet sonrası CPMM)** | 0.25% | 0.25% | CPMM dağılımını devral |

### Ücret paydası kuralı

- **AMM v4**: ücretler x/10.000 olarak kodlanır. "25" %0.25 anlamına gelir.
- **CPMM/CLMM**: ücretler x/1.000.000 olarak kodlanır. "2500" %0.25 anlamına gelir.

Doğrudan zincir üstü verileri okurken, havuzun programı için doğru paydayı kullanın.

## Protokol ücret dağılımları

Her bir ücretin "protokol" payı içinde başka dağılımlar uygulanır:

| Ürün | Swap ücretinin protokol payı | Protokol içindeki dağılım |
|---------|----------------------------|------------------------|
| AMM v4 | %12 (3/25) | Hazineye %100 |
| CPMM | AmmConfig başına yapılandırılabilir (şu anda %0) | N/A |
| CLMM | Swap ücretinin %12'si | Hazineye %100 |
| LaunchLab | Swap ücretinin %50'si (mezuniyet öncesi) | LaunchLab hazinesine %100 |

**LP payı** LP token sahiplerine otomatik olarak akar:

- AMM v4 / CPMM: havuz rezervlerine birikir, çekilişte oransal olarak dağıtılır.
- CLMM: `fee_growth_global_X/Y` içinde birikir, `collectFee` aracılığıyla talep edilebilir.

## Oluşturma maliyetleri

Bir havuz dağıtmak için SOL cinsinden ödenen tek seferlik maliyetler.

| Ürün | Kira maliyeti | Oluşturma ücreti | Tipik toplam |
|---------|-----------|--------------|---------------|
| AMM v4 | ~0,15 SOL (birden fazla hesap) | 0,2 SOL | ~0,35 SOL |
| CPMM | ~0,04 SOL (havuz + kasalar + gözlem) | 0,15 SOL | ~0,19 SOL |
| CLMM | ~0,075 SOL (havuz + gözlem + ilk tick dizileri) | 0 SOL | ~0,075 SOL |
| Farm v6 | ~0,02 SOL (çiftlik durumu + ödül kasaları) | 0,1 SOL (spam karşıtı) | ~0,12 SOL |
| LaunchLab başlatma | ~0,015 SOL (bağlama eğrisi hesabı) | 0,1 SOL (LaunchLab ücreti) | ~0,115 SOL |

Tüm kiralar geri alınabilir — havuzu / konumu / çiftliği kapatmak lamportları döndürür.

## Mezuniyet / geçiş maliyetleri

| Olay | Maliyet | Kim öder |
|-------|------|----------|
| LaunchLab → CPMM mezuniyeti | ~0,04 SOL (yeni CPMM havuzu oluşturma) | Bağlama eğrisi rezervlerinden ödenir |
| AMM v4 → CPMM geçişi | ~0,04 SOL CPMM oluşturma + ~0,000005 SOL işlem ücreti | Geçişi gerçekleştiren LP |
| CLMM konum NFT transferi | ~0,000005 SOL işlem ücreti | Devreden |

## Token-2022 ücret etkileşimi

Bir havuzun bir veya her iki tarafı transfer ücreti olan Token-2022'yi kullandığında, etkili ücret bileşik hale gelir:

```
havuza net giriş    = amount_in × (1 − transfer_fee_in)
havuz swap alır     = net × swap_fee_rate
kullanıcıya net çıkış  = out × (1 − transfer_fee_out)
```

Örnek: CPMM %0,25 havuz, giriş tarafında %1 transfer ücreti, çıkış tarafında %0,5 transfer ücreti.

```
amount_in        = 1000
net_to_pool      = 1000 × 0,99    = 990
pool swap fee    = 990 × 0,0025   = 2,475 (LP ücreti olarak tutulur)
amount_after_fee = 987,525
amount_out (AMM) = 987,525 × ratio, diyelim 987,525 × 0,98 = 967,77
net_out_to_user  = 967,77 × 0,995 = 962,94
```

Etkili ücret: **~%3,7** adlı %0,25 havuz ücretine rağmen.

SDK etkili oranı `getComputeAmountOut` aracılığıyla hesaplar — Token-2022 havuzlarını alıntılarken her zaman bunu kullanın.

## LaunchLab özellikleri

| Aşama | Ücret |
|-------|-----|
| Bağlama eğrisi işlemleri (satın alma veya satış) | %1 (havuz/hazine 50/50 dağılımı) |
| Mezuniyet (tek seferlik) | ~0,04 SOL eğri rezervlerinden |
| Mezuniyet sonrası (CPMM havuz) | 0,25% (standart CPMM ücretleri) |

Token başlatan projeler, genellikle bağlama eğrisi aşamasında LP payından çıkarılan bir "yaratıcı ücreti" seçeneği (0–%0,3) içerir; başlatmada yapılandırılabilir. Yaratıcı ücreti **yalnızca bağlama eğrisi** — başlatma bir CPMM havuzuna mezun olduğunda, elde edilen havuz standart CPMM katmanında hiçbir yaratıcı ek ücreti olmadan çalışır. Yaşam döngüsünün tamamı ve yaratıcıların LaunchLab NFT mekanizması aracılığıyla mezun olan havuzlardan nasıl kazanmaya devam ettikleri için [`products/launchlab/creator-fees`](/tr/products/launchlab/creator-fees) bölümüne bakın.

## Farm v6 ödül maliyet modeli

Çiftlikler kesinlikle ücret almazlar — bunlar ödül dağılımıdır. Ancak işletme maliyetleri vardır:

| Maliyet | Tipik |
|------|---------|
| Çiftlik oluşturma | 0,1 SOL spam karşıtı + ~0,02 SOL kira |
| Ödül kasasını doldur | Sadece tokens + ~0,0005 SOL işlem ücreti |
| Kullanılmayan ödülleri geri al | ~0,0005 SOL işlem ücreti |

Çiftlikler ayrıca çiftliğin ömrü boyunca kira ücretlerini karşılamak için ödül kasasında yeterli lamport gerektirir; kapatma bunları döndürür.

## Mevcut değerleri zincir üzerinde nasıl okuyabilirsiniz

### CPMM/CLMM AmmConfig

```ts
const config = await raydium.cpmm.getAmmConfigs();
console.log(config[0]);
// {
//   id: "...",
//   tradeFeeRate: 2500,         // 2500 / 1_000_000 = 0.25%
//   protocolFeeRate: 0,         // 0 / 1_000_000 = 0%
//   fundFeeRate: 0,
//   ...
// }
```

### AMM v4 havuzu

```ts
const poolInfo = await raydium.liquidity.getPoolInfoFromRpc({ poolId });
console.log(poolInfo.ammFeesNumerator, poolInfo.ammFeesDenominator);
// 25, 10000 → 0.25%
```

### Farm v6 ödül oranı

```ts
const farm = await raydium.farm.getFarmInfoFromRpc({ id: farmId });
console.log(farm.farmData.rewardInfos);
// Ödül başına: emissionsPerSecond, startTime, endTime
```
