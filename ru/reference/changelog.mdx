## Релизы

<Update label="2026-06-22" description="Stable AMM: удаление неиспользуемого кода OpenBook (market)">
  Stable AMM удаляет давно неиспользуемые счета и код для маркет-мейкинга OpenBook. Меньшие размеры `SwapBaseIn` / `SwapBaseOut` (18 → 9), `Deposit` (14 → 12) и `Withdraw` (21/22 → 12) (старые макеты остаются совместимыми); **критическое** изменение `WithdrawPnl` (16 → 10, без совместимости); отмена комиссии реферала; и упрощённая формула активов пула только для хранилища. Большинство других инструкций Stable больше не вызываются.

  [Читать полную запись →](/ru/reference/changelog/2026-06-22-stable-amm-openbook-cleanup)
</Update>

<Update label="2026-05-18" description="CLMM: лимитные ордера, односторонняя комиссия, динамическая комиссия">
  Три опциональные, обратно совместимые возможности CLMM: лимитные ордера первого класса (с `limit_order_admin` keeper для расчётов), односторонняя сборка комиссии (`CollectFeeOn`) и отслеживающая волатильность динамическая комиссия. Добавляет `CreateCustomizablePool`, переформатирование `PoolState` (**критическое изменение для индексатора**), новые поля `TickState`, одиннадцать новых кодов ошибок (с числовым сдвигом) и соответствующие добавления SDK / API.

  [Читать полную запись →](/ru/reference/changelog/2026-05-18-clmm-limit-orders)
</Update>

<Update label="2026-04-26" description="Первоначальная публикация">
  Первый публичный релиз набора документации Raydium, проверенный на соответствие развёртываниям на живой mainnet-beta и `@raydium-io/raydium-sdk-v2@0.2.42-alpha`.

  [Читать полную запись →](/ru/reference/changelog/2026-04-26-initial-publication)
</Update>

## Соглашения документации

- **Версионирование**: эта документация использует календарное версионирование (YYYY-MM-DD). Каждое обновление добавляет новую страницу записи и новую строку в начало хронологии выше.
- **Одна страница на релиз**: каждое резюме релиза находится на отдельной странице в `reference/changelog/`, поэтому этот индекс остаётся кратким и каждая запись независимо ссылаема.
- **Дата проверки**: каждая запись фиксирует, когда содержание было последний раз перепроверено на соответствие состоянию блокчейна / API и исходному коду программы. Если не указано, предполагайте основную дату записи.
- **Критические изменения**: выделены в боксе предупреждения на затронутых страницах и отмечены в записи.
- **Охват**: этот журнал изменений охватывает сам набор документации. Собственная историческая хронология протокола находится в [`introduction/history-and-milestones`](/ru/introduction/history-and-milestones) и является источником истины для «когда произошло X на Raydium».
