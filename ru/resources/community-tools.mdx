## Что здесь перечислено

Инструмент попадает на эту страницу, если он (a) общедоступен, (b) фактически работает с мейннет-программами Raydium на момент размещения и (c) поддерживается — то есть проект имел коммит, релиз или видимую активность в течение последних 90 дней.

Категории ниже — это начальная классификация; если у вас есть инструмент, который не подходит ни под одну из них, отправьте его через процесс ниже, и мы добавим новую категорию.

## Аналитика и данные

| Инструмент | Тип | Примечания |
|---|---|---|
| [DefiLlama](https://defillama.com/protocol/raydium) | TVL и объём протокола | Отслеживает пулы и фермы Raydium по всем продуктам. Достаточно надёжен для высокоуровневых метрик. |

| [Dune Analytics](https://dune.com/) | SQL-панели мониторинга | Несколько пользовательских панелей, охватывающих объёмы Raydium, TVL и статистику по пулам. Найдите "raydium" на Dune. |

## Агрегаторы

| Инструмент | Тип | Примечания |
|---|---|---|
| [Jupiter](https://jup.ag) | Агрегатор DEX | Маршрутизирует через пулы Raydium как один из основных источников ликвидности. |
| [DFlow](https://dflow.net) | RFQ + агрегатор | Агрегатор потока ордеров с маршрутизацией через Raydium. |

Маршрутизация агрегаторов через пулы Raydium не требует интеграции на стороне LP — как только пул открыт, агрегаторы индексируют его автоматически.

## Индексаторы и API (сообщество)

| Инструмент | Тип | Примечания |
|---|---|---|
| [Helius](https://helius.dev) | RPC + индексированные данные | Предоставляет распарсенные события Raydium и данные пулов через их расширенные API. |
| [Triton One](https://triton.one) | RPC | Высокопроизводительный RPC, обычно используемый для продакшн-интеграций Raydium. |
| [Solscan API](https://api.solscan.io) | API обозревателя блоков | Данные аккаунтов и транзакций, включая распарсенные инструкции Raydium. |
| [Bitquery](https://bitquery.io) | GraphQL данные | Свопы Raydium, события ликвидности и историю торговли можно запрашивать через GraphQL. |

Для официального API, поддерживаемого Raydium, см. [`sdk-api/rest-api`](/ru/sdk-api/rest-api). Приведённые выше инструменты полезны, когда вам нужны структуры данных, которые официальный API не предоставляет.

## Отправка инструмента

Чтобы добавить или обновить запись на этой странице:

1. Сделайте форк [репозитория документации](https://github.com/raydium-io) (источник документации находится в публичном репозитории документации Raydium — точный путь указан в README).
2. Отредактируйте `resources/community-tools.mdx`.
3. Отправьте PR с:
   - Названием инструмента и ссылкой.
   - Однострочным описанием (что это, в какую категорию).
   - GitHub или X-ручкой (дескриптором) разработчика для проверки.
   - Заметкой об активности (последний релиз / последний значимый коммит).

Записи проверяются по критериям "общедоступен / фактически работает / поддерживается". Собственнические записи без рабочего ПО или инструменты, основная цель которых противоречит безопасности пользователей (дренеры, генераторы поддельных пулов), не будут объединены.

## Политика удаления

Инструмент удаляется со страницы, если:

- Размещённый экземпляр был в офлайне более 30 дней.
- Репозиторий был заархивирован или удалён.
- Разработчик молчит более 180 дней, и инструмент перестаёт работать.
- Инструмент начинает наносить ущерб пользователям (дренеры, обманчивый UX и т. д.).

Удаление производится по усмотрению разработчиков документации; переввод в список добро пожаловать, если инструмент вернулся онлайн и поддерживается.

## Примечание о безопасности

Инструмент из списка, который просит вашу seed-фразу кошелька, требует отправки средств на адрес хранилища или обещает гарантированный доход — **это не инструмент, который вы должны использовать**. Включение в список — это низкий порог и не подтверждает безопасность. Всегда проверяйте, что вы авторизуете, перед подписанием транзакции.
