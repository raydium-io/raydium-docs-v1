## Примечание по содержанию

- Каждое руководство начинается с описания проблемы (например, «мои транзакции свопов падают под нагрузкой») и завершается проверенным рецептом.
- Включайте бенчмарки, когда они значимы (например, типичное использование CU для свопа CPMM по сравнению со свопом CLMM).
- Ссылайтесь на SDK и API для базовых примитивов; не переопределяйте их.
