Temp API — временное хранилище для эндпоинтов, которые ещё не получили постоянного места в основных API Raydium. Сервис предназначен для быстрого прототипирования, внутренних инструментов и функций, находящихся в активной разработке.

## Хосты

| Среда       | Хост                              |
|-------------|----------------------------------|
| Mainnet     | `temp-api-v1.raydium.io`         |
| Devnet      | `temp-api-v1-devnet.raydium.io`  |

## Предупреждение о стабильности

**Стабильность API не гарантируется.** Эндпоинты могут изменяться, устаревать, переноситься в другой сервис или удаляться без предварительного уведомления. Используйте этот API только для:

- внутренних инструментов и скриптов;
- функций в активной разработке;
- некритичных путей, допускающих сбои;
- тестирования и прототипирования.

Не стройте на этом API производственные dApp или критически важные интеграции.

## Обзор

В настоящее время сервис предоставляет следующие эндпоинты:

- **`/cp-creator-fee`** — получение накопленных комиссий создателя для кошелька по всем CPMM-пулам.
- **`/limit-order/order/list`** — текущие лимитные ордера CLMM кошелька (открытые и частично исполненные), отдаваемые из кеша индексатора.
- **`/limit-order/history/order/list-by-user`** — исторические (закрытые) лимитные ордера CLMM кошелька с опциональными фильтрами `poolId` / `mint1` / `mint2` / `hideCancel`; пагинация по курсору через `nextPageId`.
- **`/limit-order/history/event/list-by-pda`** — журнал событий (`open` / `increase` / `decrease` / `settle` / `close`) по одному или нескольким PDA лимитных ордеров; пагинация по курсору через `nextPageId`.

## Типичные сценарии использования

### Проверка комиссий создателя для кошелька

```bash
curl -s "https://temp-api-v1.raydium.io/cp-creator-fee?wallet=<your-wallet-pubkey>"
```

Возвращает список пулов, в которых кошелёк является создателем, а также накопленные, но ещё не востребованные комиссии в токенах A и B.

### Получение текущих лимитных ордеров CLMM кошелька

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/order/list?wallet=<your-wallet-pubkey>"
```

Возвращает как полностью неисполненные, так и частично исполненные ордера. Каждая строка — декодированное состояние `LimitOrderState` (PDA, пул, владелец, тик, сторона, фаза FIFO-когорты, суммарные/заполненные объёмы) вместе с контекстом пула, необходимым для вызова `SettleLimitOrder`, включая предварительно вычисленное значение `pendingSettle`.

Эндпоинт активных ордеров не поддерживает пагинацию — он возвращает все текущие кешированные ордера кошелька в одном ответе.

### Получение закрытых лимитных ордеров кошелька

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/order/list-by-user?wallet=<your-wallet-pubkey>&size=50"
```

Возвращает страницу исторических ордеров, каждый из которых содержит `fillStatus` (`NONE` / `PARTIAL` / `FULL`) и объёмы ввода/вывода на момент выхода ордера из активного набора. Для получения следующей страницы добавьте `&nextPageId=<nextPageId из предыдущего ответа>`.

### Получение хронологии событий по конкретному ордеру

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/event/list-by-pda?pda=<limit-order-pda>"
```

Параметр `pda` может быть одним ключом или списком ключей через запятую. Каждая строка соответствует одной мутации на блокчейне (`open` / `increase` / `decrease` / `settle` / `close`) и содержит txid, слот, время блока, признак подписи владельцем или keeper-ом (`autoRunner`), а также дельты ввода/вывода в человекочитаемых единицах.

### Пустой результат

Если для кошелька в кеше нет записей, эндпоинт активных ордеров возвращает:

```json
{
  "id": "...",
  "success": true,
  "data": { "rows": [] }
}
```

Исторические эндпоинты возвращают ту же структуру `{ rows: [], nextPageId?: ... }` с пустым массивом `rows`. Устаревший эндпоинт `/cp-creator-fee` для обратной совместимости возвращает `"data": []`.
