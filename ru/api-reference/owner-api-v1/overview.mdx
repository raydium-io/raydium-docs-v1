Owner API предоставляет представление позиций и балансов Raydium для отдельного кошелька. API получает кэшированные данные о позициях, заблокированной ликвидности, созданных пулах и участии в IDO, что делает его идеальным решением для интерфейсов кошельков и инструментов управления портфелем.

## Хосты

| Окружение | Хост                       |
|-------------|---------------------------|
| Mainnet     | `owner-v1.raydium.io`     |
| Devnet      | `owner-v1-devnet.raydium.io` |

## Обзор

API предоставляет пять основных эндпоинтов, организованных по типам ресурсов:

- **`/position/stake/{owner}`** — Позиции stake (стандартные AMM / CPMM LP)
- **`/position/clmm-lock/{owner}`** — Заблокированные позиции CLMM
- **`/create-pool/{owner}`** — Пулы, созданные этим кошельком
- **`/main/ido/{owner}`** — Данные участия в IDO

## Свежесть данных

Этот сервис кэширует данные о позициях и обновляет их периодически. Очень свежие изменения позиций (в пределах нескольких секунд) могут появиться в ответах не сразу.

## Типичные сценарии

### Получить все позиции stake кошелька

```bash
curl -s "https://owner-v1.raydium.io/position/stake/{owner}"
```

Замените `{owner}` на открытый ключ кошелька в формате base58.

### Получить заблокированные позиции

```bash
curl -s "https://owner-v1.raydium.io/position/clmm-lock/{owner}"
```

### Проверить участие в IDO

```bash
curl -s "https://owner-v1.raydium.io/main/ido/{owner}"
```

## Пустые результаты

Если кошелек не содержит данных для определенного эндпоинта, сервис возвращает ошибку 404 с метаданными запроса:

```json
{
  "id": "...",
  "success": false
}
```
