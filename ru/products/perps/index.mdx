## Что это

**Raydium Perpetual** — это бесплатный протокол перпетуальных контрактов с электронной книгой лимитных ордеров (CLOB). Позиции финансируются из кошелька SPL на Solana, но механизм маршрутизации ордеров и электронная книга управляются [Orderly Network](https://orderly.network/), который объединяет ликвидность в многоцепочечной экосистеме (Solana, Ethereum, Base, Arbitrum, BSC).

Основные свойства:

- **Бесплатная торговля.** Создание, изменение и отмена ордеров не требуют блокчейн-транзакций на Solana — подпись создаётся локально и бесплатно. На цепь попадают только депозиты и выводы.
- **Кредитное плечо до 100×** в зависимости от рынка.
- **Мульти-цепочечный коллатераль** — вы можете пополнить счёт USDC / SOL / USDT / ETH / BNB / WBTC / YUSD / USD1 с их нативной цепи; весь P&L рассчитывается в USDC.
- **Только кросс-маржин.** Весь коллатераль на счёте распределяется между всеми открытыми позициями.
- **Режим one-way.** Вы не можете одновременно держать длинную и короткую позицию на одном рынке; ордер в противоположную сторону закрывает или разворачивает существующую позицию.
- **Самостоятельная опека.** Средства остаются в perps-счёте вашего кошелька; нет отдельного кастодиального кошелька Raydium.
- **Типы ордеров уровня CEX.** Market, Limit, Stop (market / limit), Scale, а также флаги IOC / FOK / Post-Only / Reduce-Only.

## Почему отдельный продукт

Спотовые AMM Raydium (spot) и Raydium Perps (перпетуальные фьючерсы) делят бренд и точку входа в интерфейс, но на уровне протокола они почти полностью независимы:

| | Спотовые AMM Raydium | Raydium Perps |
|---|---|---|
| Площадка | On-chain пулы на Solana | Off-chain CLOB от Orderly |
| Механизм маршрутизации | AMM кривая (CPMM / CLMM / AMM v4) | Электронная книга лимитных ордеров |
| Расчёты | On-chain в момент swap | Off-chain; депозиты/выводы on-chain |
| Обнаружение цены | AMM кривая + арбитраж | Электронная книга + внешние цены |
| Вселенная токенов | Любой SPL / Token-2022 mint | Только листованные перп-рынки (BTC, ETH, SOL и др.) |
| Program IDs в [`reference/program-addresses`](/ru/reference/program-addresses) | Да | Нет — смотри контракты Orderly |
| Покрытие Raydium SDK v2 | Полное | Не покрыто |
| Баунти за ошибки Raydium | Да | Через Orderly |

Остальная часть этого сайта документации посвящена почти исключительно спотовым AMM и программам в GitHub организации Raydium. Perps документирован на уровне пользовательского интерфейса — типы ордеров, комиссии, коллатераль, ограничения — так что читатели понимают, что это такое и когда это использовать, но глубокий протокольный справочник находится на стороне Orderly.

## Примечания по интеграции

- Спотовый SDK Raydium (`@raydium-io/raydium-sdk-v2`) **не** покрывает perps. Для программного доступа используйте [SDK Orderly](https://orderly.network/docs/build-on-omnichain/evm-api/introduction) (TypeScript, REST, WebSocket).
- Нет примитива "маршрутизировать swap через перп-позицию". Если клиент хочет одновременно иметь spot и perp экспозицию, он строит две транзакции независимо.
- Параметры риска Orderly (лимиты кредитного плеча, пороги ликвидации, лимиты коллатераля) меняются со временем. Всегда запрашивайте актуальные параметры из API Orderly перед представлением рисков пользователю.
