## Зачем нужны платформы

Raydium хотела сделать LaunchLab **общей примитивной структурой**, а не единственным брендированным продуктом. Система Platform PDA позволяет экосистемам, провайдерам инструментов, сообществам и мем-командам предлагать кураторские или брендированные интерфейсы запуска, которые все в итоге взаимодействуют с одной и той же базовой программой bonding-curve LaunchLab и одним и тем же пулом CPMM (или AMM v4) после миграции.

Примеры платформ, которые вы можете встретить:

- **Собственный интерфейс LaunchLab от Raydium** на raydium.io/launchlab — эталонная реализация.
- **Партнёрские лаунчпады** — третьи стороны, которые зарегистрировали свой Platform PDA для брендирования интерфейса запуска.
- **Специализированные платформы** — например, платформа для конкретного сообщества, которая запускает только мемкоины, связанные с определённой тематикой, с пользовательским распределением комиссий, финансирующей казну.

Все эти токены остаются на одной программе LaunchLab, переходят в одну программу CPMM и могут быть маршрутизированы теми же агрегаторами.

## Platform PDA

Каждая платформа представлена Platform PDA, хранящейся в программе LaunchLab. Поля включают:

| Поле | Описание |
|---|---|
| `platform_key` | Seed / идентификатор платформы, используется для получения Platform PDA. |
| `platform_fee_destination` | Счёт, который получает долю платформы от торговых комиссий до и после миграции. |
| `platform_fee_bps` | Доля комиссии платформы при swap'ах до миграции (варьируется по платформам). |
| `creator_fee_enabled` | Включает ли эта платформа распределение комиссий создателя после миграции через Burn & Earn. |
| `creator_fee_share_bps` | Если включено, доля комиссий LP, которая поступает на Fee Key NFT создателя. |
| `cp_config_id` | Если целевой пункт после миграции — CPMM, какой `AmmConfig` использовать — выбирает уровень комиссии. |
| `name`, `description`, `image_url` | Метаданные отображения, используемые кошельками и интерфейсами, поддерживающими брендирование платформы. |
| `tokens_launched` | Счётчик или индекс в цепи токенов, созданных под этой платформой. |

Точные имена полей находятся в IDL LaunchLab ([`sdk-api/anchor-idl`](/ru/sdk-api/anchor-idl)). Таблица выше даёт описательный обзор.

## Что может контролировать платформа

**Захват комиссий:**

- **Доля платформы** при каждой покупке/продаже до миграции.
- **Доля после миграции** — когда токен выпускается, LP токены минтятся в многостороннее разделение, определённое платформой: часть на сжигание (Burn & Earn — паттерн блокировки ликвидности, который позволяет заблокированным LP продолжать получать долю комиссий пула, см. [`user-flows/burn-and-earn`](/ru/user-flows/burn-and-earn)), часть создателю (Fee Key NFT) и часть самой платформе.

**Целевой пункт после миграции:**

- **CPMM (по умолчанию, если комиссии создателя включены)** с `AmmConfig`, выбранным через `cp_config_id`. Разные `AmmConfig`ы имеют разные уровни комиссий; платформа может выбрать более высокий уровень комиссий, если её запуски — это высокооборотные спекулятивные активы.
- **AMM v4 (если комиссии создателя отключены)** — для обратной совместимости с исходным потоком LaunchLab.

**Пользовательский опыт:**

- **Метаданные брендирования**, используемые приложением Raydium и любым кошельком / обозревателем, который отображает токены LaunchLab.
- **Кастомизация страницы запуска** — платформы, которые размещают свой интерфейс, могут фильтровать токены для отображения только запусков своей собственной платформы.

## Что платформа не может контролировать

- **Математика bonding-curve.** Форма кривой (логистическая, экспоненциальная и т. д.) является частью самой программы LaunchLab. Платформы выбирают из поддерживаемых кривых; они не могут добавлять произвольные пользовательские кривые.
- **Логика порога миграции.** Условие, которое запускает переход на CPMM / AMM v4 (достигнута целевая рыночная капитализация), обеспечивается программой.
- **Основная безопасность / позиция аудита.** Каждая платформа наследует историю аудита LaunchLab; злоумышленная платформа не может её обойти, изменяя программу.

## Регистрация платформы

Программа LaunchLab предоставляет инструкцию `create_platform` (точное имя из IDL). Вызывающие передают:

- Seed, который будет использоваться для получения Platform PDA.
- Счёт назначения комиссии.
- Желаемое количество bps (подчиняется лимитам уровня программы).
- Конфигурацию после миграции (ID `AmmConfig` CPMM, если применимо).
- Метаданные брендирования.

Аренда для Platform PDA оплачивается создателем. Обновления брендирования и некоторых параметров комиссий разрешены после создания авторитетом платформы; структурные параметры (программа назначения миграции) обычно — нет.

Точную сигнатуру инструкции в текущем IDL LaunchLab см. в разделе [`products/launchlab/instructions`](/ru/products/launchlab/instructions).

## Экономическое взаимодействие

Единственный swap на bonding-curve LaunchLab распределяет стоимость примерно следующим образом (точные коэффициенты установлены платформой):

```
Покупатель платит:                100 SOL
Комиссия протокола (LaunchLab):   – small %
Комиссия платформы:               – platform_fee_bps%
Комиссия создателя (до миграции): – creator_fee_bps% (если включено)
Остаток (на кривую):              ~ rest
```

После миграции LP токены разделяются согласно конфигурации платформы:

```
Минтированные LP токены при выпуске:  100%
  → Burn & Earn (сжигаются):            X%  (по умолчанию у Raydium: 90)
  → Fee Key NFT создателя:             Y%  (по умолчанию у Raydium: 10)
  → Платформа:                         Z%  (настраивается платформой; может быть 0)
```

**Доля Burn & Earn** конвертируется в общий доход от комиссий, распределяемый держателю Fee Key — которая, согласно стандартам Raydium, является хранилищем "burn", представляющим широкий рыночный резервный фонд для доверия к токену.

## Примечания для интеграторов

- **Список токенов по платформам.** Используйте endpoints API pool-info / mint с фильтром `platform` (когда поддерживается) для вывода списка только токенов, запущенных под конкретным Platform PDA.
- **Отображение брендирования.** Если вы создаёте кошелёк или обозреватель, который отображает токены LaunchLab, чтение имени платформы / URL изображения из Platform PDA обеспечивает гораздо лучший UX, чем отображение "LaunchLab" для каждого токена.
- **Математика комиссий.** Каждое предложение цены bonding-curve должно учитывать комиссию платформы в дополнение к комиссиям протокола и создателя. Помощники по цитированию SDK автоматически это делают, когда предоставляется Platform PDA.
