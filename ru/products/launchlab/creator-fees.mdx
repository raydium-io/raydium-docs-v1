## До миграции: прямое накопление комиссий

Пока токен находится на bonding curve, каждая покупка и каждая продажа включают компонент комиссии создателя, который накапливается в хранилище для конкретного токена. **Создатель** (кошелек, который запустил токен) может требовать средства из хранилища в любое время.

Характеристики:

- Выражено в **quote token** (обычно SOL).
- Настраивается **платформой** (конфигурация платформы контролирует, включены ли комиссии создателя и в каком размере bps).
- Может быть требовано в любое время со страницы Portfolio в интерфейсе LaunchLab; программно через инструкцию LaunchLab `claim_creator_fees` (точное имя в IDL).
- Продолжается до выпуска; при выпуске остаточные комиссии до миграции могут быть требованы, но новые комиссии до миграции не будут накапливаться.

## После миграции: Fee Key NFT через Burn & Earn

Когда bonding curve токена достигает цели и ликвидность мигрирует в пул CPMM (или AMM v4), LP-токены выпускаются и **разделяются в соответствии с конфигурацией платформы**. Раздел всегда составляет 100% и всегда включает какую-либо комбинацию из:

- **Сжжено** — передано в [Burn & Earn](/ru/user-flows/burn-and-earn) для постоянной блокировки. Fee Key для этой части хранится на управляемом платформой или назначенном для сжигания счёте.
- **Создатель** — Fee Key NFT, выпущенный непосредственно на кошелёк создателя токена. Этот Fee Key представляет право на сбор торговых комиссий LP из доли LP-токенов создателя.
- **Платформа** — опционально; доля, которая поступает в казну платформы как другой Fee Key NFT.

### Стандартные значения Raydium на LaunchLab

| Доля | Стандартное значение на raydium.io/launchlab |
|---|---|
| Сжжено (Burn & Earn, заблокировано навсегда) | 90% |
| Создатель (Fee Key NFT) | 10% |
| Платформа | 0% (платформа Raydium не берёт долю LP платформы) |

При этих стандартных значениях **только создатель требует торговые комиссии из своей доли**; остальные 90% заблокированы, и их комиссии идут в Burn & Earn (возвращаясь к механике сигнализации доверия, а не в какой-либо отдельный карман).

### Другие платформы

Платформы третьих сторон, разработанные на основе LaunchLab, могут настроить свой собственный раздел. Платформа, которая хочет финансировать себя через комиссии LP после миграции, может установить (например) 70 / 10 / 20 (сжигание / создатель / платформа). См. [`products/launchlab/platforms`](/ru/products/launchlab/platforms) для полей Platform PDA, которые управляют этим.

Независимо от раздела, **механизм остаётся тем же**: каждая несожженная доля представлена как Fee Key NFT, которая может быть продана, передана или использована.

### Что происходит, если комиссии создателя после миграции отключены

Если платформа устанавливает `creator_fee_enabled = false`, выпущенный токен **не проходит** через Burn & Earn. Вместо этого он мигрирует в пул **AMM v4**, используя устаревший путь миграции LaunchLab. В этом режиме создатель не получает Fee Key NFT, и нет комиссии создателя на уровне программы после миграции. Некоторые создатели в этом режиме используют инструмент CLI вне сети для требования опциональных комиссий после миграции, настроенных через `creatorFeeOn` — см. [`products/launchlab/instructions`](/ru/products/launchlab/instructions) для поверхности на цепочке.

## Реферальный токен-шер 0.1%

Отдельно от совместного использования LP-комиссий, создатели LaunchLab (и любой пользователь, действительно) могут делиться **ссылкой на страницу токена** со своим кошельком в качестве реферера. Когда кто-то торгует этим токеном (или другими токенами LaunchLab в одной и той же сессии) по общей ссылке:

- **0.1% каждой сделки** отправляется на кошелёк реферера.
- Оплачивается в SOL или quote token пула.
- Оплачивается автоматически на каждую сделку; нет шага требования.

Это отдельный продукт от реферала 1%, рассмотренного в [`user-flows/referrals-and-blinks`](/ru/user-flows/referrals-and-blinks). Реферал 1% применяется к интерфейсу spot swap Raydium; реферал токен-шер 0.1% применяется специально к страницам токенов LaunchLab.

## Требование комиссий

### Хранилище создателя (до миграции)

```ts
import { Raydium, TxVersion } from "@raydium-io/raydium-sdk-v2";
const raydium = await Raydium.load({ owner, connection });
const { execute } = await raydium.launchlab.claimCreatorFees({
  mint,           // launched token mint
  txVersion: TxVersion.V0,
});
await execute({ sendAndConfirm: true });
```

### Fee Key NFT (после миграции)

Наличие Fee Key в вашем кошельке достаточно — требуйте со страницы Portfolio или программно через инструкцию требования комиссии соответствующей программы пула. См. [`products/cpmm/instructions`](/ru/products/cpmm/instructions) или эквивалент AMM v4 для поверхности на цепочке.

## Контрольный список для интеграторов

- **Показывайте оба потока комиссий.** Панель управления создателем, которая показывает только один из "хранилища до миграции" / "Fee Key после миграции", оставляет деньги на столе для пользователей, которые не знают о существовании другого.
- **Уважайте передачу собственности Fee Key.** Если создатель продаёт или передаёт свой Fee Key NFT, предыдущий владелец теряет доступ к будущим требованиям комиссий. Ваш интерфейс должен читать текущего держателя NFT каждый раз, а не кэшировать исходный адрес создателя.
- **Дрейф конфигурации платформы.** Стандартные значения Raydium могут измениться. Для любой платформы, отличной от самой Raydium, всегда читайте Platform PDA в режиме реального времени, чтобы увидеть фактический раздел сжигания / создателя / платформы для конкретного запуска.
- **Коллатераль Token-2022 и токены с комиссиями.** Комиссии создателя могут быть выражены в Token-2022 токенах с комиссиями передачи; учитывайте потерю комиссии при передаче в любом отображаемом числе "требуемой комиссии".
