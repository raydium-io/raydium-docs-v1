По умолчанию комиссии создателя CPMM из пула, перенесённого в LaunchLab, присваиваются первоначальному создателю токена. Платформы могут переопределить это для будущих миграций, установив `platformCpCreator`.

## Зачем перенаправлять комиссии создателя

Платформы используют перенаправление комиссии создателя в следующих случаях:

- Необходима пользовательская логика распределения между создателями, сообществами и доходом платформы.
- Требуется управляемый PDA сборщик комиссий для распределения на блокчейне.
- Нужна единая бухгалтерия по всем перенесённым пулам в рамках одной платформы.

<Warning>
  Перенаправление комиссий создателя меняет, кто может их собирать. Создатели токенов теряют прямой доступ к сбору для пулов, которые мигрируют после активации этой настройки, если платформа не распределяет им комиссии заново.
</Warning>

## Как это работает

1. Администратор платформы вызывает `updatePlatformConfig`.
2. `updateInfo.type` устанавливается на `updatePlatformCpCreator`.
3. Значение — кошелёк или PDA, который станет создателем пула CPMM для будущих перенесённых пулов.
4. Будущие миграции LaunchLab под этой платформой присвоят комиссии создателя CPMM настроенному адресу.
5. Платформа собирает комиссии создателя и распределяет их по своим правилам.

```ts
await raydium.launchpad.updatePlatformConfig({
  programId,
  platformAdmin: owner,
  updateInfo: {
    type: "updatePlatformCpCreator",
    value: new PublicKey("<PLATFORM_WALLET_OR_PDA>"),
  },
  txVersion: TxVersion.V0,
});
```

## Потенциальные проблемы

- **Всё или ничего для каждой платформы.** Настройка применяется ко всем будущим миграциям под этой платформой.
- **Только для новых миграций.** Уже перенесённые пулы сохраняют своего существующего создателя пула.
- **Обновления ограничены эпохой.** Обновления конфигурации платформы ограничены по частоте эпохой Solana.
- **Комиссии создателя должны существовать.** Если выбранный CPMM `AmmConfig` имеет `creator_fee_rate = 0`, комиссий создателя CPMM нет для сбора.
- **Обработка WSOL важна.** Если собранные комиссии — это WSOL и вам нужен нативный SOL, закройте учётную запись токена WSOL после сбора.

## Контрольный список для распределения

Если вы распределяете комиссии исходным создателям:

1. Определите каждого исходного создателя из учётной записи пула LaunchLab, связанной с монетами пула CPMM.
2. Соберите ожидающие комиссии создателя для кошелька платформы или PDA.
3. Рассчитайте согласованную долю каждого создателя.
4. Переведите SPL-токены или SOL создателям, создав ATA при необходимости.
5. Опубликуйте достаточно деталей учёта, чтобы создатели могли проверить распределения.
