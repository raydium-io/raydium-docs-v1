## 名称和大小写

| 形式 | 用法 |
|---|---|
| **Raydium** | 协议名称。始终使用标题大小写。不要写成 `raydium` 或 `RAYDIUM`。 |
| **Raydium Protocol** | 在需要与其他语境中的 Raydium 名称区分时可用（它也是一种化学元素、一款视频游戏等）。 |
| **\$RAY** 或 **RAY** | Raydium 的原生 SPL 代币。在营销文案中使用美元前缀，在技术或表格语境中使用纯 `RAY`。 |
| **Raydium AMM v4** / **Raydium CPMM** / **Raydium CLMM** / **LaunchLab** | 产品名称。LaunchLab 是一个单词，两个 L 都大写。 |
| **Burn & Earn** | 一个结构，使用符号，两个名词都大写。 |

不要写 `Raydium DEX`、`Ray protocol` 或 `Raydiums`（无复数形式）。该协议是一个单数名词。

## 标志

Raydium 标志有两种形式：

- **字标** — 完整的"Raydium"字体锁定。在标题、新闻稿和需要名称可读性的任何语境中使用。
- **符号标** — 独立的单字母标记。在应用图标、网站图标和空间受限的地方（头像、社交媒体档案图片）使用，因为字标在那些位置不能达到可读大小。

### 浅色和深色版本

每种形式都有两种风格：

| 文件 | 背景 | 使用场景 |
|---|---|---|
| `logo-light.svg` | 深色背景 | 在深色 UI 界面上进行主要放置。 |
| `logo-dark.svg` | 浅色背景 | 在浅色 UI 界面上进行主要放置。 |
| `mark-light.svg` | 深色背景 | 在深色界面上仅显示符号。 |
| `mark-dark.svg` | 浅色背景 | 在浅色界面上仅显示符号。 |

所有资源都是矢量 SVG 格式。尽可能使用 SVG。只在最终渲染尺寸将其光栅化为 PNG（避免缩放 PNG）。

### 清晰空间

在标志周围保持清晰空间，大小等于 Raydium "R" 字符的高度。不要在此边距内用文本、图标或图形挤压标志。

### 最小尺寸

字标：屏幕上不低于 **120 px** 宽，或印刷品上 **20 mm**。
符号：屏幕上不低于 **24 px** 宽，或印刷品上 **6 mm**。

低于这些阈值，标志会失去清晰度，合作伙伴应切换到备用形式（例如在空间受限时用符号替换字标）。

## 色彩调色板

Raydium 的视觉识别以紫色主色调为中心，配合更深和更浅的变体用于状态、深度和对比。

| 角色 | 十六进制 | RGB | 用法 |
|---|---|---|---|
| **主色** | `#8C6EFF` | 140 / 110 / 255 | 主要品牌色。行动号召、链接、关键强调。 |
| **浅色** | `#A88BFF` | 168 / 139 / 255 | 悬停状态、次要强调、浅色主题背景。 |
| **深色** | `#5E3BE1` | 94 / 59 / 225 | 活跃/按下状态、深色主题强调、深度。 |

辅助中立色：

| 角色 | 十六进制 | 用法 |
|---|---|---|
| **前景（浅色）** | `#0A0A0F` | 浅色界面上的正文。 |
| **前景（深色）** | `#FAFAFA` | 深色界面上的正文。 |
| **表面（浅色）** | `#FFFFFF` | 页面背景、浅色主题。 |
| **表面（深色）** | `#0F0F1A` | 页面背景、深色主题。 |
| **边框** | `#E5E5EB`（浅色）/ `#1F1F2E`（深色） | 卡片和分割线边框。 |

不要重新着色标志。字标和符号始终以浅色或深色主资源形式渲染 — 永远不要着色、不要用渐变替换、不要用品牌色调处理。

## 排版

文档和站点使用 **Inter** 作为 UI 和正文字体。标题使用 Inter 的粗体权重（600–800）。代码块使用 **JetBrains Mono** 或任何等宽备用字体。

| 风格 | 字体 | 权重 | 用法 |
|---|---|---|---|
| 展示 / H1 | Inter | 800 | 页面标题。 |
| H2–H4 | Inter | 700 | 分段标题。 |
| 正文 | Inter | 400 | 段落文本。 |
| 强调 | Inter | 600 | 行内强调、加粗文本。 |
| 代码 | JetBrains Mono | 400–500 | 行内 `code` 和代码块。 |

Inter 是开放许可字体，可从 [rsms.me/inter](https://rsms.me/inter) 获取。

## 标志的注意事项

**应该做的：**

- 使用文档仓库 `/logo/` 目录中的官方 SVG 主资源。
- 保持清晰空间和最小尺寸。
- 仅在字标和符号在视觉上平衡时将其配对（不要并排显示大小不一致的两个）。
- 在背景上渲染，确保有足够的对比 — 在深色上使用浅色主资源，在浅色上使用深色主资源。

**不应该做的：**

- 重新着色标志。不要紫色字标、不要彩虹、不要除提供的浅色/深色以外的单色变体。
- 扭曲长宽比（不要拉伸、不要倾斜）。
- 添加投影、轮廓、光晕或其他效果。
- 将标志放在繁忙的背景图像上，而不在下方有保持对比的纯色面板。
- 裁剪标志（不要只显示从字标中裁出的"R" — 应使用符号标代替）。
- 使用标志暗示对 Raydium 未正式合作的产品的认可。在中性语境中使用"Powered by Raydium"或"Integrated with Raydium"徽章，不暗示任何审查。

## "Powered by Raydium" 徽章

如果你的集成使用 Raydium 作为路由来源或流动性场所，欢迎进行简单的文本或徽章归属：

```text
Powered by Raydium
Integrated with Raydium
Liquidity via Raydium
```

这些表述已被预先批准。放在集成的定价、交换或池界面附近 — 不要放在头部突出位置，那会暗示正式合作。

如果你是聚合器或钱包通过 Raydium 路由，想要更突出的归属，请在发布前通过 Discord（`#partnerships` 频道）与团队联系。
