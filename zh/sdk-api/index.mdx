## 选择你的集成方式

有五条路线可选。正确的选择取决于什么在调用 Raydium 以及你需要多少控制权。

| 路线 | 最适合 | 延迟 | 自定义逻辑 | 语言 |
|---|---|---|---|---|
| [Trade API](/zh/sdk-api/trade-api) | 后端、机器人、Blinks——任何需要服务器构建交易的地方 | 最低 | 受限（预设路由） | 任何 |
| [TypeScript SDK](/zh/sdk-api/typescript-sdk) | 需要完整构建器访问的 Node/TS 应用和前端 | 中等 | 完整 | TS / JS |
| [Python 集成](/zh/sdk-api/python-integration) | Python 后端和 ML 管道 | 中等 | 完整（手动指令构建） | Python |
| [Rust CPI](/zh/sdk-api/rust-cpi) | 调用 Raydium 的链上程序组合 | N/A（链上） | 完整 + 可组合 | Rust |
| [Anchor IDL](/zh/sdk-api/anchor-idl) | 针对没有官方支持的语言的代码生成（Go、Kotlin、Swift 等） | 可变 | 完整 | 任何带 Anchor 代码生成工具的语言 |

如果你不确定：后端和机器人 → **Trade API**；前端和完整控制应用 → **TypeScript SDK**；链上组合 → **Rust CPI**。
