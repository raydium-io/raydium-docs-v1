## Burn & Earn 是什么

Burn & Earn 是 Raydium 的一项功能，让池创建者（或任何 LP）能够在链上**永久**存入头寸，同时保留收取交易费用的权利。头寸变为不可赎回，但新铸造的 **Raydium Fee Key** NFT 代表从该锁定头寸领取费用的持续权利。

常见用例：

- **发布信任。** 项目通过锁定部署者的 LP 来承诺不会撤网。持有者可在链上验证流动性是永久的。
- **费用产生的收藏品。** 一个可交易的 NFT，产生收益。出售 NFT；转移费用流。
- **机制级流动性。** 协议希望为了品牌 / 发现目的永久保持流动性，同时仍可通过费用端获利。

## 支持的池类型

| 程序 | Burn & Earn 支持 |
|---|---|
| CPMM | 支持 |
| CLMM | **仅全价格范围**（推荐） |
| AMM v4（混合） | 不支持 |

对于 CLMM，只有跨越完整价格范围的头寸才应该被锁定。一旦锁定，你无法重新平衡——如果价格偏离你选择的范围，你将永久停止在该头寸上赚取费用。紧范围 Burn & Earn 头寸几乎总是一个错误。

## 锁定在程序级别的工作原理

1. 原始 LP 代币（CPMM）或头寸 NFT（CLMM）被转入 Burn & Earn 程序拥有的程序所有 PDA。
2. 一个 **Fee Key NFT** 被铸造给调用者。其元数据指向被锁定头寸的 PDA。
3. Burn & Earn 暴露 `collectFees`（或每个池类型的等效调用），该调用：
   - 验证签署者持有给定锁定头寸的 Fee Key NFT。
   - 通过 CPI 调用底层池程序的费用领取指令。
   - 将收集的费用路由到 Fee Key 持有者的 ATA。
4. 没有对应的 `withdraw`——底层 LP / 头寸永远不会退出托管。

Fee Key 是标准的 SPL NFT。它可以在任何 Solana NFT 市场上出售、在钱包间转移、在接受 NFT 的借贷协议中用作抵押，或包装到另一个合约中。无论谁在 `collectFees` 调用时持有它，都会收到费用。

## UI 演练

1. 像往常一样创建 CPMM 池或具有全价格范围头寸的 CLMM 池。
2. 转到**流动性 → 创建 → Burn & Earn**。
3. 选择要锁定的头寸。
4. 验证头寸详情（NFT 地址、大小、池）并输入 UI 显示的确认句子。这故意设置了摩擦，因为该操作是不可逆的。
5. 点击**确认，永久锁定流动性**。
6. 一个 Fee Key NFT 到达你的钱包。它在 Portfolio 页面上的锁定头寸下显示。

像领取任何头寸的费用一样领取费用——通过 Portfolio 页面或通过 `collectFees`。

## API 接口

池信息端点返回一个 `burnPercent` 字段，表示该池通过 Burn & Earn 锁定的流动性份额：

```http
GET https://api-v3.raydium.io/pools/info/list
  ?poolType=standard
  &poolSortField=default
  &sortType=desc
  &pageSize=1
  &page=1
```

```json
{
  "data": [
    {
      "id": "...",
      "burnPercent": 99.17,
      ...
    }
  ]
}
```

`burnPercent` 是一个方便的信任信号表面：LP 被锁定 99%+ 的代币比 LP 被锁定 0% 的代币撤网的可能性要低得多。

## 陷阱

- **不可逆转。** LP 端永远无法取回。确认金额是你愿意永远不再接触的。
- **CLMM 紧范围是一个陷阱。** 仅锁定全价格范围 CLMM 头寸，除非你真正理解范围限制的锁定头寸在价格偏离范围后停止赚取费用。
- **丢失 Fee Key NFT 就永远丢失费用。** 把它当成契约对待。不要意外销毁它；不要在恶意合约中质押它。
- **Token-2022 铸造带有转账费用**：Burn & Earn 支持它们，但转账费用的削减在每次费用领取时应用——为此做好预算。
- **没有部分锁定。** 如果你想要 50% 锁定和 50% 可移动，先将头寸分成两个头寸，然后只锁定其中一个。
