<Info>
  农场将奖励代币分配给质押特定质押币种的用户，通常是 CPMM 或 AMM v4 LP 代币。Farm v6 是当前一代，也是唯一接受新农场创建的版本。
</Info>

## 何时创建农场

在以下情况下创建农场：

- 你需要为特定池吸引流动性。
- 你想在原生交易费用基础上向 LP 提供额外奖励。
- 你正在将国库代币分配给与协议对齐的 LP。

如果池的交易费用已经提供了足够的激励，则不需要农场。

## 所需条件

- **质押币种：** 通常是现有 CPMM 或 AMM v4 池的 LP 代币。
- **奖励币种：** 1-5 个要分配的币种。
- **奖励预算：** 每个奖励币种在整个时间表中的金额。
- **钱包 SOL：** 约 0.1 SOL，用于创建租金、代币账户和优先费用。

## 农场版本选择

| 版本 | 状态 | 用途 |
|------|------|------|
| v3 | 旧版 | 仅现有农场 |
| v5 | 旧版 | 仅现有农场 |
| v6 | 当前 | 所有新农场 |

为所有新农场使用 Farm v6。

## 设计奖励时间表

对于每个奖励币种，决定：

- **总奖励金额：** 你将预先提供的预算。
- **开始时间：** 现在或计划的未来时间戳。
- **结束时间：** 通常在开始后 30-90 天。
- **目标 APR：** 你相对于预期 TVL 的预期年化奖励。

大多数农场推出时高估了 TVL。如果流动性少于预期，实际 APR 将更高，农场将按每美元 TVL 更快地花费相同预算。

## UI 演示

在 [raydium.io/farms/create](https://raydium.io/farms/create/)：

1. **选择质押币种。** 粘贴 LP 币种地址。确认它与你想激励的池相匹配。
2. **添加奖励币种。** 添加 1-5 个奖励，包括总金额、开始时间和结束时间。
3. **查看 APR 和预算。** 检查预计 APR、总奖励支出和各奖励的明细。
4. **创建农场。** 签署农场创建交易。
5. **资金奖励。** 签署将奖励代币移入金库的资金交易。
6. **等待索引。** 农场应在几分钟内出现在 Raydium 上。

## 监控农场

### 质押进度

启动后检查总质押 LP、质押者数量和 TVL。如果 TVL 远低于预期，显示的 APR 可能高于计划。

### 实现 APR

将显示的 APR 与目标进行比较。如果代币价格或 TVL 发生重大变化，决定是否补充、延长或让农场完成。

### 剩余奖励余额

在结束日期前监控剩余奖励。如果奖励过早耗尽，用户可能会看到无法收获的待处理奖励，直到金库被重新填充。

## 补充和延长

使用农场管理 UI 在可用时添加更多奖励或延长运行时间表。保持与启动相同的公开沟通标准：在用户依赖 APR 之前宣布变更。

如果农场已结束，根据该农场可用的 UI 选项创建新的时间表或重新启动激励。

## 回收未使用的奖励

未使用的奖励只能由农场管理员回收，且只能在奖励流结束后回收。除非你已清楚地宣布了关闭计划，否则不要提前回收奖励。

## 故障排除

### 资金不足的金库

如果奖励金库余额低于时间表要求，收获可能在结束时失败。预先提供完整预算。

### 错误的质押币种

针对错误的 LP 币种创建农场是一个严重错误。签署前仔细检查池和 LP 币种。

### 农场不是池

农场不创建流动性。它只奖励质押现有 LP 代币的用户。

### 每秒计价

Farm v6 随时间发出奖励。按每天代币数或整个持续期间的总预算来考虑，而不是槽或区块。

### 具有转账费用的代币

如果你的奖励币种有转账费用，用户收到的将少于显示的总奖励。在启动沟通中披露这一点。

### CLMM 头寸不使用 Farm v6

CLMM 具有内置头寸奖励。不要为 CLMM 头寸 NFT 创建 Farm v6 农场，除非 UI 明确支持该流程。
