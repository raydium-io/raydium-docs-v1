## 五种订单类型

**市价单。** 以订单簿中最优可用价格立即买入或卖出。持续与挂单匹配，直到你的订单量成交。如果簿深不足以成交全部数量，未成交部分会被取消（不会挂单）。用于追求立即成交，代价是可能承受滑点。

**限价单。** 以你指定的价格（**限价**）或更优价格买入或卖出。限价单在簿上保留直到被匹配或取消。当价格优先于速度时使用。

**止损市价单。** 处于非激活状态，直到市场价格触及你的**触发价格**。触发后，它以市价单形式提交，如果流动性充足，将以下一个可用价格成交。当立即成交比价格控制更重要时使用。

**止损限价单。** 处于非激活状态，直到市场价格触及你的触发价格。触发后，它以限价单形式提交，限价为你指定的价格，仅在簿价达到该价格时才会成交。当你想要一个永远不会比预定底价/顶价更差的止损单时使用，但须接受在跳空情况下可能完全不成交的风险。

**分层单。** 将一个较大的订单拆分为多个较小的限价单，分布在价格范围内。用于逐步进场或退场，而不是一次性全部交易。

## 四种订单标记

这些标记可以附加到限价单或止损限价单上以改变成交语义：

**IOC — 立即成交否则取消。** 以限价（或更优价格）尽可能多地立即成交。任何未成交部分会被取消而非保留在簿上。当你想要立即成交但要求价格上限时使用。

**FOK — 全部成交否则取消。** 要么以限价（或更优价格）**全部**立即成交，要么完全取消没有部分成交。当部分成交比完全不成交更差时使用——例如该头寸只有在全量规模下才有意义。

**只做市。** 确保订单增加流动性（做市方）。如果在提交时会与簿上的挂单匹配（即成为吃单方），订单会被取消而非成交。当你想要做市成交而非吃单成交时使用。

**仅平仓。** 防止订单*增加*你的头寸。它只能平仓或缩减该市场中的现有头寸。如果你没有头寸，或订单会增加它，订单不会执行。用于自动止盈/止损逻辑，确保你永远不会意外反向开仓。

## 结合类型 + 标记

并非所有组合都被允许。常见组合如下：

| 订单类型 | 标记 | 常见用途 |
|---|---|---|
| 限价单 | — | 挂单做市报价 |
| 限价单 | 只做市 | 严格做市报价（无意外吃单） |
| 限价单 | IOC | 有价格上限的激进吃单 |
| 限价单 | FOK | 全部成交或全部取消的吃单 |
| 限价单 | 仅平仓 | 按价格平仓 |
| 市价单 | 仅平仓 | 紧急平仓，不计价格 |
| 止损市价单 | 仅平仓 | 优先成交而非反向开仓的止损 |
| 止损限价单 | 仅平仓 | 有价格底限、无反向开仓的止损 |
| 分层单 | 只做市 | 被动梯形进场 |

「只做市」和「仅平仓」可以组合在同一订单上。

## 费用影响

订单类型和标记与费率等级相互作用：

- **市价单始终支付吃单费率。** 市价单无法获得做市商回扣。
- **限价单** 如果挂单后被稍后匹配则支付做市商费率；如果在提交时立即匹配则支付吃单费率。
- **只做市** 通过拒绝任何会成交的提交来追求做市商费率。
- **止损单** 根据触发时转换成的订单类型支付费率（止损市价 → 吃单费率；止损限价 → 做市商费率，除非它成交）。

基于交易量的吃单等级和固定 0 bps 做市商费率详见 [费用](/zh/products/perps/fees)。
