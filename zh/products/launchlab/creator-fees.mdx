## 迁移前：直接费用累积

当代币仍在联合曲线上时，每一笔买入和卖出都会支付一个创作者费用部分，这些费用累积在一个按代币划分的金库中。**创作者**（启动代币的钱包）可以随时从金库中领取。

特点：

- 以**计价代币**（通常是 SOL）的形式计价。
- 由**平台**配置（平台配置控制创作者费用是否启用及其基点数）。
- 可随时从 LaunchLab UI 的投资组合页面领取；通过 LaunchLab `claim_creator_fees` 指令以编程方式领取（IDL 中的具体名称）。
- 持续到毕业；在毕业时，剩余的迁移前费用可以领取，但不会再产生新的迁移前费用。

## 迁移后：通过销毁和获利的 Fee Key NFT

当代币的联合曲线达到目标并将流动性迁移到 CPMM（或 AMM v4）池时，LP 代币被铸造并**根据平台的配置进行分割**。分割总和始终为 100%，总是涉及以下某种组合：

- **销毁** — 转入[销毁和获利](/zh/user-flows/burn-and-earn)以永久锁定。此部分的 Fee Key 由平台控制的账户或销毁指定账户持有。
- **创作者** — 直接铸造到代币创作者钱包的 Fee Key NFT。此 Fee Key 代表从创作者的 LP 代币份额中收取 LP 交易费的权利。
- **平台** — 可选；流向平台财库作为另一个 Fee Key NFT 的份额。

### Raydium 官方 LaunchLab 的默认设置

| 份额 | raydium.io/launchlab 默认值 |
|---|---|
| 销毁（销毁和获利，永久锁定） | 90% |
| 创作者（Fee Key NFT） | 10% |
| 平台 | 0%（Raydium 平台不收取平台 LP 份额） |

在这些默认设置下，**只有创作者从其份额中领取交易费**；其他 90% 被锁定，其费用流向销毁和获利（流回信任信号机制而不是任何个人）。

### 其他平台

在 LaunchLab 上构建的第三方平台可以配置自己的分割。想通过迁移后 LP 费用自筹资金的平台可以设置（例如）70 / 10 / 20（销毁/创作者/平台）。请参阅[`products/launchlab/platforms`](/zh/products/launchlab/platforms)了解驱动此功能的平台 PDA 字段。

无论分割如何，**机制是相同的**：每个非销毁份额都表示为一个 Fee Key NFT，可以交易、转移或兑现。

### 如果迁移后创作者费用被禁用会发生什么

如果平台设置 `creator_fee_enabled = false`，毕业的代币**不会**进行销毁和获利。相反，它会使用旧版 LaunchLab 迁移路径迁移到 **AMM v4** 池。在该模式下，创作者不会收到 Fee Key NFT，也没有程序级迁移后创作者费用。此模式下的某些创作者使用离线 CLI 工具来通过 `creatorFeeOn` 配置的可选迁移后费用进行领取 — 请参阅[`products/launchlab/instructions`](/zh/products/launchlab/instructions)了解链上接口。

## 0.1% 代币份额推荐返利

与 LP 费用分享分开，LaunchLab 创作者（以及任何用户）可以分享一个**代币页面链接**，其钱包作为推荐人。当任何人通过共享链接交易该代币（或同一会话中的其他 LaunchLab 代币）时：

- **每笔交易的 0.1%** 被空投到推荐人的钱包。
- 以 SOL 或池的计价代币支付。
- 在每笔交易时自动支付；无需领取步骤。

这是与[`user-flows/referrals-and-blinks`](/zh/user-flows/referrals-and-blinks)中涵盖的 1% 交换推荐返利不同的独立产品。1% 推荐返利适用于 Raydium 的现货交换 UI；0.1% 代币份额推荐返利特别适用于 LaunchLab 代币页面。

## 领取费用

### 创作者金库（迁移前）

```ts
import { Raydium, TxVersion } from "@raydium-io/raydium-sdk-v2";
const raydium = await Raydium.load({ owner, connection });
const { execute } = await raydium.launchlab.claimCreatorFees({
  mint,           // launched token mint
  txVersion: TxVersion.V0,
});
await execute({ sendAndConfirm: true });
```

### Fee Key NFT（迁移后）

在钱包中持有 Fee Key 就足够了 — 从投资组合页面领取，或通过相关池程序的费用领取指令以编程方式领取。请参阅[`products/cpmm/instructions`](/zh/products/cpmm/instructions)或 AMM v4 等效内容了解链上接口。

## 集成者检查表

- **显示两个费用流。** 仅显示"迁移前金库"/"迁移后 Fee Key"之一的创作者仪表板会让不知道另一个存在的用户损失资金。
- **尊重 Fee Key 所有权转移。** 如果创作者出售或转移其 Fee Key NFT，前任所有者将无法访问未来的费用领取。您的 UI 应该每次都读取当前 NFT 持有者，而不是缓存原始创作者地址。
- **平台配置漂移。** Raydium 的默认设置可能会改变。对于除 Raydium 官方之外的任何平台，始终实时读取平台 PDA 以查看该特定启动的实际销毁/创作者/平台分割。
- **Token-2022 抵押品和费用代币。** 创作者费用可以用具有转移费的 Token-2022 代币计价；将转移费减少计入任何显示的"可领取费用"数字。
