默认情况下，来自 LaunchLab 迁移池的 CPMM 创建者费用归属于原始代币创建者。平台可以通过设置 `platformCpCreator` 来覆盖此设置，对未来的迁移生效。

## 为什么要重定向创建者费用

平台在需要以下功能时会使用创建者费用重定向：

- 在创建者、社区和平台收入之间的自定义分配逻辑。
- 用于链上重新分配的 PDA 控制的费用收集器。
- 在一个平台下所有迁移池的统一会计。

<Warning>
  重定向创建者费用会改变谁可以收取这些费用。除非平台将费用重新分配给他们，否则代币创建者将失去对在设置激活后迁移的池的直接收取权限。
</Warning>

## 工作原理

1. 平台管理员调用 `updatePlatformConfig`。
2. `updateInfo.type` 设置为 `updatePlatformCpCreator`。
3. 该值是一个钱包或 PDA，将成为未来迁移池的 CPMM 池创建者。
4. 该平台下未来的 LaunchLab 迁移将 CPMM 创建者费用分配给配置的地址。
5. 平台收取创建者费用，并根据自己的规则重新分配这些费用。

```ts
await raydium.launchpad.updatePlatformConfig({
  programId,
  platformAdmin: owner,
  updateInfo: {
    type: "updatePlatformCpCreator",
    value: new PublicKey("<PLATFORM_WALLET_OR_PDA>"),
  },
  txVersion: TxVersion.V0,
});
```

## 需要注意的事项

- **每个平台的全部或全无。** 该设置适用于该平台下的每次未来迁移。
- **仅向前看。** 已迁移的池保留其现有的池创建者。
- **受纪元限制的更新。** 平台配置更新受 Solana 纪元的速率限制。
- **创建者费用必须存在。** 如果选定的 CPMM `AmmConfig` 的 `creator_fee_rate = 0`，则没有 CPMM 创建者费用可收取。
- **WSOL 处理很重要。** 如果收集的费用是 WSOL，且你需要本地 SOL，请在收集后关闭 WSOL 代币账户。

## 重新分配检查清单

如果你重新分配给原始创建者：

1. 从与 CPMM 池 mint 相关联的 LaunchLab 池账户中解析每个原始创建者。
2. 收集平台钱包或 PDA 的待收创建者费用。
3. 计算每个创建者的协议份额。
4. 向创建者转账 SPL 代币或 SOL，根据需要创建 ATA。
5. 发布足够的会计详情，供创建者验证分配。
