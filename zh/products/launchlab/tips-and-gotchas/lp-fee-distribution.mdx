当 LaunchLab 代币迁移到 CPMM 时，迁移后的池会铸造 LP 代币。平台配置控制这些 LP 代币的处理方式。

## LP 分配

`migrateCpLockNftScale` 有三个字段。这些值的总和必须为 `1_000_000`，代表 100%。

| 字段 | 含义 |
|---|---|
| `platformScale` | LP 代币通过 Burn & Earn 锁定。平台接收 Fee Key NFT。 |
| `creatorScale` | LP 代币通过 Burn & Earn 锁定。代币创建者接收 Fee Key NFT。 |
| `burnScale` | LP 代币被永久销毁。此份额不创建 Fee Key NFT。 |

这是一个平台级设置。通过同一平台启动的代币会继承相同的 LP 分配方式，直到平台更新该设置。

## 示例

```ts
// 40% 平台，50% 创建者，10% 销毁。
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% 创建者 Fee Key。
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% 销毁。
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## 与 CPMM 费用的关系

`migrateCpLockNftScale` 控制**谁可以声称 LP 费用权利**。它不设置交易费率。迁移后的池的交易费行为来自 `cpConfigId` 选择的 CPMM `AmmConfig`。

例如，如果迁移的 CPMM 池使用 0.25% 交易费配置，且 `creatorScale` 为 500,000，创建者的 Fee Key 代表锁定 LP 费用权利的 50%。如果 `platformScale` 为 400,000，平台的 Fee Key 代表 40%。

## 更新分配

平台配置更新按 epoch 进行速率限制。将 LP 分配视为重要的平台政策，而非临时性 UI 设置。

更新前：

- 确认三个 scale 值的总和为 `1_000_000`。
- 确认平台了解未来迁移后谁将接收 Fee Key NFT。
- 在新启动使用该配置前，为创建者记录变更。
