## 文档约定

- **版本控制**：本文档使用基于日历的版本控制（YYYY-MM-DD）。每次更新都会在上面的时间线顶部添加一个新条目页面和新行。
- **每个版本一个页面**：每个版本摘要都位于 `reference/changelog/` 下的专属页面上，这样索引保持简洁，每个条目都可以独立链接。
- **验证日期**：每个条目都记录了内容最后一次与链上 / API 状态和程序源代码交叉检查的时间。如果未说明，则假设为条目的主日期。
- **破坏性变更**：在受影响的页面上用方框警告标出，并在条目中标记。
- **覆盖范围**：此更新日志涵盖文档集本身。协议本身的历史时间线位于 [`introduction/history-and-milestones`](/zh/introduction/history-and-milestones)，是"Raydium 上何时发生 X"的真实来源。
