Raydium 不是独立存在的。它从其他 Solana 协议获取输入，也为它们产生输出。本页描绘了这些关系，帮助你理解 Raydium 责任的边界，以及其他协议的行为如何影响你作为用户或集成者的体验。

## 上游依赖

### OpenBook（原名 Serum）

- **角色**：中央限价订单簿。AMM v4 池最初是与 OpenBook 混合的——交换可以对标 AMM 曲线或匹配 OpenBook 市场进行成交——但 **OpenBook 集成已被停用**。AMM v4 池不再在 OpenBook 上发布或维护订单。
- **何时相关**：历史上仅对 AMM v4 池有关；现在，Raydium 没有任何产品依赖 OpenBook。
- **风险面**：之前，OpenBook 升级或中断可能影响 AMM v4 行为（2023 年 1 月发生过一次此类事件）。现在集成已关闭，这种耦合被移除了。CPMM 和 CLMM 从未有 OpenBook 依赖。
- **状态**：OpenBook 继续在 Solana 上运行；Raydium 对任何当前交换流量都不依赖它。现有 AMM v4 池引用的 OpenBook 市场账户保留在链上，作为不可变状态以确保向后兼容性。

### Solana 运行时

- **角色**：所有 Raydium 程序的执行基础设施。
- **何时相关**：处处相关。Solana 的最终性时间（~1 秒）、计算预算限制（每笔交易 1.4M CU）和地址查询表都直接影响 Raydium 用户体验。
- **状态**：Solana 定期升级；Raydium 跟踪验证者客户端版本（Agave、Firedancer），如果行为改变则进行调整。

### Token 程序（SPL Token 和 Token-2022）

- **角色**：所有 Raydium 池持有的 Token 原语。
- **SPL Token**：遗留程序；适用于所有产品。
- **Token-2022**：启用扩展的程序；适用于 CPMM、CLMM、Farm v6，不适用于 AMM v4。
- **风险面**：扩展行为（转账费、钩子等）可能影响池功能。详见 [`security/oracle-and-token-risks`](/zh/security/oracle-and-token-risks)。

## 下游集成

### Jupiter 和其他聚合器

- **角色**：在许多 Solana DEX（包括 Raydium）间路由交易。
- **集成方式**：Jupiter 通过 Raydium 的 REST API 和直接链上调用读取池状态，然后通过 Raydium SDK（有时直接）组合交换指令。
- **交易量占比**：大约 60% 的 Jupiter 交易量至少通过一次 Raydium 池。
- **集成者路径**：聚合器通用模式在 [`integration-guides/aggregator`](/zh/integration-guides/aggregator) 中有文档说明。

### 钱包

- **角色**：让用户进行交换、追踪 LP 头寸、参与农场，无需离开钱包。
- **集成方式**：
  - 交换：通过 Jupiter 或直接通过 Raydium SDK 路由。
  - 头寸显示：枚举用户的 ATA 和 NFT 持仓，与 Raydium 的池列表交叉引用（[`integration-guides/wallet-integration`](/zh/integration-guides/wallet-integration)）。
  - 农场显示：调用 Raydium 的 `/positions/staking` 端点。
- **用户影响**：任何带有应用内交换的钱包可能通过 Raydium 路由，而用户不会察觉。

### 借贷协议（Kamino、Marginfi、Drift）

- **角色**：需要预言机价格和清算执行场所的借贷协议。
- **集成方式**：使用 CLMM ObservationState TWAP 作为预言机；通过 Raydium 直接交换以进行清算抵押品兑换。
- **风险**：如果借贷协议使用现货 Raydium 价格而非 TWAP，可能被闪电贷风格的操纵利用（见 [`security/attack-vectors`](/zh/security/attack-vectors)）。

### MEV 搜寻者和套利者

- **角色**：维护 Raydium、Orca、Phoenix 和中心化交易所之间的价格一致性。
- **对用户的影响**：大多数时候价格紧密；对保护不足的交换进行夹心攻击。详见 [`integration-guides/routing-and-mev`](/zh/integration-guides/routing-and-mev)。

### 机器人（市场做市商、DCA 执行器、清算者）

- **角色**：提供流动性（在浅层池上做 LP）、执行定时交易、平仓头寸。
- **集成路径**：通常是使用 [`sdk-api/python-integration`](/zh/sdk-api/python-integration) 或 Rust CPI 包装器的 Python 或 Rust 机器人。

## 聚合交易量中的地位

截至 2026 年 4 月：

- Raydium 每天处理约 35–45% 的 Solana 范围内 DEX 交易量。
- 当 memecoin 活动集中在 LaunchLab + 毕业池期间，最高市场份额约 55%。
- 当大量订单簿驱动活动时低端约 25%（大型机构流量在可用时流向 Phoenix）。

这些数字会波动；关键是 Raydium 可靠地是交易量排名前两位的场所之一。

## Token-2022 生态

Raydium 是最早原生支持 Token-2022 的主要 DEX 之一。含义：

- 带有转账费的新 mint（如版税、费用授予人 Token）可以进行 LP。
- 可以存在转账钩子 mint（尽管有风险注意事项）。
- 不可转账和永久委托 mint 被阻止。

随着更多项目在 Token-2022 下启动——特别是使用受监管 mint 的「机构」阵营——Raydium 的支持将其定位为该类流动性的场所。

## 前瞻性依赖

可能改变 Raydium 生态系统地位的需关注项目：

- **Solana 验证者多样化**：如果 Firedancer 达到主导地位，性能特征可能改变。
- **跨链桥接**：Raydium 一直在研究向其他 L1/L2 的扩展。尚无部署。
- **Jito 的演进**：区块引擎与 DeFi 间的持续集成可能改变 MEV 动态。
- **链上订单簿（Phoenix）**：持续成熟可能将一些 AMM 式流量转向订单簿。
