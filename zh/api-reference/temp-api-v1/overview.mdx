Temp API 是尚未在 Raydium 主 API 中获得固定位置的端点的临时存放区，主要面向快速原型开发、内部工具以及正在活跃开发中的功能。

## 主机地址

| 环境 | 主机 |
|-------------|----------------------------------|
| 主网 | `temp-api-v1.raydium.io` |
| 开发网 | `temp-api-v1-devnet.raydium.io` |

## 稳定性警告

**本服务不提供任何 API 稳定性保证。** 端点可能随时变更、废弃、迁移至其他服务或直接下线，恕不另行通知。仅在以下场景使用本 API：

- 内部工具和脚本
- 正在活跃开发中的功能
- 可以容忍中断的非关键路径
- 测试与原型开发

请勿在生产环境 dApp 或关键集成中依赖本 API。

## 概述

本服务目前提供以下端点：

- **`/cp-creator-fee`** — 查询某个钱包在所有 CPMM 池中累积的创建者费用。
- **`/limit-order/order/list`** — 从索引器缓存中获取钱包当前挂单的 CLMM 限价订单（含未成交和部分成交）。
- **`/limit-order/history/order/list-by-user`** — 查询钱包的历史（已关闭）CLMM 限价订单，支持 `poolId` / `mint1` / `mint2` / `hideCancel` 过滤器，通过 `nextPageId` 进行游标分页。
- **`/limit-order/history/event/list-by-pda`** — 查询一个或多个限价订单 PDA 的链上事件日志（`open` / `increase` / `decrease` / `settle` / `close`），通过 `nextPageId` 进行游标分页。

## 常见用法

### 查询钱包的创建者费用

```bash
curl -s "https://temp-api-v1.raydium.io/cp-creator-fee?wallet=<your-wallet-pubkey>"
```

返回该钱包作为创建者的所有池列表，以及 Token A 和 Token B 中已累积但尚未领取的创建者费用。

### 列出钱包当前挂单的 CLMM 限价订单

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/order/list?wallet=<your-wallet-pubkey>"
```

返回完全未成交和部分成交的订单。每条记录是一个已解码的 `LimitOrderState`（包含 PDA、池、持有人、tick、方向、FIFO 批次阶段、总量/已成交量），以及调用 `SettleLimitOrder` 所需的池上下文，包含预计算的 `pendingSettle` 金额。

活跃订单端点不分页——单次返回钱包当前缓存的全部订单。

### 列出钱包的历史限价订单

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/order/list-by-user?wallet=<your-wallet-pubkey>&size=50"
```

返回一页历史订单，每条记录包含 `fillStatus`（`NONE` / `PARTIAL` / `FULL`）以及订单离开活跃集时的输入/输出金额。追加 `&nextPageId=<上一次响应中的 nextPageId>` 以获取下一页。

### 查询特定订单的事件时间线

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/event/list-by-pda?pda=<limit-order-pda>"
```

`pda` 可以是单个公钥，也可以是逗号分隔的多个公钥。每条记录对应一次链上状态变更（`open` / `increase` / `decrease` / `settle` / `close`），包含 txid、slot、区块时间、是否由持有人或 keeper 签名的标志（`autoRunner`），以及以可读单位表示的输入/输出变化量。

### 空结果

如果钱包没有缓存的记录，活跃订单端点返回：

```json
{
  "id": "...",
  "success": true,
  "data": { "rows": [] }
}
```

历史端点返回相同的 `{ rows: [], nextPageId?: ... }` 结构，`rows` 数组为空。旧版 `/cp-creator-fee` 端点为向后兼容，返回 `"data": []`。
