Temp API là khu vực lưu trữ tạm thời dành cho các endpoint chưa có vị trí cố định trong hệ thống Raydium APIs chính thức. Dịch vụ này được thiết kế phục vụ cho việc thử nghiệm nhanh, công cụ nội bộ, và các tính năng đang trong quá trình phát triển.

## Hosts

| Môi trường | Host                              |
|------------|----------------------------------|
| Mainnet    | `temp-api-v1.raydium.io`         |
| Devnet     | `temp-api-v1-devnet.raydium.io`  |

## Cảnh báo về tính ổn định

**Dịch vụ này không có bảo đảm về tính ổn định của API.** Các endpoint có thể thay đổi, bị loại bỏ, chuyển sang dịch vụ khác, hoặc xóa hoàn toàn mà không có thông báo trước. Chỉ nên sử dụng API này cho:

- Công cụ và script nội bộ
- Các tính năng đang trong giai đoạn phát triển tích cực
- Các luồng không quan trọng, có thể chịu được sự cố
- Kiểm thử và thử nghiệm

Không nên phụ thuộc vào API này cho các dApp production hoặc các tích hợp quan trọng.

## Tổng quan

Dịch vụ hiện cung cấp các endpoint sau:

- **`/cp-creator-fee`** — Truy xuất phí creator đã tích lũy cho một ví trên các CPMM pool.
- **`/limit-order/order/list`** — Danh sách các lệnh limit order CLMM đang hoạt động của ví (trạng thái mở và khớp một phần) được lấy từ bộ nhớ đệm của indexer.
- **`/limit-order/history/order/list-by-user`** — Lịch sử các lệnh limit order CLMM đã đóng của ví, hỗ trợ lọc tùy chọn theo `poolId` / `mint1` / `mint2` / `hideCancel`; phân trang theo con trỏ qua `nextPageId`.
- **`/limit-order/history/event/list-by-pda`** — Nhật ký sự kiện theo từng PDA (`open` / `increase` / `decrease` / `settle` / `close`) cho một hoặc nhiều PDA của limit order; phân trang theo con trỏ qua `nextPageId`.

## Các mẫu sử dụng phổ biến

### Kiểm tra phí creator của một ví

```bash
curl -s "https://temp-api-v1.raydium.io/cp-creator-fee?wallet=<your-wallet-pubkey>"
```

Trả về danh sách các pool mà ví là creator, cùng với phí creator đã tích lũy nhưng chưa được nhận cho Token A và Token B.

### Liệt kê các lệnh limit order CLMM đang hoạt động của ví

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/order/list?wallet=<your-wallet-pubkey>"
```

Trả về cả các lệnh chưa khớp hoàn toàn và khớp một phần. Mỗi dòng là một `LimitOrderState` đã được giải mã (PDA, pool, owner, tick, side, FIFO cohort phase, tổng số lượng/số lượng đã khớp) cùng với ngữ cảnh pool cần thiết để gọi `SettleLimitOrder`, bao gồm giá trị `pendingSettle` đã được tính sẵn.

Endpoint lấy lệnh đang hoạt động không hỗ trợ phân trang — tất cả các lệnh đang được cache của ví sẽ được trả về trong một payload duy nhất.

### Liệt kê các lệnh limit order đã đóng của ví

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/order/list-by-user?wallet=<your-wallet-pubkey>&size=50"
```

Trả về một trang lịch sử lệnh, mỗi lệnh được tóm tắt với `fillStatus` (`NONE` / `PARTIAL` / `FULL`) và số lượng input/output tại thời điểm lệnh rời khỏi tập hợp đang hoạt động. Thêm `&nextPageId=<nextPageId từ response trước>` để lấy trang tiếp theo.

### Xem lịch sử sự kiện của một lệnh cụ thể

```bash
curl -s "https://temp-api-v1.raydium.io/limit-order/history/event/list-by-pda?pda=<limit-order-pda>"
```

`pda` có thể là một khóa duy nhất hoặc danh sách nhiều khóa phân cách bằng dấu phẩy. Mỗi dòng là một lần thay đổi trạng thái on-chain (`open` / `increase` / `decrease` / `settle` / `close`) kèm theo txid, slot, thời gian block, cờ xác nhận được ký bởi owner hay keeper (`autoRunner`), và các delta input/output theo đơn vị người dùng.

### Kết quả trống

Nếu ví không có bản ghi nào trong cache, endpoint lấy lệnh đang hoạt động sẽ trả về:

```json
{
  "id": "...",
  "success": true,
  "data": { "rows": [] }
}
```

Các endpoint lịch sử trả về cùng cấu trúc `{ rows: [], nextPageId?: ... }` với mảng `rows` rỗng. Endpoint cũ `/cp-creator-fee` trả về `"data": []` để đảm bảo tương thích ngược.
