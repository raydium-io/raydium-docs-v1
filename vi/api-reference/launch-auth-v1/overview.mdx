API Xác Thực xử lý việc xác thực dựa trên ví cho các dịch vụ LaunchLab. Hãy trao đổi một thông báo ký với JWT token, sau đó sử dụng token đó để xác thực các yêu cầu đến API Forum và Mint creation.

## Các Điểm Cuối API

**Mainnet:** https://launch-auth-v1.raydium.io  
**Devnet:** https://launch-auth-v1-devnet.raydium.io

## Quy Trình Xác Thực

Không cần bất kỳ thông tin xác thực nào trước đó. Xác thực dựa trên ví:

1. **Lấy dấu thời gian hiện tại** (Unix seconds)
2. **Ký thông báo** bằng ví của bạn: `"Sign in to raydium.io: " + <timestamp>`
3. **Gửi chữ ký** đến điểm cuối `/request-token`
4. **Nhận JWT token** có hiệu lực cho khoảng thời gian bạn chỉ định
5. **Sử dụng token** trong header `ray-token` cho các điểm cuối được bảo vệ

## Định Dạng Thông Báo

Thông báo bạn ký phải tuân theo định dạng chính xác này:

```
Sign in to raydium.io: <UNIX_TIMESTAMP_SECONDS>
```

**Ví dụ:**
```
Sign in to raydium.io: 1704067200
```

- Sử dụng dấu thời gian Unix hiện tại của bạn tính bằng giây (không phải milliseconds)
- Chữ ký Ed25519 (ký ngoài Solana gốc)
- Mã hóa chữ ký dưới dạng chuỗi base58
- Chuyển dấu thời gian và chữ ký đến `/request-token`

## Xác Thực Token

Sử dụng `/check-token` để xác minh token của bạn vẫn còn hiệu lực. Điểm cuối này cũng tự động mở rộng thời gian sống của token nếu sắp hết hạn.

Thời gian sống của token có thể được định cấu hình từ phía máy chủ; mặc định thường là 24 giờ.

## Thu Hồi Token

Thu hồi token bằng điểm cuối `/del-token`.

## Các Phương Pháp Ký Được Hỗ Trợ

- **Ví tiêu chuẩn:** Chữ ký Ed25519 với `/request-token`
- **Ví Ledger:** Giao dịch ký với `/request-token-ledger`

## Các Trường Hợp Sử Dụng

- **Đăng bài trên Forum** — xác thực trước khi đăng bình luận
- **Tạo Mint** — ký quyền sở hữu ví trước khi gửi các token mới
- **Phiên người dùng** — duy trì trạng thái được xác thực trên các cuộc gọi API
- **Hỗ trợ nhiều ví** — phát hành các token riêng biệt cho mỗi địa chỉ ví
