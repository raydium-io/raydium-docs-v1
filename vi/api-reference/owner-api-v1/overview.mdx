Owner API cung cấp chế độ xem cho từng ví về các vị trí và số dư của Raydium. Nó lấy dữ liệu được lưu trong bộ nhớ cache cho các vị trí, thanh khoản khóa, các pool được tạo và tham gia IDO, rất phù hợp cho giao diện ví và bảng điều khiển danh mục.

## Hosts

| Môi trường | Host                       |
|-----------|---------------------------|
| Mainnet   | `owner-v1.raydium.io`     |
| Devnet    | `owner-v1-devnet.raydium.io` |

## Tổng quan

API cung cấp năm endpoint chính được sắp xếp theo loại tài nguyên:

- **`/position/stake/{owner}`** — Vị trí stake (AMM tiêu chuẩn / CPMM LP)
- **`/position/clmm-lock/{owner}`** — Vị trí CLMM bị khóa
- **`/create-pool/{owner}`** — Các pool được tạo bởi ví này
- **`/main/ido/{owner}`** — Dữ liệu tham gia IDO

## Độ tươi mới của dữ liệu

Dịch vụ này lưu dữ liệu vị trí vào bộ nhớ cache và cập nhật nó định kỳ. Những thay đổi vị trí rất gần đây (trong vòng vài giây) có thể không xuất hiện ngay lập tức trong các phản hồi.

## Các mẫu phổ biến

### Lấy tất cả vị trí stake cho một ví

```bash
curl -s "https://owner-v1.raydium.io/position/stake/{owner}"
```

Thay thế `{owner}` bằng khóa công khai của ví ở định dạng base58.

### Lấy các vị trí bị khóa

```bash
curl -s "https://owner-v1.raydium.io/position/clmm-lock/{owner}"
```

### Kiểm tra tham gia IDO

```bash
curl -s "https://owner-v1.raydium.io/main/ido/{owner}"
```

## Kết quả trống

Nếu một ví không có dữ liệu cho một endpoint cụ thể, dịch vụ sẽ trả về lỗi 404 kèm siêu dữ liệu yêu cầu:

```json
{
  "id": "...",
  "success": false
}
```
