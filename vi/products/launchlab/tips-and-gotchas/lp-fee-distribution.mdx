Khi token LaunchLab di chuyển sang CPMM, pool được di chuyển sẽ phát hành LP token. Cấu hình nền tảng kiểm soát những gì xảy ra với các LP token đó.

## Chia tách LP

`migrateCpLockNftScale` có ba trường. Các giá trị phải cộng lại thành `1_000_000`, tương ứng với 100%.

| Trường | Ý nghĩa |
|---|---|
| `platformScale` | LP token được khóa qua Burn & Earn. Nền tảng nhận Fee Key NFT. |
| `creatorScale` | LP token được khóa qua Burn & Earn. Người tạo token nhận Fee Key NFT. |
| `burnScale` | LP token bị đốt vĩnh viễn. Không tạo Fee Key NFT cho phần này. |

Đây là cài đặt ở cấp nền tảng. Các token được phát hành qua cùng một nền tảng sẽ kế thừa cùng một phân phối LP cho đến khi nền tảng cập nhật cài đặt.

## Ví dụ

```ts
// 40% nền tảng, 50% người tạo, 10% đốt.
migrateCpLockNftScale: {
  platformScale: new BN(400_000),
  creatorScale: new BN(500_000),
  burnScale: new BN(100_000),
}

// 100% Fee Key của người tạo.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(1_000_000),
  burnScale: new BN(0),
}

// 100% đốt.
migrateCpLockNftScale: {
  platformScale: new BN(0),
  creatorScale: new BN(0),
  burnScale: new BN(1_000_000),
}
```

## Mối quan hệ với phí CPMM

`migrateCpLockNftScale` kiểm soát **ai có thể yêu cầu quyền LP fee**. Nó không đặt tỷ lệ phí swap. Hành vi phí swap của pool được di chuyển đến từ CPMM `AmmConfig` được chọn bởi `cpConfigId`.

Ví dụ, nếu pool CPMM được di chuyển sử dụng cấu hình phí giao dịch 0,25% và `creatorScale` là 500.000, Fee Key của người tạo đại diện cho 50% quyền LP fee khóa. Nếu `platformScale` là 400.000, Fee Key của nền tảng đại diện cho 40%.

## Cập nhật chia tách

Cập nhật cấu hình nền tảng bị giới hạn theo epoch. Coi LP distribution như một chính sách nền tảng quan trọng, không phải cài đặt UI tùy tiện.

Trước khi cập nhật:

- Xác nhận ba giá trị scale cộng lại thành `1_000_000`.
- Xác nhận nền tảng hiểu rõ ai sẽ nhận Fee Key NFT sau các lần di chuyển trong tương lai.
- Ghi chép thay đổi cho các nhà tạo token trước khi các vụ phát hành mới sử dụng nó.
