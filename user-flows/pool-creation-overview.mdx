<Info>
  Creating a pool is a one-way, mostly irreversible set of choices. The pool type, fee tier, and initial price are permanent. Read this before starting the creation flow.
</Info>

## What you are creating

A Raydium pool is an on-chain program account holding two token reserves. Anyone can swap between those tokens at any time using the pool's pricing curve. You, as the pool creator, fund the initial liquidity and set the pool's permanent parameters.

Once created, the pool is permissionless — you cannot stop others from trading through it, adding liquidity, or creating competing pools at different fee tiers.

## Pool types

Raydium offers three pool programs:

| Pool | Description | Use when |
|---|---|---|
| **CPMM** | Constant-product, full-range, passive | Default for most new pools and token launches |
| **CLMM** | Concentrated liquidity, user-specified price range | Active LP, correlated pairs, higher fee efficiency |
| **AMM v4** | Constant-product + OpenBook orderbook | Legacy; only if OpenBook integration is specifically needed |

For almost all new pools, choose **CPMM** or **CLMM**. AMM v4 is no longer the default and costs more to create.

See [CPMM vs CLMM](/user-flows/cpmm-vs-clmm) for a full comparison.

## Key decisions before you start

### 1. Pool type

Pick this first — it's permanent. CPMM for passive liquidity; CLMM for active range management.

### 2. Fee tier

The fee rate is set at creation and cannot be changed. Choose based on pair volatility:

- 0.01% → stable/stable
- 0.05% → correlated
- 0.25% → standard volatile
- 1% → long-tail / exotic

See [Dynamic fees during pool creation](/user-flows/pool-creation-dynamic-fees) for guidance.

### 3. Initial price

For CPMM, price is set by your deposit ratio. For CLMM, you enter it explicitly. Use the current market price. A wrong initial price means arbitrageurs drain the underpriced token — you pay for the error through your starting position.

### 4. Initial liquidity

There is no enforced minimum, but underfunded pools produce poor price impact and bad UX for swappers. Provide enough liquidity to support meaningful trades before making the pool public.

Rules of thumb:
- For a new token launch: seed with as much as you intend the pool to hold long-term.
- For a CPMM: at least $5,000–$10,000 equivalent to reduce early price impact.
- For a CLMM: enough to support trades within the initial price range.

## Prerequisites

Before going to the creation UI, have ready:

- [ ] Both token mint addresses.
- [ ] Enough of each token to seed initial liquidity.
- [ ] 0.2–0.4 SOL for creation rent, token-account creation, and priority fees (CLMM costs slightly more than CPMM).
- [ ] Decision on pool type and fee tier.
- [ ] Current market price of the pair.

## What happens after creation

1. The pool goes live on-chain immediately.
2. The Raydium UI indexes the pool within a few minutes. It appears in search and the pool list.
3. Raydium's routing algorithm starts quoting swaps through the pool once it has meaningful liquidity.
4. Other LPs can add liquidity to the pool permissionlessly.
5. You can optionally create a farm to incentivize LPs with token rewards.
