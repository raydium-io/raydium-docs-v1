<Info>
  Pool creation requires a one-time SOL payment for on-chain rent. This SOL is not a fee to Raydium — it's locked into the program accounts as required by Solana. Most of it is non-refundable as long as the pool exists.
</Info>

## SOL cost breakdown

### CPMM pool creation

| Cost item | Approx SOL |
|---|---|
| Pool state account rent | ~0.089 SOL |
| LP mint account creation | ~0.002 SOL |
| Token vault accounts (×2) | ~0.004 SOL |
| Your LP token account | ~0.002 SOL |
| Transaction + priority fee | 0.001–0.01 SOL |
| **Total (estimated)** | **~0.10–0.15 SOL** |

### CLMM pool creation + first position

| Cost item | Approx SOL |
|---|---|
| Pool state account rent | ~0.089 SOL |
| Tick array accounts (varies) | ~0.04–0.12 SOL |
| Position NFT + metadata | ~0.015 SOL |
| Token vault accounts (×2) | ~0.004 SOL |
| Transaction + priority fee | 0.001–0.01 SOL |
| **Total (estimated)** | **~0.15–0.25 SOL** |

CLMM costs more because tick arrays (the data structures that track liquidity per price step) must be initialised. Wide ranges require more tick arrays and therefore more rent.

All estimates are for normal network conditions. Always keep at least 0.4 SOL in your wallet before attempting pool creation to avoid failures due to priority fee spikes.

## Is the SOL refundable?

Solana rents account space until the account is closed. For pools:

- **CPMM:** the pool state account rent is locked for the pool's lifetime. Closing a CPMM pool (burning all LP tokens) may eventually release some rent, but pools with any remaining liquidity from other LPs cannot be closed by the creator.
- **CLMM:** tick array rent is released when the tick array is unused and purged (rare in active pools). Your own position NFT rent (~0.015 SOL) is returned when you close the position.

Treat pool creation SOL as a sunk cost.

## Minimum liquidity recommendations

There is no on-chain enforced minimum, but underfunded pools produce bad UX:

| Pool use case | Minimum recommended initial liquidity |
|---|---|
| Testing / development | Any amount (don't share the pool) |
| Small community token | $1,000–$5,000 |
| Standard token launch | $5,000–$50,000 |
| Competitive major pair | $50,000+ |

Raydium's router considers pool depth when routing swaps. A pool with $100 of liquidity will receive no routing beyond the smallest test swaps.

## Token requirements

### Both mints must exist

You cannot create a pool for a mint that hasn't been created on-chain. Confirm the mint address is valid and the token has been initialised.

### Token-2022 mints

CPMM supports Token-2022 mints, including those with transfer fees and transfer hooks. The creation flow handles these automatically but surfaces warnings before you sign.

CLMM also supports Token-2022. Be aware of the same transfer-fee and hook considerations.

### Non-transferable mints

Mints with the non-transferable extension cannot be deposited into pools. The creation UI will reject them.

### Frozen accounts

If either token mint has a freeze authority that freezes your token account, deposits will fail. Verify mint authorities before creating a pool with an unfamiliar token.

### Sufficient balance

You must hold enough of each token to seed the initial liquidity, plus the SOL for creation costs. The UI will show the exact amounts before you sign.

## Priority fees

Pool creation is a complex transaction. During network congestion, set your priority fee to at least medium in your wallet or the Raydium UI. A failed creation transaction still costs the base fee (~0.000005 SOL) but not the rent — rent only moves if the transaction lands.
